## 3. 구현 흐름 (`.pc`)

```
W1403S01()                        ← 진입 (콜백)
  ├─ prthdr / getmid / getmod     COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정                ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1','2' → retc = W1403S01_F1(...)   목록조회
  │     '3','4' → retc = W1403S01_F2(...)   계좌비밀번호검증
  │     그외   → comhdr_o->rtyp="M", mesg="작업구분 오류." 즉시 return (MSG_NOWORK)
  ├─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
  └─ fnSetMsg(&e_msg, comhdr_o) → SEND (명시적 커밋, AUTOCOMMIT 아님)

W1403S01_F1()                     ← 결제대상계좌 목록 조회
  ├─ getmid/getmod, sndl += W1403S01_O_SZ, memset(mod)
  │  max_row = MAX_RRB / sizeof(orec[0])
  ├─ GRID key: content->keyc  K_NEXT→nkey / K_PREV→pkey / default→"0" → vKeyv
  ├─ 입력 바인딩(XPRINTF+l_rtrim): vPrGb/vWbcsRlnmAltrNo/vFxUserNo/vFndNo/
  │                               vFxSlactDcd/vPwdVrfcDcd/vActPwd/vFxStlmAcn
  ├─ 조회가능시간 체크  SELECT ... TB_FXB_CMC003D (COM_CD_NM='TRN_CNTL_TIM')
  │     SQLCODE≠0 → e_msg 10118 RET_FAILURE / vSchYn='N' → e_msg 32713 RET_FAILURE
  ├─ DECLARE W1403S01_C1 CURSOR  (인라인뷰 UNION ALL)
  │     상단 SELECT: prGb='2' 이용자별(펀드) — CSC003M+CSC001M+CSC012M(+CMC003D×2)
  │     하단 SELECT: prGb='1' 고객별       — CSC003M+CSC001M(+CMC003D×2), FX_SLACT_YN='Y'
  │     외곽 WHERE: KEY(WBCS_RLNM_ALTR_NO||FX_SLACT_DCD) > :vKeyv  (페이징)
  ├─ OPEN → (NO_DATA_FOUND→91001 return SQL_SUCCESS / else 10118 RET_FAILURE)
  ├─ FETCH loop (ii++)
  │     NO_DATA_FOUND → break / else 10118 CLOSE RET_FAILURE
  │     ii==max_row → i_isnext++, break  (다음페이지 존재)
  │     4-1. fnGetHostAcntBals(stAcntBals) — 계좌별 HOST(EAI) 잔액조회
  │           실패 → 10118 RET_FAILURE
  │     fnAcntConv() — 계좌번호 '-' 포맷 변환
  │     for j in stAcntBals.cBalCnt (≤MAX_CCY_CNT): orec[arrcount] 세팅, arrcount++
  │           crcd/cusAcntBal 은 HOST 응답값으로 덮어씀
  │     vKeyv = cKeyv (NEXT KEY)
  ├─ CLOSE W1403S01_C1
  ├─ 결과판정: ii==0&&i_isnext==0 → 91001 / i_isnext==1 → set_nkey+91002 / else 91003
  └─ grid01_cnt = arrcount, sndl += (arrcount-1)*sizeof(orec[0]), RET_SUCCESS

W1403S01_F2()                     ← 계좌비밀번호 검증
  ├─ getmid/getmod, sndl += W1403S01_O_SZ, memset(mod)
  ├─ 입력 바인딩: vFxSlactDcd/vPwdVrfcDcd/vActPwd/vFxStlmAcn
  ├─ vPwdVrfcDcd!='Y' → e_msg 10117 (단, return 안함 — 계속 진행)
  ├─ strlen(vActPwd)==32 → SEED(ECB) 복호화 → vActDcPwd
  ├─ 조회가능시간 체크 (F1 과 동일 SELECT) — 10118 / 32713
  ├─ SMPw_Encrypt(vActDcPwd→vActEcPwd)  실패 → 10818 RET_FAILURE
  ├─ fnGetHostAcntBals(stAcntBals, pwdVrfc='Y') — HOST 비번검증
  │     실패해도 return 안하고 cBalCnt=1 강제 (임시 우회, "SOON")
  ├─ cBalCnt<=0 → ii=0(오류) RET_FAILURE / else ii=1(정상) 91003
  └─ grid01_cnt = ii, sndl += (ii-1)*sizeof(orec[0]), RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `prGb 1/2`→F1, `3/4`→F2. `.h` 는 `1/2/3` 만 문서화(§6).
- **페이징**: 커서 KEY = `WBCS_RLNM_ALTR_NO || FX_SLACT_DCD`, 외곽 WHERE `KEY > :vKeyv`. `keyc` (K_NEXT/K_PREV/default) 로 vKeyv 결정. `ii==max_row` 시 `i_isnext=1`, `content_set_nkey`, e_msg=91002.
- **UNION ALL 상/하단 상호배타**: 상단은 `'2'=vPrGb`, 하단은 `'1'=vPrGb` 리터럴 조건으로 prGb 값에 따라 한쪽만 매칭. 하단(고객별)은 추가로 `FX_SLACT_YN='Y'` 필터.
- **HOST 잔액 fan-out**: fetch 된 계좌 1건마다 `fnGetHostAcntBals` 로 통화별 잔액을 받아 `cBalCnt` 만큼 orec 를 복제 생성(통화별 1 orec). `crcd`/`cusAcntBal` 는 HOST 응답으로 덮어씀. 통화 수는 `MAX_CCY_CNT` 로 상한.
- **F2 비밀번호검증**: `actPwd` 길이 32 일 때만 SEED-ECB 복호화(고정 16B 키) → NH `SMPw_Encrypt` 재암호화 → `fnGetHostAcntBals(pwdVrfc='Y')` 로 HOST 검증. 성공 시 grid01_cnt='1', 실패(계좌없음) 시 '0'.
- **트랜잭션**: 조회 전용이나 진입부에서 명시적 commit/rollback 수행(AUTOCOMMIT 아님).
- **에러코드(e_msg.code)**: `10118`(SQL/시스템 오류), `32713`(일자전환 거래제한), `91001`(자료없음), `91002`(다음조회 계속), `91003`(조회완료), `10117`(비번검증구분 오류, F2), `10818`(NH암호화 오류, F2). rtyp="M" 이므로 `fnSetMsg` 가 코드→메시지 변환해 COMHDR 로 전달.

## 6. 주의 — 문서/코드 불일치

| 위치 | 헤더/주석 표기 | 실제(.pc) |
|---|---|---|
| `.h` `prGb` 주석 | "1:일반고객계좌잔고조회, 2:펀드고객계좌잔고조회, 3:비밀번호검증" | switch 는 `'1','2'→F1`, `'3','4'→F2`. **`'4'` 미문서화**, 명칭도 '잔고조회'가 아니라 '결제대상계좌 목록조회' |
| `.h` `grid01_cnt` 주석 | "prGb [1,2] 건수, [3] 0:오류,1:정상" | `'4'` 도 F2 이므로 0/1 규칙 동일 (누락) |
| F1 `orec.crcd` / `.h` "통화코드" | cursor `ICSC003M.CRCD` 를 뜻하는 것처럼 보임 | 출력값은 커서 CRCD 가 아니라 **HOST(EAI) 잔액응답 통화**(`stAcntBals.cCrcd[j]`) 로 덮어씀 |
| F1 페이징 vs 출력 카운트 | — | 페이징 상한은 fetch 행수 `ii==max_row` 로 판정하나, 실제 출력은 통화별 `arrcount`. 1행이 다통화면 arrcount > max_row 로 커져 **`mod->orec` 버퍼 초과 위험** (통화 fan-out 이 페이징 상한을 우회) |
| F1/F2 말미 `sndl` 보정 | — | `content->sndl += (ii-1)*sizeof(orec[0])`. 건수 0 (ii=0/arrcount=0) 이면 `-1 * orec` 로 **sndl 이 orec 1건만큼 감소** (자료없음 경계 버그) |
| F1 조회가능시간 CASE | "조회 가능시간 여부" | `CASE WHEN (con1<=now) OR (con2>=now) THEN 'N' ELSE 'Y'`. OR 조건이라 거의 항상 참→'N'(불가) 로 평가되는 **로직 의심** (AND 여야 창구간 판정. 실 운영 데이터 의존) |
| F2 `pwdVrfcDcd!='Y'` 분기 | 검증 실패 시 오류 처리 기대 | e_msg 10117 만 세팅하고 **return 하지 않음** → 그대로 검증 로직 진행 (조기중단 누락) |
| F2 `fnGetHostAcntBals` 실패 처리 | HOST 비번검증 실패 시 오류 반환 기대 | 실패해도 `return` 대신 `cBalCnt=1` 강제 → 후속 분기가 **항상 정상(grid=1)** 로 귀결. 주석 "SOON 임시로 막고 정상처리" — **비밀번호검증 사실상 무력화** |
| F2 SEED 복호화 조건 | actPwd 복호화 | `strlen(vActPwd)==32` 일 때만 복호화. 그 외 길이면 `vActDcPwd` 공백 상태로 암호화·검증 진행 |
| F1 OPEN `NO_DATA_FOUND` 분기 | 커서 OPEN 시 자료없음 처리 | OPEN 은 통상 NO_DATA_FOUND 를 리턴하지 않음 → **dead branch** (자료없음은 FETCH loop 후 `ii==0` 로 판정됨) |
| `.h` 필드폭 vs `.pc` DECLARE | `actPwd[64]` | 호스트변수 `vActPwd[64+1]`. 단 복호화는 32자 hex(16B) 전제 — 64B 필드 대비 실사용 폭 상이 |
