## 3. 구현 흐름 (`.pc`)

```
W1358A01()                        ← 진입 (콜백)
  ├─ prthdr / getmid / getmod     COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               pname=rkey, ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1358A01_F1(...)   등록/수정
  │     '3' → retc = W1358A01_F3(...)   삭제
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return(sndl)
  ├─ retc==RET_SUCCESS ? (eflg="0", l_dbcommit()) : (eflg="1", l_dbrollback())
  ├─ fnSetMsg(&e_msg, comhdr_o)   e_msg.code → 응답 메시지 세팅
  └─ return(content->sndl)

W1358A01_F1()                     ← 등록/수정
  ├─ getmid/getmod, memset(mod)
  ├─ GetDate(thddYmd)            당일 YYYYMMDD
  ├─ cBaseYmd ← mid->baseYmd, vFxSysLsmdId ← comhdr_i->usid (최종변경ID)
  ├─ SELECT MAX(APSR_YMD) INTO cMaxYmd
  │         FROM TB_FXB_CMG039M WHERE FNAP_YMD='99991231'      (현재 유효건의 최신 적용시작일)
  │     SQLCODE≠0 && ≠NO_DATA → e_msg "10118"(조회오류), RET_FAILURE
  │     cMaxYmd > cBaseYmd → e_msg "10784"(적용예정건 존재), RET_FAILURE
  ├─ cnt = l_n2i(grid01_cnt); for(ii=0; ii<cnt; ii++)
  │     · orec[ii] 필드 yprintf + l_rtrim (cCrnc, cFxPdcd, cPdfTnrClsfId, 시각 4종, cUsdYn)
  │     · if (cMaxYmd == cBaseYmd)                       ← 최신 적용일과 입력일 동일
  │           UPDATE TB_FXB_CMG039M SET 시각4종 + 변경ID/TS
  │                  WHERE CRCD/FX_PDCD/PDF_TNR_CLSF_ID/APSR_YMD=cBaseYmd
  │           SQLCODE≠0 → e_msg "10122", RET_FAILURE
  │       else                                            ← 신규 적용일
  │           if (strlen(cMaxYmd)!=0)
  │               UPDATE TB_FXB_CMG039M SET FNAP_YMD=cBaseYmd-1, 변경ID/TS
  │                      WHERE ... APSR_YMD=cMaxYmd       (기존건 종료일 마감)
  │               SQLCODE≠0 → e_msg "10122", RET_FAILURE
  │           INSERT INTO TB_FXB_CMG039M (..., APSR_YMD=cBaseYmd, FNAP_YMD='99991231',
  │                  시각4종, USE_YN='Y', 변경ID, SYSDATE)
  │           SQLCODE≠0 → e_msg "10122", RET_FAILURE
  │     · [lines 356–483 전체 주석 처리 — 최종적용일 재조회 + vInsertYn 분기 + SYSDATE INSERT 는 DEAD CODE, §6]
  └─ e_msg "10150", sndl += O_SZ, RET_SUCCESS

W1358A01_F3()                     ← 삭제
  ├─ getmid/getmod, memset(mod), GetDate(thddYmd)
  ├─ cBaseYmd ← baseYmd, vFxSysLsmdId ← comhdr_i->usid
  ├─ SELECT COUNT(*) INTO nApsrCnt FROM (
  │         SELECT APSR_YMD,FNAP_YMD FROM TB_FXB_CMG039M
  │          WHERE (APSR_YMD=cBaseYmd AND FNAP_YMD='99991231')
  │             OR (FNAP_YMD = cBaseYmd-1)
  │          GROUP BY APSR_YMD,FNAP_YMD )

  │     nApsrCnt==0 → e_msg "10326"(삭제대상없음), RET_FAILURE

  │     SQLCODE≠SUCCESS → e_msg "10122", RET_FAILURE
  ├─ UPDATE TB_FXB_CMG039M SET FNAP_YMD='99991231', 변경ID/TS
  │         WHERE FNAP_YMD = cBaseYmd-1                   (직전건을 다시 유효건으로 복원)
  │     SQLCODE≠0 → e_msg "88888", mesg="적용시작일자 … 종료일자 반영중 오류", RET_FAILURE
  └─ e_msg "10150", sndl += O_SZ, RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG039M` | 고객거래가능시간 | `CMG039M` (일부 문에서만) | SELECT / INSERT / UPDATE / DELETE | 통화·상품·테너별 지정가/시장가 거래가능시각 이력 관리 (APSR_YMD~FNAP_YMD 유효기간, `'99991231'` = 현재 유효건) |

*DML 요약:*
- **F1 등록/수정**: `SELECT MAX(APSR_YMD)` 로 최신 유효건 확인 → 동일일이면 시각 `UPDATE`, 신규일이면 기존 유효건 종료일 `UPDATE`(FNAP_YMD=입력일-1) 후 신규 `INSERT`.
- **F3 삭제**: `SELECT COUNT(*)`(그룹 2건 검증) → 유효건 `DELETE` → 직전건 `UPDATE`(FNAP_YMD='99991231' 복원).
- 공통함수 경유 간접 테이블 접근 없음 (직접 EXEC SQL 만 사용).

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `prGb` `'1'`=등록/수정, `'3'`=삭제. 그 외는 COMHDR `mesg="작업구분 오류."` 반환, DML 미수행.
- **유효기간 모델**: 각 (CRCD, FX_PDCD, PDF_TNR_CLSF_ID) 조합은 `APSR_YMD`(적용시작)~`FNAP_YMD`(적용종료) 구간 이력. 현재 유효건은 `FNAP_YMD='99991231'`.
- **등록 선행검증**: 현재 유효건의 최신 `APSR_YMD`(cMaxYmd)가 입력 `baseYmd` 보다 크면(미래 예정건 존재) `e_msg "10784"` 로 거부.
- **등록 분기**:
  - `cMaxYmd == cBaseYmd` → 기존 유효건의 시각 4종만 UPDATE (덮어쓰기).
  - `cMaxYmd != cBaseYmd` → 기존 유효건 종료일을 `baseYmd-1` 로 마감 후 `baseYmd~'99991231'` 신규 INSERT (INSERT 의 `USE_YN` 은 항상 `'Y'`).
- **삭제 선행검증**: `(APSR_YMD=baseYmd AND FNAP_YMD='99991231')` 유효건과 `(FNAP_YMD=baseYmd-1)` 직전건이 GROUP BY 결과 **정확히 2세트**여야 삭제 진행. 0 또는 2가 아니면 `e_msg "10326"`.
- **삭제 처리**: 유효건 DELETE → 직전건을 `FNAP_YMD='99991231'` 로 UPDATE(유효 복원). 복원 실패 시 `e_msg "88888"` + 동적 `mesg`.
- **트랜잭션**: 서브함수는 성공 `RET_SUCCESS` / 실패 `RET_FAILURE` 만 반환, 상위에서 `l_dbcommit`/`l_dbrollback`.
- **에러코드(e_msg.code)**: `10118` 조회오류, `10784` 적용예정건 존재, `10122` 자료처리오류, `10326` 삭제대상 없음, `88888` 종료일 반영오류(동적 mesg), `10150` 정상완료.
