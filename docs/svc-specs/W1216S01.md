# 서비스 명세 — `W1216S01` (통화Pair 조회 — 코드조회 팝업)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1216S01` |
| 서비스 이름 | 통화Pair 조회 (코드조회 팝업) |
| 유형 | 조회 (SELECT 단건 커서, 그리드/페이징) |
| 모듈 | W1200 |
| 구현 | `win/src/trn/W1200/W1216S01.pc` (302 라인) |
| I/O 헤더 | `win/src/inc/trn/W1216S01.h` |
| 라우팅 alias | `W1216S01` |
| 채널 | 매매 (ChannelWeb, `/v1/tx`) |

통화페어 마스터(`TB_FXB_CMG004M`)에서 사용중(`USE_YN='Y'`) 통화페어ID 목록을 조회하는 코드조회 팝업용 서비스. 페어ID 부분검색 + K_NEXT/K_PREV 기반 페이징을 지원한다.

## 2. I/O 전문 (`.h`)

**Input `W1216S01_I`** (총 8 byte)
| 필드 | 크기 | 설명 (실제 의미) |
|------|-----|-----------------|
| `prGb` | 1 | 작업구분 — `.pc` 는 `'1'` 만 처리(조회), 그 외는 "작업구분 오류" |
| `crncPairId` | 7 | 통화페어ID — 검색 조건. 공백이면 전체, 값이 있으면 완전일치 OR `%...%` 부분일치 |

**Output `W1216S01_O`** (그리드, orec 가변)
| 필드 | 크기 | 설명 |
|------|-----|------|
| `grid01_cnt` | 6 | 그리드 건수 (조회된 orec 개수) |
| `orec[]` | 반복 | 아래 반복 레코드. `.h` 는 `orec[1]` 로 선언, 실제 전송 건수는 `sndl` 로 가변 확장 |
| └ `orec[].crncPairId` | 7 | 통화페어ID |

레코드 상한 `max_row = MAX_RRB / sizeof(orec[0])`. `ii == max_row` 도달 시 다음 페이지 존재 플래그(`i_isnext`) 세팅 후 중단.

## 3. 구현 흐름 (`.pc`)
```
W1216S01()  [진입/dispatch]
  ├ prthdr / getmid / getmod            // 공통헤더·입력·출력 주소 획득
  ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  ├ switch(mid->prGb[0])
  │    case '1' → W1216S01_F1()          // 조회
  │    default  → comhdr_o.mesg "작업구분 오류." → return sndl
  ├ retc == RET_SUCCESS → l_dbcommit(), eflg="0"
  │            else      → l_dbrollback(), eflg="1"
  ├ fnSetMsg(&e_msg, comhdr_o)           // e_msg.code → mesg/rcod/rtyp 세팅
  └ return content->sndl

W1216S01_F1()  [조회]
  ├ getmid / getmod, sndl += W1216S01_O_SZ, memset(mod)
  ├ max_row = MAX_RRB / sizeof(orec[0])
  ├ switch(content->keyc)                // 페이징 키
  │    K_NEXT → content_get_nkey → vKeyv
  │    K_PREV → content_get_pkey → vKeyv
  │    default → vKeyv="0"               // 최초
  ├ yprintf/l_rtrim(vCrncPairId)         // 입력 페어ID 정리
  ├ EXEC SQL DECLARE W1216S01_C1 CURSOR  // TB_FXB_CMG004M SELECT
  ├ EXEC SQL OPEN
  │    NO_DATA_FOUND → e_msg 91001, return RET_SUCCESS
  │    else error    → e_msg 10118, return RET_FAILURE
  ├ for(ii=0; ;ii++) FETCH
  │    NO_DATA_FOUND → break
  │    error         → CLOSE, e_msg 10118, return RET_FAILURE
  │    ii == max_row → i_isnext++, break        // 다음페이지 있음
  │    zsedit(orec[ii].crncPairId), vKeyv=cKeyv // 레코드 적재 + NEXT KEY 갱신
  ├ EXEC SQL CLOSE
  ├ 결과판정
  │    ii==0 && i_isnext==0 → e_msg 91001 (자료없음)
  │    i_isnext==1          → content_set_nkey(vKeyv), e_msg 91002 (계속)
  │    else                 → e_msg 91003 (완료), grid01_cnt=ii
  ├ eflg="0", grid01_cnt=ii
  ├ sndl += (ii-1)*sizeof(orec[0])       // 가변 전송길이 확장
  └ return RET_SUCCESS
```
트랜잭션: 조회 서비스이나 상위 dispatch 에서 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 을 명시적으로 호출(AUTOCOMMIT 아님).

## 4. 사용 DB 테이블
| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|--------|-------|------|------|------|
| `TB_FXB_CMG004M` | 통화페어 마스터 | `CMG004M` | SELECT | 사용중(`USE_YN='Y'`) 통화페어ID 목록 조회 |

*DML 요약: SELECT 전용 (커서 `W1216S01_C1` 1개). INSERT/UPDATE/DELETE 없음. 공통함수 경유 간접 테이블 접근 없음.*

## 5. 로직 / 필터 규칙
- **커서 필터**: `CRNC_PAIR_ID = NVL(:vCrncPairId, CRNC_PAIR_ID) OR CRNC_PAIR_ID like '%'||TRIM(:vCrncPairId)||'%'` (완전일치 또는 부분일치) AND `USE_YN='Y'` AND `LPAD(RPSN_SQC,7,0)||TRIM(CRNC_PAIR_ID) > :vKeyv`. `ORDER BY RPSN_SQC, CRNC_PAIR_ID`.
- **페이징 키**: NEXT_KEY = `LPAD(RPSN_SQC,7,0)||TRIM(CRNC_PAIR_ID)`. 최초 조회는 vKeyv="0", 이후 마지막 fetch 의 cKeyv 를 커서 조건에 넘겨 재조회.
- **다음 페이지 판정**: fetch 루프에서 `ii == max_row` 도달 시(= 상한+1번째 레코드가 존재) `i_isnext` 세팅하고 해당 레코드는 적재하지 않고 중단. `content_set_nkey` 로 NEXT KEY 전달.
- **에러/메시지 (e_msg.code → fnSetMsg)**:
  | code | 의미 |
  |------|------|
  | 91001 | 해당 자료가 없습니다 |
  | 91002 | 조회가 계속됩니다 (다음 페이지 존재) |
  | 91003 | 조회가 완료되었습니다 |
  | 10118 | 커서 OPEN/FETCH SQL 오류 |
- **eflg**: dispatch 에서 성공 0 / 실패 1. `W1216S01_F1` 내부에서도 성공 시 `eflg="0"` 를 별도 세팅.

## 6. 주의 — 문서/코드 불일치
| 위치 | 헤더/주석 표기 | 실제(.pc) |
|------|--------------|-----------|
| `.pc` L48 | `W1216S01_O *mod; // ... 입력값 주소` | 실제는 **출력값** 주소 (getmod). 주석 복붙 오기 |
| `.h` L12 / `.pc` L12 | `.h`: "통화Pair 조회" | `.pc`: "통화Pair 조회(코드조회 팝업)" — `.pc` 가 정확 |
| `.pc` L297 `sndl += (ii-1)*sizeof(orec[0])` | — | **버그 가능**: L151 에서 이미 `W1216S01_O_SZ`(orec[1] 1건 포함)를 더했고 여기서 `(ii-1)`건을 추가해 총 ii건이 되는 구조. 그러나 자료없음(ii==0)으로 이 라인까지 도달하면 `(0-1)*size` 만큼 sndl 이 **음수 오프셋으로 축소**됨 (한 레코드 크기만큼 언더플로우). 정상경로(ii≥1)에서는 문제 없음 |
| `.pc` L287, L295 | — | `grid01_cnt` 를 완료 분기(L287)와 무조건 라인(L295)에서 **이중 세팅** — 값은 동일(ii)해 무해하나 중복 |
| `.pc` L208~215 (OPEN 후 NO_DATA_FOUND 분기) | — | Oracle precompiler 관례상 커서 OPEN 은 통상 `NO_DATA_FOUND` 를 반환하지 않음(자료없음은 첫 FETCH 에서 발생). 사실상 **dead code** 성격, 실제 자료없음은 L270 fetch-루프 후 분기가 처리 |
| `.h` L37 `prGb` | 작업구분 (값 정의 없음) | `.pc` 는 `'1'`(조회)만 유효, 그 외 전부 "작업구분 오류." 로 거부 |
