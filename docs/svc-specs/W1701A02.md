## 3. 구현 흐름 (`.pc`)

```
W1701A02()                        ← 진입 (콜백)
  ├─ getmid / getmod             COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               ltyp=comhdr_i->ltyp, mtyp="S"(FXON), rtyp="M"
  ├─ switch(comhdr_i->ctyp[0])   접속매체 → comhdr_i->chnl 매핑 (HTS/MTS/WTS/EMP/API/SPI)
  ├─ retc = W1701A02_F1(...)      ← 실제 로직 (작업구분 분기 내부에서 처리)
  └─ retc==RET_SUCCESS ? l_dbcommit() : (l_dbrollback() + comhdr_o->eflg="1")
                                  fnSetMsg(&e_msg, comhdr_o) 로 메시지 세팅

W1701A02_F1()                     ← 실 로직 (함수 주석 "SMS통지서비스등록" 은 오기, §6)
  ├─ getmid/getmod, memset(mod), comhdr_o->rtyp="G"

  ├─ 입력 바인딩: yprintf + l_rtrim → iXxx 변수들
  ├─ 유효성: imEnuId 없음 → e_msg "42021" RET_FAILURE
  │          ilRdvMenuId 없음 → e_msg "42022" RET_FAILURE
  ├─ SELECT 1 INTO v_chk       FROM CMC011M WHERE MENU_ID=:imEnuId  (기등록 존재확인)
  ├─ SELECT COUNT(1) INTO v_scn_chk FROM CMC011M WHERE FX_SCRE_NO=NVL(:ifXScreNo,'x')
  │                            ※ v_scn_chk 계산만 하고 이후 미사용 (dead code, §6)
  │  (메뉴코드채번 SELECT 는 주석처리됨 — "채번에서 받는걸로")
  │
  ├─ wKtp=='1' (등록)
  │     v_chk!=0 → "이미 등록" e_msg "10151" RET_FAILURE
  │     INSERT INTO CMC011M (...) VALUES(... MENU_SRN=1, LRRN_SRN=1, PRGH_SPTN_YN='N' ...)
  │
  ├─ wKtp=='2' (수정)
  │     v_chk==0 → "메뉴ID 확인" e_msg "42021" RET_FAILURE
  │     UPDATE CMC011M SET ... WHERE MENU_ID=:imEnuId
  │
  └─ else (삭제; '3' 및 그 외 모든 값)
        v_chk==0 → "메뉴ID 확인" e_msg "42021" RET_FAILURE
        SELECT LRDV/MDDV/SMDV INTO :vlRdvMenuId/:vmDdvMenuId/:vsMdvMenuId
               FROM CMC011M WHERE MENU_ID=:imEnuId   (DB 실값 재조회)
        ├─ vmDdvMenuId 빈값 (대분류만 존재) → 대분류 통째 삭제
        │     DECLARE/OPEN W1701A02_C1 (SELECT MENU_ID WHERE LRDV_MENU_ID=:vlRdvMenuId)
        │     FETCH 루프: 각 MENU_ID 를 CMC012M.MENU_ID_CTLG_CON 에서 REPLACE 제거 (UPDATE)
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC011M` | 메뉴분류(관리) | `CMC011M` | **SELECT / INSERT / UPDATE / DELETE** | 기등록 존재확인·화면번호 카운트(SELECT), 등록(INSERT), 수정(UPDATE), 삭제(DELETE, 대/중/소 분류 단위). 삭제 대상 목록은 커서 `C1`(대분류)·`C2`(중분류)로 조회 |
| `TB_FXB_CMC012M` | 메뉴화면상세 | `CMC012M`, `CMC012M1`, `CMC012M2` | **UPDATE** | 삭제되는 메뉴ID 를 `MENU_ID_CTLG_CON`(메뉴ID 카탈로그, 콤마구분 목록)에서 `REPLACE`/`INSTR` 로 제거. 자기 조인(`CMC012M2`)으로 동일 (FX_USER_DCD, SCRE_ACS_ATHR_DCD) 행 참조 |

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `mid->wKtp[0]` — `'1'`=INSERT, `'2'`=UPDATE, **그 외 모든 값**(`'3'` 포함, 검증 없음)=DELETE. 즉 오타/빈값도 삭제 경로로 진입하므로 주의(§6).
- **입력 유효성**: `mEnuId`, `lRdvMenuId` 만 필수 검사(각각 `42021`/`42022`). 나머지 필드는 미검증.
- **등록 중복 방지**: `wKtp=='1'` 인데 `v_chk!=0`(이미 존재) → `10151`. 수정/삭제는 `v_chk==0`(없음) → `42021`.
- **삭제 단위 판단**: 입력값이 아니라 **DB에서 재조회한** 중/소분류 값의 공백 여부로 대/중/소 삭제 범위를 결정. 중분류 공백=대분류 통삭제, 소분류 공백=중분류 삭제, 둘 다 존재=단건 삭제.
- **카탈로그 정합성 유지**: 메뉴 삭제 시 `CMC012M.MENU_ID_CTLG_CON`(권한별 화면 메뉴 목록)에서 해당 메뉴ID 를 `REPLACE` 로 제거 — 길이<10 이면 통째, 아니면 `menuid,` 또는 `,menuid` 패턴 제거.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님 — 명시적 커밋 필수. 커서 오류 시 `CLOSE` 후 `RET_FAILURE`.
- **에러(e_msg.code→mesg, eflg)**:
  - 성공: `00000`, `eflg="0"`.
  - 입력오류: `42021`(메뉴ID) / `42022`(대분류메뉴ID) / `10151`(이미 등록).
  - SQL 오류: `95000`(전산담당자 문의) — SELECT/INSERT/UPDATE/DELETE/커서 전 구간 공통.
  - 실패 시 main 에서 `comhdr_o->eflg="1"` + `l_dbrollback`.
  - → WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, `errn`/COMHDR 본문 동봉(레거시 규약).
