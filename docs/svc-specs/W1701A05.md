## 3. 구현 흐름 (`.pc`)

```
W1701A05()                        ← 진입 (콜백)
  ├─ getmid / getmod              COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               pname=rkey, ltyp=comhdr_i->ltyp, mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(comhdr_i->ctyp[0])    HTS/MTS/WTS/EMP/API/SPI → comhdr_i->chnl[0]=ctyp[0] (이후 미사용)
  ├─ retc = W1701A05_F1(...)      ← 실제 로직
  ├─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)
  ├─ fnSetMsg(&e_msg, comhdr_o)   e_msg → 응답 COMHDR 편집
  └─ return content->sndl

W1701A05_F1()                     ← 등록/수정/삭제 (함수 주석 "SMS통지서비스등록" 은 복붙 오기)
  ├─ getmid/getmod, memset(mod), comhdr_o->rtyp="G"   ← 진입부 "M" 을 "G"(오류코드전송)로 덮어씀
  ├─ 입력 바인딩: yprintf + l_rtrim → iwKtp, ifXUserDcd, iscreAcsAthrDcd,
  │              icPyfXUserDcd, icPyscreAcsAthrDcd, ibScGrpYn, ifXAtgpDcdNm
  │              cSysUno = comhdr_i->usid[0..10) (시스템사용자번호)
  ├─ 유효성: ifXUserDcd[0]==0x00 → e_msg "32043", RET_FAILURE
  ├─ if iwKtp=='1' (등록/수정)
  │     ├─ ifXAtgpDcdNm[0]==NULL → e_msg "42038"(권한그룹명 오류), RET_FAILURE
  │     ├─ SELECT count(1) INTO :i_sndcnt FROM TB_FXB_CMC012M
  │     │        WHERE FX_USER_DCD=:ifXUserDcd AND SCRE_ACS_ATHR_DCD=:iscreAcsAthrDcd
  │     │        SQLCODE!=0 → e_msg "95000", RET_FAILURE
  │     ├─ i_sndcnt==0 → INSERT TB_FXB_CMC012M (신규 그룹)
  │     │                (MENU_ID_CTLG_CON=NULL, USE_YN='Y', LSMD_ID=cSysUno, LSMD_TS=SYSDATE)
  │     └─ else        → UPDATE TB_FXB_CMC012M SET ATHR_GRP_NM=:ifXAtgpDcdNm (그룹명 갱신)
  │                      실패 시 e_msg "95000", RET_FAILURE
  ├─ else if iwKtp=='2' (삭제)
  │     └─ DELETE TB_FXB_CMC012M WHERE FX_USER_DCD/SCRE_ACS_ATHR_DCD, 실패 "95000"
  ├─ else if iwKtp=='3' (메뉴목록 초기화)
  │     └─ UPDATE TB_FXB_CMC012M SET MENU_ID_CTLG_CON=NULL, 실패 "95000"
  │     (그 외 wKtp 값이면 아무 DML 없이 통과)
  └─ 정상 → comhdr_o->eflg="0", e_msg "00000", RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC012M` | 사용자구분별메뉴목록기본 (권한그룹 마스터) | `CMC012M` | SELECT / INSERT / UPDATE / DELETE | 권한그룹(FX사용자구분코드 × 화면접근권한구분코드) 존재확인 + 등록/그룹명수정/삭제/메뉴초기화 |

*DML 요약:*
- **SELECT** `count(1)` — 등록('1') 분기에서 PK 존재 여부 판별 → INSERT vs UPDATE 결정
- **INSERT** — PK 미존재 시 신규 권한그룹 등록 (`MENU_ID_CTLG_CON=NULL`, `USE_YN='Y'`, `BSC_GRP_YN=:ibScGrpYn`, LSMD_ID/TS 스탬프)
- **UPDATE** — (a) 존재 시 `ATHR_GRP_NM` 갱신, (b) `wKtp='3'` 시 `MENU_ID_CTLG_CON=NULL` 초기화
- **DELETE** — `wKtp='2'` 시 PK 기준 물리 삭제
- 커서/FETCH 없음 (전건 단일행 DML). 트랜잭션은 상위 `W1701A05()` 가 반환값 기준 `l_dbcommit()`/`l_dbrollback()`.

## 5. 로직 / 필터 규칙

- **PK**: `FX_USER_DCD` + `SCRE_ACS_ATHR_DCD` 조합. 모든 DML 의 WHERE 절이 이 두 컬럼.
- **작업구분(`wKtp`) 분기**:
  - `'1'` = upsert (존재하면 그룹명만 UPDATE, 없으면 INSERT)
  - `'2'` = DELETE (물리 삭제)
  - `'3'` = 메뉴목록 초기화 (`MENU_ID_CTLG_CON=NULL`)
  - 그 외(`'4'` 복사 포함) = 매칭 분기 없음 → DML 없이 정상 종료 (§6 참조)
- **필수값 검증**: `ifXUserDcd` 비어있으면 e_msg `32043` → 실패. 등록('1') 은 추가로 `ifXAtgpDcdNm` 필수(비면 `42038`).
- **감사 컬럼**: `FX_SYS_LSMD_ID` = COMHDR `usid` 앞 10바이트(`cSysUno`), `FX_SYS_LSMD_TS` = `SYSDATE`.
- **에러 코드**: 입력오류 `32043`(FX사용자구분코드)/`42038`(권한그룹명), SQL 오류 일괄 `95000`, 정상 `00000` + `eflg="0"`.
- **응답 타입**: 진입부는 rtyp `"M"`(Message전송) 설정하나 `F1` 에서 `comhdr_o->rtyp="G"`(오류코드전송)로 덮어씀 → 실제 클라이언트로는 `"G"` 로 나감.
