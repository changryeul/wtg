## 2. I/O 전문 (`.h`)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" |
| `fdmId` | 10 | 외환중개사ID — 조회 필터(빈값이면 전체). `NVL(TRIM(:vFdmId), FDM_ID)` |

| `orec[]`.`fdmId` | 10 | 외환중개사ID |
| `orec[]`.`fdmAcn` | 20 | 외환중개사계좌번호 |
| `orec[]`.`fdmAcntNm` | 50 | 외환중개사계좌명 |

> 전송 길이는 `content->sndl += W1357S01_O_SZ` (헤더+orec 1개) 후 `+= (ii-1)*sizeof(orec[0])` 로 보정한다.

## 3. 구현 흐름 (`.pc`)

```
W1357S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1357S01_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  ├─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()  (명시적 커밋)
  └─ fnSetMsg(&e_msg, comhdr_o) → COMHDR 응답 세팅

W1357S01_F1()                   ← 조회 (grid 커서 + 페이징)
  ├─ getmid/getmod, sndl += W1357S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])         한 페이지 최대 행수
  ├─ GRID key 처리 (content->keyc)
  │     K_NEXT → content_get_nkey → vKeyv
  │     K_PREV → content_get_pkey → vKeyv        (주의: PREV 도 nkey 방향 '>' 필터 사용 — §6)
  │     default(최초) → vKeyv="0"
  │     l_rtrim(vKeyv)
  ├─ 입력 바인딩: yprintf + l_rtrim → vFdmId / vUseYn  (유효성 실검증 없음 — §6)
  ├─ DECLARE W1357S01_C1 CURSOR ... OPEN
  │     SQLCODE≠SUCCESS → NO_DATA_FOUND: e_msg "91001", RET_FAILURE
  │                        기타 err     : e_msg "10118", RET_FAILURE
  ├─ for(ii=0; ; ii++) FETCH
  │     NO_DATA_FOUND → break
  │     기타 err      → CLOSE + e_msg "10118" + RET_FAILURE
  │     ii==max_row   → i_isnext++, break   (초과 1건 = 다음페이지 존재 신호, 미저장)
  │     그 외         → mod->orec[ii] 세팅 + vKeyv = cKeyv(NEXT_KEY)
  ├─ CLOSE W1357S01_C1
  ├─ 결과 판정
  │     ii==0 && i_isnext==0 → e_msg "91001" (해당 자료 없음)
  │     i_isnext==1          → content_set_nkey(vKeyv) + e_msg "91002" (계속됩니다)
  │     그 외                → e_msg "91003" (조회 완료)
  ├─ zredit(mod->grid01_cnt, ii)
  ├─ content->sndl += (ii-1)*sizeof(orec[0])     orec 길이 보정
  └─ return RET_SUCCESS                          (no-data 포함 항상 SUCCESS — §6)
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG003M` | 외환중개사계좌기본 | `CMG003M` | **SELECT** (커서 `W1357S01_C1`) | `FDM_ID`/`USE_YN` 필터 + `FDM_ID‖FDM_ACN > :vKeyv` 페이징으로 외환중개사계좌 목록 조회 |

*DML 요약: 단일 테이블 SELECT 커서 1개(`W1357S01_C1`)뿐. JOIN/INSERT/UPDATE/DELETE/공통함수 경유 접근 없음. 조회 전용이나 규약상 성공 시 `l_dbcommit`.*

## 5. 로직 / 필터 규칙

- **DML**: 단일 커서 SELECT. 데이터 변경 없음. 성공 시 규약상 `l_dbcommit`, 실패 시 `l_dbrollback`.
- **동적 필터** (모두 커서 WHERE 절, `1=1` 베이스):
  - `FDM_ID = NVL(TRIM(:vFdmId), FDM_ID)` — `fdmId` 빈값이면 전체 외환중개사.
  - `USE_YN = NVL(:vUseYn, CMG003M.USE_YN)` — `useYn` 빈값이면 전체(단 Oracle `''`=NULL 전제).
  - `FDM_ID‖FDM_ACN > :vKeyv` — 페이징 커서(첫 조회 `vKeyv="0"`).
  - `ORDER BY FDM_ID, FDM_ACN`.
- **NEXT_KEY 페이징**: `FDM_ID‖FDM_ACN` 를 키로 사용. `max_row+1` 건째를 FETCH 하면 `i_isnext=1` 로 표시하고 저장 없이 break → `content_set_nkey`. 다음 페이지는 마지막 저장행 키(`vKeyv`) 초과분을 조회. **정상 동작(W1119 계열의 dead-code 페이징과 달리 여기선 `vKeyv` 가 SQL 에 바인딩됨).**
- **입력 유효성**: 3절 주석은 "입력항목 유효성 check" 이나 실제 `fdmId/useYn` 를 `yprintf/l_rtrim` 만 하고 NULL/필수 거부 로직 없음(§6). 필터가 모두 `NVL` 이라 빈 입력은 전체 조회로 귀결.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드** (`e_msg.code` → `fnSetMsg` → COMHDR `mesg`):
  - `91003` "조회가 완료되었습니다." (마지막 페이지), `eflg=0`.
  - `91002` "조회가 계속됩니다." (다음 페이지 존재), `eflg=0`, `nkey` 세팅.
  - `91001` "해당하는 자료가 없습니다." — **`RET_SUCCESS` 반환 → eflg=0, commit** (§6).
  - `10118` "자료 조회중 오류가 발생하였습니다." — 커서 OPEN/FETCH SQL 오류, `RET_FAILURE`(eflg=1).
  - "작업구분 오류." — `prGb≠'1'`, COMHDR `mesg` 직접 세팅(eflg 미세팅).
- **결과 전달**: grid 헤더(건수) + `orec[]` 반복 + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
