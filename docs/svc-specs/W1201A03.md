## 3. 구현 흐름 (`.pc`)

```
W1201A03()                      ← 진입 (콜백, 함수 주석은 "조회"지만 실제 dispatcher)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1201A03_F1(...)     ← 등록 (INSERT)
  │     '2' → retc = W1201A03_F2(...)     ← 삭제 (DELETE)
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  ├─ retc==RET_SUCCESS ? (eflg="0", l_dbcommit()) : (eflg="1", l_dbrollback())
  │                             (명시적 커밋, AUTOCOMMIT 아님)
  ├─ fnSetMsg(&e_msg, comhdr_o)  e_msg.code → COMHDR(rcod/mesg/pflg…) 편집
  └─ return content->sndl

W1201A03_F1()                   ← 등록
  ├─ getmid/getmod, memset(mod)
  ├─ 입력 바인딩: yprintf + l_rtrim → vCrcd, vFlctHlddYmd, vFxHlddNm, vComntCon, vFxSysLsmdId
  │              (vFdmId 도 바인딩하나 INSERT 미사용)
  ├─ 유효성: strlen(vCrcd)<=0 → e_msg "32029", RET_FAILURE
  │          strlen(vFlctHlddYmd)<=0 → e_msg "10736", RET_FAILURE
  ├─ EXEC SQL INSERT INTO TB_FXB_CMG011M
  │           (FDM_ID='FRX', CRCD, FLCT_HLDD_YMD, FX_HLDD_NM, COMNT_CON, FX_SYS_LSMD_ID, FX_SYS_LSMD_TS=SYSDATE)
  ├─ SQLCODE==DUP_KEY → e_msg "10223"(중복), RET_FAILURE
  │  SQLCODE!=0        → e_msg "10122"(처리 오류), RET_FAILURE
  └─ 정상 → e_msg "10150", RET_SUCCESS

W1201A03_F2()                   ← 삭제
  ├─ getmid/getmod, memset(mod)
  ├─ 입력 바인딩: yprintf + l_rtrim → vPrGb, vFdmId, vCrcd, vFlctHlddYmd
  ├─ 유효성: strlen(vCrcd)<=0 → e_msg "32029", RET_FAILURE
  │          strlen(vFlctHlddYmd)<=0 → e_msg "10736", RET_FAILURE
  ├─ EXEC SQL DELETE FROM TB_FXB_CMG011M
  │           WHERE FDM_ID='FRX' AND CRCD=:vCrcd AND FLCT_HLDD_YMD=:vFlctHlddYmd
  ├─ SQLCODE==NO_DATA_FOUND → e_msg "32029"(삭제대상 없음), RET_FAILURE
  │  SQLCODE!=0             → e_msg "10122"(처리 오류), RET_FAILURE
  └─ 정상 → e_msg "10150", RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG011M` | 통화별 변동휴일 (FX휴일) | `CMG011M` | **INSERT** (F1) / **DELETE** (F2) | 등록: `FDM_ID='FRX'` + 통화/변동휴일년월일/휴일명/코멘트/최종변경ID/SYSDATE 1행 INSERT. 삭제: `FDM_ID='FRX' AND CRCD AND FLCT_HLDD_YMD` 매칭 행 DELETE |

*DML 요약: `TB_FXB_CMG011M` INSERT 1문(F1) / DELETE 1문(F2). SELECT·UPDATE·커서·조인·공통함수 경유 접근 없음.*

## 5. 로직 / 필터 규칙

- **작업 분기**: `prGb[0]` — `'1'`=등록(INSERT), `'2'`=삭제(DELETE). 그 외는 dispatcher 에서 "작업구분 오류" 로 즉시 반환.
- **입력 유효성** (F1·F2 공통): `crcd` 공백 → `32029` 거부, `flctHlddYmd` 공백 → `10736` 거부. 그 외 필드 미검증.
- **DML**:
  - INSERT — 7컬럼 고정. `FDM_ID` 는 리터럴 `'FRX'`, `FX_SYS_LSMD_TS` 는 `SYSDATE`, 나머지는 바인드.
  - DELETE — `FDM_ID='FRX'` + 통화코드 + 변동휴일년월일 3조건 매칭. (PK = 이 3개로 추정)
- **트랜잭션**: dispatcher 에서 `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님 — 명시적 커밋 필수.
- **결과 전달**: 데이터 대신 COMHDR — `eflg`(0 정상/1 오류) + `e_msg.code`→(`fnSetMsg`)→`mesg`/`rcod`.
  - 성공: `10150` "정상처리 되었습니다. 조회로 확인하세요."
  - 실패: `32029`(통화코드 확인 / 삭제대상 없음) · `10736`(변동휴일년월일 확인) · `10223`(중복) · `10122`(처리 오류).
  - → WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, `errn`/COMHDR 본문 동봉(레거시 규약).
