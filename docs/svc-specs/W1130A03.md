# 서비스 명세 — `W1130A03` (로그아웃처리)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석 및 `.pc` 내부 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1130A03` |
| 이름 | 로그아웃처리 |
| 유형 | 갱신 (SELECT 확인 후 UPDATE) |
| 구현 경로 | `win/src/trn/W1100/W1130A03.pc` (388 라인) |
| I/O 헤더 | `win/src/inc/trn/W1130A03.h` |
| 모듈 | W1100 |
| 라우팅 alias | (미확인 — routing store 기준) |
| 채널 | 매매 엔진 svc (ChannelWeb via `/v1/tx`) |

작업구분(`prGb`)에 따라 로그인내역(TB_FXB_CSC015L)을 로그아웃 상태로 갱신한다. `prGb`가 '1'이면 일반 로그아웃, '2'면 보안설정 자리비움 대기시간 초과에 의한 로그아웃이다. 두 경우 모두 동일 함수 `W1130A03_F1()`을 `opt`(=prGb 문자) 인자로 호출한다.

## 2. I/O 전문 (`.h`)

**Input `W1130A03_I`** (총 121 byte)

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb`       | 1  | 처리구분 — '1' 일반 로그아웃 / '2' 자리비움 초과 로그아웃 |
| `fxUserNo`   | 30 | FX사용자번호 (필수 검증) |
| `fxUserNm`   | 40 | FX사용자명 (읽기만 하고 SQL/검증에 미사용) |
| `lgnIdntCon` | 50 | 로그인식별내용 (필수 검증, WHERE 키) |

**Output `W1130A03_O`** (총 1 byte)

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `dummy` | 1 | dummy — 실제 데이터 반환 없음. 결과는 COMHDR(`eflg`/`mesg`/`rtyp`)로만 전달 |

grid/orec 없음. `content->sndl`에 `W1130A03_O_SZ`(1)만 더해진다.

## 3. 구현 흐름 (`.pc`)

```
W1130A03()                                  // 콜백 진입
  ├ prthdr / getmid / getmod               // COMHDR·입출력 구조체 주소 확보
  ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp="M"
  └ switch(mid->prGb[0])
        case '1' → W1130A03_F1(opt='1')     // 일반 로그아웃
        case '2' → W1130A03_F1(opt='2')     // 자리비움 초과 로그아웃
        default  → comhdr_o->mesg="작업구분 오류.", return sndl (커밋/롤백 없이 종료)
  ├ retc == RET_SUCCESS → l_dbcommit()
  └ else               → l_dbrollback()
  return content->sndl

W1130A03_F1(opt)                            // 실제 처리
  ├ getmid / getmod, content->sndl += W1130A03_O_SZ, mod 0-clear
  ├ switch(content->keyc): K_NEXT/K_PREV/기본 → vKeyv 세팅 (※ 이후 미사용, dead)
  ├ 입력 정규화 yprintf + l_rtrim: vFxUserNo, vFxUserNm, vLgnIdntCon
  ├ 유효성:
  │    isNullSpace(vFxUserNo)==0   → e_msg.code=10555, eflg=1, RET_FAILURE
  │    isNullSpace(vLgnIdntCon)==0 → e_msg.code=10597, eflg=1, RET_FAILURE
  ├ [SELECT] TB_FXB_CSC015L → cLgotScd, cLgnScd(CASE LGOT_YMD IS NULL), cComentCon
  │    NO_DATA_FOUND → code=91001, eflg=0, RET_FAILURE
  │    기타 SQLERR  → code=10118, eflg=1, RET_FAILURE
  ├ if cLgnScd[0]=='0'  (이미 로그아웃된 상태)
  │    [UPDATE] LGOT_SCD='0', LGN_YN='N', COMNT_CON ||= ' 중복접속 정상로그아웃 완료.'
  │        NO_DATA_FOUND → code=91001,eflg=0,FAILURE / 기타 → code=10596,eflg=1,FAILURE
  │    성공 → eflg=0, code=91003, RET_SUCCESS  (여기서 리턴)
  └ else  (로그인 상태 → 로그아웃 수행)
       cComentCon = (opt!='2') ? "정상로그아웃" : "보안설정: 자리비움 대기시간 초과"
       [UPDATE] WHERE LGN_YN='Y' AND FX_USER_NO AND LGN_IDNT_CON
                SET LGOT_SCD='1', LGN_YN='N', COMNT_CON=:cComentCon
           NO_DATA_FOUND → code=91001,eflg=0,FAILURE / 기타 → code=10596,eflg=1,FAILURE
       성공 → eflg=0, code = (opt!='2') ? 91003 : 10699, RET_SUCCESS
```

트랜잭션은 상위 `W1130A03()`에서 F1 반환값 기준으로 commit/rollback. `default` 분기는 commit/rollback 을 거치지 않고 바로 반환한다.

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-------|-------|-----|-----|-----|
| `TB_FXB_CSC015L` | 로그인내역 | (없음) | SELECT | 현재 로그아웃/로그인 상태(LGOT_SCD, LGOT_YMD NULL 여부), 코멘트 확인 |
| `TB_FXB_CSC015L` | 로그인내역 | (없음) | UPDATE | 로그아웃 처리 — LGOT_YMD/HMS(SYSDATE), LGOT_SCD, LGN_YN='N', COMNT_CON, FX_SYS_LSMD_ID/TS |

*DML 요약: 단일 테이블 TB_FXB_CSC015L 에 대해 SELECT 1회 + 상태 분기에 따른 UPDATE 1회. 커서/INSERT/DELETE 없음.*

## 5. 로직 / 필터 규칙

- **작업구분(prGb)**: '1'=일반 로그아웃, '2'=자리비움 대기시간 초과 로그아웃. 둘 다 W1130A03_F1 호출, 그 외 값은 "작업구분 오류." 로 즉시 종료.
- **필수 입력 검증**: `fxUserNo`, `lgnIdntCon` 두 개만 검증(각각 e_msg.code 10555 / 10597). `fxUserNm`은 정규화만 하고 검증·SQL 미사용.
- **상태 판정**: SELECT 의 `LGN_SCD` = `CASE WHEN LGOT_YMD IS NULL THEN '1'(로그인) ELSE '0'(로그아웃)`.
  - `cLgnScd=='0'` (이미 로그아웃됨): 로그를 남기고도 오류 반환하지 않고, LGOT_SCD='0' 으로 재-UPDATE 후 코멘트에 " 중복접속 정상로그아웃 완료." 를 append, 성공(91003) 처리. WHERE 절에 `LGN_YN='Y'` 조건이 없어 FX_USER_NO+LGN_IDNT_CON 매칭 행 전부 갱신.
  - `cLgnScd=='1'` (로그인 상태): LGOT_SCD='1' 로 UPDATE, WHERE 에 `LGN_YN='Y'` 포함. 코멘트는 opt 에 따라 "정상로그아웃" 또는 "보안설정: 자리비움 대기시간 초과".
- **트랜잭션**: SQL 수행 후 상위 함수에서 명시적 COMMIT/ROLLBACK (AUTOCOMMIT 아님). RET_SUCCESS 만 commit.
- **에러(e_msg.code → 처리)**:
  | code | 상황 | eflg |
  |-----|-----|-----|
  | 10555 | FX사용자번호 미입력 | 1 |
  | 10597 | 로그인식별내용 미입력 | 1 |
  | 91001 | 대상 자료 없음 (SELECT/UPDATE NO_DATA_FOUND) | 0 |
  | 10118 | SELECT 오류 | 1 |
  | 10596 | UPDATE(로그인내역 갱신) 오류 | 1 |
  | 91003 | 로그아웃 정상처리 성공 | 0 |
  | 10699 | 자리비움 초과(opt='2') 정상처리 성공 | 0 |
- 성공 시에도 `fnSetMsg` 로 code(91003/10699)를 메시지로 세팅하며, rtyp='M'(메시지 전송)으로 응답.
