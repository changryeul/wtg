# 서비스 명세 — `W1201S01` (통화별주말조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며,
> `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|---|---|
| 서비스 코드 | `W1201S01` |
| 이름 | 통화별주말조회 (`.pc`/`.h` Description = "통화별주말조회") — 특정 통화의 요일구분코드(`DWK_DCD`) 목록 조회 |
| 유형 | **조회 (SELECT)** — 통화코드 1건으로 요일구분 다건(grid) 커서 조회 |
| 구현 | `win/src/trn/W1200/W1201S01.pc` (313 lines) |
| I/O 헤더 | `win/src/inc/trn/W1201S01.h` |
| 라우팅 alias | `W1201S01` → `POST /v1/tx {"alias":"W1201S01","data":{…}}` |
| 채널 | 진입부에 `ctyp→chnl` 매핑 없음 (COMHDR 그대로 사용) |

## 2. I/O 전문 (`.h`)

**Input `W1201S01_I`** (총 4B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" |
| `crcd` | 3 | 통화코드 — 조회 키 (필수, 공백이면 32029 오류) |

**Output `W1201S01_O`** (grid — `grid01_cnt` 6B + `orec[]` 가변)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (`ii`, `zredit` 우측정렬 숫자) |
| `orec[].crcd` | 3 | 통화코드 (`CMG009M.CRCD`) |
| `orec[].dwkDcd` | 1 | 요일구분코드 (`CMG009M.DWK_DCD`) |

> `.h` 는 `orec[1]` 로 선언(1건) 되어 있으나 실제 코드는 `max_row = MAX_RRB / sizeof(orec[0])` 만큼 반복 적재하는 **가변 grid** 다(§6). 레코드 1건 = 4B.

## 3. 구현 흐름 (`.pc`)

```
W1201S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' → retc = W1201S01_F1(...)      ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  ├─ retc==RET_SUCCESS ? eflg="0",l_dbcommit() : eflg="1",l_dbrollback()  (명시적 커밋)
  └─ fnSetMsg(&e_msg, comhdr_o)  → COMHDR 로 결과/메시지 반영 후 return(sndl)

W1201S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1201S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ GRID key 처리 (content->keyc)  K_NEXT→nkey / K_PREV→pkey / default→"0" → vKeyv, l_rtrim
  ├─ 입력 바인딩: yprintf + l_rtrim → vCrcd
  ├─ 유효성: strlen(vCrcd)<=0 → e_msg "32029", RET_FAILURE   (실제 검증 있음)
  ├─ DECLARE/OPEN 커서 W1201S01_C1
  │      SELECT CRCD, DWK_DCD, CRCD||DWK_DCD AS NEXT_KEY
  │        FROM TB_FXB_CMG009M
  │       WHERE FDM_ID='FRX' AND CRCD=:vCrcd
  │         AND FDM_ID||CRCD||DWK_DCD > :vKeyv
  │       ORDER BY FDM_ID, CRCD, DWK_DCD
  │      OPEN 실패: NO_DATA_FOUND→"91001" / 기타→"10118", 둘다 RET_FAILURE
  ├─ FETCH 루프 for(ii=0;;ii++)
  │      FETCH INTO cCrcd, cDwkDcd, cKeyv
  │        NO_DATA_FOUND → break / 기타 err → CLOSE, "10118", RET_FAILURE
  │      ii==max_row → i_isnext++, break            (lookahead 1건, 미적재)
  │      l_rtrim → zsedit mod->orec[ii].{crcd,dwkDcd}
  │      vKeyv = cKeyv (다음키 갱신)
  ├─ CLOSE 커서
  ├─ ii==0 && i_isnext==0 → e_msg "91001" (해당 자료 없음)
  │  else i_isnext==1 → content_set_nkey(vKeyv), e_msg "91002" (조회 계속)
  │       else        → e_msg "91003" (조회 완료), zredit(grid01_cnt, ii)
  ├─ zredit(mod->grid01_cnt, ii)                     (재설정, 중복)
  └─ sndl += (ii-1)*sizeof(orec[0]), return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG009M` | 통화별 요일/주말 구분 관리 (환율 요일구분, `FDM_ID='FRX'`) *(별칭명 기준 추정)* | `CMG009M` | **SELECT** (커서) | `FDM_ID='FRX' AND CRCD=:vCrcd` 로 해당 통화의 `CRCD`/`DWK_DCD` 다건 조회 |

*DML 요약: 커서 `W1201S01_C1` 1개로 `TB_FXB_CMG009M` **SELECT** 만. INSERT/UPDATE/DELETE 없음. 규약상 성공 시 `l_dbcommit`, 실패 시 `l_dbrollback`.*

## 5. 로직 / 필터 규칙

- **DML**: 단일 커서 SELECT(다건). 조인 없음. 데이터 변경 없음이나 규약상 성공 `l_dbcommit`/실패 `l_dbrollback` (AUTOCOMMIT 아님).
- **작업구분**: `switch(mid->prGb)` — `'1'` 만 `W1201S01_F1` 호출, 그 외 COMHDR `mesg="작업구분 오류."` 후 즉시 반환.
- **입력 유효성**: `vCrcd`(통화코드) 공백/NULL 이면 `32029` 오류로 `RET_FAILURE` (W1119 와 달리 실제 검증 존재).
- **필터**: `FDM_ID='FRX'` 고정 + `CRCD=:vCrcd`. 정렬 `FDM_ID, CRCD, DWK_DCD`.
- **페이징**: `content->keyc`(NEXT/PREV/최초)로 `vKeyv` 구성 → `AND FDM_ID||CRCD||DWK_DCD > :vKeyv` 로 이어받기 의도. 그러나 커서가 반환하는 NEXT_KEY(`cKeyv`)는 `CRCD||DWK_DCD` 로 **`FDM_ID` prefix 가 빠져** WHERE 비교식과 정렬 기준이 불일치 → NEXT 페이지가 정상 동작하지 않음(§6). 실무상 통화당 요일구분 건수가 적어 `max_row` 미도달로 표면화되지 않음.
- **lookahead**: `ii==max_row` 시 1건 더 FETCH 했음을 감지(`i_isnext++`)하고 해당 건은 적재/키갱신 없이 break → 계속 여부만 판정.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / else→l_dbrollback`.
- **에러/결과 코드**:
  - 성공(완료): `91003` "조회가 완료되었습니다.", `eflg=0`, `grid01_cnt=ii`.
  - 계속: `91002` "조회가 계속됩니다.", `content_set_nkey` 세팅.
  - 자료없음: `91001` "해당하는 자료가 없습니다." (fnSetMsg 후, `RET_SUCCESS` 로 반환 — rollback 아님).
  - 통화코드 미입력: `32029` (RET_FAILURE → rollback).
  - 커서 OPEN/FETCH SQL 오류: `10118` "자료 조회중 오류..." (RET_FAILURE).
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (eflg 미세팅).
- **결과 전달**: grid(`grid01_cnt`+`orec[]`) + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).

## 6. 주의 — 문서/코드 불일치 (마이그레이션 시 확인)

| 위치 | 헤더/주석 표기 | 실제(`.pc`) |
|---|---|---|
| `.h` L48 | `orec[1]` (1건) | 실제 `max_row = MAX_RRB/sizeof(orec[0])` 만큼 반복 적재하는 가변 grid |
| `.pc` L47–48 주석 | `mid`/`mod` 둘 다 "입력값 주소" | `mod` 는 출력(Output) 주소 — 복붙 오기 |
| 서비스명 "통화별주말조회" | 주말(weekend) 조회 | 실제 조회 대상은 `DWK_DCD`(요일구분코드) 목록 — 주말/요일 구분 정의 조회 (명칭 모호) |
| 커서 NEXT_KEY | `CRCD||DWK_DCD` | WHERE 는 `FDM_ID||CRCD||DWK_DCD > :vKeyv` 로 비교 → **`FDM_ID` prefix 누락**으로 NEXT 페이징 미동작 (건수 적어 표면화 X) |
| L216 OPEN 후 `NO_DATA_FOUND` 분기 | 커서 OPEN 시 무데이터 처리 | 통상 OPEN 은 NO_DATA 를 내지 않음 — 방어적 dead branch (무데이터는 첫 FETCH 에서 판정) |
| L302 & L308 `zredit(grid01_cnt,ii)` | — | 완료 분기에서 세팅 후 루프 밖에서 재세팅 — 중복(무해) |
| L280 로그 "다음데이터=cKeyv" | 다음 키 | NO_DATA 로 break 시 `cKeyv` 는 직전 memset 으로 공백 — 로그값 부정확(무해) |
| 미사용 지역변수 | — | `i, cnt, editb, ret, error_f, i_sndcnt, log_m, i` 등 다수 미사용 |

> WTG 는 `.h` 를 파싱해 필드명/크기만 취하고 전문을 통과시키므로 위 주석/dead code 가
> 동작에 영향 없다. 단 NEXT 페이징 미동작은 grid 건수가 `max_row` 를 초과할 경우
> 실제 결함이 되므로, 마이그레이션 시 NEXT_KEY 에 `FDM_ID` 를 포함하도록 교정 권장.
