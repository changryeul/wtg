## 2. I/O 전문 (`.h`)

**Input `W1701S04_I`** (총 22B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" |
| `screNo` | 20 | 검색어 — `FX_SCRE_NO`(화면번호) **및** `SCRE_NM`(화면명) 양쪽에 `LIKE '%…%'` (OR). 헤더는 "결재요청화면번호" 로 표기되나 실제는 화면번호/화면명 통합 검색어 (§6) |

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (`zredit(mod->grid01_cnt, ii)`) |
| `orec[]` | 174 반복 | 화면정보 레코드 배열 (그리드) |

|---|---|---|
| `screNo` | 20 | 화면번호 (`CMC011M.FX_SCRE_NO`) |
| `screNm` | 100 | 화면명 (`CMC011M.SCRE_NM`) |
| `prrmId` | 50 | 프로그램ID (`CMC011M.FX_PRRM_ID`) |
| `csmINLSYn` | 1 | 고객명포함여부 (`CMC011M.CSM_INLS_YN`) |
| `tpnInlsYn` | 1 | 전화번호포함여부 (`CMC011M.TPN_INLS_YN`) |
| `emlInlsYn` | 1 | 이메일포함여부 (`CMC011M.EML_INLS_YN`) |

## 3. 구현 흐름 (`.pc`)

```
W1701S04()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' → retc = W1701S04_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)

W1701S04_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1701S04_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])   (1행=174B, 한 페이지 최대 행수)
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT → content_get_nkey / K_PREV → content_get_pkey / default → "0"
  │      → vKeyv, l_rtrim  (실제 SQL 에 바인딩되어 페이징 동작함)
  ├─ 입력 바인딩: yprintfrtrim → vScreNo, vUseYn  (유효성 실검증 없음 — §6)
  ├─ EXEC SQL DECLARE W1701S04_C1 CURSOR
  │      SELECT DISTINCT FX_SCRE_NO, SCRE_NM, FX_PRRM_ID,
  │             CSM_INLS_YN, TPN_INLS_YN, EML_INLS_YN, ACN_INLS_YN
  │        FROM TB_FXB_CMC011M                                  (화면기본)
  │       WHERE (FX_SCRE_NO LIKE '%'||:vScreNo||'%'
  │              OR SCRE_NM LIKE '%'||:vScreNo||'%')
  │         AND USE_YN = NVL(:vUseYn, USE_YN)
  │         AND FX_SCRE_NO > :vKeyv                             (다음조회 Key)
  │       ORDER BY FX_SCRE_NO
  ├─ OPEN → SQLCODE≠SUCCESS 시:
  │      NO_DATA_FOUND → e_msg "91001", eflg "0", RET_FAILURE  (OPEN 에선 사실상 미발생 — §6)
  │      기타          → e_msg "10118", eflg "1", RET_FAILURE
  ├─ FETCH 루프 (ii=0;;ii++)
  │      NO_DATA_FOUND → break
  │      기타 err      → CLOSE + e_msg "10118", eflg "1", RET_FAILURE
  │      ii == max_row → i_isnext++, break   (초과 1행 감지 → 다음페이지 존재)
  │      → l_rtrim 후 mod->orec[ii] 세팅, vKeyv = cScreNo (NEXT KEY 갱신)
  ├─ CLOSE W1701S04_C1
  ├─ 결과 판정
  │      ii==0 && i_isnext==0 → e_msg "91001"(자료없음), eflg "0"
  │      i_isnext==1          → content_set_nkey(vKeyv), e_msg "91002"(계속), eflg "0"
  │      그외                  → e_msg "91003"(완료), eflg "0", grid01_cnt=ii
  ├─ [무조건] eflg "0", grid01_cnt=ii, fnSetMsg 재호출 (위 분기 일부 덮어씀 — §6)
  ├─ sndl += (ii-1) * sizeof(orec[0])
  └─ return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC011M` | 화면기본 | `CMC011M` | **SELECT** (커서 `W1701S04_C1`) | 화면번호/화면명 부분일치 + 사용여부 + NEXT KEY(`FX_SCRE_NO`) 조건으로 화면정보 목록 조회 |

## 5. 로직 / 필터 규칙

- **DML**: 커서 `W1701S04_C1` SELECT 1문뿐. 데이터 변경 없음이나 규약상 성공 시 `l_dbcommit`, 실패 시 `l_dbrollback`.
- **검색 필터**:
  - `screNo` 1개 값으로 `FX_SCRE_NO` 또는 `SCRE_NM` 를 `LIKE '%값%'` (OR) → 화면번호·화면명 통합 검색. 빈값이면 `LIKE '%%'` 로 전체.
  - `useYn` 은 `USE_YN = NVL(:vUseYn, USE_YN)` — 빈값(Oracle NULL) 이면 사용여부 무필터.
- **페이징**: `content->keyc`(NEXT/PREV/최초)로 `vKeyv` 결정, 커서 조건 `FX_SCRE_NO > :vKeyv` 에 실제 바인딩. `ORDER BY FX_SCRE_NO` 로 정렬. 한 페이지 `max_row = MAX_RRB / 174` 행. FETCH 가 `max_row` 번째(0-base 로 초과 1행)에 도달하면 저장하지 않고 `i_isnext=1` → `content_set_nkey(vKeyv)` 로 다음 Key 전달. (W1119S01 과 달리 페이징 실동작)
- **입력 유효성**: 3절 주석은 "입력항목 유효성 check" 이나 실제 `yprintfrtrim` trim 만 하고 NULL/공백 거부 없음(§6).
- **에러/결과 코드**:
  - `91003` 조회 완료, `eflg=0`, `grid01_cnt=ii`.
  - `91002` 조회 계속(다음페이지), `eflg=0`, NEXT KEY 세팅.
  - `91001` 해당 자료 없음, `eflg=0`.
  - `10118` 커서 OPEN/FETCH SQL 오류, `eflg=1`, RET_FAILURE.
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (eflg 미세팅).
- **결과 전달**: 그리드 레코드 + `grid01_cnt` + COMHDR(`eflg`/`e_msg.code`→`mesg`) + NEXT KEY. WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
