# 서비스 명세 — `W1115S01` (표준상품정보조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1115S01` |
| 서비스 이름 | 표준상품정보조회 (사용자 거래가능 표준상품 목록) |
| 유형 | 조회 (SELECT 전용, 그리드 + NEXT KEY 페이징) |
| 구현 경로 | `win/src/trn/W1100/W1115S01.pc` (407 라인) |
| I/O 헤더 | `win/src/inc/trn/W1115S01.h` |
| 라우팅 alias | `content->rkey` (= `W1115S01`) 기반, `/v1/tx` 통과 |
| 채널 | 매매 엔진 broker (`ChannelWeb`) — WTG `/v1/tx` passthrough |

작업구분(`prGb`)은 `'1'`(조회) 만 처리하며, 그 외 값은 "작업구분 오류." 를 반환한다.

## 2. I/O 전문 (`.h`)

### Input `W1115S01_I`

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 처리구분 — `'1'`=조회 만 유효 |
| `userId` | 8 | 유저ID — 커서 바인드 `:vUserId` (CSC005R.FX_USER_NO 매칭) |

### Output `W1115S01_O`

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `grid01_cnt` | 6 | 그리드 건수 (조회된 레코드 수 `ii`) |
| `orec[]` | 가변 | 상품 레코드 배열 — 헤더는 `orec[1]` 로 선언, 런타임에 `content->sndl` 을 `(건수-1)*sizeof(orec)` 만큼 확장하여 가변 반복 |

**`orec[]` 요소 (한 상품 = 102 바이트)**

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `fdmId` | 10 | 외환중개사ID (CMG002M.FDM_ID) |
| `fxPdcd` | 3 | FX상품코드 (CMG002M.FX_PDCD) |
| `crncPairId` | 8 | 통화페어ID (CMG002M.CRNC_PAIR_ID) |
| `stdTn` | 1 | STD_TN — 쿼리에서 상수 `'S'` 로 하드코딩 |
| `tnrId` | 3 | TENOR ID (CMG002M.TNR_ID) |
| `expiFnshYmd` | 8 | 만기종료년월일 (CMG006M.EXPI_FNSH_YMD) |
| `prvDt` | 8 | 이전일자 — 쿼리에서 상수 공백 `' '` 로 하드코딩 (미사용) |
| `nxtDt` | 8 | 다음일자 — 쿼리에서 상수 공백 `' '` 로 하드코딩 (미사용) |
| `trnAblYn` | 1 | 거래가능여부 (CMG002M.TRN_ABL_YN, 필터로 항상 `'Y'`) |
| `symbol` | 60 | Market Data Symbol — `FDM_ID‖FX_PDCD‖CRNC_PAIR_ID‖TNR_ID‖EXPI_FNSH_YMD` 연결 문자열 |

## 3. 구현 흐름 (`.pc`)

```
W1115S01()                         # 콜백 진입
 ├ prthdr / getmid / getmod        # COMHDR·입력·출력 주소 확보
 ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp="M"
 ├ switch(mid->prGb[0])
 │   ├ '1' → W1115S01_F1()         # 조회
 │   └ default → comhdr_o->mesg="작업구분 오류." + return(sndl)   # commit/rollback 없이 즉시 반환
 ├ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()
 └ return(content->sndl)

W1115S01_F1()                      # 조회
 ├ content->sndl += W1115S01_O_SZ ; memset(mod)
 ├ max_row = MAX_RRB / sizeof(orec[0])                    # 한 페이지 최대 건수
 ├ switch(content->keyc)                                  # 페이징 방향
 │   ├ K_NEXT → content_get_nkey → vKeyv
 │   ├ K_PREV → content_get_pkey → vKeyv
 │   └ 최초  → vKeyv = "0"
 ├ yprintf/l_rtrim → vUserId
 ├ EXEC SQL DECLARE ST_W1115S01_C1 CURSOR ...            # ROW_NUMBER() 서브쿼리 + ROW_NUM > :vKeyv
 ├ EXEC SQL OPEN ST_W1115S01_C1
 │   └ NO_DATA_FOUND → e_msg 91001 / eflg=0 / return(RET_FAILURE)
 ├ for(ii=0; ; ii++)
 │   ├ FETCH INTO cFdmId..cKeyv
 │   │   ├ NO_DATA_FOUND → break                          # 정상 종료
 │   │   └ else → CLOSE + e_msg 10118 / eflg=1 / return(RET_FAILURE)
 │   ├ if(ii==max_row){ i_isnext++; break; }              # 다음 페이지 존재
 │   ├ zsedit(mod->orec[ii].*)                            # 레코드 적재
 │   └ vKeyv = cKeyv (ROW_NUM)                            # NEXT KEY 갱신
 ├ EXEC SQL CLOSE ST_W1115S01_C1
 ├ if(ii==0 && i_isnext==0) → e_msg 91001 (자료없음)
 │  elif(i_isnext==1)       → content_set_nkey + e_msg 91002 (계속)
 │  else                    → e_msg 91003 (완료) + grid01_cnt=ii
 ├ comhdr_o->eflg="0" ; grid01_cnt=ii                     # 무조건 재설정
 ├ content->sndl += (ii-1) * sizeof(orec[0])
 └ return(RET_SUCCESS)
```

## 5. 로직 / 필터 규칙

- **커서 `ST_W1115S01_C1`**: 내부 서브쿼리(`W1115S02`)에서 `ROW_NUMBER() OVER (ORDER BY FDM_ID, FX_PDCD DESC, RPSN_SQC, EXPI_FNSH_YMD)` 로 순번을 매기고, 외부에서 `ROW_NUM > :vKeyv` 로 keyset 페이징한다.
- **조인/필터 조건**:
  - `CSC005R.FX_USER_NO = :vUserId` (요청 사용자)
  - `CSC001M.WBCS_RLNM_ALTR_NO = CSC005R.WBCS_RLNM_ALTR_NO`
  - `CSC001M.FX_CUS_STTS_DCD = '1'` (고객 상태 정상)
  - `CMG002M.TRN_ABL_YN = 'Y'` (거래가능 상품만)
  - `CMG002M.(FX_PDCD, CRNC_PAIR_ID, TNR_ID) = CMG006M.(...)`, `CMG006M.USE_YN = 'Y'`
  - `CMG002M.CRNC_PAIR_ID = CMG004M.CRNC_PAIR_ID`
  - **개인고객 제약**: `CASE WHEN CSC001M.CIF_DCD != '11' THEN 1 WHEN CMG002M.TNR_ID = 'TOD' THEN 1 ELSE 0 END = 1` — 개인고객(`CIF_DCD='11'`)은 `TOD` 테너만 조회 가능 (주석: "2017.06.28 개인고객은 TOD만 가능").
  - `FDM_ID = 'FRX'` 조건은 **주석 처리되어 비활성**.
- **페이징 방향**: `content->keyc` 가 `K_NEXT`/`K_PREV`/최초. 단 쿼리는 `ROW_NUM > :vKeyv` 단방향만 존재하므로 K_PREV 도 실질적으로 전진 조회로 동작(진정한 역방향 페이징 아님).
- **페이지 경계**: `max_row = MAX_RRB / sizeof(orec[0])`. `ii == max_row` 도달 시 마지막으로 FETCH 한 행은 **적재하지 않고** `i_isnext=1` 로 표시하며, NEXT KEY 는 직전(ii=max_row-1) 행의 ROW_NUM 이므로 다음 페이지에서 경계 행이 다시 조회된다(표준 keyset 패턴).
- **트랜잭션**: 상위 콜백이 `RET_SUCCESS` 면 `l_dbcommit`, 실패면 `l_dbrollback`. `prGb` default 분기는 commit/rollback 없이 즉시 반환.
- **에러/메시지 코드 (`e_msg.code` → `fnSetMsg`)**:
  | code | eflg | 의미 |
  |-----|------|-----|
  | `10118` | 1 | 커서 DECLARE/OPEN(비 NO_DATA)/FETCH SQL 오류 |
  | `91001` | 0 | 조회 자료 없음 (OPEN NO_DATA 또는 `ii==0`) |
  | `91002` | 0 | 조회 계속됨 (다음 페이지 존재, NEXT KEY 설정) |
  | `91003` | 0 | 조회 완료 (마지막 페이지) |
  - default 분기: `comhdr_o->mesg = "작업구분 오류."` (`e_msg.code` 미설정).
