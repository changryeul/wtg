## 2. I/O 전문 (`.h`)

**Input `W1359A01_I`** (헤더 정의: prGb 1 + wbcsRlnmAltrNo 16 + fxUserNo 30 + grid01_cnt 6 + irec[1]{62} = 115B, 단 irec 는 가변 grid)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 등록, `'3'` 삭제. (`'2'` 수정은 비활성 → default "작업구분 오류") |
| `wbcsRlnmAltrNo` | 16 | 전행고객번호(전행고객실명대체번호) — 마진그룹/펀드 매칭 키. 필수 |

| `grid01_cnt` | 6 | 그리드 건수 — 처리할 펀드(irec) 개수. `atoi` 로 정수화 후 루프 상한 |
| `irec[n]` | 62 × n | 펀드 그리드(가변). `.h` 는 `irec[1]` 로 선언하나 실제 `grid01_cnt` 만큼 반복 접근 |
| ├ `fndNo` | 12 | 펀드번호 (DB 바인딩 변수는 30자, §6) |
| └ `fndNm` | 50 | 펀드명 |

**Output `W1359A01_O`** (총 100B)

| 필드 | 크기 | 설명 |
|---|---|---|
| `dummy` | 100 | 미사용 placeholder. `memset(0)` 만 하고 값 채우지 않음. 실 결과는 COMHDR(`eflg`/`mesg`) 로 전달 |

> grid/orec 출력 없음. 성공/실패 전부 COMHDR 로 회신. 게다가 `content->sndl` 을 `W1359A01_O_SZ` 만큼 증가시키지 않으므로 출력 본문은 사실상 COMHDR 뿐(§6).

## 3. 구현 흐름 (`.pc`)

```
W1359A01()                        ← 진입 (콜백), 주석은 "조회" 이나 실제 등록/삭제 디스패처
  ├─ prthdr / getmid / getmod     COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정                ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1359A01_F1(...)     ← 등록 (upsert)
  │     '2' → (switch 케이스 주석처리 + F2 자체 #if 0) → 미도달, default 로
  │     '3' → retc = W1359A01_F3(...)     ← 삭제 (soft delete)
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return (eflg 미설정)
  └─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
        fnSetMsg(&e_msg, comhdr_o)  후 return content->sndl   (명시적 커밋, AUTOCOMMIT 아님)

W1359A01_F1()                     ← 등록 (등록/중복시 갱신)
  ├─ getmid/getmod, memset(mod)
  ├─ 입력 바인딩: usid→vFxSysLsmdId, wbcsRlnmAltrNo, fxUserNo (yprintf + l_rtrim)
  ├─ 유효성: vWbcsRlnmAltrNo 빈값→10201 / vFxUserNo 빈값→10197  (RET_FAILURE)
  ├─ SELECT TB_FXB_CMG020R  → 고객 기존 본점마진그룹(vHdomAplyGrpTcd/Id) 조회
  │      SQL err(≠NO_DATA) → 로그만 남기고 vHdomAplyGrpId="100"(디폴트) 로 진행
  │      (NO_DATA_FOUND 시엔 vHdomAplyGrpId 공백 그대로 — §6)
  ├─ for ii in [0, grid01_cnt)                     ← 펀드 grid 반복
  │     ├─ INSERT INTO TB_FXB_CSC011M (USE_YN='Y', ...)
  │     │     SQLCODE==DUP_KEY → UPDATE TB_FXB_CSC011M (동일 PK upsert)
  │     │           UPDATE 실패 → 10122 RET_FAILURE
  │     │     기타 SQL err     → 10122 RET_FAILURE
  │     └─ f_set_cusgrp(고객,펀드,그룹ID,…,s_chg_yn)  ← 본점마진그룹 고객매칭 설정
  │           실패 → RET_FAILURE / s_chg_yn=='Y' → s_chg_yn2='Y'
  ├─ if s_chg_yn2=='Y'  (그룹 변경 발생 시)
  │     ├─ fnCstBobMrgnLoad(고객)     ← 고객 본점마진 SHM 재적재 (실패시 로그만)
  │     └─ fnHdomPushSendQry(고객)    ← 조회TR 처리 PUSH 메시지 전송 (실패시 로그만)
  └─ e_msg="10150" RET_SUCCESS

W1359A01_F3()                     ← 삭제 (soft delete)
  ├─ getmid/getmod, memset(mod)
  ├─ 입력 바인딩: usid→vFxSysLsmdId, wbcsRlnmAltrNo, fxUserNo
  ├─ 유효성: vWbcsRlnmAltrNo 빈값→10201 / vFxUserNo 빈값→10197
  ├─ for ii in [0, grid01_cnt)
  │     ├─ UPDATE TB_FXB_CSC011M SET USE_YN='N' WHERE 고객+사용자+펀드
  │     │     NO_DATA_FOUND → 10326(삭제대상 없음) RET_FAILURE
  │     │     기타 SQL err  → 10122 RET_FAILURE
  │     └─ UPDATE TB_FXB_CMG020R SET GRP_FNAP_TS=SYSDATE (마진그룹 매칭 종료처리)
  │           SQL err(≠NO_DATA) → 로그만 (미중단)
  └─ e_msg="10150" RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CSC011M` | 사용자별 펀드 기본 | `CSC011M` | **INSERT / UPDATE** | F1: 펀드 등록(INSERT), 중복(DUP_KEY) 시 UPDATE upsert. F3: `USE_YN='N'` soft delete UPDATE |
| `TB_FXB_CMG020R` | 본점마진그룹 고객매칭 | — | **SELECT / UPDATE** | F1: 고객 기존 본점마진그룹(`HDOM_APLY_GRP_TCD/ID`) 조회. F3: `GRP_FNAP_TS=SYSDATE` 로 매칭 종료 (`WBCS_RLNM_ALTR_NO_FND_NO = 고객∥펀드`) |
| `TB_FXB_CMG020R` (간접) | 본점마진그룹 고객매칭 | — | **INSERT/UPDATE (추정)** | F1 의 `f_set_cusgrp()` 공통함수가 고객-펀드별 본점마진적용그룹을 설정 — 내부에서 CMG020R 갱신 (본 `.pc` 미노출) |

## 5. 로직 / 필터 규칙

- **작업 분기**: `prGb='1'`(등록)·`'3'`(삭제)만 동작. `'2'`(수정)은 switch case 와 함수 본체 모두 `#if 0`/주석으로 비활성 → default 로 떨어져 "작업구분 오류".
- **등록(F1) upsert**: `INSERT` 시 `DUP_KEY`(PK=FX_USER_NO+FND_NO 추정)면 동일 키 `UPDATE` 로 재활성(`USE_YN='Y'`). 즉 등록은 사실상 upsert.
- **삭제(F3) soft delete**: 물리 삭제 아님. `USE_YN='N'` UPDATE + 본점마진그룹 매칭 `GRP_FNAP_TS=SYSDATE`(적용 종료일 마감).
- **본점마진그룹 연계**: F1 은 등록 후 `f_set_cusgrp` 로 고객-펀드 마진그룹을 설정하고, 그룹 변경이 하나라도 있으면(`s_chg_yn2='Y'`) commit 전 함수 내에서 `fnCstBobMrgnLoad`(SHM 재적재) + `fnHdomPushSendQry`(조회TR PUSH) 를 호출. 두 후처리 실패는 로그만 남기고 RET_FAILURE 로 이어지지 않음.
- **grid 반복**: `grid01_cnt` 만큼 `irec[ii]` 를 순회. 하나라도 SQL 오류면 즉시 RET_FAILURE → 상위에서 전체 rollback (부분 커밋 없음).
- **입력 유효성**: `wbcsRlnmAltrNo`·`fxUserNo` 빈값만 거부(10201/10197). 펀드번호·그리드 건수 유효성 검증은 없음.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback` (AUTOCOMMIT 아님). `eflg` 는 상위 `W1359A01()` 에서 `0/1` 세팅.
- **에러/결과 코드**:
  - 성공: `10150` "정상처리 되었습니다. 조회로 확인하세요.", `eflg=0`.
  - 고객번호 빈값: `10201`, `eflg=1`.
  - 사용자번호 빈값: `10197`, `eflg=1`.
  - 자료 처리 오류(INSERT/UPDATE 실패): `10122`, `eflg=1`.
  - 삭제대상 없음(F3 NO_DATA): `10326`, `eflg=1`.
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (eflg 미세팅, `e_msg.code` 미세팅).
- **결과 전달**: 출력 레코드 없음. COMHDR(`eflg`/`e_msg.code`→`mesg`) 로만 회신. WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
