## 3. 구현 흐름 (`.pc`)

```
W1404A02()                        ← 진입 (콜백, 함수 주석 "조회"는 오기)
  ├─ prthdr / getmid / getmod     COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정                ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' | '2' → retc = W1404A02_F2(...)   ← 실제 로직(둘 다 동일 함수)
  │     그외      → comhdr_o->mesg="작업구분 오류." 후 즉시 return
  └─ retc==RET_SUCCESS ? (eflg="0", l_dbcommit()) : (eflg="1", l_dbrollback())
        → fnSetMsg(&e_msg, comhdr_o)  로 e_msg.code→mesg 반영

W1404A02_F2()                     ← 등록/변경(함수 주석 "등록수정")
  ├─ getmid/getmod, memset(mod)
  ├─ 입력 바인딩: XPRINTF + l_rtrim → v* 로컬, hls_upd_id=comhdr_i->usid
  ├─ 필수 유효성(공백 거부):
  │     wbcsRlnmAltrNo→"32017", fxSlactDcd→"10168", fxUserNo→"10744",
  │     fxStlmAcn→"10169", fxSlactYn→"10170"  (각각 RET_FAILURE)
  │
  ├─ [A] vPrGb=='1'  고객별 결제계좌 변경 -----------------------------------
  │     ├─ UPDATE TB_FXB_CSC003M (FX_SLACT_YN/COMNT_CON/FRCY_ACNT_PDF_DCD/수정자·시각)
  │     │     NO_DATA_FOUND → code "91001"(자료없음) 후 return RET_SUCCESS(=commit)
  │     │     기타 오류      → code "10122" RET_FAILURE
  │     ├─ UPDATE TB_FXB_CSC012M (FRCY_ACNT_PDF_DCD) — WTR05 주문용
  │     └─ if fxSlactYn=='Y':  (결제기본계좌 세팅 로직)
  │           ├─ SELECT TB_FXB_CSC003M (기본계좌 외 다른 계좌 존재?)
  │           ├─ 있으면 UPDATE TB_FXB_CSC003M SET FX_SLACT_YN='N' (기존 기본 해제)
  │           ├─ 고객약정 계좌 반영:
  │           │     원화('1') → UPDATE TB_FXB_CSC001M SET WON_ACN
  │           │     외화('2') → UPDATE TB_FXB_CSC001M SET FRCY_ACN(16자 절단)+PDF_DCD
  │           ├─ fnCustInfoQry(custInfo)         ← 간접 고객정보 조회
  │           ├─ fnSetCusAcnChange(stCusChg)     ← 계정계로 약정계좌변경(01) 전송
  │           └─ [실시간] CURSOR W1404A02_C1: SELECT FX_USER_NO FROM TB_FXB_CSC005R
  │                 (wbcsRlnmAltrNo 동일한 전체 FX_USER_NO — 본인 포함)
  │                 FETCH loop → fxSlactYn=='Y' 이면 fnFrxSendSetlAcnt("U") push
  │
  └─ [B] vPrGb!='1' (=='2') 이용자(펀드)별 결제계좌 -------------------------
        ├─ if fxSlactYn=='Y':  등록/변경
        │     ├─ SELECT FRCY_ACNT_PDF_DCD FROM TB_FXB_CSC003M
        │     ├─ UPDATE TB_FXB_CSC012M (FX_STLM_ACN/PDF_DCD/수정자·시각)
        │     ├─ UPDATE 가 NO_DATA_FOUND → INSERT TB_FXB_CSC012M (신규 등록)
        │     ├─ fnCustInfoQry(custInfo)
        │     ├─ SELECT 원화계좌(FX_SLACT_DCD='1') / 외화계좌('2') FROM TB_FXB_CSC012M
        │     └─ fnSetCusAcnChange(stCusChg)  변경처리구분 "02"(펀드계좌변경) 전송
        │            (이용자 계좌 없으면 custInfo.wonAcn/frcyAcn 로 대체)
        └─ else (fxSlactYn!='Y'):  삭제
              ├─ DELETE FROM TB_FXB_CSC012M (FX_USER_NO/FND_NO/wbcsRlnmAltrNo/DCD)
              │     NO_DATA_FOUND → code "10179"(등록된 계좌 없음), 미return(계속)
              └─ 삭제 성공 시: vFxSlactYn='Y' 강제 후
                    SELECT FX_STLM_ACN FROM TB_FXB_CSC003M (고객별 기본계좌로 대체)
                    → 3201 화면에 고객 기본결제계좌 push 하도록 준비

        [실시간] CURSOR W1404A02_C2: SELECT FX_USER_NO FROM TB_FXB_CSC012M
              (FND_NO + FX_SLACT_DCD + wbcsRlnmAltrNo 동일한 이용자 전체)
              FETCH loop → fxSlactYn=='Y' 이면 fnFrxSendSetlAcnt("U") push

  └─ Z. zredit(mod->grid01_cnt, 1)  (항상 1)
       if(e_msg.code != 0x00) code "10150"(정상처리)  ← 조건 항상 참(§6)
       return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CSC003M` | 고객별 결제계좌(고객결제계좌) | — | **SELECT / UPDATE** | 고객별 결제계좌 여부·코멘트·상품구분 UPDATE, 기존 기본계좌 해제(FX_SLACT_YN='N'), 기본계좌 SELECT (이용자 삭제 시 대체계좌 조회 포함) |

## 5. 로직 / 필터 규칙

- **작업구분(prGb)**: `'1'`=고객별, `'2'`=이용자(펀드)별. 두 값 모두 `W1404A02_F2` 하나로 처리하고 함수 내부에서 다시 `vPrGb[0]=='1'` 로 분기.
- **필수 입력 유효성** (공백/길이0 거부, 각각 `RET_FAILURE`):
  `wbcsRlnmAltrNo`→`32017`, `fxSlactDcd`→`10168`, `fxUserNo`→`10744`, `fxStlmAcn`→`10169`, `fxSlactYn`→`10170`.
- **고객별(`'1'`) 흐름**: CSC003M 결제여부/코멘트 UPDATE → CSC012M 상품구분 UPDATE → `fxSlactYn='Y'` 이면 기존 기본계좌 해제 + CSC001M(원/외화 약정계좌) 반영 + 계정계 전송(구분 "01") + CSC005R 커서로 동일 고객의 모든 FX_USER_NO 에 실시간 push.
  - 첫 CSC003M UPDATE 가 `NO_DATA_FOUND` → `91001`(자료 없음) 반환하지만 **`RET_SUCCESS` 이므로 commit** (rollback 아님).
- **이용자별(`'2'`) 흐름**:
  - `fxSlactYn='Y'`: CSC012M UPDATE, 갱신행 없으면(`NO_DATA_FOUND`) INSERT. 이후 원/외화 계좌 조회 → 계정계 전송(구분 "02", 이용자 계좌 없으면 custInfo 값 대체).
  - `fxSlactYn≠'Y'`: CSC012M DELETE. 삭제 후 `vFxSlactYn`를 강제로 `'Y'` 로 바꿔 고객별 기본계좌(CSC003M)를 조회해 실시간 push. 삭제 대상 없으면 `10179`.
  - 마지막에 CSC012M 커서(C2)로 동일 펀드/구분/고객의 FX_USER_NO 전체에 실시간 push.
- **트랜잭션**: `l_dbcommit`/`l_dbrollback` (AUTOCOMMIT 아님). `W1404A02_F2` 반환값이 `RET_SUCCESS` 면 commit, 그 외 rollback.
- **에러/메시지(e_msg.code → mesg)**: 입력검증 `32017/10168/10744/10169/10170`, 처리오류 `10122/10171/10172/10173/10745/10750`, 조회오류 `10118/10166`, 자료없음 `91001/10179`, push오류 `32142`, 정상 `10150`.
- **결과 전달**: `grid01_cnt` 는 항상 `1`. 실질 성공/실패는 COMHDR `eflg`(0/1) + `mesg`. → WTG `/v1/tx` 는 errn≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
