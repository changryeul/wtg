# 서비스 명세 — `W1111S01` (사용자정보조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1111S01` |
| 서비스 이름 | 사용자정보조회 |
| 유형 | 조회 (단건 SELECT INTO) |
| 구현 경로 | `win/src/trn/W1100/W1111S01.pc` (355 라인) |
| I/O 헤더 | `win/src/inc/trn/W1111S01.h` |

## 3. 구현 흐름 (`.pc`)

```
W1111S01()                         // 콜백 진입
  prthdr / getmid / getmod         // 공통헤더·입력·출력 주소 확보
  e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp="M"
  switch(mid->prGb[0])
    '1'    → W1111S01_F1()         // 조회
    default→ comhdr_o->mesg="작업구분 오류." + return  (commit/rollback 없이 반환)
  retc == RET_SUCCESS ? l_dbcommit() : l_dbrollback()
  return content->sndl

W1111S01_F1()                      // 조회
  getmid/getmod, content->sndl += O_SZ, memset(mod)
  max_row = MAX_RRB / O_SZ          // 계산만 하고 미사용
  switch(content->keyc)            // K_NEXT/K_PREV/최초 → vKeyv 세팅 (미사용)
  yprintf(vFxUserNo, mid->fxUserNo); l_rtrim
  출력 host 변수 memset
  EXEC SQL SELECT ... INTO ...      // 단건 (커서 아님)
    SQLCODE != 0 ?
      NO_DATA_FOUND → e_msg.code="91001", eflg="0", fnSetMsg, return RET_FAILURE
      기타          → e_msg.code="10118", eflg="1", fnSetMsg, return RET_FAILURE
  l_rtrim(각 컬럼) → zsedit(mod->*)  // 출력 전문에 채움
  eflg="0", e_msg.code="91003"(조회완료), fnSetMsg
  content->sndl += O_SZ             // ★ 두 번째 가산 (§6 참조)
  return RET_SUCCESS
```

트랜잭션: 조회 서비스지만 콜백 말미에 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 를 명시 호출 (AUTOCOMMIT 아님).

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `prGb == '1'` → 조회 F1. 그 외 값은 "작업구분 오류" 메시지 세팅 후 즉시 반환(commit/rollback 미수행).
- **단건 조회**: 커서/FETCH 없이 `SELECT ... INTO` 단건. F1 이름은 "조회"지만 grid 페이징(vKeyv, keyc, max_row)은 세팅만 하고 실제 사용하지 않는다 (§6).
- **조인/필터**:
  - `CSC005R.FX_USER_NO = CSC004M.FX_USER_NO`, `CSC005R.WBCS_RLNM_ALTR_NO = CSC001M.WBCS_RLNM_ALTR_NO` 로 3테이블 inner join.
  - `CSC001M.MNGM_BRCD = CMC001F_1.MNGM_BRCD(+)`, `CSC004M.OPN_BRCD = CMC001F_2.MNGM_BRCD(+)` outer join 으로 부점명.
  - `CSC004M.FX_USER_NO = :vFxUserNo` (입력), `CSC004M.USER_STTS_DCD = '1'` (활성 사용자만).
  - `ORDER BY CSC005R.WBCS_RLNM_ALTR_NO` — 단건 INTO 라 정렬 의미는 사실상 없음(다중행 시 too_many_rows 위험).
- **파생 컬럼**:
  - `screAcsAthrDcd` = `DECODE(RISK_AGRM_YN,'Y','Y','0') || FWEX_TRN_RNG_DCD` (리스크약정+선물환범위 합성).
  - `ctRgNdYn` = 이메일ID/도메인/휴대폰/발송매체 4개 concat 이 공백이면 `'Y'`(등록필요), 아니면 `'N'`.
- **에러 / 메시지** (`fnSetMsg(&e_msg, comhdr_o)`):
  | 상황 | e_msg.code | comhdr_o->eflg | 의미 |
  |-----|-----|-----|-----|
  | NO_DATA_FOUND | `91001` | `0` | 건수없음 (오류플래그는 정상 0 — §6) |
  | 기타 SQL error | `10118` | `1` | Select Error (비정상) |
  | 정상 | `91003` | `0` | 조회완료 |
- rtyp="M" (Message 전송) 으로 세팅되어 code 에 대응하는 메시지 문구를 클라이언트에 전송.
