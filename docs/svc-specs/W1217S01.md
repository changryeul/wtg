# 서비스 명세 — `W1217S01` (통화 조회 — 통화페어 등록 통화 팝업)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며,
> `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|---|---|

| 이름 | 통화 조회 (`.pc` Description = "통화 조회(코드조회 팝업)") — 통화페어기본에 등록된 통화만 코드조회 팝업용으로 나열 |
| 유형 | **조회 (SELECT)** — 커서 기반 grid 조회 (커밋/롤백은 수행하나 DML 없음) |
| 구현 | `win/src/trn/W1200/W1217S01.pc` (324 lines) |

## 2. I/O 전문 (`.h`)

**Input `W1217S01_I`** (총 4B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업(처리)구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" 후 즉시 return |
| `ccyId` | 3 | 통화ID — 조회 필터. 값 있으면 해당 통화 단건, 비었으면 전체(`NVL(:vCcyId, CRCD)`) |

**Output `W1217S01_O`** (base 39B = grid01_cnt 6 + orec 1개 33B, grid 가변)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (`zredit` 로 정수 문자열) |
| `orec[].ccyId` | 3 | 통화ID (`CMG005M.CRCD`) |
| `orec[].ccyNm` | 30 | 통화명 (`CMG005M.CRCD_NM`) |

> `.h` 는 `orec[1]` 로 선언되나 `.pc` 가 `max_row = MAX_RRB / sizeof(orec[0])` 만큼
> 반복 append 하는 가변 grid. `content->sndl` 을 `(ii-1)*sizeof(orec[0])` 만큼 늘려 실제 건수 반영.

## 3. 구현 흐름 (`.pc`)

```
W1217S01()                       ← 진입 (콜백)
  ├─ prthdr / getmid / getmod    COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1217S01_F1(...)      ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return(sndl)
  │            (eflg 미설정, MSG_NOWORK 로그)
  ├─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님; DML 은 없음)
  └─ fnSetMsg(&e_msg, comhdr_o) → return(content->sndl)

W1217S01_F1()                    ← 조회
  ├─ getmid/getmod, sndl += W1217S01_O_SZ, memset(mod), max_row = MAX_RRB/sizeof(orec[0])
  ├─ GRID key 처리 (content->keyc)  K_NEXT→content_get_nkey / K_PREV→content_get_pkey / default→"0"
  │      → vKeyv 세팅 후 l_rtrim   (계산만 하고 실제 SQL WHERE 엔 미사용 — §6 페이징 미동작)
  ├─ 입력 바인딩: yprintf + l_rtrim → vCcyId  (유효성 실검증 없음, 로그만)
  ├─ EXEC SQL DECLARE W1217S01_C1 CURSOR FOR
  │      (TB_FXB_CMG005M ⨝ TB_FXB_CMG004M) UNION 서브쿼리 → CRCD/CRCD_NM/RPSN_SQC
  │      WHERE CRCD = NVL(:vCcyId, CRCD)  ORDER BY RPSN_SQC
  ├─ OPEN → SQLCODE!=SUCCESS 분기
  │      NO_DATA_FOUND → e_msg "91001", eflg "0", RET_FAILURE
  │      기타 err      → e_msg "10118", eflg "1", RET_FAILURE
  ├─ for(ii=0;;ii++) FETCH INTO :cCcyId,:cCcyNm
  │      NO_DATA_FOUND → break
  │      기타 err      → CLOSE, e_msg "10118", eflg "1", RET_FAILURE
  │      ii==max_row   → i_isnext++, break (다음 페이지 존재 표시)
  │      → l_rtrim 후 mod->orec[ii].{ccyId,ccyNm} 세팅
  │      → sprintf(vKeyv,"%.64s", cKeyv)  (cKeyv 미할당 — §6 NEXT KEY 버그)
  ├─ CLOSE
  ├─ (ii==0 && i_isnext==0) → e_msg "91001"(자료없음), eflg "1", RET_FAILURE
  │  else i_isnext==1 → content_set_nkey(vKeyv), e_msg "91002"(계속)
  │       else        → e_msg "91003"(완료), grid01_cnt=ii
  └─ eflg "0", grid01_cnt=ii, sndl += (ii-1)*sizeof(orec[0]), RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG005M` | 통화기본 | `CMG005M` | **SELECT** | 통화코드(`CRCD`)/통화명(`CRCD_NM`)/대표순번(`RPSN_SQC`) 조회. 정렬 기준 |
| `TB_FXB_CMG004M` | 통화페어기본 | `CMG004M` | **SELECT** | 통화페어에 base(`BASE_CRCD`) 또는 상대(`COCU_CD`)로 등록된 통화만 필터 (통화기본과 outer join + UNION) |

*DML 요약: 커서(`W1217S01_C1`) SELECT 1문뿐. INSERT/UPDATE/DELETE 없음. `l_dbcommit/l_dbrollback` 은 관례상 호출되나 변경 대상 없음.*

## 5. 로직 / 필터 규칙

- **필터**: `WHERE CRCD = NVL(:vCcyId, CRCD)` — `ccyId` 입력 시 단건, 미입력 시 전체.
- **대상 통화 산정**: `CMG004M.BASE_CRCD = CMG005M.CRCD(+)` 서브쿼리와
  `CMG004M.COCU_CD = CMG005M.CRCD(+)` 서브쿼리를 **UNION** — 통화페어의 base 통화와 상대 통화를 합집합으로 모아, 페어에 실제 쓰이는 통화만 노출.
- **정렬**: `ORDER BY RPSN_SQC` (대표순번).
- **페이징 (설계상 존재, 실동작 X)**: `content->keyc` 로 NEXT/PREV key 를 읽어 `vKeyv` 를 만들지만 SQL WHERE 에 바인딩하지 않고, NEXT KEY 로 세팅하는 `cKeyv` 는 fetch 되지 않는 미할당 변수 → 페이징 미동작 (§6).
- **max_row**: `MAX_RRB / sizeof(orec[0])` 초과 시 `i_isnext` 세워 `91002`(계속) 반환. 커서 컬럼에 key 가 없어 다음 페이지를 재조회할 수단은 실제로 없음.
- **트랜잭션**: `RET_SUCCESS` 면 `l_dbcommit`, 아니면 `l_dbrollback` (조회 전용이라 실질 효과 없음).
- **에러 (e_msg.code → mesg, eflg)**:
  - `91001` 자료없음 — OPEN NO_DATA 시 `eflg="0"`, 최종 ii==0 시 `eflg="1"` (§6 불일치)
  - `10118` 조회중 오류 (`eflg="1"`)
  - `91002` 조회 계속 / `91003` 조회 완료 (정상, `eflg="0"`)
