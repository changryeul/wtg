## 3. 구현 흐름 (`.pc`)

```
W1404S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' / '2' → retc = W1404S01_F1(...)   ← 실제 로직 (두 케이스 동일 호출)
  │     그외      → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │                (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? eflg="0",l_dbcommit() : eflg="1",l_dbrollback()
     → fnSetMsg(&e_msg, comhdr_o)  (e_msg.code → COMHDR 메시지)

W1404S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1404S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ GRID key: content->keyc  K_NEXT→content_get_nkey / K_PREV→content_get_pkey
  │            / default→"0"  → vKeyv, l_rtrim
  ├─ 입력 바인딩: yprintfrtrim → vPrGb / vWbcsRlnmAltrNo / vFxUserNo / vFndNo
  │            (COMHDR usid/loip/scrn 도 추출하나 SQL 미사용 — §6)
  ├─ DECLARE W1404S01_C1 CURSOR  (UNION ALL 2블록)
  │     블록1(펀드이용자, '2'=:vPrGb): CSC003M ⋈ CSC001M
  │            LEFT JOIN CSC012M(이용자별 결제계좌기본, vFxUserNo/vFndNo 매칭)
  │            LEFT JOIN CMC003D×2 → FX_SLACT_YN = CSC012M 존재여부
  │     블록2(일반이용자, '1'=:vPrGb): CSC003M ⋈ CSC001M
  │            LEFT JOIN CMC003D×2 → FX_SLACT_YN = CSC003M.FX_SLACT_YN
  │     공통 필터: FX_SLACT_DCD IN ('1','2'), WBCS_RLNM_ALTR_NO=:vWbcsRlnmAltrNo,
  │              STLM_ACSS_DCD='1'(정상계좌만)
  │     외부: WHERE FX_SLACT_DCD||FX_STLM_ACN > :vKeyv  ← keyset 페이징
  │           ORDER BY FX_SLACT_DCD, FX_SLACT_YN DESC
  ├─ OPEN → SQLCODE NO_DATA → e_msg "91001", RET_SUCCESS 반환
  │         기타 err → e_msg "10118", RET_FAILURE
  ├─ for(ii=0;;ii++) FETCH
  │     NO_DATA → break
  │     기타 err → CLOSE, e_msg "10118", RET_FAILURE
  │     ii==max_row → i_isnext++, break (이 행은 미저장 = 다음 존재 표식)
  │     zsedit 로 orec[ii] 전 필드 세팅
  │     cKeyv = FX_SLACT_DCD[0]+FX_STLM_ACN
  │     fxStlmAcn ≠ 공백 → fnAcntConv 로 표시형 변환하여 orec[ii].fxStlmAcn 재세팅
  │     vKeyv = cKeyv (NEXT KEY 갱신)
  ├─ CLOSE W1404S01_C1
  ├─ ii==0 && i_isnext==0 → e_msg "91001" (자료 없음)
  │  i_isnext==1          → content_set_nkey(vKeyv), e_msg "91002" (계속)
  │  그외                  → e_msg "91003" (완료), grid01_cnt=ii
  └─ grid01_cnt=ii (재세팅), sndl += (ii-1)*sizeof(orec[0]), RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CSC003M` | 고객계좌정보기본 | `CSC003M` | **SELECT** | 결제계좌 원장 (계좌구분/번호/통화/개설일/상태/코멘트 등). `FX_SLACT_DCD IN ('1','2')`, `STLM_ACSS_DCD='1'`, 고객번호 필터. UNION ALL 양 블록 공통 |

| `TB_FXB_CSC012M` | 이용자별 결제계좌기본 | `CSC012M` | **SELECT** | 블록1(`prGb='2'`)만 LEFT JOIN. `WBCS_RLNM_ALTR_NO/FX_SLACT_DCD/FX_STLM_ACN` + `FX_USER_NO=:vFxUserNo` + `FND_NO=:vFndNo` 매칭 → 존재 여부로 FX_SLACT_YN 산출 |
| `TB_FXB_CMC003D` | 코드정보상세 | `CMC003D1`,`CMC003D2` | **SELECT** | LEFT JOIN ×2. `COM_CD_NM='FX_SLACT_DCD'`(구분코드명), `='STLM_ACSS_DCD'`(상태코드명) 디코드 |

*DML 요약: SELECT 전용 커서 1개(`W1404S01_C1`, UNION ALL 2블록·최대 4테이블 조인). INSERT/UPDATE/DELETE 없음. 데이터 변경 없으나 규약상 성공 시 `l_dbcommit`. `fnAcntConv`(계좌번호 표시형 변환)는 테이블 미접근 유틸.*

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `prGb='1'`(일반이용자)·`'2'`(펀드이용자) 둘 다 동일하게 `W1404S01_F1` 호출. 실제 분기는 SQL 내부 `'1'=:vPrGb` / `'2'=:vPrGb` 상수 비교로 UNION ALL 블록 중 하나만 활성.
- **FX_SLACT_YN(결제계좌여부) 산출식**:
  - `prGb='2'`(블록1): `CSC012M.FX_STLM_ACN IS NULL → 'N'/'부'`, 아니면 `'Y'/'여'` (이용자별 등록 계좌를 결제계좌로 판정).
  - `prGb='1'`(블록2): `CSC003M.FX_SLACT_YN` 원값 사용, `'N'→'부'` 그외 `'여'`.
- **디코드**: `FX_SLACT_DCD_NM`/`STLM_ACSS_DCD_NM` 은 `NVL(CMC003D.CMCD_VL_NM, 원코드)`. `HOST_ACNT_EXSN_YN_NM`/`TRN_USE_YN_NM` 은 CASE 인라인.
- **정렬/페이징**: `ORDER BY FX_SLACT_DCD, FX_SLACT_YN DESC`. keyset 커서(`FX_SLACT_DCD||FX_STLM_ACN > :vKeyv`) → NEXT/PREV 실동작. 한 페이지 최대 `max_row=MAX_RRB/sizeof(orec[0])` 건, 초과 시 `i_isnext` 로 다음 페이지 표식.
- **계좌번호 변환**: 저장 전 `fnAcntConv` 로 표시형 변환하나, NEXT KEY(`cKeyv`)는 변환 전 원 계좌번호로 산출 → 커서 WHERE 와 정합.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드 (e_msg.code → fnSetMsg)**:
  - `91003` 조회 완료 (`eflg=0`).
  - `91002` 조회 계속 (다음 페이지 존재, `content_set_nkey`).
  - `91001` 자료 없음 — 커서 OPEN NO_DATA(RET_SUCCESS 반환) 또는 종료 시 `ii==0 && i_isnext==0`.
  - `10118` 커서 OPEN/FETCH SQL 오류 (`RET_FAILURE`).
  - "작업구분 오류." COMHDR mesg (진입부, eflg 미세팅).
- **결과 전달**: grid(헤더+orec N) + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
