# 서비스 명세 — `W1702S01` (사용자조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1702S01` |
| 서비스 이름 | 사용자조회 (직원번호 단건 조회) |
| 유형 | 조회 (단건 SELECT INTO) |
| 구현 | `win/src/trn/W1700/W1702S01.pc` (286줄) |
| I/O 헤더 | `win/src/inc/trn/W1702S01.h` |
| 라우팅 alias | `W1702S01` |
| 채널 | 매매(ChannelWeb) — `/v1/tx` 경유 |

지정한 직원번호(`FX_USER_NO`)로 사용자정보기본(CSC004M) 1건을 조회하고, 외환중개사 계좌정보(CMG059M)를 LEFT OUTER JOIN 하여 SMB/KMB/EBS 거래소계좌번호를 함께 돌려주는 단건 조회 서비스다.

## 3. 구현 흐름 (`.pc`)

```
W1702S01()                       # 진입/dispatch
  prthdr / getmid / getmod       # COMHDR·mid·mod 주소 획득
  e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  switch(mid->prGb[0])
    '1' -> W1702S01_F1()         # 조회
    default -> comhdr_o.mesg "작업구분 오류." + return (분기 미커밋)
  retc == RET_SUCCESS ? l_dbcommit() + eflg="0"
                      : l_dbrollback() + eflg="1"
  fnSetMsg(&e_msg, comhdr_o)     # e_msg.code -> 응답 메시지 세팅
  return content->sndl

W1702S01_F1()                    # 조회 본체
  getmid / getmod
  content->sndl += W1702S01_O_SZ   # (주의: §6 — 여기 + 하단에서 2회 가산)
  memset(mod, 0, O_SZ)
  switch(content->keyc)          # NEXT KEY 수집 (K_NEXT/K_PREV/최초) — vKeyv (사용 안 됨, §6)
  yprintfrtrim(vEmn, mid->emn)   # 직원번호 추출·rtrim
  if strlen(vEmn)==0 -> e_msg.code "10416" + return RET_FAILURE   # 직원번호 필수
  EXEC SQL SELECT ... INTO ...   # 단건 조회 (CSC004M LEFT JOIN CMG059M)
    SQLCODE != SUCCESS
      NO_DATA_FOUND -> e_msg.code "91001" + return RET_SUCCESS    # (주의: 성공 리턴 -> commit)
      else          -> e_msg.code "90000" + return RET_FAILURE    # 시스템오류 -> rollback
  l_rtrim(각 컬럼) -> zsedit(mod->*)   # 결과 mod 채움
  e_msg.code "91003"             # 정상
  content->sndl += W1702S01_O_SZ  # (주의: 2회째 가산 — §6)
  return RET_SUCCESS
```

트랜잭션: 조회 전용이나 상위 함수가 `RET_SUCCESS` 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 을 명시 호출한다(AUTOCOMMIT 아님).

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `prGb='1'` 만 `W1702S01_F1` 로 조회. 그 외 값은 `comhdr_o.mesg="작업구분 오류."` 세팅 후 즉시 return (커밋/롤백 트리 미도달).
- **입력 검증**: `emn`(직원번호) 이 공백이면 `e_msg.code="10416"` 세팅 후 `RET_FAILURE` → 상위에서 rollback + `eflg="1"`.
- **조회 필터** (고정, 동적 아님):
  - `CSC004M.FX_USER_NO = TRIM(:vEmn)`
  - `CSC004M.USER_STTS_DCD = '1'` (사용자상태구분코드 정상)
  - JOIN 조건: `CSC004M.FX_USER_NO = CMG059M.FX_EMP_NO AND CMG059M.USE_YN = 'Y'` (LEFT OUTER — 중개사 계좌 없어도 사용자정보는 반환)
- **에러/메시지 코드** (`e_msg.code` → `fnSetMsg` 로 응답 mesg 변환):
  | code | 의미 | 리턴 | 후처리 |
  |------|------|------|--------|
  | `10416` | 직원번호 입력값 없음 | RET_FAILURE | rollback, eflg=1 |
  | `91001` | 해당 데이터 없음(NO_DATA_FOUND) | RET_SUCCESS | commit, eflg=0 |
  | `90000` | 시스템오류(그 외 SQLCODE) | RET_FAILURE | rollback, eflg=1 |
  | `91003` | 정상 조회 | RET_SUCCESS | commit, eflg=0 |
- **페이징**: `content->keyc`(K_NEXT/K_PREV/최초)로 `vKeyv` 를 수집하나, SELECT 문에 페이징/키 조건이 전혀 없어 실제로는 사용되지 않는다(단건 조회이므로 무의미 — §6).
