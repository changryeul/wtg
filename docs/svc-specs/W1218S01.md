## 3. 구현 흐름 (`.pc`)

```
W1218S01()                       ← 진입 (콜백)
  ├─ getmid / getmod            COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              pname=rkey, ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  │                             comhdr_o->rtyp = "M"
  ├─ retc = W1218S01_F1(...)     ← 실제 조회 로직 (작업구분 분기 없음, 단일 조회)
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)

W1218S01_F1()                    ← 조회
  ├─ getmid/getmod, sndl += W1218S01_O_SZ, memset(mod)
  │  max_row = MAX_RRB / sizeof(orec[0])            페이지당 최대 행
  ├─ GRID 키 처리 (content->keyc)
  │      K_NEXT → content_get_nkey(nextKey)
  │      K_PREV → content_get_pkey(nextKey)         (NEXT 와 동일 커서식 사용 — §6)
  │      default(최초) → nextKey = "99991231ZZZZZZ" (내림차순 조회 상한)
  │      l_rtrim(nextKey)
  ├─ 입력 바인딩: yprintf + l_rtrim → iBsopYmdFrom/iBsopYmdTo/iBtchId/iStatTp
  ├─ 유효성: len(iBsopYmdFrom)<8 → e_msg "10160", eflg "1", RET_FAILURE
  │          len(iBsopYmdTo)  <8 → e_msg "10160", eflg "1", RET_FAILURE
  ├─ EXEC SQL DECLARE TB_FXB_STG006L_C1 CURSOR   (커서명은 STG006L 이나 실제 대상은 SYG002G — §6)
  │      SELECT … FROM TB_FXB_SYG002G SYG002G,
  │                    TB_FXB_CMC003D CMC003D  (ERR_LVL_DCD 코드명 outer join),
  │                    TB_FXB_CMC003D CMC003D1 (BTCH_ID 코드명 outer join)
  │      WHERE JBEX_YMD BETWEEN :from AND :to
  │        AND TRIM(BTCH_ID) LIKE '%'||TRIM(:iBtchId)||'%'
  │        AND ERR_LVL_DCD = NVL(TRIM(:iStatTp), ERR_LVL_DCD)   (빈값=전체)
  │        AND JBEX_YMD||JBEX_SQN < :nextKey                    (페이징 커서)
  │      ORDER BY JBEX_YMD DESC, JBEX_SQN DESC
  │      DECLARE 실패 → e_msg "10118", eflg "1", RET_FAILURE
  ├─ OPEN
  │      NO_DATA_FOUND → e_msg "91001", eflg "0" (return 하지 않고 진행)
  │      기타 err      → e_msg "10118", eflg "1", RET_FAILURE
  ├─ for(ii=0; ; ii++)  FETCH … INTO 호스트변수 18개
  │      NO_DATA_FOUND → break
  │      기타 err      → CLOSE + e_msg "10118", eflg "1", RET_FAILURE
  │      ii == max_row → i_isnext++; break   (초과 1행으로 다음 페이지 유무 판정, 미저장)
  │      → l_rtrim 후 zsedit 로 mod->orec[ii] 채움
  │      → nextKey = jbexYmd + jbexSqn        (마지막 저장 행 키)
  ├─ CLOSE TB_FXB_STG006L_C1
  ├─ 결과 판정
  │      ii==0 && i_isnext==0 → e_msg "91001"(자료없음), eflg "0"
  │      i_isnext==1          → content_set_nkey(nextKey), e_msg "91002"(계속), eflg "0"
  │      그외                 → e_msg "91003"(완료), eflg "0", grid01_cnt=ii
  ├─ fnSetMsg(&e_msg, comhdr_o); eflg="0"; grid01_cnt=ii   (무조건 재설정)
  └─ content->sndl += (ii-1) * sizeof(orec[0]);  RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_SYG002G` | 배치로그 | `SYG002G` | **SELECT** (커서) | 영업일 구간·배치ID·에러레벨 조건으로 배치로그 본문 조회 (main) |
| `TB_FXB_CMC003D` | 코드정보상세(공통코드) | `CMC003D` | **SELECT** (outer join) | `COM_CD_NM='ERR_LVL_DCD' AND CMCD_VL=ERR_LVL_DCD` 로 에러레벨코드명(`errLvlDcdNm`) 매핑 |
| `TB_FXB_CMC003D` | 코드정보상세(공통코드) | `CMC003D1` | **SELECT** (outer join) | `COM_CD_NM='BTCH_ID' AND CMCD_VL=BTCH_ID` 로 배치ID명(`btchIdNm`) 매핑 |

*DML 요약: 단일 커서(`TB_FXB_STG006L_C1`) SELECT 1문 — `TB_FXB_SYG002G` 1회 + `TB_FXB_CMC003D` 2회 self-outer-join. INSERT/UPDATE/DELETE 없음. 데이터 변경 없으나 규약상 성공 시 `l_dbcommit`.*

## 5. 로직 / 필터 규칙

- **DML**: 조회 전용. `DECLARE → OPEN → FETCH(loop) → CLOSE` 단일 커서. 실제 변경 없음.
- **동적 필터**:
  - `JBEX_YMD BETWEEN :from AND :to` — 필수 8자리 영업일 구간.
  - `TRIM(BTCH_ID) LIKE '%'||TRIM(:iBtchId)||'%'` — 배치ID 부분일치(빈값이면 `%%` = 전체).
  - `ERR_LVL_DCD = NVL(TRIM(:iStatTp), ERR_LVL_DCD)` — 입력 있으면 해당 에러레벨, 빈값이면 자기 컬럼과 비교 = 전체.
- **페이징 (커서식)**: `JBEX_YMD||JBEX_SQN < :nextKey` + `ORDER BY … DESC`. 최초는 `nextKey="99991231ZZZZZZ"`(상한). 매 저장 행마다 `nextKey`를 갱신. `ii==max_row` 시 초과 1행을 fetch 하여 다음 페이지 존재 판정(`i_isnext`) 후 저장 없이 break. 다음 페이지는 `content_set_nkey` 로 클라이언트에 전달, `K_NEXT` 재요청 시 `content_get_nkey` 로 회수.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드** (`e_msg.code` → `fnSetMsg` → `comhdr_o->mesg`, `eflg`):
  - `10160` 조회일자 확인 (from/to 길이<8), `eflg=1`, RET_FAILURE.
  - `10118` 커서 DECLARE/OPEN(기타)/FETCH(기타) 오류, `eflg=1`, RET_FAILURE.
  - `91001` 해당 자료 없음(0건), `eflg=0`.
  - `91002` 조회 계속(다음 페이지 있음), `eflg=0`, `content_set_nkey`.
  - `91003` 조회 완료, `eflg=0`, `grid01_cnt` 세팅.
- **결과 전달**: 그리드(`grid01_cnt`+`orec[]`) + COMHDR(`eflg`/`mesg`). WTG `/v1/tx` 는 errn≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
