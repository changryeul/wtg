# 서비스 명세 — `W1202S01` (시장환율인터페이스)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1202S01` |
| 서비스 이름 | 시장환율인터페이스 (시장환율 고시 조회) |
| 유형 | 조회 (grid, 페이징) — 실 DML 은 SELECT 전용 |
| 구현 | `win/src/trn/W1200/W1202S01.pc` (586줄) |
| I/O 헤더 | `win/src/inc/trn/W1202S01.h` |
| 라우팅 alias | `W1202S01` |
| 채널 | `ChannelWeb` (`mci-api` `/v1/tx` 경유 broker → trn AP) |

시장환율 고시 테이블(`TB_FXB_CMG014F`)을 통화쌍·통화 마스터와 조인해 고시년월일/회차 기준 환율 리스트를 그리드로 반환한다. 처리구분(`prGb`) `1`/`2` 로 조회 회차 산정 방식만 달라지며 두 값 모두 조회(`W1202S01_F1`)로 분기한다.

## 3. 구현 흐름 (`.pc`)

```
W1202S01()                              // 콜백 진입
  prthdr / getmid / getmod             // 공통헤드 + I/O 주소 획득
  e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  switch(mid->prGb[0])
    case '1' / '2' → W1202S01_F1()      // 조회 (둘 다 동일 진입)
    default        → comhdr_o.mesg="작업구분 오류." return  // 커밋/롤백 없이 종료
  retc==RET_SUCCESS ? eflg="0", l_dbcommit() : eflg="1", l_dbrollback()
  fnSetMsg(&e_msg, comhdr_o)           // e_msg.code → mesg/rcod 편집
  return content->sndl

W1202S01_F1()                           // 조회
  getmid/getmod, sndl += O_SZ, memset(mod)
  max_row = MAX_RRB / sizeof(orec[0])   // 페이지 크기
  switch(content->keyc)                 // NEXT/PREV/최초 → vKeyv 세팅
    K_NEXT → content_get_nkey
    K_PREV → content_get_pkey
    default→ vKeyv="0"
  입력값 yprintf → 호스트변수(vprGd/vBltnYmd/vBaseCrcd/vCocuCd/vBltnTob/vXrtBltnMrktDcd) + l_rtrim
  ① SELECT ... FROM CMG014F WHERE BLTN_YMD<=:vBltnYmd ... ROWNUM=1  // 최근 고시일 스냅
     → cCurrencyBltnYmd, 값 있고 다르면 vBltnYmd 교체
  ② SELECT MAX/MIN(BLTN_TOB) FROM CMG014F WHERE BLTN_YMD=:vBltnYmd  // 회차 범위
     → cMaxBltnTob / cMinBltnTob
  ③ DECLARE W1202S01_C1 CURSOR                                      // 본 조회 (CMG014F+CMG004M+CMG005M)
  OPEN C1  (NO_DATA_FOUND→91001 return / 기타→10118 return)
  for(ii=0;;ii++)
     FETCH C1 (NO_DATA_FOUND→break / 기타→CLOSE+10118 return)
     if(ii==max_row){ i_isnext++; break; }   // 페이지 초과 → 다음 있음 표시, 미기록
     zsedit/sprintf 로 orec[ii] 채움, vKeyv=cKeyv  // NEXT KEY 갱신
  CLOSE C1
  결과 판정:
     ii==0 && i_isnext==0 → 91001 (자료없음)
     i_isnext==1          → content_set_nkey(vKeyv), 91002 (계속됨)
     else                 → 91003 (완료), grid01_cnt=ii
  grid01_cnt=ii; sndl += (ii-1)*sizeof(orec[0])
  return RET_SUCCESS
```

트랜잭션: 조회 전용이나 상위 `W1202S01()` 이 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 을 항상 호출.

## 4. 사용 DB 테이블

| 테이블 | 한글명(추정) | 별칭 | 접근 | 용도 |
|-----|-----|-----|-----|-----|
| `TB_FXB_CMG014F` | 시장환율(환율고시) | `CMG014F` / `ICMG014F` | SELECT | 고시년월일/회차별 환율 본 데이터. 최근 고시일 스냅(①), MAX/MIN 회차(②), 본 조회 커서(③) |
| `TB_FXB_CMG004M` | 통화쌍 마스터 | `CMG004M` | SELECT | `CRNC_PAIR_ID`(base/cocu) 로 LEFT OUTER JOIN → `RPSN_SQC`(정렬순), `CUS_CLCL_DCPU_RCN`(고객계산 소수점 자릿수) |
| `TB_FXB_CMG005M` | 통화 마스터 | `CMG005M` | SELECT | `CRCD=BASE_CRCD` 등가조인 → `RPSN_SQC`(통화 정렬순) |

*DML 요약: 순수 조회. INSERT/UPDATE/DELETE 없음. 커서 `W1202S01_C1` (multi-record FETCH). 공통함수 간접 테이블 접근 없음.*

## 5. 로직 / 필터 규칙

- **고시일 스냅(①)**: `INDEX_DESC(PK_FXB_CMG014F)` + `BLTN_YMD <= :vBltnYmd` + `ROWNUM=1` 로 요청일 이하 최근 고시일을 구해, 요청일과 다르면 `vBltnYmd` 를 그 값으로 교체. base/cocu 는 NVL 로 선택 필터.
- **회차 산정(②→③)**: 스냅된 `vBltnYmd` 기준 `MAX/MIN(BLTN_TOB)` 산출. 본 커서(③)의 `BLTN_TOB` 조건은:
  - `vprGd` 가 NULL 또는 `'1'` → `NVL(:vBltnTob, :cMaxBltnTob)` (요청 회차, 없으면 최신 회차)
  - `vprGd = '2'` → `:cMinBltnTob` (최초 회차, 신규요청)
- **동적 필터(NVL 패턴)**: `baseCrcd` / `cocuCd` / `xrtBltnMrktDcd` 는 빈값(NULL)이면 해당 컬럼 자기참조로 전체 통과.
- **정렬**: `SORT_KEY`(COCU_CD='KRW'→1, else 2) → `PAIR_SORT_KEY`(CMG004M.RPSN_SQC, NVL 20) → `CRCD_SORT_KEY`(CMG005M.RPSN_SQC) → `BASE_CRCD`.
- **페이징**: NEXT_KEY = `SORT_KEY||PAIR_SORT_KEY||CRCD_SORT_KEY||BASE_CRCD`. 커서 WHERE `... > :vKeyv` (최초 `vKeyv="0"`). `ii==max_row` 도달 시 초과 1건은 미기록하고 `i_isnext` 세팅.
- **금액 편집**: `fxTusaXrt`/`fxAcbkPrc`/`fxTscnRt`/`fxCoinBngRt`/`fxCoinSellRt`/`fxTvchSellRt`/`fxPobeBngRt` 는 `sprintf("%*lf", size, val)` 폭 편집. 나머지 매매/매입/매도율(`fxBgbsRt` 등)은 SQL 이 만든 동적 포맷 `cDcpuRcn`(예: `"%.4f"`) 로 편집 — **§6 버그 주의**.
- **에러/상태 코드** (`e_msg.code` → `fnSetMsg`):
  - `10118` 자료 조회중 오류 (SELECT/커서 오류)
  - `91001` 해당하는 자료가 없습니다 (건수 0)
  - `91002` 조회가 계속됩니다 (NEXT_KEY 존재, `i_isnext==1`)
  - `91003` 조회가 완료되었습니다 (마지막 페이지)
  - 작업구분 오류: `comhdr_o.mesg = "작업구분 오류."` (default 분기, `e_msg.code` 미설정)

| 위치 | 헤더/주석 표기 | 실제(.pc) |
|-----|-----|-----|
