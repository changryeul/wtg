# 서비스 명세 — `W1253A01` (SMS알림메세지관리 — 등록/수정/삭제)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 .pc 실 구현 기준이며, .h 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1253A01` |
| 이름 | SMS알림메세지관리 |
| 유형 | 복합 (등록 / 수정 / 삭제) — 실 DML 은 INSERT / UPDATE (삭제는 `USE_YN='N'` soft delete) |
| 구현 | `win/src/trn/W1250/W1253A01.pc` (359 줄) |
| I/O 헤더 | `win/src/inc/trn/W1253A01.h` |
| 라우팅 alias | `W1253A01` |
| 채널 | `ChannelWeb` (`/v1/tx`) |
| 모듈 | W1250 |

FX(SMS) 알림 메시지 유형별 발송대상/제목/본문을 관리하는 트랜잭션. 조회 기능은 없고 실제로는 **등록·수정·삭제**만 수행한다 (§6 참조).

## 2. I/O 전문 (`.h`)

**Input `W1253A01_I`**

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 작업구분 — `'2'` 만 처리(등록수정), 그 외 "작업구분 오류" |
| `fxMsgPtrnDcd` | 2 | FX메시지유형구분코드 (PK, 필수). 숫자면 `%02d` zero-pad |
| `smssTgtDcd` | 1 | 발송대상구분코드 (등록/수정 시 필수, `'0'` 불가) |
| `fxMsgTtlNm` | 100 | FX제목내용 (등록/수정 시 필수) |
| `fxMsgCon` | 2100 | FX메시지내용/본문 (등록/수정 시 필수) |
| `useYn` | 1 | 사용여부 (필수). `'N'` 이면 삭제(soft delete) 분기 |

**Output `W1253A01_O`**

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `grid01_cnt` | 6 | 헤더 주석은 "그리드건수" 지만, 실제로는 정상처리 flag 로 `1` 을 세팅 (`zredit(mod->grid01_cnt,1)`). grid 아님 (§6) |

## 3. 구현 흐름 (`.pc`)

```
W1253A01()  (진입, 함수 주석은 "조회" 이나 실제는 등록수정 dispatch)
  ├─ prthdr / getmid / getmod            # COMHDR + mid/mod 획득
  ├─ E_MSG 기본설정 (ltyp KR, mtyp S, rtyp M)
  └─ switch(mid->prGb[0])
       case '2' → W1253A01_F2()          # 유일한 처리 분기 (등록수정)
       default  → comhdr_o.mesg="작업구분 오류." + return (DML 없음)
  ├─ retc==RET_SUCCESS → eflg="0" + l_dbcommit()
  │  else               → eflg="1" + l_dbrollback()
  └─ fnSetMsg(&e_msg, comhdr_o) → return content->sndl

W1253A01_F2()  (등록수정)
  ├─ getmid/getmod + memset(mod)
  ├─ yprintfrtrim 로 입력 각 필드 로컬버퍼 이관 (+ comhdr_i->usid → hls_upd_id)
  ├─ iMsgPtrnDcd=atoi(vFxMsgPtrnDcd); >0 이면 sprintf "%02d" zero-pad
  ├─ 입력 유효성 검증
  │    vFxMsgPtrnDcd 빈값 → e_msg.code "10527" + RET_FAILURE
  │    vUseYn 빈값        → e_msg.code "10528" + RET_FAILURE
  │    vUseYn != 'N' 일 때만: smssTgtDcd(빈/'0'), fxMsgTtlNm, fxMsgCon 필수
  │       → 각각 "10713" / "10530" / "10531" + RET_FAILURE
  └─ 메시지설정목록 처리
       vUseYn=='N' (삭제) → UPDATE TB_FXB_CMC004M SET USE_YN, FNAP_YMD=오늘, 변경ID/TS
                              오류 → "10532"
       else (등록/수정)   → SELECT COUNT(1) TB_FXB_CMC004M (기등록 확인)  오류 "10533"
                              i_cnt>0  → UPDATE TB_FXB_CMC004M (FNAP_YMD='99991231')  오류 "10532"
                              i_cnt==0 → INSERT TB_FXB_CMC004M (APSR_YMD=오늘, FNAP='99991231') 오류 "10534"
       (#if 0 로 비활성화된 TB_FXB_CMC003D UPDATE 블록 — dead code, §6)
  └─ zredit(mod->grid01_cnt,1) + e_msg.code "10150"(정상) + return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-------|-------|-----|-----|-----|
| `TB_FXB_CMC004M` | SMS메시지설정기본 | — | SELECT / INSERT / UPDATE | 유형코드(PK) 기등록 확인 후 신규 INSERT 또는 UPDATE. `USE_YN='N'` 삭제는 UPDATE 로 처리 |
| `TB_FXB_CMC003D` | (공통코드 상세, 추정) | — | UPDATE (dead code) | `#if 0` 로 비활성. 발송매체구분 참조컬럼 세팅 시도였으나 컴파일 제외 (§6) |

*DML 요약: 실 접근 테이블은 `TB_FXB_CMC004M` 단일. INSERT/UPDATE(수정)/UPDATE(soft delete) 3 경로. 물리 DELETE 없음. 트랜잭션은 진입함수에서 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()`.*

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb=='2'` 만 등록수정으로 처리. 그 외 값은 DML 없이 "작업구분 오류." 응답.
- **유형코드 정규화**: `fxMsgPtrnDcd` 를 atoi 후 양수면 `%02d` 로 zero-pad (`"1"→"01"`). PK 로 사용.
- **삭제 vs 등록/수정 분기**: `useYn=='N'` 이면 soft delete (`USE_YN='N'`, `FNAP_YMD`=오늘). 그 외 값이면 등록/수정 경로.
- **등록/수정 판정**: `SELECT COUNT(1)` 결과 `i_cnt>0` → UPDATE, `==0` → INSERT (upsert 패턴).
- **적용기간**: 등록/수정 시 `FNAP_YMD='99991231'` (무기한). 등록 시 `APSR_YMD`=오늘. 삭제 시 `FNAP_YMD`=오늘(적용종료).
- **감사컬럼**: `FX_SYS_LSMD_ID` = `comhdr_i->usid`, `FX_SYS_LSMD_TS` = SYSDATE.
- **트랜잭션**: 서브함수는 RET_SUCCESS/RET_FAILURE 만 반환, commit/rollback 은 진입함수 `W1253A01()` 이 결정.
- **에러코드 (e_msg.code → mesg, eflg)**:
  | code | 의미 |
  |-----|-----|
  | 10527 | FX메시지유형구분코드 미입력 |
  | 10528 | 사용여부 미입력 (로그는 "메시지삭제여부") |
  | 10713 | 발송대상구분코드 미입력/`'0'` |
  | 10530 | 제목내용 미입력 |
  | 10531 | 본문내용 미입력 |
  | 10532 | SMS메시지설정 수정중 오류 (삭제 UPDATE / 수정 UPDATE 공용) |
  | 10533 | 기등록 여부 확인중 오류 (SELECT) |
  | 10534 | 등록중 오류 (INSERT) |
  | 10150 | 정상 처리 |
- 정상 시 `eflg="0"` + commit, 실패 시 `eflg="1"` + rollback.
