## 3. 구현 흐름 (`.pc`)

```
W1701A03()                       ← 진입 (콜백)
  ├─ getmid / getmod             COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              pname=rkey, ltyp(입력), mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(comhdr_i->ctyp[0])   접속매체 → comhdr_i->chnl 매핑
  │     '1'HTS '2'MTS 'W'WTS 'E'EMP 'A'API 'S'SPI → chnl = ctyp
  │     default → 매핑 없음 로그만
  ├─ retc = W1701A03_F1(...)     ← 실제 로직
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)
     └─ fnSetMsg(&e_msg, comhdr_o)  결과 메시지 세팅 후 return content->sndl

W1701A03_F1()                    ← 등록(갱신) 로직
  ├─ getmid/getmod, memset(mod), comhdr_o->rtyp="G"
  ├─ 입력 바인딩: i_grid01_cnt=atoi(grid01_cnt), ifXAthrDcd, cSysUno(usid 16B)
  ├─ 유효성: ifXAthrDcd[0]==NULL → e_msg.code="42038", RET_FAILURE
  ├─ [3] 그룹존재 확인 SELECT COUNT(1) FROM TB_FXB_CMC012M
  │       WHERE FX_USER_DCD=SUBSTR(:ifXAthrDcd,1,2) AND SCRE_ACS_ATHR_DCD=SUBSTR(:ifXAthrDcd,3,2)
  │       sqlcode!=0 or count==0 → e_msg.code="95000", RET_FAILURE
  ├─ [4] 메뉴리스트 생성 (for ii < i_grid01_cnt)
  │       각 menuId 존재확인 SELECT 1 FROM TB_FXB_CMC010M WHERE MENU_ID=:imenuId
  │         sqlcode!=0 → e_msg.code="95000", RET_FAILURE
  │       마지막 건이 아니면 ',' 부착 → vMenuIdCtlgCon 에 strcat, icnt++
  ├─ [5] 권한그룹목록 갱신 UPDATE TB_FXB_CMC012M
  │       SET MENU_ID_CTLG_CON=(trailing ',' 제거) , FX_SYS_LSMD_ID=:cSysUno , FX_SYS_LSMD_TS=SYSDATE
  │       WHERE FX_USER_DCD/SCRE_ACS_ATHR_DCD = SUBSTR 분해값
  │       sqlcode!=0 → e_msg.code="95000", RET_FAILURE
  └─ [Z] comhdr_o->eflg="0", e_msg.code="00000", RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC012M` | 권한그룹(화면접근권한) 마스터 | CMC012M | SELECT / UPDATE | 그룹 존재확인(COUNT) 후 MENU_ID_CTLG_CON(메뉴목록내용)·최종수정자/시각 갱신 |
| `TB_FXB_CMC010M` | 메뉴 마스터 | CMC010M | SELECT | 입력 menuId 각 건의 존재(MENU_ID) 검증 |

*DML 요약: TB_FXB_CMC012M 을 존재확인(SELECT COUNT)한 뒤, 입력 menuId 를 TB_FXB_CMC010M 로 건별 검증하여 콤마 구분 문자열로 조립, 최종적으로 TB_FXB_CMC012M.MENU_ID_CTLG_CON 을 1회 UPDATE. INSERT/DELETE 없음. 커서 미사용(전건 단일 SELECT). 트랜잭션은 진입함수에서 성공 시 `l_dbcommit`, 실패 시 `l_dbrollback`.*

## 5. 로직 / 필터 규칙

- **권한그룹코드 분해**: `fXAthrDcd`(4B) 를 SUBSTR(1,2)=FX_USER_DCD, SUBSTR(3,2)=SCRE_ACS_ATHR_DCD 로 나눠 WHERE 조건에 사용. SQL 안에서 분해하므로 로컬 파싱 변수 없음.
- **입력 유효성**: `ifXAthrDcd[0]==NULL` 만 검증 → `42038`. grid01_cnt·menuId 형식 검증은 없음.
- **메뉴목록 조립**: `vMenuIdCtlgCon`(40000+1B) 에 유효 menuId 를 콤마로 이어붙임. 마지막 건은 콤마 미부착. UPDATE 시 혹시 남은 trailing ',' 는 CASE/SUBSTR 로 한 번 더 제거(이중 안전장치).
- **빈 menuId 스킵**: `imenuId[0]!=0x00` 인 건만 처리(공백 건은 목록·검증에서 제외).
- **트랜잭션**: 진입함수 `W1701A03()` 에서 `_F1` 결과에 따라 명시적 commit/rollback. AUTOCOMMIT 아님.
- **에러 코드(e_msg.code → 메시지)**:
  - `42038` — 권한그룹코드 미입력(입력값 오류)
  - `95000` — 그룹 미존재/SELECT 오류, menuId SELECT 오류, UPDATE 오류(공통 DB 오류)
  - `00000` — 정상, 이 때만 `comhdr_o->eflg="0"`
