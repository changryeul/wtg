## 3. 구현 흐름 (`.pc`)

```
W1354S01()                       ← 진입 (콜백)
  ├─ prthdr / getmid / getmod    COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1354S01_F1(...)     ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  ├─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
  │                             (명시적 커밋, AUTOCOMMIT 아님)
  └─ fnSetMsg(&e_msg, comhdr_o) → return content->sndl

W1354S01_F1()                    ← 조회
  ├─ getmid/getmod, sndl += W1354S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])   ← 페이지당 최대 건수
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT → content_get_nkey → vKeyv
  │      K_PREV → content_get_pkey → vKeyv   (ORDER BY DESC 와 조합 불안정 — §6)
  │      default(최초) → vKeyv = "0"
  ├─ 입력 바인딩: yprintf + l_rtrim → vFxPdcd/vCrcd/vFxCusLmtDcd/vUseYn
  │      (주석은 "입력항목 유효성 check" 이나 NULL/공백 거부 없음 — trim 만, §6)
  ├─ DECLARE/OPEN W1354S01_C1  (CMG008M ⋈ CMG005M + CMC003D×3 outer join)
  │      각 필터는 NVL(:bind, 컬럼) 패턴 → 빈값이면 전체
  │      페이징: FX_PDCD||CRCD||CRCD > :vKeyv  (NEXT_KEY 열과 동일 표현)
  │      ORDER BY FX_PDCD DESC, CMG005M.RPSN_SQC
  │      OPEN 실패: NO_DATA→"91001" 반환 / 기타→"10118" 반환 (RET_FAILURE)
  ├─ FETCH 루프 (ii=0..)
  │      SQLCODE NO_DATA → break / 기타 err → CLOSE + "10122" + RET_FAILURE
  │      ii == max_row  → i_isnext++, break (다음 페이지 존재; 이 행은 미저장)
  │      mod->orec[ii].* ← zsedit(c*)  각 컬럼 세팅
  │      vKeyv = cKeyv (다음키 갱신)
  ├─ CLOSE W1354S01_C1
  ├─ 결과 코드:
  │      ii==0 && i_isnext==0 → "91001" 해당 자료 없음
  │      i_isnext==1          → content_set_nkey(vKeyv) + "91002" 조회 계속
  │      그외                 → "91003" 조회 완료
  ├─ zredit(mod->grid01_cnt, ii)
  └─ content->sndl += (ii-1) * sizeof(orec[0]) → return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='1'`(조회) 만 처리. 그 외는 상위 `W1354S01()` 에서 "작업구분 오류" 로 즉시 반환(F1 미진입).
- **동적 필터**: 4개 입력(`fxPdcd`/`crcd`/`fxCusLmtDcd`/`useYn`)을 각각 `AND col = NVL(:bind, col)` 로 바인딩 → 빈값(trim 후 "")이면 해당 조건 무력화(전체). NULL 처리 방식이라 빈 문자열 바인딩 시에도 전체 조회.
- **조인**: 본 테이블 `CMG008M` 을 `CMG005M`(통화기본)과 `CRCD` 로 **inner join**, `CMC003D` 3벌과는 **outer join(`(+)`)** 하여 각 코드 한글명 부착.
- **페이징**: `content->keyc`(NEXT/PREV/최초)로 `vKeyv` 산정 → `WHERE FX_PDCD||CRCD||CRCD > :vKeyv` 로 커서 진행. 한 페이지 = `max_row = MAX_RRB / sizeof(orec[0])` 건. `max_row` 초과 감지 시 `i_isnext=1` 로 "조회 계속(91002)" + `content_set_nkey`.
- **정렬**: `ORDER BY CMG008M.FX_PDCD DESC, CMG005M.RPSN_SQC`.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드** (`e_msg.code`, `fnSetMsg` 로 COMHDR `mesg`/`rcod` 반영):
  - `91003` 조회가 완료되었습니다 (성공, `eflg=0`).
  - `91002` 조회가 계속됩니다 (다음 페이지 존재, `eflg=0`, nkey 세팅).
  - `91001` 해당하는 자료가 없습니다 (커서 OPEN NO_DATA 또는 `ii==0` — 이 경우 F1 은 `RET_FAILURE` 로 반환되어 상위서 `eflg=1`+rollback, §6).
  - `10118` 자료 조회중 오류 (커서 OPEN 기타 SQL 오류, `RET_FAILURE`).
  - `10122` 자료 처리중 오류 (FETCH 기타 SQL 오류, CLOSE 후 `RET_FAILURE`).
  - "작업구분 오류." (COMHDR `mesg`, `prGb≠'1'`).
- **결과 전달**: `grid01_cnt` + `orec[]` + COMHDR(`eflg`/`rcod`/`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).

## 6. 주의 — 문서/코드 불일치 (마이그레이션 시 확인)

| 위치 | 헤더/주석 표기 | 실제(`.pc`) |
|---|---|---|
| `.pc` L47 변수 주석 | `mod` = "W1354S01 의 입력값 주소" | 출력값(Output) 주소 — 복붙 오기 |
| `.pc` L228 / L246 SELECT | `NEXT_KEY = FX_PDCD \|\| CRCD \|\| CRCD` | `CRCD` 가 중복 concat (고객한도구분코드 `FX_CUS_LMT_DCD` 의 오기로 추정). SELECT 열/WHERE 양쪽 동일 표현이라 페이징은 동작하나 키가 `(상품,통화,통화)` 로 의미상 불완전 — 동일 상품·통화에 한도구분이 여러 건이면 페이지 경계에서 누락/중복 가능 |
| L247 정렬 vs L246 페이징 | `ORDER BY FX_PDCD DESC` + `key > :vKeyv` | 정렬은 내림차순인데 페이징 비교는 `>`(오름차순 진행). 최초키 `"0"` 로 시작하면 DESC 결과 첫 페이지는 잡히나, NEXT 진행 방향과 정렬 방향이 불일치 — 정렬 보조키(`RPSN_SQC`)가 키에 포함되지 않아 다중건 페이징 시 불안정 |
| L67 함수 주석 / L187 섹션 주석 | "통화 목록 조회" / "입력항목 유효성 check" | 실제는 상품별거래한도 조회이며, 유효성 검증 없이 trim 만 수행 (복붙 주석) |
| K_PREV 처리 (`content_get_pkey`) | 이전 페이지 페이징 의도 | vKeyv 산정만 하고 SELECT 는 항상 `> :vKeyv` (단방향). PREV 시 이전 페이지가 아니라 해당 키 이후를 다시 조회 — 실질 이전페이지 동작 아님 |
| L163 `max_row` | 페이지 크기 | `MAX_RRB / sizeof(orec[0])` 로 산정. `orec[0]`≈1034B 라 페이지당 건수가 버퍼 한도에 종속 — 헤더/문서에 명시 없음 |
| L381 `sndl` 계산 | — | `content->sndl += (ii-1)*sizeof(orec[0])`. `ii==0`(자료 없음) 시 `-1`배로 orec 1건분을 되빼 base(=grid01_cnt만) 로 귀결 → 결과적으로 정상. 단 `RET_FAILURE` 반환 경로라 상위서 rollback+eflg=1 로 처리됨 |
| 미사용 지역변수 | — | `i, cnt, ret, editb, log_m, max_row 외 error_f, i_sndcnt, retc` 등 다수 미사용 (동작 무해) |

> WTG 는 `.h` 를 파싱해 필드명/크기만 취하고 전문을 통과시키므로 위 주석/키 버그가
> 게이트웨이 동작 자체에는 영향 없다. 단 페이징 키(`CRCD` 중복 / DESC-vs-`>`)는
> 실제 다건 페이징 정확도에 영향이 있으므로, 클라이언트 페이지네이션 노출 전 엔진측 확인 권장.
