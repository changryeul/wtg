## 3. 구현 흐름 (`.pc`)

```
W1252S01()                       ← 진입 (콜백)
  ├─ prthdr / getmid / getmod    COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1252S01_F1(...)      ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? (eflg "0" + l_dbcommit()) : (eflg "1" + l_dbrollback())
     → fnSetMsg(&e_msg, comhdr_o)  로 e_msg → COMHDR 반영 후 send

W1252S01_F1()                    ← 조회
  ├─ getmid/getmod, sndl += W1252S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(mod->orec[0])   페이지당 최대 건수
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT → content_get_nkey → vKeyv
  │      K_PREV → content_get_pkey → vKeyv
  │      default(최초) → vKeyv = "0|0"
  ├─ 입력 바인딩: yprintf + l_rtrim → vprGb / vComCdNm / vCmcdVl / vKeyv
  │      (유효성 실검증 없음 — 로그만, §6)
  ├─ [SEED 복호화 테스트 블록]  하드코딩 키로 vCmcdVl 를 hex→bytes→SeedDecrypt
  │      결과는 로그만 남기고 미사용 — dead/leftover 코드 (§6)
  ├─ NEXT KEY 분해: SELECT SUBSTR(:vKeyv,…'|'…) → vKeyv1 / vKeyv2  (FROM DUAL)
  ├─ DECLARE W1252S01_C1 CURSOR  (TB_FXB_CMC003D, COM_CD_NM/CMCD_VL LIKE 필터
  │      + keyset 페이징 조건 + ORDER BY COM_CD_NM, CMCD_VL)
  ├─ OPEN W1252S01_C1
  │      OPEN err(≠SUCCESS,≠NO_DATA) → e_msg "10118"
  │      NO_DATA_FOUND → e_msg "91001"  (OPEN 시엔 사실상 미발생, §6)
  ├─ for(ii=0; ; ii++)
  │      FETCH → c* 지역변수
  │        NO_DATA_FOUND → break
  │        기타 SQL err → CLOSE, e_msg "10118", return RET_FAILURE
  │      if (ii == max_row) { i_isnext++; break; }   ← 초과 1건 감지, 저장 안 함
  │      zsedit 로 mod->orec[ii] 각 필드 채움
  │      vKeyv = cKeyv (마지막 저장행의 NEXT KEY)
  ├─ CLOSE W1252S01_C1
  ├─ 결과 판정
  │      ii==0 && i_isnext==0 → e_msg "91001" (자료 없음)
  │      i_isnext==1          → content_set_nkey(vKeyv), e_msg "91002" (계속됨)
  │      그외                 → e_msg "91003" (조회 완료)
  ├─ zredit(mod->grid01_cnt, ii)
  ├─ content->sndl += (ii - 1) * sizeof(orec[0])   ← ii==0 시 언더플로 (§6)
  └─ return RET_SUCCESS   (자료 없음/계속됨 포함 항상 SUCCESS → 상위서 commit)
```

| `DUAL` | (Oracle 의사 테이블) | — | **SELECT** | `vKeyv`("k1&#124;k2") 를 `SUBSTR/INSTR` 로 `vKeyv1`/`vKeyv2` 분해 |

*DML 요약: 조회 전용. 커서 SELECT 1문(`TB_FXB_CMC003D`) + NEXT KEY 파싱용 `FROM DUAL` SELECT 1문. INSERT/UPDATE/DELETE 없음. `l_dbcommit`/`l_dbrollback` 은 조회지만 규약상 명시 호출.*

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='1'` (조회) 만 유효. 그 외는 상위 `W1252S01()` 에서 "작업구분 오류" 로 즉시 반환.
- **동적 필터**:
  - `COM_CD_NM LIKE '%' || NVL(UPPER(:vComCdNm), COM_CD_NM) || '%'` — 대문자 부분일치, 빈값이면 전체.
  - `CMCD_VL LIKE UPPER(:vCmcdVl) || '%'` — 대문자 접두일치, 빈값이면 전체(`'%'`).
- **정렬**: `ORDER BY COM_CD_NM, CMCD_VL`.
- **커서 페이징 (keyset)**: `(COM_CD_NM > :vKeyv1) OR (COM_CD_NM = :vKeyv1 AND CMCD_VL > :vKeyv2)`.
  - 최초 조회: `vKeyv="0|0"` → `vKeyv1="0"`, `vKeyv2="0"` (문자열 비교이므로 사실상 전체 시작).
  - NEXT KEY 는 `COM_CD_NM || '|' || CMCD_VL` 로 각 행에서 산출, `content_set_nkey` 로 클라이언트 전달.
  - 한 페이지 최대 `max_row = MAX_RRB / sizeof(orec[0])` 건. `ii == max_row` 도달 시 초과 1건을 FETCH 하되 저장하지 않고 `i_isnext` 로 다음 페이지 존재만 표시.
- **트랜잭션**: `F1` 은 자료 없음/계속됨 포함 항상 `RET_SUCCESS` 반환 → 상위에서 `eflg="0"` + `l_dbcommit`. FETCH SQL 오류만 `RET_FAILURE` → `eflg="1"` + `l_dbrollback`.
- **에러/상태 코드 (e_msg.code → fnSetMsg → COMHDR)**:
  - `10118` — 자료 조회중 오류 (커서 OPEN/FETCH 실패). FETCH 실패는 RET_FAILURE.
  - `91001` — 해당하는 자료가 없습니다 (`ii==0 && i_isnext==0`). RET_SUCCESS.
  - `91002` — 조회가 계속됩니다 (다음 페이지 있음).
  - `91003` — 조회가 완료되었습니다 (마지막 페이지).
