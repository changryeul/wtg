## 2. I/O 전문 (`.h`)

**Input `W1502S01_I`** (총 51B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `StartYmd` | 8 | 조회시작일자 — `A.BSOP_YMD BETWEEN :StartYmd AND :EndYmd` |
| `EndYmd` | 8 | 조회종료일자 |
| `WbcsRlnmAltrNo` | 16 | 전행고객실명대체번호(고객번호) — 빈값이면 전체(NVL) |
| `FndNo` | 12 | 펀드번호 — 빈값이면 전체, 있으면 고객번호에 concat 하여 매칭 |
| `ByselDcd` | 1 | 매입매도구분(B:매입,S:매도) — 빈값이면 전체(NVL) |
| `MngmBrcd` | 6 | 관리부점코드 — B(고객) 테이블 기준 필터(NVL) |

**Output `W1502S01_O`** (헤더 6B + `orec[]` 가변, orec 1건 = 134B)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (`ii`, zredit) |
| `orec[].BsnsDt` | 8 | 작성일자(영업일자 `A.BSOP_YMD`) |
| `orec[].WbcsRlnmAltrNo` | 16 | 전행고객실명대체번호 |
| `orec[].FndNo` | 12 | 펀드번호 (`SUBSTR(WBCS_RLNM_ALTR_NO_FND_NO, len(고객번호)+1, 12)`) |
| `orec[].FwdPrechkNo` | 15 | 선물환사전점검번호(`FWEX_PSRN_NO`) |
| `orec[].FwdPrechkSeq` | 5 | 선물환사전점검세부번호(`FWEX_PSRN_DTLS_NO`) |
| `orec[].ByselDcd` | 1 | 매입매도구분 |
| `orec[].CustIm` | 40 | 고객명 (B 테이블 `CUS_ABBR_NM` 고객약칭명) |
| `orec[].InvClcd` | 1 | 투자구분코드(`INVS_DCD`) |
| `orec[].InvInfoVddd` | 8 | 투자정보유효기일(`INVM_INFO_VLDU_YMD`) |
| `orec[].InvInfoMadd` | 8 | 투자정보작성일(`INVM_INFO_WRTN_YMD`) |
| `orec[].FxSysLsmdTs` | 20 | FX시스템최종변경일시 (`TO_CHAR(FX_SYS_LSMD_TS,'YYYY/MM/DD HH24:MI:SS')`) |

> grid 형. 성공/실패·페이징 여부는 COMHDR(`eflg`) + `e_msg.code`(→`mesg`) 로 함께 전달.
> 커서는 34개 컬럼을 FETCH 하지만 그중 위 11개만 orec 로 세팅한다(§6 참조).

## 3. 구현 흐름 (`.pc`)

```
W1502S01()                       ← 진입 (콜백)
  ├─ prthdr / getmid / getmod    COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ retc = W1502S01_F1(...)     ← 실제 조회 로직
  ├─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()
  └─ (같은 commit/rollback 블록이 한 번 더 반복 — 중복, §6)

W1502S01_F1()                    ← 사전점검내역 조회
  ├─ getmid/getmod, sndl += W1502S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT → content_get_nkey → nextKey
  │      K_PREV → content_get_pkey → nextKey  (역방향 미구현, §6)
  │      default(최초) → nextKey = "999…9"(38자리) 후 l_rtrim
  ├─ 입력 바인딩: yprintf + l_rtrim → iStartYmd/iEndYmd/iWbcsRlnmAltrNo/
  │      iByselDcd/iFndNo/iMngmBrcd  (필수값 검증 없음 — 주석 블록 /* */ 처리, §6)
  ├─ DECLARE W1502S01_C1 CURSOR  (TB_FXB_TRC003L A LEFT JOIN TB_FXB_CSC001M B)
  ├─ OPEN W1502S01_C1
  │      SQLCODE≠0 → NO_DATA면 91001/eflg"0", 그외 10118/eflg"1" → RET_FAILURE
  │      (OPEN 은 NO_DATA_FOUND 를 내지 않음 → 해당 분기 dead, §6)
  ├─ for(ii=0; ;ii++)
  │      FETCH → 34개 host 변수
  │        NO_DATA_FOUND → break
  │        기타 err → CLOSE + 10118/eflg"1" → RET_FAILURE
  │      if (ii == max_row) { i_isnext++; break; }   ← 한 건 더 있으면 next 표시
  │      l_rtrim(전 컬럼) → zsedit(mod->orec[ii].{11개 필드})
  │      nextKey = oBsnsDt + oWbcsRlnmAltrNo + oFndNo   ← 다음 조회 키(§6)
  ├─ CLOSE W1502S01_C1
  ├─ 결과 코드
  │      ii==0 && i_isnext==0 → 91001 "해당 자료 없음" eflg"0"
  │      i_isnext==1          → 91002 "조회 계속" + content_set_nkey(nextKey) eflg"0"
  │      그외                 → 91003 "조회 완료" eflg"0" + grid01_cnt=ii
  ├─ (무조건) eflg="0", grid01_cnt=zredit(ii)   ← 위 분기 무력화, §6
  └─ content->sndl += (ii-1)*sizeof(orec[0]) ; return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_TRC003L` | 선물환사전점검등록내역 | A | **SELECT** (커서) | 기간(`BSOP_YMD`)·고객번호·펀드·매매구분·`USE_YN='Y'` 조건으로 사전점검 등록내역 조회 (주 테이블) |
| `TB_FXB_CSC001M` | 고객약정정보기본 | B | **SELECT** (커서, OUTER JOIN) | 고객약칭명(`CUS_ABBR_NM`) 및 관리부점코드(`MNGM_BRCD`) 필터. `A.WBCS_RLNM_ALTR_NO = B.WBCS_RLNM_ALTR_NO(+)` |

*DML 요약: 단일 커서 `W1502S01_C1` 의 SELECT(2 테이블 outer join) 1문뿐. INSERT/UPDATE/DELETE 없음. 데이터 변경 없으나 규약상 성공 시 `l_dbcommit`, 실패 시 `l_dbrollback`.*

## 5. 로직 / 필터 규칙

- **동적 필터(모두 NVL/CASE 로 optional)**:
  - `A.BSOP_YMD BETWEEN :iStartYmd AND :iEndYmd` (기간 필수 성격 — 빈값이면 사실상 무결과)
  - `A.BYSEL_DCD = NVL(:iByselDcd, A.BYSEL_DCD)` — 빈값이면 전체
  - `A.WBCS_RLNM_ALTR_NO = NVL(:iWbcsRlnmAltrNo, A.WBCS_RLNM_ALTR_NO)` — 빈값이면 전체
  - `A.WBCS_RLNM_ALTR_NO_FND_NO = CASE WHEN :iFndNo IS NULL THEN 그대로 ELSE 고객번호||:iFndNo END`
  - `A.USE_YN = 'Y'` (사용중만)
  - `B.MNGM_BRCD = NVL(:iMngmBrcd, B.MNGM_BRCD)` — 관리부점 필터 (outer join 무력화 이슈 §6)
  - Oracle 에서 빈 문자열 = NULL 이므로 memset 후 미입력 host 변수가 NVL 분기를 탄다.
- **정렬/페이징**: `ORDER BY A.BSOP_YMD DESC, A.WBCS_RLNM_ALTR_NO_FND_NO DESC` + `A.BSOP_YMD || A.WBCS_RLNM_ALTR_NO_FND_NO < :nextKey`.
  최초 조회는 `nextKey="999…9"` 로 전체 시작, NEXT 는 마지막 행 키로 이어감(하향 seek 방식).
- **NEXT 여부 판정**: `max_row = MAX_RRB / orec크기`. 루프에서 `ii == max_row` 도달(= 한 건 더 존재) 시 그 행은 출력하지 않고 `i_isnext=1` 로 break → 91002.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님. (진입 함수에서 동일 블록 중복 실행 — §6)
- **에러/결과 코드**:
  - 성공(마지막 페이지): `91003` "조회가 완료되었습니다.", `eflg=0`.
  - 다음 페이지 존재: `91002` "조회가 계속됩니다." + `content_set_nkey`, `eflg=0`.
  - 자료 없음: `91001` "해당하는 자료가 없습니다.", `eflg=0` (RET_FAILURE 아님 — CLOSE 후 정상 return).
  - 커서 OPEN/FETCH 오류: `10118`, `eflg=1`, RET_FAILURE.
- **결과 전달**: grid(`grid01_cnt`+orec[ii]) + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).

| L215~217 K_PREV | 이전(PREV) 페이징 | `content_get_pkey` 로 키만 받고 정렬/`<` 방향은 그대로(DESC+`<`) → **역방향 페이징 미동작**, `content_set_pkey` 호출도 없음. NEXT 단방향만 유효 |
| L538 nextKey 조합 | 다음 조회 키 | trim 된 `oBsnsDt+oWbcsRlnmAltrNo+oFndNo` 로 조립. WHERE 는 고정폭 `BSOP_YMD||WBCS_RLNM_ALTR_NO_FND_NO` 와 비교 — 고객번호/펀드가 고정폭 미만일 때 키 자릿수 불일치로 **페이징 경계 어긋날 소지** |
| L263~270 유효성 블록 | "입력항목 유효성 check" (고객번호 필수) | 전체가 `/* */` 주석 처리 → 실제 필수값 검증 없음 (trim 만) |
| L200·L579 sndl 산정 | grid 크기 반영 | `sndl += W1502S01_O_SZ`(orec 1건 포함) 후 `+= (ii-1)*orec크기`. ii≥1 이면 6+ii*orec 로 정확하나 로직이 오프바이 형태 — 검토 권장 |
| 미사용 지역변수 | — | `i,cnt,ret,editb,ii(외곽),max_row 외 i_sndcnt,log_m,error_f,vKeyv,iBsopYmd` 등 다수 미사용 |

> WTG 는 `.h` 를 파싱해 필드명/크기만 취하고 전문을 통과시키므로 위 주석/데드코드는
> 동작에 직접 영향 없다. 단 outer join 무력화·PREV 미동작·페이징 키 자릿수 이슈는
> 조회 결과 자체에 영향을 줄 수 있어 마이그레이션/재현 검증 시 확인이 필요하다.
