## 2. I/O 전문 (`.h`)

**Input `W1500A02_I`** (총 270B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 처리구분 — `'1'`조회 `'2'`등록 `'3'`수정 `'4'`삭제. 그 외 "작업구분 오류" |
| `errCd` | 12 | 에러코드 (`EAI_LNK_EROR_ID`). F1 에선 LIKE `%errCd%` 검색어, F2/F3/F4 에선 PK 값(필수) |
| `lnggDcd` | 1 | 언어구분 — 헤더 주석 "1:한글 2:영어". F1 에선 `'0'` 이면 전체 검색, 그 외 접두 LIKE. F2/F3/F4 필수 |
| `errMsg` | 256 | 에러메시지 (`FX_ERMG_CON`). F1 에선 UPPER LIKE 검색어, F2/F3 에선 저장 내용. F4 미사용 |

**Output `W1500A02_O`** (grid — 가변)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (조회 반환 건수 `ii`. 등록/수정 성공 시 1) |
| `orec[]` | — | 반복 레코드 (헤더는 `orec[1]` 선언, 실제는 `MAX_RRB/sizeof(orec[0])` 만큼 채워 sndl 확장) |
| ├ `errCd` | 12 | 에러코드 |
| ├ `lnggDcd` | 1 | 언어구분 |
| └ `errMsg` | 256 | 에러메시지 |

> 조회(F1) 만 grid 를 채운다. 등록/수정/삭제는 `grid01_cnt` 만 세팅하고 레코드 없이 COMHDR(`eflg`/`mesg`) 로 결과 전달.

## 3. 구현 흐름 (`.pc`)

```
W1500A02()                         ← 진입 (콜백)
  ├─ prthdr / getmid / getmod       COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정                  ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → W1500A02_F1()   조회
  │     '2' → W1500A02_F2()   등록
  │     '3' → W1500A02_F3()   수정
  │     '4' → W1500A02_F4()   삭제
  │     그외 → comhdr_o->mesg="작업구분 오류." 후 즉시 return
  ├─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)
  └─ fnSetMsg(&e_msg, comhdr_o) → return(content->sndl)

W1500A02_F1()  조회
  ├─ sndl += W1500A02_O_SZ, memset(mod), max_row = MAX_RRB/sizeof(orec[0])
  ├─ 페이징 키: content->keyc
  │     K_NEXT → content_get_nkey  / K_PREV → content_get_pkey  / default → vKeyv="0"
  ├─ 입력 바인딩: yprintf + l_rtrim → vErrCd, vLnggDcd, vErrMsg
  ├─ DECLARE TB_FXB_SYG003M_C1  (WHERE errCd LIKE %..% AND lnggDcd DECODE(0→전체) AND
  │                              UPPER(errMsg) LIKE %..% AND (errCd||lnggDcd) > :vKeyv
  │                              ORDER BY errCd, lnggDcd)
  ├─ OPEN → NO_DATA_FOUND 이면 e_msg "91001", eflg "0", RET_FAILURE
  ├─ FETCH 루프 (ii=0..)
  │     ii==max_row → i_isnext++, break (초과 1건은 저장 안 함, 다음존재 판정용)
  │     저장 → orec[ii], vKeyv = cKeyv[:13]
  ├─ CLOSE
  ├─ 결과 메시지
  │     ii==0 && i_isnext==0 → e_msg "91001" (자료 없음)
  │     i_isnext==1          → content_set_nkey(vKeyv), e_msg "91002" (계속됨)
  │     그외                  → e_msg "91003" (완료), grid01_cnt=ii
  └─ eflg "0", grid01_cnt=ii, sndl += (ii-1)*sizeof(orec[0]), RET_SUCCESS

W1500A02_F2()  등록
  ├─ 유효성: vErrCd 공백 → "32468" / vErrMsg 공백 → "32469" / vLnggDcd 공백 → "00001" (eflg"1", FAIL)
  ├─ INSERT TB_FXB_SYG003M (errCd, lnggDcd, errMsg, 최종수정자=usid[6], SYSDATE)
  │     SQLCODE==-1(중복) → "00108" / 그외 오류 → "00105" (eflg"1", FAIL)
  └─ 정상 → grid01_cnt=1, eflg"0", e_msg "10150", RET_SUCCESS

W1500A02_F3()  수정
  ├─ 유효성: vErrCd/"32468", vErrMsg/"32469", vLnggDcd/"00001" (F2 와 동일)
  ├─ UPDATE TB_FXB_SYG003M SET FX_ERMG_CON=:vErrMsg WHERE errCd AND lnggDcd
  │     NO_DATA_FOUND → "00107"(정보없음) / SQLCODE!=0 → "00106" (eflg"1", FAIL)
  └─ 정상 → grid01_cnt=1, eflg"0", e_msg "10150", RET_SUCCESS

W1500A02_F4()  삭제
  ├─ 유효성: vErrCd 공백 → "32468" / vLnggDcd 공백 → "00001" (eflg"1", FAIL)  (errMsg 미검증)
  ├─ DELETE TB_FXB_SYG003M WHERE errCd AND lnggDcd   ← SQLCODE 미검사
  └─ 무조건 eflg"0", e_msg "10150", RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분(`prGb`)**: `'1'`~`'4'` 만 유효, 그 외는 커밋 없이 "작업구분 오류" 즉시 반환.
- **F1 동적 필터**:
  - `errCd LIKE '%' || :vErrCd || '%'` (부분일치).
  - `lnggDcd LIKE DECODE(:vLnggDcd,'0','',:vLnggDcd) || '%'` — `'0'` 입력 시 전체 언어, 그 외 접두 일치.
  - `UPPER(FX_ERMG_CON) LIKE '%' || UPPER(:vErrMsg) || '%'` (대소문자 무시 부분일치).
  - 페이징 키 `EAI_LNK_EROR_ID || LNGG_DCD > :vKeyv`, `ORDER BY EAI_LNK_EROR_ID, LNGG_DCD` (전방 스캔).
- **F1 페이징**: `max_row = MAX_RRB / sizeof(orec[0])`. `ii==max_row` 초과분 1건을 fetch 해 다음존재(`i_isnext`) 만 판정하고 저장하지 않음. 다음존재 시 `content_set_nkey(vKeyv)`.
- **입력 유효성(F2/F3)**: `errCd`, `errMsg`, `lnggDcd` 공백 불가. F4 는 `errCd`, `lnggDcd` 만 검사.
- **트랜잭션**: 각 서브함수 RET_SUCCESS → `l_dbcommit`, RET_FAILURE → `l_dbrollback` (진입 콜백에서 일괄, AUTOCOMMIT 아님).
- **에러/메시지 코드(e_msg.code → mesg, eflg)**:

| 상황 | code | eflg | 함수 |
|---|---|---|---|
| 조회 계속(NEXT) | 91002 | 0 | F1 |
| 조회 완료 | 91003 | 0 | F1 |
| 자료 없음 | 91001 | 0 | F1 |
| 커서 DECLARE/OPEN/FETCH 오류 | 10118 | 1 | F1 |
| 등록/수정/삭제 성공 | 10150 | 0 | F2/F3/F4 |
| 이미 등록됨(중복 -1) | 00108 | 1 | F2 |
| 등록 오류 | 00105 | 1 | F2 |
| 등록된 정보 없음 | 00107 | 1 | F3 |
| 수정 오류 | 00106 | 1 | F3 |
| 에러코드 미입력 | 32468 | 1 | F2/F3/F4 |
| 에러메시지 미입력 | 32469 | 1 | F2/F3 |
| 언어구분 미입력 | 00001 | 1 | F2/F3/F4 |

| `.h` `orec[1]` (L42) | 1건 고정처럼 선언 | grid. 실제로 `max_row` 만큼 채우고 `sndl` 을 `(ii-1)*sizeof(orec[0])` 만큼 확장 |

| F1 OPEN NO_DATA_FOUND (L230) | OPEN 시 무데이터 처리 | Oracle 커서 OPEN 은 통상 NO_DATA_FOUND 를 리턴하지 않음 → 사실상 dead code (무데이터는 FETCH 루프에서 걸림) |

| F1 FETCH 오류 로그 (L269) | — | 로그 문자열이 `"W3300S99_C1 CURSOR Fetch Error"` (다른 서비스명 복붙 오기) |
| F1 no-data 경로 sndl (L327) | — | `ii==0` 이면 `sndl += (ii-1)*sizeof(orec[0])` 가 음수 → 레코드 한 개분 감산 (엣지, 무데이터 시 grid 만 반환되나 sndl 계산 상 주의) |
| F2/F3/F4 로컬 `vPrGb`,`vSysLastPrcssYms` 선언 | 사용 의도 | 선언만 되고 사용 안 함 (dead variable) |
