## 3. 구현 흐름 (`.pc`)

```
W1251A01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1251A01_F1(...)   ← 등록 (INSERT)
  │     '2' → retc = W1251A01_F2(...)   ← 수정 (UPDATE)
  │     '3' → retc = W1251A01_F3(...)   ← 삭제 (USE_YN='N' UPDATE, 논리삭제)
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? (eflg="0", l_dbcommit()) : (eflg="1", l_dbrollback())
     → fnSetMsg(&e_msg, comhdr_o)   (e_msg.code → COMHDR mesg 변환)   (명시적 커밋, AUTOCOMMIT 아님)

W1251A01_F1()                   ← 등록
  ├─ getmid/getmod, memset(mod)
  ├─ yprintf + l_rtrim → vComCdNm/vCmcdKrnNm/vUseYn/vComCdDescCon/vFxSysLsmdId
  ├─ 유효성: vComCdNm 빈값 → e_msg "10301", RET_FAILURE
  │          vCmcdKrnNm 빈값 → e_msg "10302", RET_FAILURE
  ├─ INSERT INTO TB_FXB_CMC002M
  │      (COM_CD_NM, CMCD_KRN_NM, USE_YN, COM_CD_DESC_CON, FX_SYS_LSMD_ID, FX_SYS_LSMD_TS)
  │      VALUES (TRIM(:vComCdNm), TRIM(:vCmcdKrnNm), 'Y', :vComCdDescCon, :vFxSysLsmdId, SYSDATE)
  │      SQLCODE==DUP_KEY → e_msg "20002"(중복), RET_FAILURE
  └─ 정상 → e_msg "10150", RET_SUCCESS

W1251A01_F2()                   ← 수정
  ├─ getmid/getmod, memset(mod)
  ├─ yprintf + l_rtrim → 동일 5개 바인딩
  ├─ 유효성: vComCdNm 빈값 → "10301" / vCmcdKrnNm 빈값 → "10302", RET_FAILURE
  ├─ UPDATE TB_FXB_CMC002M
  │      SET CMCD_KRN_NM=TRIM(:vCmcdKrnNm), USE_YN=NVL(:vUseYn,'N'),
  │          COM_CD_DESC_CON=:vComCdDescCon, FX_SYS_LSMD_ID=:vFxSysLsmdId, FX_SYS_LSMD_TS=SYSDATE
  │      WHERE COM_CD_NM=TRIM(:vComCdNm)
  │      SQLCODE==NO_DATA_FOUND → e_msg "10221"(수정할 자료 없음), RET_FAILURE
  └─ 정상 → e_msg "10150", RET_SUCCESS

W1251A01_F3()                   ← 삭제 (논리삭제)
  ├─ getmid/getmod, memset(mod)
  ├─ yprintf + l_rtrim → vComCdNm/vFxSysLsmdId (2개만)
  ├─ 유효성: vComCdNm 빈값 → "10301", RET_FAILURE
  ├─ UPDATE TB_FXB_CMC002M
  │      SET USE_YN='N', FX_SYS_LSMD_ID=TRIM(:vFxSysLsmdId), FX_SYS_LSMD_TS=SYSDATE
  │      WHERE COM_CD_NM=RTRIM(:vComCdNm)
  │      SQLCODE==NO_DATA_FOUND → e_msg "10307"(삭제대상 없음), RET_FAILURE
  │      SQLCODE!=0            → e_msg "10122"(처리중 오류), RET_FAILURE
  └─ 정상 → e_msg "10150", RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업 분기**: `mid->prGb[0]` 로 F1(등록)/F2(수정)/F3(삭제) 라우팅. 미지정 값은 "작업구분 오류" 로 즉시 반환(eflg 미세팅, MSG_NOWORK 로그).
- **키**: 세 분기 모두 `comCdNm`(공통코드명 `COM_CD_NM`) 가 식별 키. 등록은 PK, 수정/삭제는 `WHERE COM_CD_NM = TRIM/RTRIM(:vComCdNm)`.
- **입력 유효성**: F1/F2 는 `comCdNm`+`cmcdKrnNm` 빈값 거부(`10301`/`10302`), F3 는 `comCdNm` 만 검증. `useYn`/`comCdDescCon`/`fxSysLsmdId` 는 빈값 검증 없음.
- **USE_YN 처리 차이**: F1 은 입력 `useYn` 을 무시하고 `'Y'` 리터럴 저장, F2 는 `NVL(:vUseYn,'N')` 로 입력 반영, F3 은 `'N'` 리터럴(논리삭제). → 등록 시 사용여부 지정 불가(§6).
- **삭제 시맨틱**: F3 은 행을 지우지 않고 `USE_YN='N'` 으로 비활성화하는 논리삭제. 후속 조회에서 `USE_YN='Y'` 필터로 걸러내는 것을 전제(§6).
- **트랜잭션**: 각 F 함수 성공 시 상위 `W1251A01()` 가 `eflg="0"`+`l_dbcommit`, 실패 시 `eflg="1"`+`l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드** (e_msg.code → fnSetMsg → COMHDR mesg):
  - 성공: `10150` (정상처리, 조회로 확인).
  - 공통코드명 빈값: `10301`.
  - 공통코드한글명 빈값: `10302` (등록/수정).
  - 등록 중복키: `20002` (DUP_KEY).
  - 수정 대상 없음: `10221` (NO_DATA_FOUND).
  - 삭제 대상 없음: `10307` (NO_DATA_FOUND).
  - 삭제 중 기타 SQL 오류: `10122`.
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (e_msg 미경유, eflg 미세팅).
- **결과 전달**: 출력 레코드 없음(dummy 1B). COMHDR(`eflg`/`mesg`) 로만 결과 전달. WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
