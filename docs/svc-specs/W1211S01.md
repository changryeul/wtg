# 서비스 명세 — `W1211S01` (부점정보인터페이스)

## 2. I/O 전문 (`.h`)

**Input `W1211S01_I`** (총 57B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" 후 즉시 return |
| `mngmBrcd` | 6 | 관리부점코드 — 부분일치(LIKE) 검색어 |
| `mngmBrm` | 50 | 관리부점명 — 부분일치(LIKE) 검색어 |

**Output `W1211S01_O`** (base = 6 + orec[1]×56 = 62B) — **grid (가변 orec)**

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (조회된 행 수, `zredit` 우측정렬 숫자) |
| `orec[].mngmBrcd` | 6 | 관리부점코드 |
| `orec[].mngmBrm` | 50 | 관리부점명 |

> `orec` 는 `.h` 상 `[1]` 이나 실 응답은 `content->sndl` 을 `(ii-1)×sizeof(orec[0])`
> 만큼 늘려 조회 건수만큼 가변 전송한다 (grid). 페이징 계속 여부는 COMHDR NEXT KEY
> 및 결과코드(91002/91003)로 전달.

## 3. 구현 흐름 (`.pc`)

```
W1211S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  │                             comhdr_o->rtyp = "M"
  ├─ switch(mid->prGb)
  │     '1' → retc = W1211S01_F1(...)     ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
       → fnSetMsg(&e_msg, comhdr_o)  (e_msg.code → COMHDR mesg/rcod)

W1211S01_F1()                   ← 조회 (grid)
  ├─ getmid/getmod, sndl += W1211S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])      ← 1 페이지 최대 행수
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT → content_get_nkey → vkeyv
  │      K_PREV → content_get_pkey → vkeyv   (조건은 NEXT 와 동일 '>' — §6)
  │      default(최초) → vkeyv="0"
  │      → l_rtrim(vkeyv)
  ├─ 입력 바인딩: yprintf + l_rtrim → vmngmBrcd / vmngmBrm  (실 유효성 검증 없음 — §6)
  ├─ DECLARE W1211S01_C1 CURSOR
  │      SELECT MNGM_BRCD, MNGM_BRM, MNGM_BRCD AS NEXT_KEY
  │        FROM TB_FXB_CMC001F
  │       WHERE (MNGM_BRCD LIKE '%'||:vmngmBrcd||'%' OR MNGM_BRM LIKE '%'||:vmngmBrm||'%')
  │         AND USE_YN='Y'
  │         AND MNGM_BRCD > :vkeyv           ← 페이징 커서 (오름차순)
  │       ORDER BY MNGM_BRCD
  ├─ OPEN → SQLCODE!=0 : NO_DATA_FOUND→91001 / 기타→10118, RET_FAILURE
  ├─ for(ii=0;;ii++) FETCH INTO cmngmBrcd, cmngmBrm, ckeyv
  │      SQLCODE : NO_DATA_FOUND→break / 기타→CLOSE+10122, RET_FAILURE
  │      ii == max_row → i_isnext++ ; break   ← 다음 페이지 존재 표시(행 미저장)
  │      mod->orec[ii] = {cmngmBrcd, cmngmBrm}
  │      ckeyv = cmngmBrcd ; vkeyv = ckeyv(%.64s)   ← NEXT KEY 갱신
  ├─ CLOSE W1211S01_C1
  ├─ ii==0 && i_isnext==0 → 91001 "해당 자료 없음", RET_FAILURE
  │  else i_isnext==1     → content_set_nkey(vkeyv) + 91002 "조회 계속"
  │       else            → 91003 "조회 완료" + grid01_cnt=ii
  └─ grid01_cnt=ii ; sndl += (ii-1)*sizeof(orec[0]) ; RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC001F` | 관리부점정보 (부점 마스터, 추정 — SQL 내 별칭주석 없음) | `CMC001F` | **SELECT** (커서) | `USE_YN='Y'` 인 관리부점을 `MNGM_BRCD/MNGM_BRM` LIKE 검색, `MNGM_BRCD > vkeyv` 커서로 페이징 |

*DML 요약: `TB_FXB_CMC001F` 단일 테이블 커서 SELECT 1문. JOIN / INSERT / UPDATE / DELETE / 공통함수 경유 접근 없음. 데이터 변경은 없으나 규약상 성공 시 `l_dbcommit`, 실패 시 `l_dbrollback`.*

## 5. 로직 / 필터 규칙

- **DML**: 커서 SELECT 1문(`TB_FXB_CMC001F`)만. 조인/변경 없음.
- **동적 필터**:
  - `MNGM_BRCD LIKE '%'||vmngmBrcd||'%' OR MNGM_BRM LIKE '%'||vmngmBrm||'%'` (두 조건 **OR**).
  - `USE_YN='Y'` (사용중 부점만).
  - `MNGM_BRCD > :vkeyv` 페이징 커서 (최초 vkeyv="0").
  - `ORDER BY MNGM_BRCD` 오름차순.
- **페이징**: `max_row = MAX_RRB / sizeof(orec[0])` 건마다 `i_isnext` 세워 다음 존재 표시.
  다음 페이지는 `content_set_nkey` 로 마지막 `MNGM_BRCD` 를 NEXT KEY 로 전달, 재호출 시
  `K_NEXT` 로 이어짐. **K_PREV(이전)도 조건이 `>`(forward) 라 실제 역방향 페이징 미동작 — §6.**
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드** (`e_msg.code` → `fnSetMsg` → COMHDR):
  - 성공(완료): `91003` "조회가 완료되었습니다.", `eflg=0`.
  - 성공(계속): `91002` "조회가 계속됩니다.", NEXT KEY 세팅, `eflg=0`.
  - 자료 없음: `91001` "해당하는 자료가 없습니다.", `eflg=1`(RET_FAILURE→rollback).
  - 커서 OPEN 오류: `10118` "자료 조회중 오류".
  - FETCH 오류: `10122` "자료 처리중 오류".
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (eflg 미세팅, e_msg 미사용).
- **결과 전달**: grid orec + `grid01_cnt` + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는
  `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).

## 6. 주의 — 문서/코드 불일치 (마이그레이션 시 확인)

| 위치 | 헤더/주석 표기 | 실제(`.pc`) |
|---|---|---|
| `.pc` L41 변수 주석 | `mod` = "W1211S01 의 입력값 주소" | 출력값(Output) 주소 — 복붙 오기 |
| L193 필터 (LIKE OR) | 관리부점코드 **와** 명으로 검색 의도 | `OR` + 빈 검색어는 `'%%'`(전건 매치) → **한쪽만 입력하면 다른 쪽 `'%%'` 가 항상 참이 되어 필터가 무력화(전건 반환)**. 실질 필터는 두 필드 모두 입력해야 의미. |
| L156 K_PREV(이전) | 이전 페이지 조회 | 커서 조건이 NEXT 와 동일한 `MNGM_BRCD > :vkeyv` (forward) → 역방향 페이징 미동작. pkey 를 forward 키로 사용. |
| L167 유효성 섹션 주석 | "입력항목 유효성 check" | 실제 NULL/공백 검증 없음 — `yprintf/l_rtrim` 만 수행 |
| L199 OPEN 후 `NO_DATA_FOUND` 분기 | OPEN 시 데이터 없음 처리 | 커서 OPEN 은 통상 `NO_DATA_FOUND` 를 내지 않음 → 사실상 dead branch (실 없음 판정은 L269 fetch 루프 후 `ii==0`) |
| L227/L254 `ckeyv` | FETCH 한 NEXT_KEY 사용 | fetch 직후 `strcpy(ckeyv, cmngmBrcd)` 로 덮어씀 — `NEXT_KEY(=MNGM_BRCD)` 와 동일값이라 무해하나 중복 |
| L287 / L294 `grid01_cnt` | — | 완료 분기(L287)와 공통(L294)에서 `zredit(grid01_cnt, ii)` 이중 세팅 (동작 무해) |
| L97 로그 | 오류여부 = `comhdr_o->eflg` | 출력 폭에 `sizeof(comhdr_o->pflg)` 사용 — 로그 폭 산정 복붙 버그(동작 무해) |
| `.h` L16 vs `.pc` L8 | 최초작성 2023-10-20 | `.pc` 는 2023-10-23 — 날짜 불일치(무해) |
| 미사용 지역변수 | — | `i, cnt, editb, retc, error_f, i_sndcnt, log_m` 등 다수 선언만 하고 미사용 |

> WTG 는 `.h` 를 파싱해 필드명/크기만 취하고 전문을 통과시키므로 위 주석/데드코드가
> 동작에 영향 없다. 다만 **필터 OR 무력화**와 **K_PREV 미동작**은 실제 조회 결과에
> 영향을 주는 로직 버그이므로 마이그레이션/재사용 시 반드시 검토할 것.
