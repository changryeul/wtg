## 3. 구현 흐름 (`.pc`)

```
W1253A02()                        진입 dispatch (Description="조회" 주석이나 실제는 등록수정 라우팅만)
  ├ prthdr / getmid / getmod      COMHDR·mid·mod 주소 획득
  ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  ├ switch(mid->prGb[0])
  │    case '2' → W1253A02_F2()   등록수정
  │    default  → comhdr_o.mesg="작업구분 오류." + return (commit/rollback 안 탐)
  ├ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
  ├ fnSetMsg(&e_msg, comhdr_o)    e_msg.code → 응답 메시지 세팅
  └ return content->sndl

W1253A02_F2()                     등록수정 본체
  ├ getmid / getmod / memset(mod)
  ├ vFxMsgPtrnDcd = mid->fxMsgPtrnDcd (atoi>0 이면 %02d), hls_upd_id = comhdr_i->usid
  ├ grid01_cnt = atoi(mid->grid01_cnt)
  ├ strlen(vFxMsgPtrnDcd)<=0 → e_msg.code="10527" + RET_FAILURE
  ├ [DELETE] TB_FXB_CMC015M WHERE FX_MSG_PTRN_DCD=:vFxMsgPtrnDcd  (기존 대상 전삭제)
  │    SQLCODE!=0 && !=NO_DATA_FOUND → e_msg.code="10535" + RET_FAILURE
  ├ for ii in 0..grid01_cnt-1
  │    ├ yprintf/l_rtrim 각 필드 정리
  │    ├ strlen(vSndgTgtEmn)<=0 → e_msg.code="10536" + RET_FAILURE
  │    └ [INSERT] TB_FXB_CMC015M (cpn 은 REPLACE '-' 제거, upd_id=usid, ts=SYSDATE)
  │         SQLCODE!=SQL_SUCCESS → e_msg.code="10537" + RET_FAILURE
  ├ zredit(mod->grid01_cnt, 1)    결과 flag=1
  └ e_msg.code="10150" + RET_SUCCESS
```

트랜잭션: F2 내부에서 commit/rollback 하지 않고 `RET_SUCCESS/RET_FAILURE` 만 반환.
상위 `W1253A02()` 가 결과에 따라 `l_dbcommit()`/`l_dbrollback()` 를 한 번 수행 (전삭제+전건 INSERT 가 단일 트랜잭션으로 원자 처리).

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC015M` | SMS메시지발송대상직원목록 | (없음) | DELETE | 해당 `FX_MSG_PTRN_DCD` 의 기존 발송대상 전삭제 |
| `TB_FXB_CMC015M` | SMS메시지발송대상직원목록 | (없음) | INSERT | 그리드 건수만큼 발송대상 직원 재등록 |

*DML 요약: 커서 없음. `TB_FXB_CMC015M` 에 대해 유형코드 기준 DELETE 1회 후, 그리드 loop 안에서 INSERT 를 반복하는 전체 교체(replace-all) 패턴. 공통함수 경유 간접 테이블 접근 없음.*

INSERT 컬럼: `FX_MSG_PTRN_DCD, SNDG_TGT_EMN, CPN, EML_ID, EML_DMN_NM, EMP_NM, EMP_BRCD, FX_SYS_LSMD_ID(=usid), FX_SYS_LSMD_TS(=SYSDATE)`.

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='2'` 만 등록수정 처리. 그 외 값은 COMHDR 에 "작업구분 오류." 세팅 후 즉시 반환(DB 무접근).
- **유형코드 정규화**: `fxMsgPtrnDcd` 를 `atoi` 하여 0 초과일 때만 `%02d` 로 재포맷. 빈값(strlen<=0)이면 등록 진행 안 함.
- **전체 교체**: 특정 `FX_MSG_PTRN_DCD` 의 기존 행을 모두 DELETE 한 뒤 그리드 전건을 INSERT — 부분 갱신이 아니라 유형별 목록 스냅샷 교체.
- **행 검증**: 각 그리드 행에서 `sndgTgtEmn`(직원번호) 이 공백이면 즉시 실패 → 이미 수행한 DELETE/INSERT 는 상위에서 ROLLBACK.
- **핸드폰번호 정제**: INSERT 시 `REPLACE(:vCpn,'-','')` 로 하이픈 제거 후 저장.
- **변경이력**: `FX_SYS_LSMD_ID` = `comhdr_i->usid`, `FX_SYS_LSMD_TS` = `SYSDATE`.
- **에러 코드 (e_msg.code → fnSetMsg)**:
  - `10527` — FX메시지유형구분코드 미입력/유효성 오류
  - `10535` — SMS메시지설정(발송대상목록) 삭제중 오류
  - `10536` — 직원번호 미입력
  - `10537` — SMS메시지발송대상직원 등록중 오류
  - `10150` — 정상 처리 완료
- **eflg**: 성공 `0` / 실패 `1`.
