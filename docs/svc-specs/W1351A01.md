# 서비스 명세 — `W1351A01` (외환중개사 기본)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1351A01` |
| 서비스 이름 | 외환중개사 기본 (등록/수정/삭제) |
| 유형 | 복합 (처리구분 분기 — 등록/수정/삭제). 실 DML 은 INSERT 1 + UPDATE 2 (삭제는 soft-delete UPDATE) |
| 구현 | `win/src/trn/W1350/W1351A01.pc` (515 라인) |
| I/O 헤더 | `win/src/inc/trn/W1351A01.h` |
| 라우팅 alias | `W1351A01` |
| 모듈 | W1350 |
| 채널 | 매매 엔진 AP (broker 경유, WTG `/v1/tx`) |

핵심: `mid->prGb[0]` 처리구분에 따라 등록(`W1351A01_F1`) / 수정(`W1351A01_F2`) / 삭제(`W1351A01_F3`) 로 분기하고, 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()`. 단일 테이블 `TB_FXB_CMG001M`(외환중개사기본) 만 다룬다. 조회(SELECT) 는 없다.

## 2. I/O 전문 (`.h`)

### Input `W1351A01_I`

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 처리구분 — 실제 코드: `1`=등록, `2`=수정, `3`=삭제 (헤더 주석과 다름, §6 참조) |

| `fnshYmd` | 8 | 종료년월일 (미입력 시 `99991231`) |
| `useYn` | 1 | 사용여부 (미입력 시 `Y`) |
| `oprtYn` | 1 | 운영여부 (미입력 시 `Y`) |

| `trnmTrnTgisId` | 50 | 송신거래대상기관ID |
| `trnmTrnSubTgisId` | 50 | 송신거래서브대상기관ID |

| `rctxSubTgisId` | 50 | 수신거래서브대상기관ID |

`FX_SYS_LSMD_ID`(최종변경ID) 는 입력 전문이 아니라 `comhdr_i->usid`(공통헤더 사용자ID) 에서 채운다.

### Output `W1351A01_O`

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `dummy` | 1 | Dummy — 본문 무의미. 등록/수정/삭제 결과는 COMHDR(`eflg`/`rtyp`/`mesg`/`rcod`) + e_msg 로 전달 |

grid/orec 가변영역 없음 (단건 처리, 출력 body 없음).

## 3. 구현 흐름 (`.pc`)

```
W1351A01()                                 // 진입 dispatch (헤더 주석은 "조회"지만 실제는 등록/수정/삭제 라우터)
 ├ prthdr / getmid / getmod               // 공통헤더 + 입력 mid + 출력 mod 주소 확보
 ├ e_msg 기본설정 (pname/ltyp=KR/mtyp=S/rtyp=M)
 ├ switch(mid->prGb[0])
 │    case '1' → W1351A01_F1()  // 등록
 │    case '2' → W1351A01_F2()  // 수정
 │    case '3' → W1351A01_F3()  // 삭제
 │    default  → comhdr_o->mesg "작업구분 오류." + MSG_NOWORK, 즉시 return (commit/rollback 안 탐)
 ├ retc == RET_SUCCESS → eflg="0" + l_dbcommit()
 │            else      → eflg="1" + l_dbrollback()
 ├ fnSetMsg(&e_msg, comhdr_o)             // e_msg.code → 사용자 메시지/응답코드 세팅
 └ return content->sndl

W1351A01_F1() 등록
 ├ getmid/getmod, memset mod
 ├ yprintf(mid→ 로컬 v*) + l_rtrim, vFxSysLsmdId ← comhdr_i->usid
 ├ if strlen(vFdmId)<=0 → e_msg.code=10205, RET_FAILURE
 ├ EXEC SQL INSERT INTO TB_FXB_CMG001M (...15 col..., FX_SYS_LSMD_TS=SYSDATE)
 │    NVL 기본값: OPN_YMD=SYSDATE, FNSH_YMD=99991231, USE/OPRT/HLDD_YN=Y
 ├ SQLCODE==DUP_KEY → e_msg.code=10223 (중복), RET_FAILURE
 ├ SQLCODE!=0       → e_msg.code=10122 (오류), RET_FAILURE
 └ 성공 → e_msg.code=10150, RET_SUCCESS

W1351A01_F2() 수정
 ├ (F1 과 동일한 입력 전처리)
 ├ if strlen(vFdmId)<=0 → 10205, RET_FAILURE
 ├ EXEC SQL UPDATE TB_FXB_CMG001M SET (13개 컬럼 + FX_SYS_LSMD_TS=SYSDATE) WHERE FDM_ID=:vFdmId
 ├ SQLCODE==NO_DATA_FOUND → 10221 (수정할 데이터 없음), RET_FAILURE
 ├ SQLCODE!=0             → 10122, RET_FAILURE
 └ 성공 → 10150, RET_SUCCESS

W1351A01_F3() 삭제 (soft delete)
 ├ vFdmId, vFxSysLsmdId(←usid) 만 처리
 ├ if strlen(vFdmId)<=0 → 10205, RET_FAILURE
 ├ EXEC SQL UPDATE TB_FXB_CMG001M SET USE_YN='N', FX_SYS_LSMD_ID, FX_SYS_LSMD_TS=SYSDATE WHERE FDM_ID=:vFdmId
 ├ SQLCODE==NO_DATA_FOUND → 10208 (NOTFOUND), RET_FAILURE
 ├ SQLCODE!=0             → 10122, RET_FAILURE
 └ 성공 → 10150, RET_SUCCESS
```

커서(DECLARE/OPEN/FETCH/CLOSE) 는 사용하지 않는다. 트랜잭션 경계는 dispatcher `W1351A01()` 한 곳(commit/rollback)에서만 결정된다.

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-------|-------|-----|-----|-----|
| `TB_FXB_CMG001M` | 외환중개사기본 | `CMG001M` | INSERT | 등록(F1) — 신규 외환중개사 1건 삽입 |
| `TB_FXB_CMG001M` | 외환중개사기본 | `CMG001M` | UPDATE | 수정(F2) — `FDM_ID` 기준 속성 갱신 |
| `TB_FXB_CMG001M` | 외환중개사기본 | `CMG001M` | UPDATE | 삭제(F3) — `USE_YN='N'` soft delete |

*DML 요약: INSERT ×1 (등록), UPDATE ×2 (수정 / 삭제=사용여부 N). 물리 DELETE 없음. SELECT 없음. 공통함수 경유 간접 테이블 접근 없음 (`fnSetMsg` 은 메시지 세팅만).*

## 5. 로직 / 필터 규칙

- **처리구분 분기**: `mid->prGb[0]` — `1`=등록 / `2`=수정 / `3`=삭제. 그 외는 "작업구분 오류." 메시지로 즉시 반환하며 commit/rollback 을 타지 않는다.
- **입력 전처리**: 모든 입력을 `yprintf` 로 로컬 host 변수에 복사 → `l_rtrim` 우측 공백 제거. SQL 안에서도 `RTRIM`/`TRIM` 재적용.
- **필수 검증**: 세 경로 모두 `strlen(vFdmId) <= 0` 이면 `10205`(외환중개사ID 확인) 반환.
- **기본값(NVL)**: 등록/수정 시 `OPN_YMD`=SYSDATE, `FNSH_YMD`=`99991231`, `USE_YN/OPRT_YN/HLDD_YN`=`Y` 로 보정.
- **감사 컬럼**: `FX_SYS_LSMD_ID` ← `comhdr_i->usid`, `FX_SYS_LSMD_TS` ← SYSDATE (세 경로 공통).
- **트랜잭션**: dispatcher 에서 `RET_SUCCESS` → `l_dbcommit()` + `eflg="0"`, 그 외 → `l_dbrollback()` + `eflg="1"`. AUTOCOMMIT 아님.
- **에러코드(e_msg.code → fnSetMsg)**:

| code | 상황 | 발생 위치 |
|-----|-----|-----|
| `10150` | 정상 처리 (MSG_REQUERY) | F1/F2/F3 성공 |
| `10205` | 외환중개사ID 미입력 | F1/F2/F3 검증 |
| `10223` | 등록 중복(DUP_KEY) | F1 |
| `10221` | 수정할 데이터 없음(NO_DATA_FOUND) | F2 |
| `10208` | 삭제 대상 없음(NO_DATA_FOUND) | F3 |
| `10122` | 그 외 SQL 오류 | F1/F2/F3 |
