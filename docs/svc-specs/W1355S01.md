# 서비스 명세 — `W1355S01` (외환중개사운영상태 조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1355S01` |
| 서비스 이름 | 외환중개사운영상태 조회 |
| 유형 | 조회 (단건 SELECT) |
| 구현 | `win/src/trn/W1350/W1355S01.pc` (258 라인) |
| I/O 헤더 | `win/src/inc/trn/W1355S01.h` (59 라인) |
| 라우팅 alias | `W1355S01` |
| 채널 | `ChannelWeb` (매매 svc, `/v1/tx` 경유) |

외환중개사(FX 중개사) 마스터 테이블에서 특정 중개사 1건의 **운영여부 + 운영여부명 + FX시스템 최종변경 ID/일시** 를 조회하는 단건 서비스. 그리드/페이징 없음.

## 2. I/O 전문 (`.h`)

**Input `W1355S01_I`**

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 작업구분 — `'1'` 만 유효(조회). 그 외 "작업구분 오류" |
| `fmdId` | 10 | 외환중개사ID — SQL `WHERE FDM_ID = :vFmdId` 의 **유일 필터** |
| `oprtYn` | 1 | 운영여부 — 읽어서 로그만 찍고 **쿼리에 미사용** (§6) |
| `fxSysLsmdId` | 30 | FX시스템최종변경ID — 읽어서 로그만 찍고 **쿼리에 미사용** (§6) |

**Output `W1355S01_O`** (단건, grid 아님)

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `oprtYn` | 1 | 운영여부 (`Y`/`N`) |
| `oprtYnNm` | 10 | 운영여부명 (`Y`→정상운영중, `N`→거래중단중, 그 외 원값) |
| `fxSysLsmdId` | 30 | FX시스템최종변경ID |
| `fxSysLsmdTs` | 20 | FX시스템최종변경일시 (`YYYY/MM/DD HH24:MI:SS`) |

## 3. 구현 흐름 (`.pc`)

```
W1355S01()                       // 진입 dispatch
  ├ prthdr / getmid / getmod     // 공통헤더 + I/O 구조체 주소 확보
  ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  ├ switch(mid->prGb[0])
  │    case '1' → W1355S01_F1()   // 조회 본체
  │    default  → comhdr_o.rtyp="M", mesg="작업구분 오류", return sndl
  ├ retc==RET_SUCCESS ? eflg="0" + l_dbcommit()
  │                    : eflg="1" + l_dbrollback()
  ├ fnSetMsg(&e_msg, comhdr_o)   // e_msg.code → 응답 메시지 세팅
  └ return content->sndl

W1355S01_F1()                    // 조회
  ├ getmid / getmod, content->sndl += W1355S01_O_SZ, memset(mod)
  ├ switch(content->keyc)         // K_NEXT/K_PREV/default → vKeyv 세팅
  │    (vKeyv l_rtrim 까지 하지만 쿼리에서 미사용 — 페이징 dead, §6)
  ├ yprintf/l_rtrim: vFmdId, vOprtYn, vFxSysLsmdId
  ├ EXEC SQL SELECT ... FROM TB_FXB_CMG001M WHERE FDM_ID=:vFmdId  // 단건
  │    ├ NO_DATA_FOUND → e_msg.code="91001", return RET_FAILURE
  │    └ 기타 SQLCODE≠0 → e_msg.code="10118", return RET_FAILURE
  ├ zsedit → mod->{oprtYn, oprtYnNm, fxSysLsmdId, fxSysLsmdTs}
  ├ e_msg.code="91003", content->sndl += W1355S01_O_SZ  // sndl 2회 가산(§6)
  └ return RET_SUCCESS
```

커서(DECLARE/OPEN/FETCH) 없음 — 단일 행 `SELECT ... INTO`. 순수 조회라 commit/rollback 은 형식적.

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-------|-------|-----|-----|-----|
| `TB_FXB_CMG001M` | 외환중개사 마스터 | `CMG001M` | SELECT | `FDM_ID` 로 중개사 1건의 운영여부 + FX시스템 최종변경 ID/일시 조회 |

*DML 요약: SELECT 1건. INSERT/UPDATE/DELETE 없음. 공통함수 경유 간접 테이블 접근 없음.*

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='1'` 만 조회 수행. 그 외는 즉시 "작업구분 오류" 응답.
- **필터**: `WHERE FDM_ID = :vFmdId` 단 하나. `oprtYn`/`fxSysLsmdId` 입력은 검증·조합에 미사용.
- **운영여부명 파생**: SQL `CASE` — `OPRT_YN='Y'`→`정상운영중`, `'N'`→`거래중단중`, 그 외 원값.
- **일시 포맷**: `TO_CHAR(FX_SYS_LSMD_TS,'YYYY/MM/DD HH24:MI:SS')`.
- **트랜잭션**: 성공 시 `l_dbcommit()`+eflg="0", 실패 시 `l_dbrollback()`+eflg="1". SELECT 전용이라 실질 효과 없음.
- **에러/메시지 코드**:
  | e_msg.code | 상황 | 의미 |
  |-----|-----|-----|
  | `91003` | 정상 조회 | 정상(재조회) — `MSG_REQUERY` |
  | `91001` | `NO_DATA_FOUND` | 해당하는 자료가 없습니다 |
  | `10118` | 기타 SQL 오류 | 자료 조회중 오류가 발생하였습니다 |
- e_msg: `ltyp=KR`, `mtyp=S`(FXON), `rtyp=M`(Message 전송). `fnSetMsg` 가 최종 comhdr_o 반영.
- **페이징**: `content->keyc`(K_NEXT/K_PREV)로 `vKeyv` 를 구성하나 쿼리에 반영 안 됨 → 단건 조회라 페이징 자체가 무의미 (dead).
