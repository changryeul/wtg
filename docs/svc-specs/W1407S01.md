# 서비스 명세 — `W1407S01` (상품/통화별 거래한도 조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 .pc 실 구현 기준이며, .h 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1407S01` |
| 이름 | 상품/통화별 거래한도 조회 |

| 모듈 | W1400 |
| 구현 | `win/src/trn/W1400/W1407S01.pc` (366 라인) |

FX 상품(SPT/FWD 등)과 통화별로 등록된 고객 거래한도(최소/최대 거래금액, 한도구분)를 코드명과 조인해 그리드로 조회한다. 처리구분(`prGb`)은 `1`(조회)만 유효하다.

## 3. 구현 흐름 (`.pc`)

```
W1407S01()  — 진입/디스패치
  ├ prthdr / getmid / getmod  (COMHDR·입출력 구조체 주소 획득)
  ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  ├ switch(mid->prGb[0])
  │     case '1' → retc = W1407S01_F1()          // 조회
  │     default  → comhdr_o.mesg="작업구분 오류." return sndl (커밋 안 함)
  ├ retc==RET_SUCCESS → eflg="0", l_dbcommit()
  │            else   → eflg="1", l_dbrollback()
  ├ fnSetMsg(&e_msg, comhdr_o)                    // e_msg.code → mesg/rcod 세팅
  └ return content->sndl

W1407S01_F1()  — 조회 본체
  ├ getmid/getmod, content->sndl += W1407S01_O_SZ, mod 0 초기화
  ├ max_row = MAX_RRB / sizeof(orec[0])           // 그리드 상한
  ├ 페이징 키 획득: keyc
  │     K_NEXT → content_get_nkey → vKeyv
  │     K_PREV → content_get_pkey → vKeyv
  │     default(최초) → vKeyv="0"
  ├ 입력 필터 rtrim: vFxPdcd, vCrcd
  ├ DECLARE W1407S01_C1 CURSOR (아래 §4·§5)
  ├ OPEN
  │     SQLCODE≠SUCCESS: NO_DATA_FOUND → e_msg.code="91001" return RET_SUCCESS
  │                      그 외        → e_msg.code="10118" return RET_FAILURE
  ├ for(ii=0;;ii++)
  │     FETCH → NO_DATA_FOUND break / 그 외 CLOSE + "10118" RET_FAILURE
  │     if(ii==max_row){ i_isnext++; break; }     // 초과 1건은 버리고 next 플래그만
  │     zsedit/sprintf 로 orec[ii] 채움, vKeyv = cKeyv (NEXT KEY 갱신)
  ├ CLOSE
  ├ 결과 판정
  │     ii==0 && i_isnext==0 → "91001" (자료 없음)
  │     i_isnext==1          → content_set_nkey(vKeyv) + "91002" (계속됨)
  │     그 외                → "91003" (완료) + grid01_cnt=ii
  ├ grid01_cnt=ii (재설정), content->sndl += (ii-1)*sizeof(orec[0])
  └ return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb[0]=='1'` 만 조회 수행. 그 외는 커서 진입 없이 "작업구분 오류." 메시지 후 반환(커밋/롤백 없음, `return content->sndl`).
- **동적 필터**: `FX_PDCD = NVL(RTRIM(:vFxPdcd), FX_PDCD)`, `CRCD = NVL(RTRIM(:vCrcd), CRCD)` — 입력이 빈값이면 자기 자신과 비교되어 전체 조회. `USE_YN='Y'` 고정.
- **정렬/페이징 키**: `NEXT_KEY = DECODE(FX_PDCD,'SPT','111','FWD','222','333') || TO_CHAR(CMG005M.RPSN_SQC)`. `WHERE NEXT_KEY > :vKeyv ORDER BY NEXT_KEY` 로 forward seek 페이징. 최초는 `vKeyv="0"`.
- **NEXT 판정**: 그리드에 `max_row`(=`MAX_RRB/sizeof(orec)`)건까지만 담고, `ii==max_row` 인 추가 1건 FETCH는 담지 않고 `i_isnext` 만 올린 뒤 break → 다음 페이지 존재 표식. `content_set_nkey(vKeyv)` 로 다음 키 반환.
- **트랜잭션**: 조회 전용이나 `retc` 에 따라 진입부에서 commit/rollback 명시 호출(AUTOCOMMIT 아님).
- **에러/결과 코드(e_msg.code → fnSetMsg)**:
  - `91001` — 자료 없음 (OPEN 시 NO_DATA_FOUND 또는 FETCH 0건)
  - `91002` — 조회 계속됨(다음 페이지 있음, nkey 세팅)
  - `91003` — 조회 완료
  - `10118` — 커서 OPEN/FETCH SQL 오류 → `RET_FAILURE` → rollback + eflg="1"
  - eflg: 정상 "0" / 비정상 "1"
