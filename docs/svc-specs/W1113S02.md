## 3. 구현 흐름 (`.pc`)
```
W1113S02()  (진입 콜백)
  ├ prthdr / getmid / getmod                 # COMHDR·입출력 주소 획득
  ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M) + comhdr_o->rtyp="M"
  ├ switch(mid->prGb[0])
  │    case '1' → W1113S02_F1()               # 조회
  │    default  → comhdr_o->mesg="작업구분 오류." return sndl
  └ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()
       (조회 전용이라 commit/rollback 은 형식적)

W1113S02_F1()  (조회)
  ├ getmid/getmod, content->sndl += W1113S02_O_SZ, memset(mod)
  ├ max_row = MAX_RRB / sizeof(orec[0])
  ├ content->keyc 로 vKeyv 세팅 (K_NEXT/K_PREV/최초="0") — 이후 쿼리에 미사용(§6)
  ├ vFxUserNo = rtrim(mid->fxUserNo)          # 미사용 더미
  ├ DECLARE W1113S02_C1  CURSOR (SELECT … FROM TB_FXB_CMG024M WHERE USE_YN='Y' ORDER BY PU_CD)
  │    SQLCODE!=0 → e_msg 10118, eflg=1 (return 없이 진행)
  ├ OPEN W1113S02_C1
  │    NO_DATA_FOUND → e_msg 91001, eflg=0, return RET_FAILURE
  │    기타 err     → e_msg 10118, eflg=1, return RET_FAILURE
  ├ for(ii=0; ; ii++)
  │    ├ 호스트변수 memset
  │    ├ FETCH … INTO (28개 컬럼)
  │    │    NO_DATA_FOUND → break
  │    │    기타 err → CLOSE, e_msg 10118, eflg=1, return RET_FAILURE
  │    ├ if(ii==max_row){ i_isnext++; break; }   # 페이지 초과
  │    ├ l_rtrim 전 컬럼 → zsedit(mod->orec[ii].*)
  │    └ sprintf(vKeyv,"%.64s",cKeyv)            # cKeyv 미초기화 사용(§6 버그)
  ├ CLOSE W1113S02_C1
  ├ 결과 분기
  │    ii==0 && i_isnext==0 → e_msg 91001 (자료 없음), eflg=0
  │    i_isnext==1          → content_set_nkey, e_msg 91002 (계속), eflg=0
  │    else                 → e_msg 91003 (완료), eflg=0, grid01_cnt=ii
  ├ CLOSE W1113S02_C1   (중복 CLOSE — §6)
  ├ eflg="0", grid01_cnt=ii
  └ content->sndl += (ii-1)*sizeof(orec[0]); return RET_SUCCESS
```

## 4. 사용 DB 테이블
| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-------|-------|-----|-----|-----|
| TB_FXB_CMG024M | PU기본 | CMG024M | SELECT | PU 마스터 전체 리스트 조회 (USE_YN='Y', PU_CD 정렬) |

*DML 요약: SELECT 1건 (커서 W1113S02_C1). INSERT/UPDATE/DELETE 없음. 조회 전용.*

## 5. 로직 / 필터 규칙
- **작업구분 분기**: `prGb='1'` 만 조회 실행. 그 외는 즉시 "작업구분 오류." 메시지로 반환.
- **필터**: 커서는 `USE_YN='Y'` 고정 필터 + `ORDER BY PU_CD`. 입력 `fxUserNo` 는 **필터에 반영되지 않음** — 헤더/서비스명("직원 PU리스트")과 달리 특정 직원/사용자 조건 없이 사용중 PU 전체를 반환한다.
- **페이지네이션**: `content->keyc` (K_NEXT/K_PREV/최초) 로 `vKeyv` 세팅. `ii==max_row` 도달 시 `i_isnext++` 후 중단하고 `content_set_nkey` 로 NEXT KEY 저장. 단 NEXT KEY 값(`vKeyv`)이 fetch 결과와 연결되지 않아 실제 이어보기는 동작 불명확(§6).
- **트랜잭션**: 진입 콜백에서 RET_SUCCESS 시 `l_dbcommit()`, 실패 시 `l_dbrollback()`. 조회 전용이라 형식적.
- **건수**: `grid01_cnt` 및 `content->sndl` 확장은 fetch 성공 행 수 `ii` 기준.
- **에러/메시지 코드**:
  | e_msg.code | 의미 | eflg |
  |-----|-----|-----|
  | 10118 | SQL 오류 (DECLARE/OPEN/FETCH) | 1 |
  | 91001 | 해당 자료 없음 (OPEN NO_DATA / ii==0) | 0 |
  | 91002 | 조회 계속됨 (NEXT KEY 존재) | 0 |
  | 91003 | 조회 완료 | 0 |

## 6. 주의 — 문서/코드 불일치
| 위치 | 헤더/주석 표기 | 실제(.pc) |
|-----|-------------|----------|
| 서비스명 / 필터 | "직원 PU리스트 조회" + 입력 `fxUserNo` | `fxUserNo` 를 쿼리에 전혀 쓰지 않음(주석 "입력값 사용하지 않음"). 실제는 USE_YN='Y' 인 PU 마스터 **전체** 조회 |
| `.h` orec.puLnknDcd | 주석 "PU연결구분코드" | SQL 은 `CMG024M.PU_PTRN_DCD` (PU패턴구분코드) 를 매핑 — 필드명/주석과 컬럼명 불일치 |
| orec.rprsYn (대표여부) | "대표여부" 컬럼처럼 표기 | DB 미관리, SELECT 에서 상수 `'Y'` 하드코딩 (.pc 주석 "대표여부는 관리하지 않음") |
| `.pc` L41 `mod` 선언 주석 | "입력값 주소" | 실제는 출력값(mod) — 복붙 오기 |
| `.pc` L446 `sprintf(vKeyv,"%.64s",cKeyv)` | NEXT KEY 세팅 의도 | `cKeyv` 는 선언만 되고 어디에도 값이 할당되지 않은 **미초기화 변수** → NEXT KEY 가 쓰레기값. 페이지 이어보기 사실상 오동작 (버그) |
| `.pc` keyc 페이지 처리 | K_NEXT/K_PREV 로 vKeyv 획득 | 획득한 vKeyv 를 커서 WHERE 절에 넣지 않음 → 이전/다음 방향 조회가 실제 커서에 반영되지 않음 |

| `.pc` L450, L477 | — | `EXEC SQL CLOSE W1113S02_C1` 가 중복 호출됨 (무해하나 불필요) |
| `.pc` DECLARE 에러처리 | — | DECLARE 실패 시 e_msg/eflg 만 세팅하고 `return` 없이 OPEN 으로 진행 (일관성 결여) |
