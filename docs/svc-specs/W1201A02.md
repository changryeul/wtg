## 3. 구현 흐름 (`.pc`)

```
W1201A02()  [dispatch]
  ├─ prthdr / getmid / getmod             (comhdr·mid·mod 주소 획득)
  ├─ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  ├─ switch(mid->prGb[0])
  │    ├ '1' → W1201A02_F1()  등록
  │    ├ '2' → W1201A02_F2()  삭제
  │    └ default → comhdr_o.mesg="작업구분 오류." + 즉시 return (commit/rollback 없음)
  ├─ retc==RET_SUCCESS → eflg="0" + l_dbcommit()
  │  else            → eflg="1" + l_dbrollback()
  ├─ fnSetMsg(&e_msg, comhdr_o)
  └─ return content->sndl

W1201A02_F1()  [등록]
  ├─ getmid/getmod, memset(mod)
  ├─ yprintf → l_rtrim (vFdmId/vCrcd/vFxngHlddMd/vFxHlddNm/vComntCon/vFxSysLsmdId)
  ├─ 유효성: vCrcd 빈값 → e_msg.code=32029 → RET_FAILURE
  │          vFxngHlddMd 빈값 → e_msg.code=10729 → RET_FAILURE
  ├─ INSERT INTO TB_FXB_CMG010M  (FDM_ID 리터럴 'FRX', FX_SYS_LSMD_TS=SYSDATE)
  │    ├ SQLCODE==DUP_KEY → e_msg.code=10223 → RET_FAILURE
  │    └ SQLCODE!=0       → e_msg.code=10122 → RET_FAILURE
  └─ e_msg.code=10150 → RET_SUCCESS

W1201A02_F2()  [삭제]
  ├─ getmid/getmod, memset(mod)
  ├─ yprintf → l_rtrim (vPrGb/vFdmId/vCrcd/vFxngHlddMd)
  ├─ 유효성: vCrcd 빈값 → e_msg.code=32029 → RET_FAILURE
  │          vFxngHlddMd 빈값 → e_msg.code=10729 → RET_FAILURE
  ├─ DELETE FROM TB_FXB_CMG010M WHERE FDM_ID='FRX' AND CRCD=:vCrcd AND FXNG_HLDD_MD=:vFxngHlddMd
  │    ├ SQLCODE==NO_DATA_FOUND → e_msg.code=32029 → RET_FAILURE
  │    └ SQLCODE!=0             → e_msg.code=10122 → RET_FAILURE
  └─ e_msg.code=10150 → RET_SUCCESS
```

commit/rollback 는 상위 dispatch(`W1201A02`) 한 곳에서만 수행 (서브함수는 RET_SUCCESS/FAILURE 만 반환).

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|--------|--------|------|------|------|
| `TB_FXB_CMG010M` | 통화별 고정휴일 (마스터) | CMG010M | INSERT (F1) / DELETE (F2) | 통화별 FX 고정휴일 등록·삭제 |

*DML 요약:*
- **F1 등록** — `INSERT INTO TB_FXB_CMG010M (FDM_ID, CRCD, FXNG_HLDD_MD, FX_HLDD_NM, COMNT_CON, FX_SYS_LSMD_ID, FX_SYS_LSMD_TS) VALUES ('FRX', :vCrcd, :vFxngHlddMd, :vFxHlddNm, :vComntCon, :vFxSysLsmdId, SYSDATE)` — `FDM_ID` 는 바인드 변수가 아닌 리터럴 `'FRX'`.
- **F2 삭제** — `DELETE FROM TB_FXB_CMG010M WHERE FDM_ID='FRX' AND CRCD=:vCrcd AND FXNG_HLDD_MD=:vFxngHlddMd` — 키(통화코드+고정휴일월일) 단건 삭제.
- 커서(DECLARE/OPEN/FETCH) 및 공통함수 경유 간접 접근 없음. 단일 INSERT/DELETE 뿐.

## 5. 로직 / 필터 규칙

- **작업구분(`prGb`) dispatch**: `1`=등록, `2`=삭제. 그 외 값은 "작업구분 오류." 메시지 후 commit/rollback 없이 즉시 반환.
- **트랜잭션**: AUTOCOMMIT 아님. 서브함수 RET_SUCCESS → `l_dbcommit()`, 그 외 → `l_dbrollback()`. `eflg` 는 성공 "0" / 실패 "1".
- **필수값 검증** (F1/F2 공통): `crcd`(통화코드) 빈값 → `32029`, `fxngHlddMd`(고정휴일월일) 빈값 → `10729`. 두 검증 모두 문자열 `strlen` 기준.
- **에러/메시지 코드 (e_msg.code)**:
  | code | 의미 | 발생 |
  |------|------|------|
  | 10150 | 정상 처리 (조회로 확인) | F1/F2 성공 |
  | 32029 | 통화코드 확인 / 삭제대상 없음 | crcd 빈값(F1·F2), 삭제 NO_DATA_FOUND |
  | 10729 | 고정휴일월일 확인 | fxngHlddMd 빈값(F1·F2) |
  | 10223 | 중복(DUP) | INSERT DUP_KEY |
  | 10122 | 자료 처리중 오류 | INSERT/DELETE 기타 SQL 오류 |
- 최종 메시지는 `fnSetMsg(&e_msg, comhdr_o)` 로 COMHDR 에 실려 전송.
