# 서비스 명세 — `W1114S01` (고객스프레드조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1114S01` |
| 이름 | 고객스프레드조회 (고객별 통화페어 SPT/FWD 영업점마진 조회) |
| 유형 | 조회 (SELECT 전용, DML 없음) |
| 구현 경로 | `win/src/trn/W1100/W1114S01.pc` (519 라인) |
| I/O 헤더 | `win/src/inc/trn/W1114S01.h` |
| 라우팅 alias | broker rkey = `content->rkey` (서비스 코드 그대로), 채널 라우팅은 admin 룰에서 resolve |
| 채널 | ChannelWeb (매매 AP, `/v1/tx` passthrough) |

전행고객실명대체번호(+CustomerId)를 키로, USE_YN='Y' 이고 CNH 를 포함하지 않는 모든 통화페어에 대해 그 고객의 SPT/FWD 영업점마진(고객 개별 마진 없으면 영업점 표준마진 fallback)을 그리드로 반환한다.

## 2. I/O 전문 (`.h`)

**Input `W1114S01_I`** (16 byte)
| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `wbcsRlnmAltrNo` | 16 | 전행고객실명대체번호 |

> 추가로 `.pc` 는 공통헤더 `comhdr_i->fdid` (CustomerId, 최대 12B) 를 읽어 `wbcsRlnmAltrNo + fdid` 를 이어붙여 조회키(`vWbcsRlnmAltrNoFndNo`)로 사용한다. 즉 실제 조회키는 입력 전문 필드 하나만으로 결정되지 않는다.

**Output `W1114S01_O`** (그리드)
| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `grid01_cnt` | 6 | 그리드 건수 (반환 orec 개수) |
| `orec[]` | 가변 | 아래 레코드의 반복 (헤더는 `orec[1]` 로 선언, 실제 개수는 `grid01_cnt`) |

orec 레코드 구조 (69 byte/건):
| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `crncPairId` | 7 | 통화페어ID |
| `sptBobmgPtrnDcd` | 1 | SPT 영업점마진유형구분코드 (1=금액형, else 율형) |
| `sptFxBymg` | 15 | SPT FX 매입마진 (`FM9999990.099999`) |
| `sptFxSlmg` | 15 | SPT FX 매도마진 |
| `fwdBobmgPtrnDcd` | 1 | FWD 영업점마진유형구분코드 |
| `fwdFxBymg` | 15 | FWD FX 매입마진 |
| `fwdFxSlmg` | 15 | FWD FX 매도마진 |

## 3. 구현 흐름 (`.pc`)
```
W1114S01()                       // 콜백 진입 (36~85)
 ├ prthdr/getmid/getmod          // 공통헤더·입출력 주소 획득
 ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp="M"
 ├ retc = W1114S01_F1(...)       // 조회 본체
 └ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   // (SELECT 전용이라 명목상)

W1114S01_F1()                    // 조회 (91~518)
 ├ getmid/getmod, sndl += W1114S01_O_SZ, mod 초기화
 ├ max_row = MAX_RRB / sizeof(orec[0])
 ├ switch(content->keyc): K_NEXT→nkey / K_PREV→pkey / else→"0"  ⇒ vKeyv  (※SQL 미사용)
 ├ vWbcsRlnmAltrNoFndNo = wbcsRlnmAltrNo + comhdr_i->fdid
 ├ DECLARE/OPEN W1114S01_C1  (CMG004M 통화페어 목록, ORDER BY RPSN_SQC)
 │    OPEN 실패 → e_msg.code=95000, eflg=1, return RET_FAILURE
 └ for(ii=0; ; ii++)
      ├ FETCH C1 INTO cCrncPairId
      │    NO_DATA_FOUND→break / 기타 SQLCODE→95000,eflg=1,return FAILURE
      ├ SELECT ... INTO cSpt*  (SPT 고객/표준 마진, CMG015M+CMG016M+CMG004M, outer join)
      │    SQLCODE!=0 → 로그만 (에러 반환 블록은 주석처리)
      ├ SELECT ... INTO cFwd*  (FWD 동일 구조)
      │    SQLCODE!=0 && !=NO_DATA_FOUND → 로그만 (반환 블록 주석처리)
      ├ if (ii == max_row) { i_isnext++; break; }   // row 미기록 후 탈출
      └ zsedit → mod->orec[ii].*
   ├ ii==0 && i_isnext==0 → e_msg.code=91001 (자료없음)
   ├ i_isnext==1 → content_set_nkey(vKeyv), 91002 (계속)
   ├ else → 91003 (완료), grid01_cnt=ii
   ├ eflg="0" (항상)  ; grid01_cnt=ii
   └ sndl += (ii-1) * sizeof(orec[0]) ; return RET_SUCCESS
```

## 4. 사용 DB 테이블
| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-------|-------|-----|-----|-----|
| `TB_FXB_CMG004M` | 통화페어정보(추정) | CMG004M | SELECT | 커서로 대상 통화페어 목록(USE_YN='Y', CNH 제외, RPSN_SQC 순) + 마진 조인 시 활성 페어 필터 |
| `TB_FXB_CMG015M` | 영업점표준마진정보 | CMG015M / STND_SPR | SELECT | 영업점 표준마진 (고객 개별마진 없을 때 fallback) |
| `TB_FXB_CMG016M` | 고객영업점마진기본 | CMG016M / CUST_SPR | SELECT | 고객 개별 영업점마진 (ALL 기본행 + 통화페어별 오버라이드, 마진 유효기간 SYSDATE BETWEEN MRGN_APSR_TS~MRGN_FNAP_TS) |

*DML 요약: SELECT 전용. INSERT/UPDATE/DELETE 없음. commit/rollback 은 프레임워크 컨벤션상 호출되나 조회라 상태 변경 없음.*

## 5. 로직 / 필터 규칙
- **대상 통화페어**: `CMG004M` 에서 `USE_YN='Y' AND CRNC_PAIR_ID NOT LIKE '%CNH%'`, `RPSN_SQC` 정렬. 페어마다 SPT·FWD 마진 2회 조회.
- **마진 결정 (SPT/FWD 각각)**: outer join 으로 `CUST_SPR`(고객마진) 우선, `CUST_SPR.BOBMG_PTRN_DCD IS NULL` 이면 `STND_SPR`(표준마진) 값 사용.
- **CUST_SPR 구성 (UNION ALL)**: (a) `CRNC_PAIR_ID='ALL'` 기본 마진행 중, 해당 페어에 개별 오버라이드가 없는 경우(NOT IN) + (b) `CRNC_PAIR_ID<>'ALL'` 페어별 마진행. 둘 다 SYSDATE 유효기간 필터.
- **마진값 산출**: `BOBMG_PTRN_DCD='1'` → SPR(스프레드) 컬럼, else → 율(RT) 컬럼. `TO_CHAR(..., 'FM9999990.099999')` 로 편집.
- **페이징**: `max_row = MAX_RRB/sizeof(orec)`. `ii==max_row` 도달 시 `i_isnext=1` 후 탈출. (단 §6 참조 — NEXT/PREV 키가 SQL 에 반영되지 않아 실질 페이징은 매번 동일 목록.)
- **트랜잭션**: `W1114S01_F1` 성공 → `l_dbcommit`, 실패 → `l_dbrollback`.
- **에러/메시지 코드**:
  | code | 의미 | eflg | 발생 |
  |-----|-----|-----|-----|
  | 95000 | 시스템(커서 OPEN/FETCH) 오류 | 1 | C1 OPEN/FETCH 실패 |
  | 91001 | 해당 자료 없음 | 0 | ii==0 (조회 0건) |
  | 91002 | 조회 계속 (NEXT) | 0 | max_row 초과 |
  | 91003 | 조회 완료 | 0 | 정상 종료 |
  - SPT/FWD INTO SELECT 의 SQLCODE 오류는 로그만 남기고 진행 (에러 반환 코드 20019 블록은 전부 주석처리됨).

## 6. 주의 — 문서/코드 불일치
| 위치 | 헤더/주석 표기 | 실제(.pc) |
|-----|-------------|----------|
| `W1114S01.h` L42 | `mod` 주석 "서비스 코드 W1114S01 의 **입력값** 주소" | 실제로는 출력값(mod) 주소. 복붙 오기. |
| Input 전문 vs 조회키 | `.h` Input 은 `wbcsRlnmAltrNo` 1개뿐 | `.pc` 는 여기에 `comhdr_i->fdid`(CustomerId) 를 strcat 한 `WBCS_RLNM_ALTR_NO_FND_NO` 로 조회. 전문 필드만 봐선 키를 재현 불가. |
| NEXT/PREV 페이징 | keyc 로 nkey/pkey 를 vKeyv 로 받고 91002 시 `content_set_nkey(vKeyv)` | `vKeyv` 가 어떤 SELECT/커서 WHERE 에도 쓰이지 않음. 커서는 항상 처음부터 재조회하며, set 되는 nkey 도 원본("0"/이전값) 그대로 → **다음페이지 조회가 동일 목록을 반복**하는 페이징 결함. |
| L516 `content->sndl += (ii-1)*sizeof(orec[0])` | — | `ii==0`(자료없음)일 때 `(ii-1)=-1` 로 orec 1건 크기만큼 **음수 가산** → 전송길이 언더플로 위험. 최소 baseline(orec[1]) 을 깎음. |
| SPT NOT IN 서브쿼리 L303 vs FWD L406 | 동일 로직이어야 할 SPT/FWD 두 블록 | SPT 는 `AND FX_PDCD = CMG016M.FX_PDCD`, FWD 는 `AND FX_PDCD = CMG015M.FX_PDCD`. 상관 서브쿼리에서 참조 alias 가 서로 달라 복붙 불일치(오버라이드 판정이 상품별로 다르게 동작할 소지). |
| SPT 상관 서브쿼리 L302 | outer `WBCS_RLNM_ALTR_NO_FND_NO = :vWbcsRlnmAltrNoFndNo` 로 필터 | NOT IN 서브쿼리는 `WBCS_RLNM_ALTR_NO`(FND_NO 없는 컬럼)로 상관 — 키 컬럼 혼용. 의도한 오버라이드 존재판정과 어긋날 수 있음. |
| SPT 에러체크 L335 vs FWD L438 | 동일해야 함 | SPT 는 `if (SQLCODE != 0)` 라 `NO_DATA_FOUND` 도 에러 로그, FWD 는 `SQLCODE != 0 && != NO_DATA_FOUND` 로 정상 처리. 일관성 없음(단 둘 다 반환은 안 함). |
| L80 로그 | eflg 출력에 `sizeof(comhdr_o->pflg)` 사용 | eflg 를 찍으면서 길이는 `pflg` sizeof 사용 — 로깅 길이 오기(기능 영향은 없음). |
| grid01_cnt 세팅 | 완료 분기 L507 + 무조건 L514 | 두 번 zredit. 중복이나 값 동일(ii)이라 무해. |
| 미사용 선언 | `CUST_SPRD CUST_SPR`, `cKeyv`, `editb`, `log_m`, `cnt`, `i_sndcnt`, `error_f` | 선언만 되고 사용 안 됨(호스트변수 `CUST_SPR` 와 SQL alias `CUST_SPR` 는 별개). |
