## 3. 구현 흐름 (`.pc`)

```
W1352S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1'  → retc = W1352S01_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? eflg="0",l_dbcommit() : eflg="1",l_dbrollback()
        → fnSetMsg(&e_msg, comhdr_o)  (e_msg.code → COMHDR mesg 매핑)

W1352S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1352S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])       (그리드 최대 행)
  ├─ GRID key 처리 (content->keyc)  K_NEXT→nkey / K_PREV→pkey / default→"0"
  │      → vKeyv 세팅 후 l_rtrim  (커서 WHERE 에 미바인딩 — §6 페이징 미동작)
  ├─ 입력 바인딩: yprintf + l_rtrim → vFdmId/vFxPdcd/vCrncPairId/vUseYn/vTrnAblYn
  │      ("입력항목 유효성 check" 주석이나 실검증 없음 — §6)
  ├─ DECLARE W1352S01_C1 CURSOR (5테이블 조인, NVL 선택 필터)
  ├─ OPEN C1
  │      SQLCODE!=SUCCESS → NO_DATA_FOUND: e_msg "91001", RET_FAILURE
  │                         기타:          e_msg "10118", RET_FAILURE
  ├─ for(ii=0; ; ii++)  FETCH C1 INTO :c...(12개 컬럼)
  │      NO_DATA_FOUND → break
  │      기타 err       → CLOSE, e_msg "10118", RET_FAILURE
  │      ii==max_row   → i_isnext++, break  (초과 1행 fetch 후 폐기)
  │      → mod->orec[ii].* 세팅 (zsedit)
  │      → sprintf(vKeyv, "%.64s", cKeyv)   (cKeyv 는 FETCH INTO 대상 아님 — §6)
  ├─ CLOSE C1
  ├─ 결과 판정
  │      ii==0 && i_isnext==0 → e_msg "91001" (해당 자료 없음)  [단 RET_SUCCESS 반환]
  │      i_isnext==1          → content_set_nkey(vKeyv), e_msg "91002" (계속)
  │      else                 → e_msg "91003" (조회 완료)
  ├─ zredit(mod->grid01_cnt, ii)
  ├─ sndl += (ii-1)*sizeof(orec[0])
  └─ return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **DML**: 커서 `W1352S01_C1` SELECT 1문(5테이블 조인). 데이터 변경 없음.
- **조인 조건**:
  - `CMG001M.FDM_ID = CMG002M.FDM_ID` (중개사 기본)
  - `CMG002M.FX_PDCD = CMC003D1.CMCD_VL_REF_CON3` AND `CMG002M.TNR_ID = CMC003D1.CMCD_VL` AND `CMC003D1.COM_CD_NM = 'TNR_ID'` (TENOR 코드)
  - `CMG002M.CRNC_PAIR_ID = CMG004M.CRNC_PAIR_ID` (통화페어)
  - `CMG002M.FX_PDCD = CMC003D2.CMCD_VL` AND `CMC003D2.COM_CD_NM = 'FX_PDCD'` (상품코드명)
- **동적 필터**: `fdmId / fxPdcd / crncPairId / trnAblYn / useYn` 각각 `= NVL(:v..., 컬럼)` — 입력 공백이면 해당 조건 무시(전체 조회). (참고: `USE_YN='Y'`, `TRN_ABL_YN='Y'` 고정 조건은 주석 처리되어 비활성)
- **정렬**: `FDM_ID, FX_PDCD DESC, CMG004M.RPSN_SQC, CMC003D1.CMCD_VL_REF_CON2`(TENOR 출력순서).
- **페이징(그리드)**: `content->keyc`(NEXT/PREV/최초)로 `vKeyv` 를 만들고 `max_row` 초과 시 `i_isnext=1` + `content_set_nkey` 로 계속 신호(`91002`)를 준다. **그러나 (a) 커서 WHERE 에 키 범위 조건이 없고 (b) NEXT KEY 로 쓰는 `cKeyv` 가 FETCH INTO 대상에 없어 항상 빈값** → 실제 다음 페이지 이동 불가(§6).
- **입력 유효성**: 3절 주석은 "입력항목 유효성 check" 이나 실제 NULL/공백 거부 로직 없음 — `yprintf/l_rtrim` 만 수행(§6).
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드** (`e_msg.code` → `fnSetMsg` → COMHDR `mesg`):
  - `91003` "조회가 완료되었습니다." (정상 완료, `eflg=0`)
  - `91002` "조회가 계속됩니다." (`max_row` 초과, NEXT KEY 세팅)
  - `91001` "해당하는 자료가 없습니다." (OPEN 시 NO_DATA / 결과 0건)
  - `10118` "자료 조회중 오류..." (OPEN/FETCH SQL 오류)
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (진입부, eflg 미세팅)
- **결과 전달**: 그리드(`grid01_cnt` + `orec[]`) + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
