# 서비스 명세 — `W1701S06` (화면그룹별메뉴조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 .pc 실 구현 기준이며, .h 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|------|----|
| 서비스 코드 | `W1701S06` |
| 서비스 이름 | 화면그룹별메뉴조회 |
| 유형 | 조회 (grid, SELECT 전용 — INSERT/UPDATE/DELETE 없음) |
| 구현 | `win/src/trn/W1700/W1701S06.pc` (742 라인) |
| I/O 헤더 | `win/src/inc/trn/W1701S06.h` |
| 라우팅 alias | `W1701S06` |
| 채널 | comhdr_i->ctyp 기반 chnl 매핑 (HTS/MTS/WTS/EMP/API/SPI) — 실제 분기에는 미사용 |

메뉴 마스터(대/중/소분류)와 화면접속권한(권한그룹) 카탈로그를 조인해 화면그룹별 메뉴 목록을
그리드로 반환한다. 조회구분(sElectTp)에 따라 두 갈래로 나뉜다.

## 2. I/O 전문 (`.h`)

**Input `W1701S06_I`** (총 19B)

| 필드 | 크기 | 헤더 주석 | 실제(.pc) 의미 |
|------|-----|----------|---------------|
| `fXAthrDcd`  | 4 | 권한그룹코드 | 권한그룹코드. F1 에서 `SUBSTR(1,2)`=FX_USER_DCD, `SUBSTR(3,2)`=SCRE_ACS_ATHR_DCD 로 분해해 TB_FXB_CMC012M 조회 |
| `sElectTp`   | 1 | 조회구분 | dispatch 키. `'3'`→F2, 그 외→F1. F1 내부에서 `'1'`=카탈로그 메뉴만(IN), `'2'`=카탈로그 제외(NOT IN) |
| `fXuSerDcd`  | 2 | 메뉴구분코드 | 실제로는 사용자구분(FX_USER_DCD). F2 의 UNION where 절 필터 |
| `lRdvMenuId` | 6 | 대분류 | 대분류 메뉴ID (F2 필터) |
| `mDdvMenuId` | 6 | 중분류 | 중분류 메뉴ID (F2 필터) |

**Output `W1701S06_O`** (grid — `grid01_cnt`(6) + `orec[1]` 가변 반복)

| 필드 | 크기 | 설명 |
|------|-----|------|
| `grid01_cnt` | 6 | 그리드 건수 (zredit(ii)) |
| `orec[].mEnuIid`      | 6   | 메뉴ID |
| `orec[].fXuSerDcdNm`  | 10  | 메뉴구분명 (`FX_USER_DCD='AM'`→'관리자' else '고객') |
| `orec[].lRdvMenuIdNm` | 100 | 대분류명 |
| `orec[].mDdvMenuIdNm` | 100 | 중분류명 |
| `orec[].sMdvMenuIdNm` | 100 | 소분류명 |

`orec` 는 헤더상 `[1]` 이나 실제로는 `content->sndl` 을 건수만큼 늘려 가변 그리드로 송신
(`content->sndl += W1701S06_O_SZ` 후 `+= (ii-1)*sizeof(orec[0])`).

## 3. 구현 흐름 (`.pc`)

```
W1701S06()                                   // 진입/dispatch
  ├ getmid / getmod                          // 입출력 주소 확보
  ├ e_msg 기본 세팅 (pname=rkey, ltyp, mtyp="S", rtyp="M")
  ├ switch(comhdr_i->ctyp) → comhdr_i->chnl 매핑 (HTS/MTS/WTS/EMP/API/SPI, 이후 미사용)
  ├ if (sElectTp=='3') → W1701S06_F2()        // 대/중/소분류 UNION 조회
  │  else              → W1701S06_F1()        // 권한그룹 카탈로그 기반 동적 조회
  ├ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()
  └ fnSetMsg(&e_msg, comhdr_o) → return content->sndl

W1701S06_F1()  (동적 SQL)
  ├ content->keyc → v_keyv (K_NEXT:nkey / K_PREV:pkey / else "0")
  ├ 입력 yprintf+l_rtrim (fXAthrDcd/sElectTp/fXuSerDcd/lRdvMenuId/mDdvMenuId)
  ├ fXAthrDcd 있으면 → TB_FXB_CMC012M SELECT MENU_ID_CTLG_CON INTO vmenuIdCtlgCon
  │     (에러시 e_msg.code="95000", RET_FAILURE)
  ├ strStmt 조립: TB_FXB_CMC011M A + 스칼라서브(TB_FXB_CMC010M) × 대/중/소분류명
  │     WHERE USE_YN='Y' AND MENU_ID > :v_keyv
  │     · sElectTp=='1' → 카탈로그 있으면 IN(list), 없으면 MENU_ID='999999999'(=결과 0)
  │     · sElectTp=='2' → 카탈로그 있으면 NOT IN(list)
  │     ORDER BY A.MENU_ID
  ├ PREPARE sql_stmt → DECLARE W1701S06_C1 → OPEN USING :v_keyv
  ├ FETCH 루프 (8열 INTO, 그 중 5열만 orec 저장) → ii==max_row 이면 i_isnext++/break
  ├ CLOSE C1
  └ 건수 판정: ii==0 →"91001", ii>=max_row →nkey set +"53102", else →"91003"; eflg="0"

W1701S06_F2()  (정적 SQL)
  ├ v_keyv / 입력 편집 (F1 과 동일)
  ├ DECLARE W1701S06_C3 : (TB_FXB_CMC010M ⋈ TB_FXB_CMC011M) UNION (TB_FXB_CMC011M + 스칼라서브)
  │     바인드 :ilRdvMenuId :ifXuSerDcd :imDdvMenuId :v_keyv, ORDER BY MENU_ID
  ├ OPEN C3
  ├ FETCH 루프 (F1 과 동일 구조)
  ├ CLOSE C3
  └ 건수 판정 (F1 과 동일) → RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **dispatch**: `sElectTp=='3'` → F2 (분류 트리 조회), 그 외 → F1 (권한그룹 카탈로그 기반).
- **페이징**: `content->keyc` 로 next/prev 키(`v_keyv`) 획득, `MENU_ID > :v_keyv ORDER BY MENU_ID` 로 커서 전진. `max_row = MAX_RRB / sizeof(orec[0])`. 매 행마다 `v_keyv = MENU_ID`.
- **F1 카탈로그 필터**:
  - `fXAthrDcd` 존재 시 CMC012M 에서 카탈로그(MENU_ID_CTLG_CON) 선조회.
  - `sElectTp=='1'`: 카탈로그 있으면 `MENU_ID IN (카탈로그)`, 없으면 `MENU_ID='999999999'` 로 강제 0건.
  - `sElectTp=='2'`: 카탈로그 있으면 `MENU_ID NOT IN (카탈로그)` (없으면 필터 없음).
- **F2 트리 필터**: `lRdvMenuId`/`fXuSerDcd`/`mDdvMenuId` 를 `like '%'||값` 또는 `CASE WHEN ... IS NULL/'' THEN 원본 ELSE 값 END` 로 선택적 적용.
- **트랜잭션**: 서브함수 반환값 기준 commit/rollback (§3). 조회만 하므로 사실상 무의미.
- **에러/메시지(e_msg.code → mesg, comhdr_o->eflg)**:
  - `"95000"` : SQL 오류(SELECT/PREPARE/DECLARE/OPEN/FETCH 실패), RET_FAILURE → rollback.
  - `"91001"` : 해당 자료 없음(ii==0). eflg="0", RET_SUCCESS (return 은 주석 처리, 계속 진행).
  - `"53102"` : 다음 페이지 존재(ii>=max_row). next key set, eflg="0".
  - `"91003"` : 정상 조회 완료. eflg="0".
  - 정상 판정(91001/53102/91003) 모두 `comhdr_o->eflg="0"` (0:정상).
