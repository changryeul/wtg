## 2. I/O 전문 (`.h`)

**Input `W1130A02_I`**
| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 처리구분 — `'1'` 만 유효(로그인). 그 외는 "작업구분 오류" |
| `fxUserNo` | 30 | FX사용자번호 — 유일한 필수 입력값. 모든 조회/등록의 키 |
| `fxUserNm` | 40 | FX사용자명 — **입력값은 실제로 사용되지 않음** (§6). 등록 시엔 CSC004M 재조회값 사용 |

입력 전문 외에 `COMHDR` 에서 추가로 읽는 값:
- `comhdr_i->loip` → 로그인IP (`vLgnIp`)
- `comhdr_i->maca` → MAC주소 (`vMacAdr`)
- `comhdr_i->cont[0]` → 로그인구분코드 `notiTagetDcd` (`'F'`직원 / `'H'`고객)
- `comhdr_i->fdid` → 펀드번호 (`vFndNo`, 선물환 사전점검 조인 키)

## 4. 사용 DB 테이블
| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-----|-----|-----|-----|-----|
| `TB_FXB_CMG012M` | 영업일정보 | CMG012M | SELECT | 당/전/익 영업년월일 |
| `TB_FXB_CSC015L` | 로그인내역 | CSC015L | SELECT / UPDATE / INSERT | 최종로그아웃일시 조회, 활성세션 조회, 직전세션 로그아웃 UPDATE, 신규 로그인 INSERT |
| `TB_FXB_CSC005R` | 사용자별고객번호관계 | CSC005R | SELECT | 커서 조인 + 식별번호 채번 조인 (FX사용자번호↔실명대체번호) |
| `TB_FXB_CSC001M` | 고객약정정보기본 | CSC001M | SELECT | 커서의 고객약어명, 선물환 거래범위구분(FWEX_TRN_RNG_DCD) |
| `TB_FXB_TRC003L` | 선물환사전점검내역 | TRC003L | SELECT | 당일 사전점검 존재여부(outer join) → 팝업여부 판정 |
| `TB_FXB_CSC004M` | 사용자정보기본 | CSC004M | SELECT | 이용자별칭명·로그인ID·사용자명 (INSERT 소스) |

*DML 요약: SELECT ×5 (영업일 / 최종로그아웃 / 커서 / 채번·팝업 / 사용자기본) + 커서 내 UPDATE(직전세션 로그아웃) + INSERT(신규 로그인내역). 정상 시 최종적으로 상위 `W1130A02()` 가 COMMIT, 실패 시 ROLLBACK.*

## 5. 로직 / 필터 규칙
- **처리구분 분기**: `prGb='1'` 만 처리(로그인). 그 외 즉시 "작업구분 오류"(`MSG_NOWORK`), F1 미진입.
- **필수 입력**: `fxUserNo` 만 검증(`isNullSpace`). 공백이면 `10555`. `fxUserNm`/`lgnId` 입력값은 검증·사용 안 함(§6).
- **로그인구분** `notiTagetDcd`(COMHDR `cont`): `'F'`직원 / `'H'`고객. 강제종료 메시지 문구, 팝업여부 강제 N(직원)을 가른다.
- **중복 로그인 처리**: 커서로 `LGN_YN='Y'` 인 기존 세션을 모두 FETCH → 각 건 UPDATE 로 `LGN_YN='N'`, `LGOT_SCD='1'`(비정상), 로그아웃 일시·`COMNT_CON` 세팅. WHERE 키는 `FX_USER_NO + LGN_YN='Y' + LGN_IDNT_CON`.
  - **실제 강제종료 푸시(`mymq_broadcast`)와 SMS 알림(`fnSndSmsMsg`)은 전 구간 주석처리** — DB 상 로그아웃만 반영되고 대상 단말로의 강제종료/알림 송신은 일어나지 않는다.
- **식별번호 채번**: `SYSDATE(YYYYMMDDHH24MISS)` + MAC(`:`제거) + IP(`.`제거) + FX사용자번호.
- **선물환사전점검 팝업(`fwdPreChkPopYn`)**: 당일 CSC015L 로그인건수 0(=당일 최초) **그리고** CSC001M `FWEX_TRN_RNG_DCD='3'` **그리고** 당일 TRC003L 사전점검 미존재일 때 `Y`. 직원은 무조건 `N`.
- **신규 로그인 INSERT**: `LGN_YN='Y'`, `FX_MDIA_DCD='1X'`, `COMNT_CON='정상접속'`, `FX_SYS_LSMD_ID='SYSTEM'`. 실패 시 `10594`.
- **트랜잭션**: F1 은 RET_SUCCESS/RET_FAILURE 만 반환, COMMIT/ROLLBACK 은 상위 `W1130A02()` 가 수행(AUTOCOMMIT 아님).
- **에러 코드(e_msg.code → mesg, eflg='1')**: `10555`(입력값 확인) · `91001`(해당 자료 없음) · `10118`(조회 중 오류) · `10594`(등록 중 오류). 정상 종료는 `eflg='0'` + `91003`(재조회). 커서 OPEN 의 NO_DATA_FOUND 분기는 no-op(주석). `31008`/`10177` 는 주석처리된 알림/파생잔액 블록에만 존재.
