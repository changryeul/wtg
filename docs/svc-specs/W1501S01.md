## 3. 구현 흐름 (`.pc`)

```
W1501S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ retc = W1501S01_F1(...)     ← 실제 로직 (작업구분 분기 없음, 단일 조회)
  └─ retc==RET_SUCCESS ? l_dbcommit()+eflg"0" : l_dbrollback()+eflg"1"
        (SELECT/외부 IF 만 있어도 규약상 명시적 커밋 — AUTOCOMMIT 아님)
     → fnSetMsg(&e_msg, comhdr_o)   e_msg.code → COMHDR mesg/rcod 반영

W1501S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1501S01_O_SZ, memset(mod), comhdr_o->rtyp="G"
  ├─ 입력 바인딩: yprintf+l_rtrim → iBsnsDt/iCustId/iBsTp, comhdr_i->fdid → iFndNo
  ├─ [4] 유효성: len(iBsnsDt)<8 → 10507 실패 / len(iCustId)<1 → 10201 실패
  │      (실패 시 mod->Custd="0000000000")
  ├─ [5.1] SELECT WBCS_RLNM_ALTR_NO FROM TB_FXB_CSC005R WHERE FX_USER_NO=:iCustId
  │        → oWbcsRlnmAltrNo(전행고객실명대체번호). NO_DATA→31008 만 세팅하고 진행(§6),
  │          기타 SQL err→10118 실패
  │      fnCustInfoQry(wbcsRlnmAltrNo) ← 고객약정정보 단건(간접). 실패→10177 실패
  │      custInfo.fwexTrnRngDcd != '3'(선물환) → 10506 "선물거래 불가 고객" 실패
  ├─ [5.2] fnBzdyInq(baseYmd=iBsnsDt, crcd="KRW", bsopNdd=0) → 영업일(oHlsPdyDt). 실패→RET_FAILURE
  ├─ [5.3] SELECT CASE(day-switch 시간대) FROM TB_FXB_CMC003D
  │        WHERE COM_CD_NM='TRN_CNTL_TIM' AND CMCD_VL='0' → vSchYn
  │        SQL err→10118 실패 / vSchYn='N'(조회불가시간)→32713 실패
  ├─ [6] IF-019: API_REQ_INIT → EAI_SET_STR(cusno/rep_cusno/tr_dsc="2"/inq_yn="Y"/
  │        ac_amn_brc/bas_dt/itnbnk_cus_id=iFndNo/cus_rlno) → fefnpl19i0(&req,&res)
  │        실패→RET_FAILURE  (한도/신용등급 등 핵심 값의 실제 출처)
  ├─ [7] SELECT MAX(FWEX_PSRN_NO) FROM TB_FXB_TRC003L
  │        WHERE BSOP_YMD=:iBsnsDt AND WBCS_RLNM_ALTR_NO_FND_NO=:oWbcsRlnmAltrNo||:iFndNo
  │              AND BYSEL_DCD=:iBsTp → oFwexPsrnNo. SQL err→10118 실패
  ├─ [8] IF-019 신용등급에 "일반" 포함 시:
  │        SELECT MAX(PXNT_NO), CASE(있으면 'Y' else 'N') FROM TB_FXB_TRC002L
  │        WHERE BSOP_YMD=:iBsnsDt AND WBCS_RLNM_ALTR_NO=:oWbcsRlnmAltrNo AND BYSEL_DCD=:iBsTp
  │        → oPxntSrn/oPxntCnfaYn, InvClcd='1'  |  "기업"→'2' | "전문"→'3'
  ├─ mod 세팅: BsTpN/InvClnm/InvInfoVddd/LssLmt*/Pxnt*/FwexPsrnNo + iBsTp=='B' 분기별 Lmt* 세팅
  │        (Custd/CustIm 은 세팅 코드 주석처리 — §6)
  └─ 정상 → eflg"0", e_msg"91003", RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **DML**: 전부 SELECT(단건/집계). 조인·커서 없음. 데이터 변경 없으나 규약상 성공 시 `l_dbcommit`.
- **조회 체인**: ① `CSC005R` 로 FX사용자번호→전행고객실명대체번호 ② `fnCustInfoQry` 로 고객약정정보(선물환 자격 체크) ③ `fnBzdyInq` 영업일 ④ `CMC003D` 뱅킹 조회가능시간 ⑤ **IF-019** 한도/신용정보 ⑥ `TRC003L` 사전점검번호 ⑦ (일반소비자) `TRC002L` 상품설명서.
- **선물환 자격 필터**: `custInfo.fwexTrnRngDcd != '3'`(1:바로환전/2:현물환/3:선물환) 이면 10506 으로 거절.
- **뱅킹 시간 필터**: `CMC003D` 의 두 기준시각으로 **자정 넘김(day-switch) 시간대**를 판정. `CMCD_VL_REF_CON1 <= now OR CMCD_VL_REF_CON2 >= now → 'N'`(조회불가). 감싸는(wrap-around) 창을 OR 로 처리하는 정상 로직. `'N'` 이면 32713.
- **매입/매도 분기**: `iBsTp=='B'`(은행기준 매도/수입) vs else(은행기준 매입/수출) 로 IF-019 응답 필드 세트가 갈린다(§2 표). `BsTpN` 은 입력 에코, `LmtVddd` 도 분기별(`rg_dt`/`rg_dt2`).
- **투자구분(InvClcd/InvClnm)**: IF-019 `itnbnk_crdrk_dsc` 문자열 부분매칭("일반"/"기업"/"전문"). 상품설명서 조회는 "일반" 분기에서만 수행.
- **사전점검번호 키**: `WBCS_RLNM_ALTR_NO_FND_NO = :oWbcsRlnmAltrNo || :iFndNo` (전행고객번호+펀드ID 연결) + 영업일 + 매매구분.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드**:
  - 성공: `91003` "조회가 완료되었습니다.", `eflg=0`.
  - 영업일자 미입력 `10507` / 고객번호 미입력 `10201` (둘 다 `eflg=1`, `Custd="0000000000"`).
  - `CSC005R` NO_DATA → `31008` "정보가 없습니다" 세팅만 하고 **계속 진행**(§6), 기타 SQL 오류 `10118`.
  - `fnCustInfoQry` 실패 → `10177` "고객정보 없음"(eflg=1).
  - 선물환 불가 고객 `10506`, 뱅킹 조회불가시간 `32713`, `TRC003L`/`TRC002L`/`CMC003D` SQL 오류 `10118`.
  - IF-019(`fefnpl19i0`) 실패 → e_msg.code 는 함수 내부 세팅값 유지, RET_FAILURE.
- **결과 전달**: 출력 레코드(23필드) + COMHDR(`eflg`/`e_msg.code`→`mesg`/`rcod`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
