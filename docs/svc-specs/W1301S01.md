## 2. I/O 전문 (`.h`)

**Input `W1301S01_I`** (총 5B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'`=통화리스트(그리드) 조회, `'2'`=전체통화콤보SET 조회. 둘 다 `W1301S01_F1` 로 진입. 그 외 값은 "작업구분 오류". 조회 SQL 의 `vInqGb` 로도 사용되어 NEXT_KEY 정렬 방식을 분기 |
| `crcd` | 3 | 통화코드 — 조회 필터(단건 필터, 빈값이면 전체) |
| `useYn` | 1 | 사용여부 — 조회 필터(빈값이면 전체) |

**Output `W1301S01_O`** — grid (가변, `orec[]` 반복)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드건수 (`zredit(mod->grid01_cnt, ii)` — 실제 담은 행 수 `ii`) |
| `orec[].crcd` | 3 | 통화코드 (`CMG005M.CRCD`) |
| `orec[].crcdNm` | 30 | 통화코드명 (`CMG005M.CRCD_NM`) |
| `orec[].dcpnPcsnDcd` | 1 | 소수점처리구분코드 (`CMG005M.DCPN_PCSN_DCD`, RoundingType) |
| `orec[].dcpnPcsnDcdNm` | 20 | 소수점처리구분코드명 (`TB_FXB_CMC003D` COM_CD_NM='DCPN_PCSN_DCD' 코드명) |
| `orec[].dcpuRcn` | 5 | 소수점이하자리수 (`CMG005M.DCPU_RCN`, int → `%d`) |
| `orec[].rpsnSqc` | 7 | 표시순서 (`CMG005M.RPSN_SQC`, int → `%d`) |

## 3. 구현 흐름 (`.pc`)

```
W1301S01()                          ← 진입 (콜백)
  ├─ prthdr / getmid / getmod       COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정                  ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' | '2' → retc = W1301S01_F1(...)     ← 실제 로직 (둘 다 동일 함수)
  │     그외      → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │                 (MSG_NOWORK 로그, eflg 미설정)
  ├─ retc==RET_SUCCESS ? l_dbcommit()+eflg="0" : l_dbrollback()+eflg="1"
  │                                 (명시적 커밋, AUTOCOMMIT 아님)
  └─ fnSetMsg(&e_msg, comhdr_o)     e_msg.code → COMHDR mesg/rcod 세팅 후 SEND

W1301S01_F1()                       ← 조회 본체
  ├─ getmid/getmod, sndl += W1301S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ GRID key 처리 (content->keyc)  K_NEXT→nkey / K_PREV→pkey / default→"0"
  │      → vKeyv 세팅 후 l_rtrim
  ├─ 입력 바인딩: yprintf + l_rtrim → vCrcd / vUseYn / vInqGb(=prGb)
  │      (유효성 실검증 없음 — 로그만 남김, §6)
  ├─ DECLARE W1301S01_C1 CURSOR (CMG005M + CMC003D ×2 LEFT OUTER JOIN)
  │      WHERE CRCD = NVL(:vCrcd,CRCD) AND USE_YN = NVL(:vUseYn,USE_YN)
  │            AND LPAD(RPSN_SQC,5,0) > :vKeyv
  │      NEXT_KEY = (vInqGb='1' ? LPAD(RPSN_SQC,5,0) : CRCD)  ORDER BY NEXT_KEY
  ├─ OPEN → SQLCODE 체크 (NO_DATA_FOUND→91001 return / else→10118 return)
  ├─ FETCH 루프 (for ii=0;;ii++)
  │      NO_DATA_FOUND → break
  │      SQL error     → CLOSE + 10118 return
  │      ii==max_row   → i_isnext++; break  (초과 1행은 담지 않고 '계속' 감지용)
  │      → zsedit/sprintf 로 mod->orec[ii] 채움, vKeyv = cKeyv(NEXT_KEY) 갱신
  ├─ CLOSE W1301S01_C1
  ├─ 결과 판정
  │      ii==0 && i_isnext==0 → e_msg 91001 (자료 없음)
  │      i_isnext==1          → content_set_nkey(vKeyv) + e_msg 91002 (계속)
  │      else                 → e_msg 91003 (완료)
  ├─ zredit(grid01_cnt, ii)
  ├─ sndl += (ii - 1) * sizeof(orec[0])    (§6 — ii==0 시 언더플로 위험)
  └─ return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분(prGb)**: `'1'`(그리드)·`'2'`(콤보) 모두 `W1301S01_F1` 실행. `vInqGb`(=prGb) 값에 따라 SQL 의 `NEXT_KEY` 계산 컬럼만 달라진다 — `'1'`=`LPAD(RPSN_SQC,5,0)`(표시순서), 그 외=`CRCD`(통화코드). ORDER BY 도 이 NEXT_KEY 를 따른다.
- **동적 필터**: `crcd`/`useYn` 는 `NVL(:bind, 컬럼)` 패턴 — 빈값(널)이면 해당 조건을 사실상 무시(전체).
- **페이징**: `content->keyc`(K_NEXT/K_PREV/default) 로 `vKeyv` 결정. WHERE 절은 항상 `LPAD(RPSN_SQC,5,0) > :vKeyv`. 한 행 초과(`ii==max_row`) 감지 시 `i_isnext=1`, 다음 키를 `content_set_nkey` 로 반환.
- **트랜잭션**: 조회 전용이나 진입 함수에서 성공→`l_dbcommit`, 실패→`l_dbrollback` 을 명시 수행(엔진 AUTOCOMMIT 아님).
- **에러/메시지 (e_msg.code → fnSetMsg)**:
  - `91001` 해당 자료 없음 (커서 OPEN NO_DATA_FOUND, 또는 FETCH 0행)
  - `91002` 조회가 계속됩니다 (다음 페이지 존재)
  - `91003` 조회 완료
  - `10118` 커서 OPEN/FETCH SQL 오류
  - "작업구분 오류." (진입부 default 분기, COMHDR mesg 직접 세팅)
- **eflg**: 진입 함수에서 `retc==RET_SUCCESS`→"0", 아니면 "1". 단 자료 없음(91001)도 `W1301S01_F1` 이 `RET_SUCCESS` 로 반환하므로 eflg="0" + commit 경로를 탄다.
