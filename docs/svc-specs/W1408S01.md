## 3. 구현 흐름 (`.pc`)

```
W1408S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ retc = W1408S01_F1(...)     ← prGb switch 없이 무조건 호출 (§6)
  ├─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
  └─ fnSetMsg(&e_msg, comhdr_o) 후 return content->sndl

W1408S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1408S01_O_SZ, memset(mod)
  ├─ 입력 바인딩: yprintf + l_rtrim → iBsnsDt/iWbcsRlnmAltrNo/iBsTp/iFndNo
  ├─ 유효성: strlen(iBsnsDt)<8 → 10507 / strlen(iWbcsRlnmAltrNo)<1 → 10201
  │           (iBsTp / iFndNo 검증 없음)
  ├─ fnCustInfoQry(&custInfo)   ← 고객약정정보(TB_FXB_CSC001M) 단건 조회
  │       실패 → e_msg "10177"(고객정보 없음), RET_FAILURE
  ├─ [3] 뱅킹 조회가능시간 체크
  │       SELECT CASE ... FROM TB_FXB_CMC003D WHERE COM_CD_NM='TRN_CNTL_TIM' AND CMCD_VL='0'
  │       SQL err → 10118 / vSchYn='N' → 32713(일자전환 작업으로 거래 제한)  (§6 CASE 로직 주의)
  ├─ [4] IF-019 EAI 조회
  │       API_REQ_INIT → EAI_SET_STR(고객번호/대표고객번호/tr_dsc="2"/inq_yn="Y"/
  │                       ac_amn_brc/bas_dt/itnbnk_cus_id/cus_rlno)
  │       ret = fefnpl19i0(&api_req,&api_res,&e_msg)   ← 손실한도·실수요한도 원장
  │       실패 → RET_FAILURE (e_msg 는 fefnpl19i0 내부 set)
  ├─ [5] 손실한도 누적금액 SELECT (인라인뷰 C/D/F 3개 조인 없는 카티션 단건 집계)

  │       D: TB_FXB_TRG003L + TB_FXB_TRG001L (체결금액 USD환산 SUM, CNTT_STTS_DCD IN('5','6'))
  │       F: TB_FXB_TRG003L + TB_FXB_TRG001L + TB_FXB_CMG004M + FXPL.TB_FXB_CMG014F
  │          (체결전송 안된 '5' 건, 최신 고시환율 MUX_RT 곱해 BANK_SEL/BUY_CUR)

  └─ 정상 → eflg "0", e_msg "91003"(조회완료), RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분 분기 없음**: `W1119S01` 등과 달리 `switch(prGb)` 없이 `W1408S01_F1` 을 무조건 1회 호출하는 순수 조회.
- **입력 유효성**: `BsnsDt`(8자 미만 10507), `WbcsRlnmAltrNo`(빈값 10201) 만 검증. `BsTp`/`FndNo` 는 검증 없음 — `BsTp` 는 출력 에코, `FndNo` 는 NVL(TRIM(...),' ') 로 필터(빈값 허용).
- **조회 체인**: ① `fnCustInfoQry` 로 고객 마스터(대표고객번호/관리점) 확보 → ② 조회가능시간(`TB_FXB_CMC003D`) 게이트 → ③ IF-019 EAI 로 한도 설정/소진 원장 확보 → ④ 로컬 DB(TRG/CMG)에서 당일 미체결·체결·미전송 금액을 합산해 플랫폼 값과 합쳐 최종 소진/여유/소진율 재산정.
- **환율 환산(F 뷰)**: 최신 고시일(`MAX(BLTN_YMD)`, `BLTN_TOB='1'`) 대고객 환율을 통화쌍(`CRNC_PAIR_DCD`)·매매구분(`BYSEL_DCD`)로 DECODE 하여 원화(`BANK_SEL_CUR`/`BANK_BUY_CUR`) 환산, `FX_PDCD in ('FWD','SWP')`, `CNTT_STTS_DCD='5'` 만 대상.
- **매도/매수 재계산**: `당행사용금액 = IF-019값 + BANK환산`, `사용가능금액 = IF-019값 − BANK환산`. 사용가능<0 이면 초과여부·거래가능여부를 'N' 으로 강제.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드**:
  - 성공: `91003` "조회가 완료되었습니다.", `eflg=0`.
  - 영업일자 미입력: `10507`, `eflg=1`.
  - 고객번호 미입력: `10201`, `eflg=1`.
  - 고객정보 없음: `10177` (fnCustInfoQry 실패), `eflg=1`.
  - 조회가능시간 SQL 오류: `10118`.
  - 조회불가 시간: `32713` "은행 일자전환 작업으로 거래 제한".
  - 한도내역 SQL 오류: `32650`.
  - IF-019 오류: `fefnpl19i0` 내부 `e_msg.code` 그대로.
  - 상위 진입부에서 실패 시 `eflg="1"` + rollback, 성공 시 `eflg="0"` + commit 재설정.
- **결과 전달**: 출력 레코드(27필드) + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
