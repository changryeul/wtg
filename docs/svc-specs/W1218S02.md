# 서비스 명세 — `W1218S02` (배치로그파일 조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석 및 `.pc` 파일 상단 주석(`TARGET TABLE : TB_FXB_SYG002G`)과 어긋나는 부분은 §6 주의에 정리했다. **이 서비스는 DB 테이블을 직접 SELECT/DML 하지 않고 OS 파일시스템의 배치 로그파일을 fopen/fgets 로 읽는 파일 조회 서비스**이다.

## 2. I/O 전문 (`.h`)

**Input `W1218S02_I`** (총 99 byte)

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `procTp`  | 1  | 처리(작업)구분: `1`=배치 전체조회(F1), `2`=시간범위 로그조회(F2), `3`=다음 문자열 검색위치(F3), `4`=이전 문자열 검색위치(F4), `5`=from~to 시간범위+검색 그리드조회(F5) |
| `bsopYmd` | 8  | 작업일자(조회일자). 8자리 미만이면 오류(10160) |
| `btchId`  | 8  | 배치ID. 8자리 미만이면 오류(10797). 로그파일명 구성에 사용 |
| `findTxt` | 50 | 검색단어(문자열). F1 은 존재 시 검색, F2 는 strstr 조건에만 사용, F3/F4 는 필수 검색어, F5 는 선택 |
| `strtLen` | 20 | 검색시작위치(파일 byte offset). F3/F4/F5 에서 atoi 로 사용. F1/F2 는 미사용(NEXT KEY 로 대체) |
| `fromTm`  | 6  | 조회 시작시각 HHMMSS. **F5 에서만 사용**, F1/F3/F4 미사용, F2 는 무시(하드코딩) |
| `toTm`    | 6  | 조회 종료시각 HHMMSS. **F5 에서만 사용** |

**Output `W1218S02_O`** (grid, 헤더 26 byte + `orec[]` 가변)

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `grid01_cnt` | 6  | 그리드 건수(`zredit(ii)`) |
| `findStrt`   | 20 | 검색 결과 시작위치. **F3/F4 만 세팅** (`sprintf(mod->findStrt,"%d",findStrt)`) |
| `orec[].strtLen` | 20  | 해당 조각의 파일 내 시작 byte offset |
| `orec[].endLen`  | 20  | 해당 조각의 파일 내 끝 byte offset |
| `orec[].logTxt`  | 200 | 로그 한 줄을 200 byte 단위로 자른 조각(한글 경계 보정 포함) |

`.h` 는 `orec[1]` 로 선언, 실제로는 `content->sndl` 을 `max_row`(=`MAX_RRB/sizeof(orec[0])`) 만큼 확장해 다건 반환.

## 3. 구현 흐름 (`.pc`)

```
W1218S02()  [dispatch]
  getmid / getmod, E_MSG 기본설정(KR/S/M), comhdr_o->rtyp="M"
  switch(mid->procTp[0])
     '1' -> W1218S02_F1  (전체 로그 조회)
     '2' -> W1218S02_F2  (시간범위 로그 조회 — 시간 하드코딩)
     '3' -> W1218S02_F3  (다음 문자열 검색위치)
     '4' -> W1218S02_F4  (이전 문자열 검색위치)
     '5' -> W1218S02_F5  (from~to 시간 + 검색어 그리드 조회)
   //'6' -> W1218S02_F6  (주석처리, 미구현)
     default -> comhdr_o->mesg="작업구분 오류." return
  retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()
  return content->sndl

W1218S02_F1/F2/F3/F4/F5 공통 골격:
  1. init: sndl += O_SZ, memset(mod), max_row 계산
  2. NEXT/PREV/최초 → content_get_nkey/pkey 로 nextKey(=byte offset) 획득
  3. 입력검증: bsopYmd>=8(10160), btchId>=8(10797)  [F5 는 fromTm/toTm>=6 추가]
  3.1 fnBsopYmdQry(BSOP, FDM_ID_FRX)  → 당영업일 조회 (실패 10118)   [공통함수, 간접 DB]
  4. 로그파일 경로/파일명 조립:
       당영업일 == bsopYmd  → {path}/{btchId}-{bsopYmd}.log
       그 외(과거일)        → fnBzdyInq() 로 익영업일 계산  [공통함수, 간접 DB]
                              → {path}/{bsopYmd}/{btchId}-{bsopYmd}.log   (폴더는 bsopYmd)
  5. FileOpen(logFileName,"r")  (없으면 10812)
       F1: 위치지정(next offset / findTxt strstr 선검색 / 처음부터)
       F2: 14:00:00~14:10:00 하드코딩 시간창 스캔 → startPos/endPos 확정 후 재open+fseek
       F3: fseek(strtLen) 후 findTxt strstr 순방향 1건 → findStrt 세팅
       F4: 처음~strtLen 범위 findTxt strstr → 마지막 매칭의 findStrt 세팅
       F5: fromTm~toTm 초 환산 시간창 스캔 → startPos/endPos 확정 후 재open+fseek
  7. READ FILE (F1/F2/F5): fgets 로 줄 읽고 logTxt(200B) 단위 분할,
       l_hnamcpy + 한글잘림(offset) 보정, orec[ii] 채움, ii>=max_row+1 이면 i_isnext=1 break
  8. 결과 세팅:
       ii==0 && !i_isnext → e_msg 91001 (해당자료없음)
       i_isnext==1        → content_set_nkey, 91002 (계속)
       else               → 91003 (완료), grid01_cnt=ii
     fnSetMsg, eflg="0", grid01_cnt=ii,  sndl += (ii-1)*sizeof(orec[0])
     return RET_SUCCESS
```

## 4. 사용 DB 테이블

**직접 접근 테이블 없음** — `EXEC SQL` DECLARE/OPEN/FETCH/CLOSE/INSERT/UPDATE/DELETE 문이 전무하다. 파일 I/O(`FileOpen`/`fopen`/`fgets`/`fseek`)로만 동작한다. DB 접근은 아래 공통함수 경유의 간접 SELECT 뿐이다.

| 접근 경로 | 한글명 | 접근 | 용도 |
|-----|-----|-----|-----|
| `fnBsopYmdQry()` (공통함수) | 영업일/기준일 마스터 | SELECT (간접) | 당영업일(`apllBsopYmd`) 조회 — 당일/과거일 판정 |
| `fnBzdyInq()` (공통함수) | 영업일 캘린더 | SELECT (간접) | 과거일자의 익영업일 계산 (실제 파일 경로엔 미반영, §6) |

*DML 요약: 직접 DML 0건. dispatcher 는 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 를 호출하나 트랜잭션 대상 DML 이 없어 사실상 no-op 성격.*

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `switch(mid->procTp[0])` 로 F1~F5 라우팅. `default` 는 "작업구분 오류." 메시지 후 즉시 종료.
- **페이징**: `content->keyc`(K_NEXT/K_PREV/최초) → `content_get_nkey/pkey` 로 파일 byte offset(nextKey) 획득. `max_row = MAX_RRB / sizeof(orec[0])`. `ii >= max_row+1` 도달 시 `i_isnext=1` + `content_set_nkey`(다음 offset) + 91002. K_PREV 도 nkey 와 동일 처리(전용 역방향 로직 없음).
- **로그 줄 분할**: 한 줄(`logRowText` 최대 2048B)을 `logTxt`(200B) 조각으로 잘라 orec 로 반환. `l_hnamcpy` + 마지막 byte 가 공백이면 `offset++` 로 한글(2byte) 잘림 보정. 각 조각의 `strtLen`/`endLen` 은 파일 내 byte offset.
- **F2 시간창**: `startH/M/S=14:00:00`, `endH/M/S=14:10:00` **하드코딩**. `fromTm`/`toTm` 입력 무시. 로그 줄의 `chkText[7,8]/[10,11]/[13,14]` 위치를 HH/MM/SS 로 파싱(로그 포맷 의존). `strlen(chkText)<16` 줄 skip.
- **F5 시간창**: `fromTm`/`toTm` 를 `sscanf("%2d%2d%2d")` 로 파싱해 초 환산. `startPos`/`endPos` 확정 후 재 open+fseek 하여 그리드 read. 함수 본문에 대형 `#if 0` 死코드 블록 존재.
- **문자열 검색**: `strstr(chkText, iFindTxt)`. `iFindTxt` 가 빈 문자열이면 `strstr` 이 항상 매칭(비검색과 동일). F1/F5 는 `isNullSpace(iFindTxt)!=0` 로 검색어 유무를 구분하나 F2 는 구분 없이 항상 strstr 실행.
- **트랜잭션**: dispatcher 에서 retc 기준 commit/rollback (직접 DML 없음).
- **에러(e_msg.code → 메시지, comhdr_o->eflg)**:
  | code | 의미 | eflg |
  |-----|-----|-----|
  | 10160 | 조회일자를 확인하세요 (bsopYmd<8) | 1 |
  | 10797 | 입력값[배치ID]을 확인하세요 (btchId<8, F5 는 fromTm/toTm<6 도 동일코드) | 1 |
  | 10118 | 자료 조회중 오류 (fnBsopYmdQry/fnBzdyInq 실패, 경로 NULL) | 1 |
  | 10812 | 로그파일이 존재하지 않습니다 (FileOpen 실패) | 1 |
  | 10798 | 문자열을 찾을 수 없습니다 (F1/F3/F4 검색 실패) | 1 |
  | 91001 | 해당하는 자료가 없습니다 (ii==0 && !i_isnext) | 0 |
  | 91002 | 조회가 계속됩니다 (i_isnext==1) | 0 |
  | 91003 | 조회가 완료되었습니다 | 0 |
