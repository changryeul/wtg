## 2. I/O 전문 (`.h`)

**Input `W1105S01_I`** (총 31B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'`:사용자ID 조회 / `'2'`:실명대체번호 조회. 그 외 "작업구분 오류" (헤더엔 "1:조회" 로만 표기, §6) |
| `fxUserNo` | 30 | 조회 키 — `prGb='1'` 이면 FX사용자번호, `prGb='2'` 이면 실명대체번호(`WBCS_RLNM_ALTR_NO`) 로 매칭. 필수 |

**Output `W1105S01_O`** — `grid01_cnt`(6B) + 가변 `orec[]` (건당 158B)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (fetch 행 수, `zredit`) |
| `orec[].fxUserNo` | 30 | FX사용자번호 (`FX_USER_NO`) |
| `orec[].fndNo` | 30 | 펀드번호 (`FND_NO`) |
| `orec[].fndNm` | 50 | 펀드명 (`FND_NM`) |
| `orec[].lastTs` | 14 | 최종 사용일시 (`LAST_TS`, `YYYYMMDDHH24MISS`) — 헤더엔 "최종선택일시", §6 |
| `orec[].fxSysLsmdId` | 30 | 최종 등록/변경 ID (`FX_SYS_LSMD_ID`) |
| `orec[].fxSysLsmdTs` | 14 | 최종 변경일시 (`FX_SYS_LSMD_TS`, `YYYYMMDDHH24MISS`) |

> `.h` 는 `orec[1]` 단일 선언이지만 `content->sndl += (i_idx-1)*sizeof(orec[0])` 로 가변 확장 → 실제로는 fetch 건수만큼 가변 grid.

## 3. 구현 흐름 (`.pc`)

```
W1105S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  │                             comhdr_o->rtyp = "M"
  ├─ switch(mid->prGb)
  │     '1' / '2' → retc = W1105S01_F1(...)     ← 실제 로직 (둘 다 동일 함수)
  │     그외      → comhdr_o->mesg="작업구분 오류." 후 즉시 return
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)

W1105S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1105S01_O_SZ, memset(mod)
  ├─ 입력 바인딩: yprintf + l_rtrim → vPrGb, vFxUserNo
  ├─ 유효성: isNullSpace(vFxUserNo)==0 → e_msg "10555"(입력값 확인), eflg "1", RET_FAILURE
  ├─ EXEC SQL DECLARE W1105S01_C1 CURSOR  (TB_FXB_CSC011M, USE_YN='Y')
  │           prGb='1' → FX_USER_NO=:x  /  prGb='2' → WBCS_RLNM_ALTR_NO=:x
  ├─ OPEN → SQLCODE!=0 → e_msg "10118"(자료 없음), eflg "1", RET_FAILURE
  ├─ while(1) FETCH
  │     NO_DATA_FOUND → CLOSE, break
  │     기타 오류     → CLOSE, e_msg "10118", eflg "1", RET_FAILURE
  │     정상          → zsedit 로 orec[i_idx] 채우고 i_idx++
  ├─ i_idx==0 → e_msg "91001"(해당 자료 없음)  /  else → e_msg "91003"(조회 완료)
  └─ eflg "0", fnSetMsg, grid01_cnt=i_idx, sndl += (i_idx-1)*sizeof(orec[0]), RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CSC011M` | 사용자 펀드내역 | — | **SELECT** | `USE_YN='Y'` 인 사용자 펀드 행을 커서로 조회. `prGb` 에 따라 `FX_USER_NO`(1) 또는 `WBCS_RLNM_ALTR_NO`(2) 로 필터 |

*DML 요약: `TB_FXB_CSC011M` 커서 SELECT 1건 (INSERT/UPDATE/DELETE 없음). 조회 전용이나 진입부에서 명시적 commit/rollback 은 수행.*

## 5. 로직 / 필터 규칙

- **커서/DML**: `W1105S01_C1` 단일 커서. 조인 없음. 조건 = `USE_YN='Y'` + `prGb` 분기(`FX_USER_NO` / `WBCS_RLNM_ALTR_NO` 택1).
  - `prGb`, `FX_USER_NO`, `WBCS_RLNM_ALTR_NO` 를 SQL 안의 `OR` 조합으로 처리 — 바인드 `:vPrGb` 값에 따라 한쪽 절만 활성.
- **입력 유효성**: `fxUserNo` 공백/NULL → `10555` 거부. `prGb` 는 진입부 `switch` 에서 `'1'/'2'` 외 거부(별도 e_msg 없이 COMHDR mesg="작업구분 오류.").
- **날짜 포맷**: `LAST_TS`, `FX_SYS_LSMD_TS` 는 `TO_CHAR(..,'YYYYMMDDHH24MISS')` 14자리 문자열.
- **트랜잭션**: 조회지만 진입부에서 `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **결과 전달**: grid + COMHDR. 성공은 항상 `eflg='0'`, `e_msg.code` 는 건수에 따라 `91003`(있음)/`91001`(없음). 실패는 `eflg='1'` + `10555`/`10118`.
  - 0건이어도 `eflg='0'`(정상) 으로 나가고 `91001` 안내 메시지만 실린다 — 오류 아님.
  - → WTG `/v1/tx` 는 `errn`≠0 여도 전문 있으면 200, `errn`/COMHDR 본문 동봉(레거시 규약).
