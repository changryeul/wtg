  ├─ prthdr / getmid / getmod    COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1354A01_F1(...)     ← 등록 (INSERT)
  │     '2' → retc = W1354A01_F2(...)     ← 수정 (UPDATE)

  ├─ retc==RET_SUCCESS ? (eflg="0", l_dbcommit()) : (eflg="1", l_dbrollback())
  │                                       (명시적 커밋, AUTOCOMMIT 아님)
  ├─ fnSetMsg(&e_msg, comhdr_o)  e_msg.code → COMHDR(mesg/rcod/…) 매핑

  └─ 정상 → e_msg 10150, RET_SUCCESS

W1354A01_F2()                    ← 수정
  ├─ (F1 과 동일 바인딩/유효성; vFxSysLsmdId ← comhdr_i->usid)

  │     FX_SYS_LSMD_TS=SYSDATE) WHERE FX_PDCD=:x AND CRCD=:y AND FX_CUS_LMT_DCD=:z

  ├─ 바인딩: vFxPdcd/vCrcd/vFxCusLmtDcd, vFxSysLsmdId ← mid->fxSysLsmdId (F1/F2 와 다름!)

## 5. 로직 / 필터 규칙

- **DML**: 커서·조인 없음. `prGb` 분기별 단일 DML.
  - 키 조건(수정/삭제 WHERE) = `FX_PDCD AND CRCD AND FX_CUS_LMT_DCD` (3-컬럼 복합키 가정).
  - 등록 INSERT 는 `USE_YN='Y'` / `FX_SYS_LSMD_TS=SYSDATE` 서버 시각 스탬프, 금액은 `NVL(...,0)`.
  - 삭제는 물리삭제가 아닌 `USE_YN='N'` UPDATE (soft-delete).
- **입력 유효성** (세 경로 공통): `fxPdcd` 공백→`32006`, `crcd` 공백→`32029`, `fxCusLmtDcd` 공백→`10710`. 어느 하나라도 걸리면 `RET_FAILURE`(→ rollback).
- **트랜잭션**: `RET_SUCCESS→eflg="0"+l_dbcommit / 실패→eflg="1"+l_dbrollback`. AUTOCOMMIT 아님 — 명시적 커밋 필수.
- **에러/결과 전달**: 데이터 대신 COMHDR — `fnSetMsg` 가 `e_msg.code`→`mesg`/`rcod` 매핑. 성공 코드 `10150`(정상처리, 조회로 확인).
  - 등록: `10223`(중복) / `10122`(처리오류)
  - 수정: `10221`(수정대상 없음) / `10122`
  - 삭제: `10326`(삭제대상 없음) / `10122`
  - → WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, `errn`/COMHDR 본문 동봉(레거시 규약).
