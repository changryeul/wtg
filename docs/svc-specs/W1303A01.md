# 서비스 명세 — `W1303A01` (표준상품정보 등록/수정)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 .pc 실 구현 기준이며, .h 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1303A01` |
| 서비스 이름 | 표준상품정보 등록/수정 (헤더 표기는 "표준상품정보 등록") |
| 유형 | 복합 (등록 INSERT / 수정 UPDATE — `prGb` 로 분기) |
| 구현 | `win/src/trn/W1300/W1303A01.pc` (723 라인) |
| I/O 헤더 | `win/src/inc/trn/W1303A01.h` |
| 라우팅 alias | `W1303A01` |
| 채널 | 매매 엔진 trn (COMHDR 기반 고정폭 전문) |

FX 표준상품(통화페어 × 테너 조합) 마스터 정보를 등록(신규)하거나 수정한다. 단일 행 CRUD 서비스로, 조회/그리드 반환은 하지 않는다 (출력은 처리 건수 1건만).

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-----|-----|-----|-----|-----|
| TB_FXB_CMG006M | 표준상품정보 | (없음) | SELECT | 등록 전 중복 존재 확인 (CRNC_PAIR_ID + TNR_ID COUNT) |
| TB_FXB_CMG006M | 표준상품정보 | (없음) | INSERT | 등록(prGb=1) — 14컬럼 신규 행 |
| TB_FXB_CMG006M | 표준상품정보 | (없음) | UPDATE | 수정(prGb=2) — FX_PDCD+CRNC_PAIR_ID+TNR_ID 키로 갱신 |

*DML 요약: 단일 테이블 `TB_FXB_CMG006M` 만 사용. 등록 경로는 SELECT COUNT → INSERT, 수정 경로는 UPDATE. 커서/JOIN/DELETE 없음. 트랜잭션은 진입 콜백에서 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 명시 수행 (AUTOCOMMIT 아님).*

## 5. 로직 / 필터 규칙

- **작업 분기**: `mid->prGb[0]` — `'1'`=등록(F1), `'2'`=수정(F2), 그 외 "작업구분 오류." 반환.
- **키**: 논리 키는 (FX_PDCD, CRNC_PAIR_ID, TNR_ID). 단 등록 중복검사는 (CRNC_PAIR_ID, TNR_ID) 두 컬럼만으로 판단 (§6 참조).
- **필수값 검증** (`isNullSpace`):
  - 등록(F1): fxPdcd, crncPairId, tnrId, mrktDataDcd, expiFnshYmd, fxXrtFnnlMthdDcd. useYn 은 주석처리로 미검증.
  - 수정(F2): 위 6개 + useYn. refDfcrPairId/refWndlCrncPairId 는 주석처리로 미검증.
- **작업자 자동세팅**: fxSysLsmdId 공란이면 COMHDR 의 usid 앞 6바이트 사용.
- **감사 컬럼**: FX_SYS_LSMD_TS 는 SYSDATE (INSERT/UPDATE 공통).
- **트랜잭션**: F1/F2 는 RET_SUCCESS/RET_FAILURE 만 반환, commit/rollback 은 진입 콜백이 일괄 처리.
- **에러 코드 (e_msg.code / rcod → mesg, eflg=1)**:
  - 등록 필수값: FX상품코드 `32006`, 통화페어ID `10183`, 테너ID `10583`, 시장데이터구분코드 `10584`, 만기종료년월일 `43014`, FX환율재정방식구분코드 `10584`(중복 코드).
  - 수정 필수값: 전부 `90001`.
  - 중복 등록: `00113` (SELECT dCnt!=0 또는 INSERT SQLCODE==-1).
  - SQL 오류: 등록 `rcod=00114`, 수정 `rcod=90000`, 수정 NO_DATA_FOUND `rcod=90000`.
  - 정상: `e_msg.code=10150`, eflg="0".
