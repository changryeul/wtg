## 3. 구현 흐름 (`.pc`)

```
W1509A02()                         // 콜백 진입
  ├─ prthdr / getmid / getmod      // COMHDR·입력·출력 주소 확보
  ├─ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  ├─ retc = W1509A02_F1(...)       // 실제 등록 로직
  ├─ retc==RET_SUCCESS → l_dbcommit()
  │  else               → l_dbrollback()
  └─ return content->sndl

W1509A02_F1()                      // 등록 (prototype 주석은 "조회"로 오기)
  ├─ getmid/getmod, sndl += O_SZ, mod memset, comhdr_o->rtyp="G"
  ├─ SELECT APLL_BSOP_YMD → iBsopYmd  FROM TB_FXB_CMG012M (FDM_ID='FRX')  // 영업일
  ├─ 입력항목 MOVE (yprintf/l_n2d/l_rtrim) + INPUT LOG
  ├─ [채번] SELECT NVL(COUNT(PXNT_NO),0)+1 → oSrn
  │         FROM TB_FXB_TRC002L
  │         WHERE BSOP_YMD/WBCS_RLNM_ALTR_NO/BYSEL_DCD 일치
  │     └ SQLCODE!=0 → e_msg 91001(NO_DATA)/95000, return RET_FAILURE
  ├─ iPxntNo = GetDate+GetTime+oSrn(%02d)
  ├─ if (oSrn <= 1)                 // 당일·고객·매매구분 최초 등록만
  │     ├─ INSERT INTO TB_FXB_TRC002L (대부분 컬럼 ' '/0 하드코딩)
  │     │     └ SQLCODE!=SUCCESS → e_msg 95000, eflg=1, fnSetMsg, return RET_FAILURE
  │     ├─ (§5 UPDATE TB_FXB_TRC003L — 전체 주석 처리, dead)
  │     └─ mod->PxntNo = iPxntNo   // 출력 세팅
  │  else
  │     └─ "기등록 데이터 존재" 로그만 (mod->PxntNo 미세팅)
  └─ eflg="0", e_msg.code="00000", rcod=COD_SUCCESS, return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-------|-------|-----|-----|-----|
| `TB_FXB_CMG012M` | FX 기준(영업)일자 관리 | — | SELECT | `FDM_ID='FRX'` 의 적용영업일자(`APLL_BSOP_YMD`) 조회 |
| `TB_FXB_TRC002L` | 선물환상품설명서 제출내역 | — | SELECT / INSERT | 채번용 COUNT 조회 + 신규 제출내역 등록 |
| `TB_FXB_TRC003L` | 선물환사전점검등록내역 | A | (UPDATE — **주석 처리 dead code**) | 상품설명서확인여부/번호 갱신 (미동작) |

*DML 요약: SELECT ×2 (`CMG012M` 영업일, `TRC002L` 채번), INSERT ×1 (`TRC002L`). `TRC003L` UPDATE 는 전체 블록 주석 처리되어 실행되지 않음. 참조만 되고 미사용인 include 구조체: `WCM001.h`(TNRBZDYINQ), `WLM005.h`(LQDNPRC), `WCM202.h`(FWDBALS).*

## 5. 로직 / 필터 규칙

- **영업일**: 입력 `BsopYmd` 를 쓰지 않고 `TB_FXB_CMG012M`(`FDM_ID='FRX'`)의 `APLL_BSOP_YMD` 를 `iBsopYmd` 로 재조회하여 채번·INSERT 에 사용.
- **채번 (일련번호 oSrn)**: `BSOP_YMD + WBCS_RLNM_ALTR_NO + BYSEL_DCD` 기준으로 기존 건수 COUNT + 1. 상품설명서번호 `PxntNo = 오늘일자(8) + 현재시각(6) + oSrn(2자리)` 형식 = 16자리.
- **최초 등록 조건**: `oSrn <= 1` 일 때만 INSERT 수행하고 `mod->PxntNo` 를 세팅. `oSrn > 1`(이미 등록 존재)이면 로그만 남기고 정상 종료하되 출력번호는 빈값.
- **INSERT 값**: 대부분 업무 컬럼(`ARR_YMD`, `BASE_CRCD`, `FX_ARR_AMT`, `EXPI_FNSH_YMD`, `FX_ARR_XRT`, `FX_STLM_MTHD_DCD`)을 입력값이 아닌 `' '` / `0` 상수로 저장. 실제 저장되는 입력값은 `BSOP_YMD`(DB조회), `WBCS_RLNM_ALTR_NO`, `PXNT_NO`(채번), `BYSEL_DCD`, `FX_SYS_LSMD_ID`(=COMHDR usid), `FX_SYS_LSMD_TS`(SYSDATE) 뿐.
- **트랜잭션**: AUTOCOMMIT 아님. 콜백에서 `RET_SUCCESS` → `l_dbcommit()`, 실패 → `l_dbrollback()`.
- **에러 처리**:
  - 채번 SELECT `NO_DATA_FOUND` → `e_msg.code=91001`, `RET_FAILURE`
  - 채번 SELECT 기타 오류 → `e_msg.code=95000`, `RET_FAILURE`
  - INSERT 오류 → `e_msg.code=95000`, `comhdr_o->eflg="1"`, `fnSetMsg`, `RET_FAILURE`
  - 정상 → `comhdr_o->eflg="0"`, `e_msg.code="00000"`, `comhdr_o->rcod=COD_SUCCESS`
