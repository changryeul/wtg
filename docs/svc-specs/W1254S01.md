## 3. 구현 흐름 (`.pc`)

```
W1254S01()                       ← 진입 (콜백)
  ├─ prthdr / getmid / getmod    COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' → retc = W1254S01_F1(...)     ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return (MSG_NOWORK)
  ├─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()  (명시적, AUTOCOMMIT 아님)
  └─ fnSetMsg(&e_msg, comhdr_o) → sndl 반환

W1254S01_F1()                    ← 조회
  ├─ getmid/getmod, sndl += W1254S01_O_SZ, memset(mod)
  │   max_row = MAX_RRB / sizeof(orec[0])   ← 페이지당 최대 건수
  ├─ GRID key 처리 (content->keyc)
  │     K_NEXT → content_get_nkey → vKeyv
  │     K_PREV → content_get_pkey → vKeyv   (단, SQL 은 `NEXT_KEY > :vKeyv` 정방향만 — §6)
  │     default(최초) → vKeyv="0"
  ├─ 입력 바인딩: yprintfrtrim → vBsopYmdFrdt/Todt/WbcsRlnmAltrNo/MngmBrcd/
  │                              FxMsgPtrnDcd/Cpn/SmssTgtDcd/SmtsSttsDcd
  │              usid→hls_upd_id, loip→hls_upd_ip
  ├─ 유효성: vBsopYmdFrdt/vBsopYmdTodt 둘 다 필수 (빈값 → e_msg "43037", RET_FAILURE)
  ├─ EXEC SQL DECLARE W1254S01_C1 CURSOR
  │     인라인 뷰(ICMC006L): TB_FXB_CMC006L
  │       LEFT JOIN TB_FXB_CSC001M   (고객약정정보 → 고객명/관리부점)
  │       LEFT JOIN TB_FXB_CMC003D ×4 (FX_MSG_PTRN_DCD / SMTS_STTS_DCD / SMSS_TGT_DCD / FX_SNDG_TCD 명칭)
  │       WHERE BSOP_YMD BETWEEN Frdt AND Todt + 선택필터(NVL 패턴)
  │       외곽 WHERE NEXT_KEY > :vKeyv  ORDER BY NEXT_KEY
  │       NEXT_KEY = (99991231-BSOP_YMD)||(99999999-MSG_SQN)  ← 최신순 정렬 키
  ├─ OPEN → SQLCODE≠0 : NO_DATA→"91001"+RET_SUCCESS / 기타→"10118"+RET_FAILURE
  ├─ for(ii=0; ; ii++)
  │     c-버퍼 memset → FETCH INTO (27개 컬럼, 마지막 cKeyv=NEXT_KEY)
  │       NO_DATA → break / 기타 err → CLOSE+"10118"+RET_FAILURE
  │       ii==max_row → i_isnext++ ; break  (이 건은 저장 안 함 — 다음페이지 존재 표시)
  │       mod->orec[ii].* 필드별 세팅 (fxMsgCon 만 sprintf), vKeyv = cKeyv
  ├─ CLOSE W1254S01_C1
  ├─ 결과 분기
  │     ii==0 && i_isnext==0 → "91001" (자료없음)
  │     i_isnext==1          → content_set_nkey(vKeyv) + "91002" (계속됨)
  │     그외(완료)           → "91003" + grid01_cnt=ii
  ├─ #if 0 : 고객정보조회이력저장 fnInsCusInfoInqPrhs (비활성 — §6)
  └─ grid01_cnt=ii ; sndl += (ii-1)*sizeof(orec[0]) ; return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='1'`(조회) 만 처리. 그 외 "작업구분 오류" 후 반환.
- **필수 입력**: `bsopYmdFrdt` / `bsopYmdTodt` 두 값 모두 비어 있으면 `43037` 로 실패. (SQL 은 NVL 로 미입력 시 SYSDATE 로 대체하나, 코드가 앞단에서 빈값을 먼저 거부한다.)
- **선택 필터(NVL 패턴)**: `wbcsRlnmAltrNo`, `smssTgtDcd`, `fxMsgPtrnDcd`, `smtsSttsDcd`, `mngmBrcd`(CSC001M) — 값 있으면 등가필터, 없으면 컬럼 자기값과 비교(=전체). `cpn` 은 바인딩되나 **SQL 필터가 주석처리되어 미동작**(§6).
- **페이징**: NEXT_KEY = `(99991231-BSOP_YMD)||(99999999-MSG_SQN)` 로 최신 영업일·최신 순번이 앞서도록 오름차순 정렬. 외곽 `NEXT_KEY > :vKeyv`. `max_row` 초과 1건을 더 FETCH 하여 `i_isnext` 로 다음 페이지 존재를 판정하고, 그 건은 저장하지 않는다(표준 look-ahead). `K_PREV`(이전) 는 `content_get_pkey` 로 키만 바꿀 뿐 SQL 비교는 `>` 정방향 고정이라 실질 역방향 페이징은 되지 않는다(§6).
- **출력 세팅**: 대부분 `zsedit`. `fxMsgCon`(2100B) 만 `sprintf("%s")` 사용 — 주석에 "길이 2100이라 zsedit 먹통" 명시.
- **트랜잭션**: `RET_SUCCESS→eflg="0"+l_dbcommit`, 실패→`eflg="1"+l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드**:
  - 필수 영업일 누락: `43037` (영업년월일 확인), RET_FAILURE.
  - 커서 OPEN 오류: NO_DATA→`91001`+RET_SUCCESS / 기타→`10118`+RET_FAILURE.
  - FETCH 오류: `10118` + CLOSE + RET_FAILURE.
  - 자료없음(ii==0): `91001`.
  - 조회 계속: `91002` + `content_set_nkey`.
  - 조회 완료: `91003` + grid01_cnt.
- **결과 전달**: 그리드(건별 orec) + COMHDR(`eflg`/`e_msg.code`→`mesg`) + NEXT_KEY. WTG `/v1/tx` 는 errn≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
