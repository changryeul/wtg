## 2. I/O 전문 (`.h`)

**Input `W1201S03_I`** (총 12B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 통화별변동휴일조회(F1), `'2'` 휴일여부조회(이전영업일, F2), `'3'` 휴일여부조회(이후영업일, F2). 그 외 "작업구분 오류." |
| `crcd` | 3 | 통화코드 — 필수(공백이면 e_msg 32029) |
| `flctHlddYmd` | 8 | **F1**: 변동휴일 연도 필터(코드상 4자리 year 로 비교 — §6 참조). **F2**: 기준(만기)년월일 — 8자리 필수(e_msg 32007) |

**Output `W1201S03_O`** — grid (가변 orec, 헤더상 `orec[1]` 최소 선언)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (`zredit` 로 세팅) |
| `orec[].crcd` | 3 | **F1**: 통화코드. **F2**: 세팅 안 함(공백) |
| `orec[].flctHlddYmd` | 8 | **F1**: 변동휴일년월일. **F2**: 산정된 직전/직후 영업일자(YYYYMMDD) |
| `orec[].fxHlddNm` | 50 | **F1**: FX휴일명. **F2**: 기준일이 영업일이면 `'Y'`, 비영업일이면 `'N|<휴일명>'` |

> orec 는 가변 반복. `content->sndl` 은 `W1201S03_O_SZ`(=orec[1] 포함) 후 `(ii-1)*sizeof(orec[0])` 로 조정 → 실제 반환 orec 개수만큼 전문 길이 확정.

## 3. 구현 흐름 (`.pc`)

```
W1201S03()  [진입, L43]
  prthdr / getmid / getmod                       // COMHDR·mid·mod 주소 획득
  e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  switch(mid->prGb[0])
    '1'      → W1201S03_F1()                      // 통화별변동휴일 목록 조회
    '2','3'  → W1201S03_F2()                      // 휴일여부 + 직전/직후 영업일
    default  → eflg='1', mesg="작업구분 오류.", return
  retc==RET_SUCCESS ? eflg='0'+l_dbcommit() : eflg='1'+l_dbrollback()
  fnSetMsg(&e_msg, comhdr_o)                      // e_msg.code → mesg 세팅
  return content->sndl

W1201S03_F1()  [L118] 통화별변동휴일조회
  max_row = MAX_RRB / sizeof(orec[0])            // 페이지 크기
  keyc(K_NEXT/K_PREV/default) → vKeyv (NEXT KEY, default "0")
  입력검증: strlen(vCrcd)<=0 → e_msg 32029, RET_FAILURE
  DECLARE/OPEN W1201S03_C1 (CMG011M 변동휴일)
    OPEN 시 NO_DATA_FOUND → e_msg 91001, RET_SUCCESS
          기타오류        → e_msg 10118, RET_FAILURE
  FETCH loop (ii=0..)
    ii==max_row → i_isnext++, break            // 다음페이지 존재 감지(1건 초과 fetch)
    orec[ii] ← crcd/flctHlddYmd/fxHlddNm
    vKeyv ← cKeyv (NEXT KEY 갱신)
  CLOSE C1
  ii==0 && i_isnext==0 → e_msg 91001 (자료없음)
  i_isnext==1          → content_set_nkey, e_msg 91002 (조회 계속)
  else                 → e_msg 91003 (조회 완료)
  grid01_cnt=ii, sndl += (ii-1)*sizeof(orec[0]), RET_SUCCESS

W1201S03_F2()  [L327] 휴일여부조회
  입력검증: strlen(vCrcd)<=0 → e_msg 32029; strlen(vExpiYmd)!=8 → e_msg 32007
           (둘 다 eflg='1'+fnSetMsg 후 RET_FAILURE)
  Q1: 기준일 영업일여부 cBzdyYn 산정 (CMG009M 주말 + CMG010M 고정휴일 + CMG011M 변동휴일 3-way 조인)
  cBzdyYn=='N' → Q2: 휴일명 조회(3 UNION ALL, ROWNUM=1) → orec[0].fxHlddNm = "N|휴일명"
  cBzdyYn=='Y' → orec[0].fxHlddNm = 'Y'
  loop: nBzdyYn!='Y' 동안
    Q3: prGb='2'면 -1일, else +1일 이동한 nExpiYmd 의 영업일여부 재산정
    nExpiYmd ← 이동일자
  orec[0].flctHlddYmd = nExpiYmd (직전/직후 영업일)
  ii=1, e_msg 91003, eflg='0', grid01_cnt=1, RET_SUCCESS
  (각 SELECT NO_DATA_FOUND → e_msg 91001+RET_SUCCESS / 기타 → 10118+RET_FAILURE)
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG009M` | 통화별주말기본 | (F2 서브쿼리) | SELECT | 요일(DWK_DCD) 기준 주말여부 + 주말 코멘트명(COMNT_CON) |
| `TB_FXB_CMG010M` | 통화별고정휴일정보기본 | (F2 서브쿼리) | SELECT | 월일(FXNG_HLDD_MD) 기준 고정휴일여부 + 고정휴일명(FX_HLDD_NM) |
| `TB_FXB_CMG011M` | 통화별변동휴일정보기본 | `CMG011M` | SELECT | 변동휴일 목록(F1 커서) + 변동휴일여부/휴일명(F2) |

*DML 요약: INSERT/UPDATE/DELETE 없음. 순수 조회 서비스. F1 은 `W1201S03_C1` 커서(CMG011M) FETCH, F2 는 3개 단건 SELECT(3-way 조인 + UNION ALL). 마스터 필터는 전부 `FDM_ID='FRX'` + `CRCD`. 진입부 `l_dbcommit/l_dbrollback` 은 관례적 트랜잭션 종결(조회만이므로 실질 변경 없음).*

## 5. 로직 / 필터 규칙

- **F1 커서(W1201S03_C1)**: `FDM_ID='FRX' AND CRCD=:vCrcd AND SUBSTR(FLCT_HLDD_YMD,1,4)=NVL(:vFlctHlddYmd, SUBSTR(FX_HLDD_NM,1,4)) AND FDM_ID||CRCD||FLCT_HLDD_YMD > :vKeyv` 순서 `FDM_ID, CRCD, FLCT_HLDD_YMD`. NEXT KEY = `FDM_ID||CRCD||FLCT_HLDD_YMD`.
- **페이징(F1)**: `max_row = MAX_RRB / sizeof(orec[0])`. FETCH 시 `ii==max_row` 도달하면 1건 초과분을 저장하지 않고 `i_isnext++` 후 break → `content_set_nkey`+e_msg 91002 로 다음 페이지 유도. `K_NEXT/K_PREV` 는 `content_get_nkey/pkey`, 최초는 vKeyv="0".
- **F2 영업일 산정(Q1/Q3)**: 3개 인라인 뷰(WKND_YN·FXNG_HLDD_YN·FLCT_HLDD_YN)의 카티전 조인. 셋 다 'N' 이어야 BZDY_YN='Y'(영업일). `DWK_DCD = TO_CHAR(TO_DATE(:ymd,'YYYYMMDD'),'D')-1` 로 요일 매핑.
- **F2 휴일명(Q2)**: 주말코멘트/고정휴일명/변동휴일명을 `UNION ALL` 후 `ROWNUM=1` 로 1건 → `:cBzdyYn||'|'||HLDD_NM` 형식(`"N|휴일명"`)으로 반환.
- **F2 영업일 이동 loop**: `nBzdyYn=='Y'` 될 때까지 prGb='2'면 -1일, '3'(및 그 외)면 +1일씩 이동. 종료 시 `orec[0].flctHlddYmd`=산정된 영업일.
- **트랜잭션**: retc==RET_SUCCESS → eflg='0'+`l_dbcommit`, 아니면 eflg='1'+`l_dbrollback`.
- **에러/메시지(e_msg.code → fnSetMsg)**:
  - `32029` 입력값[통화코드] 확인
  - `32007` 입력값[만기년월일] 확인 (F2)
  - `91001` 해당하는 자료가 없습니다 (RET_SUCCESS 로 반환 — 자료없음은 오류 아님)
  - `91002` 조회가 계속됩니다 (F1 다음페이지)
  - `91003` 조회가 완료되었습니다
  - `10118` 자료 조회중 오류 (SQL 오류, RET_FAILURE)

## 6. 주의 — 문서/코드 불일치

| 위치 | 헤더/주석 표기 | 실제(.pc) |
|---|---|---|
| `.h` `flctHlddYmd` 주석 | "변동휴일년월일" | F2(prGb 2/3)에서는 **기준(만기)년월일** 로 사용(`vExpiYmd`, 8자리 필수). F1 에서만 변동휴일 연도 필터. 동일 필드가 분기별로 의미 다름 |
| `.h` `orec.flctHlddYmd` 주석 | "변동휴일년월일" | F2 출력은 변동휴일이 아니라 **산정된 직전/직후 영업일자**. 코드 주석(L640 `// 영업일여부`)도 오기(실제는 일자) |
| `.h` `orec.fxHlddNm` 주석 | "FX휴일명" | F2 출력은 휴일명이 아니라 **`'Y'` 또는 `'N|<휴일명>'`** 문자열(영업일여부+휴일명 결합). 코드 주석(L550 `// 영업일여부`)이 실제 의미 |
| `.h` `orec.crcd` | (통화코드 항상 채워짐 가정) | F2 는 `orec[0].crcd` 를 세팅하지 않음 → 공백 반환 |
| F1 커서 WHERE (L214) | (연도 필터 의도) | `SUBSTR(FLCT_HLDD_YMD,1,4)=NVL(:vFlctHlddYmd, SUBSTR(FX_HLDD_NM,1,4))` — 좌변이 4자리 year 인데 입력 `flctHlddYmd` 는 8자리 스펙. 8자리 값을 그대로 넣으면 4자리와 불일치로 무결과. 또 NVL fallback 이 `SUBSTR(FX_HLDD_NM,1,4)`(휴일명 앞4자)로 잡혀 있어 의미상 오류로 추정(정상이라면 `SUBSTR(FLCT_HLDD_YMD,...)` 이어야 함). 실무상 4자리 연도 입력 전제로 동작 |
| F2 이동 loop Q3 CMG010M (L601) | 최초 Q1(L458)은 `FXNG_HLDD_MD = SUBSTR(:vExpiYmd,5,8)`(MMDD) | 이동 loop Q3 은 `FXNG_HLDD_MD = :nExpiYmd`(8자리 YYYYMMDD 전체)로 비교 → 월일 컬럼과 자릿수 불일치로 **고정휴일이 loop 단계에서 매칭되지 않는 버그**. 최초 검사와 loop 검사의 고정휴일 판정 기준이 다름 |
| F1 `content->sndl` 조정 (L319) | — | 자료없음(ii==0)에도 `sndl += (0-1)*sizeof(orec[0])` 로 orec 1개분 **음수 가산** → 전문 길이 축소. (91001 반환 경로에서 발생) |
| F2 CMG010M SUBSTR (L458) | — | `SUBSTR(:vExpiYmd,5,8)` 은 8자리 문자열에서 5번째부터 8자 → 실효 4자(MMDD). length 인자가 컬럼 길이 초과지만 Oracle 은 남은 문자만 반환하여 결과적으로 MMDD 로 동작(의도된 quirk) |
| F2 무한 loop 가능성 | — | 이동 loop 는 영업일을 만날 때까지 무한 반복. 데이터 이상으로 계속 비영업일이면 NO_DATA_FOUND(91001) 로만 탈출. 최대 반복 상한 가드는 없음 |
