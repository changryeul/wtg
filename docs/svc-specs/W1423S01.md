## 5. 로직 / 필터 규칙

- **입력 검증**: `iWbcsRlnmAltrNo` trim 후 길이 0 이면 `10201`(고객번호를 입력하세요) 실패.
- **고객 존재**: `fnCustInfoQry` 실패 시 `10177`(고객정보 없음). 조회된 `custInfo` 는 이후 사용되지 않고 존재검증에만 쓰인다.
- **거래통제시간**: `TB_FXB_CMC003D`(`TRN_CNTL_TIM`, `CMCD_VL='0'`) 의 `CMCD_VL_REF_CON1/CON2` 를 현재 `HH24MISS` 와 비교해 `vSchYn` 산출. `'N'`(조회불가) 이면 `32713`(은행 일자전환 거래 제한) 실패. (§6: CASE 조건식 논리 의심)
- **영업일자**: `TB_FXB_CMG012M`(FDM_ID='FRX') 의 `APLL_BSOP_YMD` 를 소진액 필터 기준일로 사용.
- **한도 설정액**: `TB_FXB_TRC001M` 에서 `FX_USER_NO='ALL'`, `FX_PDCD='SPT'`, `CRCD='USD'` 고정 조건 단건.
- **소진액 산정**: 서브쿼리 UNION ALL —
  - 미체결(`TRG001L`): `ORDN_VALD_YMD >= 영업일`, `FX_PDCD='SPT'`, `FX_ORDN_BALN_QTY > 0`, `USE_YN='Y'`.
  - 체결(`TRG003L ⨝ TRG001L`): `CNTT_YMD = 영업일`, `FX_PDCD='SPT'`, `CNTT_STTS_DCD='6'`, `USE_YN='Y'`, `ORDN_NO` 조인.
  - 두 금액 모두 `FX_SDCU_AMT * MUX_RT`(최신 고시환율, JPY 는 /100) 로 USD 환산 합산.
  - 잔액 = 설정액 − 소진액 (동일 SELECT 내 `:dDd1MxmmAmt` 바인드 재사용).
- **초과여부**: 잔액 ≥ 0 → `'Y'`, < 0 → `'N'` (§6 의미 역전 의심).
- **에러코드(e_msg.code → mesg)**: `10201` 고객번호 입력, `10177` 고객정보 없음, `10118` 조회가능시간 조회오류, `32713` 일자전환 거래제한, `10119` 영업일정보 오류, `10310` 고객별거래한도 조회오류, `91003` 정상.
- **eflg**: 정상 `"0"`, 실패 `"1"`. 진입부에서 성공 시 commit, 실패 시 rollback.
