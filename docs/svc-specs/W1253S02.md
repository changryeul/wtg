# 서비스 명세 — `W1253S02` (SMS발송직원정보조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 .pc 실 구현 기준이며, .h 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1253S02` |
| 이름 | SMS발송직원정보조회 |
| 유형 | 조회 (그리드, 페이징) |
| 구현 | `win/src/trn/W1250/W1253S02.pc` (382 lines) |
| I/O 헤더 | `win/src/inc/trn/W1253S02.h` (70 lines) |
| 라우팅 alias | `W1253S02` |
| 채널 | 매매(broker) — COMHDR 기반 고정폭 전문 |
| 작업구분 | `prGb='1'` → `W1253S02_F1` (조회). 그 외 "작업구분 오류." |

특정 FX 메시지유형(`fxMsgPtrnDcd`)에 등록된 SMS 발송대상 직원 목록을, 사용자정보/부점/코드 마스터와 조인하여 직원번호·직원명·구분·부점·레벨·휴대폰·이메일까지 그리드로 반환한다.

## 3. 구현 흐름 (`.pc`)

```
W1253S02(dispatch)
 ├ prthdr / getmid / getmod                        # 공통헤드 + I/O 주소
 ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
 └ switch(mid->prGb[0])
     ├ '1'  → W1253S02_F1()                        # 조회
     └ else → comhdr_o.mesg="작업구분 오류." + MSG_NOWORK, 즉시 return
   [F1 반환 후] retc==RET_SUCCESS → eflg='0' + l_dbcommit()
                else             → eflg='1' + l_dbrollback()
   fnSetMsg(&e_msg, comhdr_o)  → return content->sndl

W1253S02_F1(조회)
 ├ getmid/getmod, sndl += O_SZ, mod 0-clear
 ├ max_row = MAX_RRB / sizeof(orec[0])             # 페이지당 최대 건수
 ├ NEXT KEY 취득: switch(content->keyc)
 │    K_NEXT → content_get_nkey / K_PREV → content_get_pkey / else → "0"
 ├ 입력 정규화: yprintfrtrim(vFxMsgPtrnDcd, vSndgTgtEmn), atoi>0 이면 %02d
 ├ 유효성: strlen(vFxMsgPtrnDcd)<=0 → e_msg.code=10527, RET_FAILURE
 ├ DECLARE W1253S02_C1 CURSOR  (CMC015M ⟕ CSC004M ⟕ CMC001F ⟕ CMC003D)
 ├ OPEN → SQLCODE 체크 (NO_DATA_FOUND→10709 RET_SUCCESS / 그외→10708 RET_FAILURE)
 └ FETCH loop (ii=0..)
      ├ NO_DATA_FOUND → break / 그외 → CLOSE + 10118 RET_FAILURE
      ├ ii==max_row → i_isnext++, break (초과분 미저장)
      ├ l_rtrim 후 mod->orec[ii] 채움
      └ vKeyv = cFxUserNo                          # 다음 페이지 커서 키
    CLOSE W1253S02_C1
    ├ ii==0 && i_isnext==0 → e_msg.code=91001 (자료없음)
    ├ i_isnext==1          → content_set_nkey, e_msg.code=91002 (계속)
    └ else                 → e_msg.code=91003 (완료), grid01_cnt=ii
    grid01_cnt=ii (재설정), sndl += (ii-1)*sizeof(orec[0])
    return RET_SUCCESS
```

트랜잭션: 조회 서비스이나 dispatch 레벨에서 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 을 명시적으로 호출 (AUTOCOMMIT 아님).

## 5. 로직 / 필터 규칙

- **커서 필터** (`W1253S02_C1`):
  - `CMC015M.FX_MSG_PTRN_DCD = TRIM(:vFxMsgPtrnDcd)` — 메시지유형 필수 매칭
  - `CMC015M.SNDG_TGT_EMN = NVL(:vSndgTgtEmn, CMC015M.SNDG_TGT_EMN)` — 직원번호 지정 시 단건, 빈값이면 전체
  - `CMC015M.SNDG_TGT_EMN > :vKeyv` — 페이징 커서 (`ORDER BY SNDG_TGT_EMN` 오름차순)
- **입력 정규화**: `fxMsgPtrnDcd` 를 atoi 후 값>0 이면 `%02d` 로 2자리 zero-pad. 최종 길이 0 이면 조회 거부(10527).
- **조인 정책**: CSC004M/CMC001F/CMC003D 는 모두 LEFT JOIN → CMC015M 에 있으나 사용자 마스터에 없어도 행 반환 (직원명 등 NULL). 구분코드명은 `NVL(CMCD_VL_NM, FX_USER_DCD)` 로 코드값 fallback.
- **페이징**: `max_row = MAX_RRB / sizeof(orec[0])`. `ii==max_row` 도달 시 해당 행은 저장하지 않고 `i_isnext=1`, `content_set_nkey(vKeyv)` 로 다음 키 전달.
- **트랜잭션**: F1 성공(RET_SUCCESS)이면 commit, 실패면 rollback (조회지만 명시 호출).
- **에러/상태코드 (`e_msg.code` → fnSetMsg)**:
  | code | 의미 | eflg |
  |-----|-----|-----|
  | 10527 | FX메시지유형구분코드 미입력 | 1 (실패) |
  | 10709 | OPEN 시 NO_DATA_FOUND (대상직원 없음) | 0 (성공 반환) |
  | 10708 | CMC015M 조회(OPEN) 중 오류 | 1 |
  | 10118 | 커서 FETCH 오류 | 1 |
  | 91001 | 해당 자료 없음 (0건) | 0 |
  | 91002 | 조회 계속됨 (다음 페이지 존재) | 0 |
  | 91003 | 조회 완료 | 0 |
  - dispatch 기본 case: `comhdr_o.mesg="작업구분 오류."` + `MSG_NOWORK`.
