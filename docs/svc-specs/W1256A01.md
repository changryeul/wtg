## 3. 구현 흐름 (`.pc`)

```
W1256A01()                          ← 진입 (콜백, 함수 주석은 "조회"지만 dispatcher, §6)
  ├─ prthdr / getmid / getmod       COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정                  ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1256A01_F1()   등록 (INSERT)
  │     '2' → retc = W1256A01_F2()   수정 (UPDATE)
  │     '3' → retc = W1256A01_F3()   삭제 (DELETE)
  │     그외 → comhdr_o->mesg="작업구분 오류." 후 즉시 return
  ├─ retc==RET_SUCCESS ? eflg "0" + l_dbcommit()
  │                    : eflg "1" + l_dbrollback()     (명시적 커밋, AUTOCOMMIT 아님)
  └─ fnSetMsg(&e_msg, comhdr_o) → return content->sndl

W1256A01_F1()  등록
  ├─ 입력 바인딩 yprintf + l_rtrim (v*)  ; vFxSysLsmdId ← comhdr_i->usid
  ├─ 유효성: rgsnYmd 없음 → "10224" RET_FAILURE / pbnsCon 없음 → "10748" RET_FAILURE
  ├─ SELECT NVL(MAX(PBNT_RGSN_SQN),0)+1 INTO :vPbntRgsnSqn  (당일 채번, SQLCODE 미검사 §6)
  ├─ INSERT TB_FXB_CMC008L (순번/일자/시각=SYSDATE/제목/내용/대상구분/등록자/변경ID/변경일시=SYSDATE)
  │     SQLCODE!=0 → "10122"(등록 중 오류) RET_FAILURE
  ├─ push #1 화면 OPEN  : 커서 C1 → 로그인 고객 FETCH loop → mymq_broadcast(msgtp="2",target="0102",data3="1")
  ├─ push #2 화면 재조회: 커서 C2 → 동일 loop → broadcast(msgtp="1",subtp="2",data3="C")
  ├─ push #3 티커바     : 커서 C3 → 동일 loop → broadcast(msgtp="3",data3=제목명)
  └─ e_msg "10150"(정상처리) RET_SUCCESS

W1256A01_F2()  수정
  ├─ 입력 바인딩 (F1 과 동일)
  ├─ 유효성: rgsnYmd "10224" / pbnsCon "10748" / pbntRgsnSqn "10746"
  ├─ UPDATE TB_FXB_CMC008L SET 시각=SYSDATE,제목,내용,대상구분,변경ID
  │     WHERE RGSN_YMD=:vRgsnYmd AND PBNT_RGSN_SQN=:vPbntRgsnSqn
  │     NO_DATA_FOUND → "10800"(수정대상 없음) eflg"1"+fnSetMsg RET_FAILURE
  │     기타 SQLCODE!=0 → "10815"(수정중 오류) eflg"1"+fnSetMsg RET_FAILURE
  ├─ push #1/#2/#3 (F1 과 동일 3-단계, 커서 F2C1/F2C2/F2C3)
  └─ e_msg "10150" RET_SUCCESS

W1256A01_F3()  삭제
  ├─ 입력 바인딩: pbntRgsnSqn, rgsnYmd 만
  ├─ 유효성: rgsnYmd "10224" / pbntRgsnSqn "10746"
  ├─ DELETE TB_FXB_CMC008L WHERE RGSN_YMD=:vRgsnYmd AND PBNT_RGSN_SQN=TO_NUMBER(:vPbntRgsnSqn)
  │     NO_DATA_FOUND → "10326"(삭제대상 없음) RET_FAILURE
  │     기타 SQLCODE!=0 → "10122"(처리중 오류) RET_FAILURE
  ├─ push 화면 재조회 : 커서 F3C3 → 로그인 고객 loop → broadcast(msgtp="1",subtp="2",data3="C")
  └─ e_msg "10150" RET_SUCCESS
```

*DML 요약: 주 테이블은 `TB_FXB_CMC008L` 단일 — `prGb` 에 따라 INSERT('1')/UPDATE('2')/DELETE('3'). 등록 시 채번용 SELECT(MAX+1) 선행. 나머지 4개 테이블은 실시간 push 수신 대상(로그인 고객) 추출용 read-only 커서(`TB_FXB_CSC004M_C1~C3`, `_F2C1~F2C3`, `_F3C3`)로만 접근. 커밋/롤백은 dispatcher `W1256A01()` 가 서브함수 retc 로 일괄 결정.*

## 5. 로직 / 필터 규칙

- **작업 분기**: `prGb[0]` — `'1'` 등록 / `'2'` 수정 / `'3'` 삭제, 그 외 즉시 "작업구분 오류" 응답 (서브함수 미호출, 커밋/롤백 없음).
- **트랜잭션**: 서브함수는 DML 만 수행하고 `RET_SUCCESS/RET_FAILURE` 만 반환. 실제 `l_dbcommit()`/`l_dbrollback()` 은 dispatcher 가 단 한 번 수행.
- **채번(등록)**: 당일(`RGSN_YMD`) 기준 `NVL(MAX(PBNT_RGSN_SQN),0)+1`. 입력 `pbntRgsnSqn` 은 이 시점에 덮어써짐.
- **실시간 push**: 각 처리 성공 후 로그인 고객 커서를 FETCH loop 하며 `mymq_broadcast(mymq, BROADCAST, NULL, NULL, "HTS", <FX_USER_NO>, &RTS_DA, RTS_SCRNPUSH_SZ)` 호출. `RTS_SCRNPUSH` 필드로 화면 동작 제어:
  - **등록/수정**: 3-단계 — (1) `[0102]` 화면 OPEN(`msgtp="2"`, `data3="1"`), (2) 열려있으면 재조회(`msgtp="1"`,`subtp="2"`,`data3="C"`), (3) 티커바 공지(`msgtp="3"`, `data3=제목명`).
  - **삭제**: 재조회 단계(`data3="C"`) 1건만.
- **push 대상 쿼리**: 최근 로그인(당일-1 ~ 당일, `LGN_YN='Y'`) 사용자 ∩ (사용자정보·사용자별고객번호관계·고객약정정보 조인) 사용자.
- **에러(e_msg code → 메시지)**:
  - `10224` 입력값[등록일자] 확인 · `10748` 입력값[공지사항내용] 확인 · `10746` 입력값[공지등록순번] 확인
  - `10122` 공지사항등록/처리 중 오류 · `10800` 수정대상 없음 · `10815` 수정중 오류 · `10326` 삭제대상 없음
  - `10118` 자료조회 중 오류(커서) · `91001` 건수없음(커서 NO_DATA)
  - `10150` 정상처리 되었습니다(성공)
  - `eflg` "0"=정상 / "1"=비정상 (dispatcher 및 일부 서브함수에서 설정)
