# 서비스 명세 — `W1109S01` (내부직원 사용자정보 조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1109S01` |
| 이름 | 내부직원 사용자정보 조회 |
| 유형 | 조회 (단건 SELECT — DML 변경 없음) |
| 구현 경로 | `win/src/trn/W1100/W1109S01.pc` (329 줄) |
| I/O 헤더 | `win/src/inc/trn/W1109S01.h` |
| 모듈 | W1100 |
| 채널 | 매매 엔진 AP (broker `mymqd` 경유, COMHDR 기반) |
| 작업구분 | `prGb` = `'1'` 만 유효 (조회) → `W1109S01_F1` |

로그인 시 내부직원(EMP) 사용자의 마스터 정보와 관리부점명을 조회하고, 동시에 해당 직원의 **근태(휴가) 등록 여부**를 검사해 근태가 등록돼 있으면 로그인 불가 오류를 반환하는 로그인 보조 조회 서비스.

## 3. 구현 흐름 (`.pc`)

```
W1109S01()  (콜백 진입)
  ├─ prthdr / getmid / getmod            # COMHDR·입력·출력 주소 확보
  ├─ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp="M"
  ├─ switch(mid->prGb[0])
  │     case '1' → W1109S01_F1()          # 조회
  │     default  → comhdr_o->mesg="작업구분 오류." + return (커밋/롤백 없이 종료)
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()
        (조회 전용이라 실 변경은 없으나 관례적으로 commit/rollback 호출)

W1109S01_F1()  (조회)
  ├─ getmid/getmod, content->sndl += W1109S01_O_SZ, mod 0x00 초기화
  ├─ switch(content->keyc)                # vKeyv(NEXT/PREV/최초) 계산 — 그러나 이후 미사용 (§6)
  │     K_NEXT→content_get_nkey / K_PREV→content_get_pkey / default→"0"
  ├─ vLgnId = rtrim(yprintf(mid->lgnId))
  ├─ 유효성: isNullSpace(vLgnId)==0 → e_msg.code="43065", eflg="1", return RET_FAILURE
  ├─ [SQL#1] SELECT ... INTO 사용자 마스터 (CSC004M + CMC001F(+) + CMC003D filter)
  │     SQLCODE==NO_DATA_FOUND → e_msg.code="10585"(자료없음), eflg="1", return FAILURE
  │     기타 SQL 오류        → e_msg.code="10118", eflg="1", return FAILURE
  ├─ [SQL#2] SELECT 'Y' INTO cAttdYn FROM CMG061M (직원근태) — 현재시각이 근태구간 내면 'Y'
  │     SQL 오류(≠NO_DATA) → e_msg.code="10118", eflg="1", return FAILURE
  │     cAttdYn=='Y'        → e_msg.code="10152"(근태등록→로그인불가), eflg="1", return FAILURE
  ├─ 각 컬럼 l_rtrim → mod 필드에 zsedit 로 세팅
  └─ 정상: eflg="0", e_msg.code="91003"(조회완료), return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-----|-----|-----|-----|-----|
| `TB_FXB_CSC004M` | 사용자정보기본 | CSC004M | SELECT | 조회 주 대상 — FX사용자번호/명/구분/레벨/관리부점코드. `USER_STTS_DCD='1'`, `LGN_ID=:vLgnId` |
| `TB_FXB_CMC001F` | 부점정보인터페이스 | CMC001F | SELECT | 관리부점명(`MNGM_BRM`) — `MNGM_BRCD(+)` outer join |
| `TB_FXB_CMC003D` | 코드정보상세 | CMC003D | SELECT | 코드 필터 — `COM_CD_NM='FX_USER_DCD'` + `CMCD_VL_REF_CON1='EMP'`(직원만) + `CMCD_VL=CSC004M.FX_USER_DCD` |
| `TB_FXB_CMG061M` | 직원근태 | (별칭없음) | SELECT | 근태(휴가) 등록 여부 — `ENO=:vLgnId` 이고 현재시각이 `ATTD_ST~ATTD_ED` 구간 내면 'Y' |

*DML 요약: 순수 조회 서비스. INSERT/UPDATE/DELETE 및 커서(DECLARE/OPEN/FETCH) 없음. 두 개의 단건 `SELECT ... INTO` 만 존재. 종료 시 `l_dbcommit()`/`l_dbrollback()` 는 관례적 호출 (실 변경 없음).*

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `prGb='1'` 만 조회 수행. 그 외는 `comhdr_o->mesg="작업구분 오류."` 로 즉시 반환하며 commit/rollback 도 타지 않는다.
- **입력 검증**: `lgnId` 가 공백/널이면 `43065` 오류.
- **SQL#1 (마스터 조회)**: `CSC004M` 를 로그인ID·활성상태(`USER_STTS_DCD='1'`)로 조회. `CMC003D` 는 join 컬럼 없이 WHERE 절 코드 조건(`FX_USER_DCD` 코드의 참조값 `EMP`)으로 **직원 여부를 필터** — 즉 직원이 아닌 사용자는 `CMC003D` 조건 불충족으로 NO_DATA_FOUND(→`10585`) 처리. `CMC001F` 는 outer join 이라 부점정보 없어도 조회는 성공.
- **SQL#2 (근태 검사)**: 로그인 직원의 현재 진행중 근태(휴가)가 있으면 `cAttdYn='Y'` → `10152` 오류로 **로그인 차단**. 데이터 없으면 기본값 'N' 유지하고 통과. 날짜/시각 비교는 `ATTD_ST_DT||ATTD_ST_TM` 문자열 concat 을 `SYSDATE` 문자열과 비교.
- **트랜잭션**: `W1109S01()` 레벨에서 `RET_SUCCESS` 면 commit, 아니면 rollback.
- **에러/메시지 코드 (e_msg.code → comhdr_o)**:
  | code | 의미 | eflg |
  |-----|-----|-----|
  | `43065` | 입력값[로그인ID] 확인 | 1 |
  | `10585` | 해당 자료 없음 (직원 아님/미존재) | 1 |
  | `10118` | 자료 조회중 오류 | 1 |
  | `10152` | 근태(휴가) 등록 → 로그인 불가 | 1 |
  | `91003` | 조회 완료 (정상) | 0 |
