# 서비스 명세 — `W1302S01` (통화Pair 조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1302S01` |
| 이름 | 통화Pair 조회 |
| 유형 | 조회 (SELECT 전용, DML 없음) |
| 구현 | `win/src/trn/W1300/W1302S01.pc` (853 라인) |
| I/O 헤더 | `win/src/inc/trn/W1302S01.h` |
| 라우팅 alias | `W1302S01` |
| 채널 | 매매 (trn) — grid 응답 (페이징 지원) |

통화페어 마스터(TB_FXB_CMG004M)를 조회한다. 작업구분(`prGb`)에 따라 두 경로로 분기:
- **F1** (통화페어 등록화면용): 전체 통화페어 + 원화베이스 필터 (prGb 1/4/5)
- **F2** (주문화면용): 외환중개사(FDM)별 등록 상품 통화페어만 (prGb 2/3)

## 2. I/O 전문 (`.h`)

**Input `W1302S01_I`**

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 작업구분. **실 코드 기준** 1/4/5→F1, 2/3→F2. `4`=원화베이스(COCU=KRW), `5`=원화베이스+선물환(FWD) 거래가능. 그 외=오류 |
| `crncPairId` | 7 | 통화페어ID (빈값이면 전체) |
| `useYn` | 1 | 사용여부 (빈값이면 전체) |
| `fdmId` | 10 | 외환중개사ID (**F2에서만 사용**, prGb=2에서 유효. F1은 무시) |

## 3. 구현 흐름 (`.pc`)

```
W1302S01(entry)
 ├─ prthdr / getmid / getmod  (comhdr·mid·mod 주소 확보)
 ├─ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
 ├─ switch(mid->prGb[0])
 │    ├─ '1' '4' '5' → W1302S01_F1()   // 전체/원화베이스/원화베이스+선물환
 │    ├─ '2' '3'     → W1302S01_F2()   // 주문 — FDM 상품 통화페어
 │    └─ default     → comhdr_o mesg "작업구분 오류.", eflg=1, return (분기 즉시 종료)
 ├─ retc==RET_SUCCESS → l_dbcommit(), eflg=0
 │           else      → l_dbrollback(), eflg=1
 ├─ fnSetMsg(&e_msg, comhdr_o)
 └─ return content->sndl

W1302S01_F1 / _F2 (공통 골격)
 ├─ getmid/getmod, content->sndl += W1302S01_O_SZ, memset mod
 ├─ max_row = MAX_RRB / sizeof(orec[0])
 ├─ content->keyc 로 페이징 키 획득 (K_NEXT=nkey, K_PREV=pkey, 최초="0")
 ├─ mid 입력값 → 호스트변수 (yprintf + l_rtrim)
 ├─ EXEC SQL DECLARE C1(F1) / C2(F2) CURSOR FOR ... ORDER BY RPSN_SQC
 ├─ EXEC SQL OPEN → SQLCODE 검사 (NO_DATA→91001/eflg0, else→10118/eflg1)
 ├─ for(ii=0;;ii++) FETCH INTO 호스트변수
 │    ├─ NO_DATA_FOUND → break
 │    ├─ 기타 오류 → CLOSE, 10118/eflg1, return RET_FAILURE
 │    ├─ ii==max_row → i_isnext++, break   (다음페이지 존재 표시)
 │    ├─ l_rtrim + zsedit/sprintf → mod->orec[ii]
 │    └─ vKeyv = cKeyv (NEXT_KEY = RPSN_SQC)
 ├─ EXEC SQL CLOSE (§6 참조 — F2는 C1 을 닫는 버그)
 ├─ 결과 판정
 │    ├─ ii==0 && i_isnext==0 → 91001 (자료없음)
 │    ├─ i_isnext==1 → content_set_nkey, 91002 (계속됨)
 │    └─ else       → 91003 (완료), grid01_cnt=ii
 ├─ eflg="0", grid01_cnt=ii (무조건 재세팅)
 └─ content->sndl += (ii-1)*sizeof(orec[0]); return RET_SUCCESS
```

트랜잭션: 조회 전용이나 관례상 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` (AUTOCOMMIT 아님).

## 5. 로직 / 필터 규칙

### F1 (커서 `W1302S01_C1`, prGb 1/4/5)
- 필터: `CRNC_PAIR_ID = NVL(:vCrncPairId, ...)`, `USE_YN = NVL(:vUseYn, ...)`, `RPSN_SQC > :vKeyv` (페이징)
- `prGb='4'`: `COCU_CD='KRW'` 강제 (원화베이스) + `CRNC_PAIR_ID NOT LIKE '%CNH%'`
- `prGb='5'`: `NOT LIKE '%CNH%'` + `CRNC_PAIR_ID = (SELECT ... CMG002M WHERE FDM_ID='FRX' AND FX_PDCD='FWD' ...)` (선물환 거래가능 통화페어만)
  - 주석 잔재: prGb='5' 의 COCU_CD='KRW' 조건은 "전체 조회로 변경요청 (김호현 주임)" 으로 주석처리됨 → prGb=5는 KRW 강제 없음
- `prGb='1'`: 추가 필터 없음 (전체)

### F2 (커서 `W1302S01_C2`, prGb 2/3)
- `prGb<>'3'` (즉 2): `CRNC_PAIR_ID <> 'USD/CNY'` 제외
- `prGb='3'`: `CRNC_PAIR_ID <> 'CNH/KRW'` 제외 (동일 조건이 두 번 걸림 — 633/636·652 중복)
- FDM 상품 IN 필터 (`CMG002M`):
  - `prGb='2'`: `vFdmId` 있으면 해당 FDM, 없으면 `'KMB SMB EBS'` LIKE 매칭
  - `prGb='3'`: `'FRX'` 고정
  - + `CMG002M.USE_YN = NVL(:vUseYn, ...)`
- 공통: `CRNC_PAIR_ID = NVL(:vCrncPairId, ...)`, `USE_YN = NVL(:vUseYn, ...)`, `RPSN_SQC > :vKeyv`

### 페이징
- `max_row = MAX_RRB / sizeof(orec[0])` 만큼 채우고 1건 더 FETCH 되면 `i_isnext=1` → `91002`(계속) + NEXT_KEY(=`RPSN_SQC`) 세팅.
- 정렬 `ORDER BY RPSN_SQC`, NEXT_KEY 도 `RPSN_SQC`.

### 에러 / 메시지 (e_msg.code → fnSetMsg)
| code | 의미 | eflg |
|-----|-----|-----|
| `91001` | 해당 자료 없음 | 0 |
| `91002` | 조회 계속됨 (다음페이지 존재) | 0 |
| `91003` | 조회 완료 | 0 |
| `10118` | 커서 OPEN/FETCH 오류 | 1 |
| (comhdr 직접) | "작업구분 오류." (prGb 미해당) | 1 |

## 6. 주의 — 문서/코드 불일치

| 위치 | 헤더/주석 표기 | 실제(.pc) |
