## 3. 구현 흐름 (`.pc`)

```
W1401S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1401S01_F1(...)     ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return (MSG_NOWORK)
  └─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
        (명시적 커밋/롤백, AUTOCOMMIT 아님) → fnSetMsg(&e_msg, comhdr_o)

W1401S01_F1()                   ← 조회 본체
  ├─ getmid/getmod, sndl += W1401S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ 페이징 키: switch(content->keyc)
  │     K_NEXT → content_get_nkey → vKeyv
  │     K_PREV → content_get_pkey → vKeyv  (SQL 단방향이라 역방향 미동작, §6)
  │     default(최초) → vKeyv="0"
  ├─ 입력 바인딩: XPRINTF + l_rtrim → vWbcsRlnmAltrNo/vMngmBrcd/vCusAbbrNm/
  │              vCifNo/vOpnFrdt/vOpnTodt/vTktFrdt/vTktTodt  (실제 유효성 검증 없음, §6)
  ├─ DECLARE W1401S01_C1 CURSOR  (아래 §4 조인 구조, KEY = wbcs||abbr||mngm)
  ├─ OPEN → SQLCODE≠0:  NO_DATA_FOUND → e_msg 91001 + RET_SUCCESS
  │                     그외         → e_msg 10118 + RET_FAILURE
  ├─ for(ii=0; ;ii++)
  │     memset(c*) → FETCH INTO c* (마지막 컬럼 cKeyv = NEXT KEY)
  │       SQLCODE≠0: NO_DATA_FOUND → break / 그외 → CLOSE + 10118 + RET_FAILURE
  │       ii==max_row → i_isnext++ ; break       (max_row 초과분 미저장, 다음키 존재 표시)
  │       zsedit(mod->orec[ii].*, c*)            (각 컬럼 복사)
  │       원화/외화계좌: 공백 아니면 fnAcntConv 로 변환 후 저장
  │                     (한도계좌 fnAcntConv 블록은 주석처리)
  │       vKeyv = cKeyv                          (다음 페이지 키 갱신)
  ├─ CLOSE W1401S01_C1
  ├─ ii==0 && i_isnext==0 → e_msg 91001 (자료없음)
  │  else i_isnext==1     → content_set_nkey(vKeyv) + e_msg 91002 (계속)
  │       else            → e_msg 91003 (완료) + grid01_cnt=ii
  ├─ grid01_cnt = ii (재설정)
  └─ sndl += (ii-1)*sizeof(orec[0]) ; return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='1'` 조회만. 그 외 즉시 "작업구분 오류" 반환.
- **동적 필터** (T1 인라인뷰 WHERE):
  - `MNGM_BRCD = NVL(vMngmBrcd, MNGM_BRCD)` — 부점 미입력 시 전체.
  - 고객약어명 / 전행고객번호: `cusAbbrNm` 또는 `wbcsRlnmAltrNo` 부분일치(`LIKE '%..%'`). 세 번째 "둘 다 NULL" 브랜치는 Oracle NULL 결합으로 항상 거짓 → **둘 중 하나는 사실상 필수**(§6).
  - `CIF_NO LIKE '%vCifNo%'` (NVL/TRIM 처리).
  - `CUS_JNNG_YMD BETWEEN NVL(vOpnFrdt, CUS_JNNG_YMD) AND NVL(vOpnTodt, SYSDATE)` — 가입일자 범위.
  - 원/외화계좌 outer join 조건: `FX_STLM_ACN='1'/'2'`, `FX_SLACT_YN='Y'`, `STLM_ACSS_DCD='1'`.
  - 코드명 6종 outer join: `CMC003D.CMCD_VL=코드값 AND COM_CD_NM='<코드그룹>'`.
- **최초거래일자(T2/T3)**: `FX_PDCD IN ('FWD','SPT')`, `CNTT_YMD < NVL(vTktTodt, SYSDATE)`. 단
  `1 = CASE WHEN NVL(vOpnFrdt,'20111201')='20111201' THEN 0 ELSE 1 END` 스위치가 걸려 있어
  **vOpnFrdt 가 미입력이거나 '20111201' 이면 T2/T3 는 0건 → 선물환/현물환 최초거래일자 공백**(§6).
- **페이징**: `ORDER BY WBCS_RLNM_ALTR_NO, CUS_ABBR_NM, MNGM_BRCD`, 다음키 `KEY = wbcs||abbr||mngm > :vKeyv`.
  `ii==max_row` 도달 시 `i_isnext` 세우고 중단, 마지막 저장행 키를 `content_set_nkey` 로 반환.
- **계좌 변환**: 원화/외화계좌가 공백(5칸) 아니면 `fnAcntConv` 로 표시포맷 변환 후 저장. 한도계좌는 SELECT '' 고정 + 변환/세팅 주석처리 → 항상 빈값.
- **에러/상태 코드** (`e_msg.code` → `fnSetMsg`):
  - `91001` 자료없음 / `91002` 조회계속(NEXT KEY) / `91003` 조회완료
  - `10118` 커서 OPEN/FETCH SQL 오류 (RET_FAILURE → 롤백, eflg="1")
