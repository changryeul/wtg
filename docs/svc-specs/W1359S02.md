## 3. 구현 흐름 (`.pc`)

```
W1359S02()  [진입 dispatch]
  ├─ prthdr / getmid / getmod          # COMHDR·입력·출력 주소 획득
  ├─ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  ├─ switch(mid->prGb[0])
  │     case '1' → W1359S02_F1()       # 조회
  │     default  → comhdr_o.mesg="작업구분 오류." + return (커밋/롤백 없이 즉시)
  ├─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
  ├─ fnSetMsg(&e_msg, comhdr_o)         # e_msg.code → mesg/rcod/pflg 세팅
  └─ return content->sndl

W1359S02_F1()  [조회 서브함수]
  ├─ getmid/getmod, content->sndl += W1359S02_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ NEXT KEY 결정: keyc = K_NEXT→nkey / K_PREV→pkey / else→"0"
  ├─ 입력 정리: yprintf+l_rtrim(vWbcsRlnmAltrNo)
  ├─ DECLARE W1359S02_C1 CURSOR (CSC004M ⋈ CSC005R)
  ├─ OPEN → SQLCODE≠0 시 e_msg=10118 + return RET_FAILURE
  ├─ for(ii): FETCH
  │     NO_DATA_FOUND → break
  │     기타 오류 → CLOSE + e_msg=10118 + return RET_FAILURE
  │     ii==max_row → i_isnext++ , break   # 다음 페이지 존재 감지 (미저장)
  │     orec[ii] 채움 + vKeyv = cKeyv       # NEXT KEY 갱신
  ├─ CLOSE W1359S02_C1
  ├─ ii==0 && i_isnext==0 → e_msg=91001 (자료 없음)
  │  else i_isnext==1 → content_set_nkey + e_msg=91002 (계속)
  │       else        → e_msg=91003 (완료)
  ├─ zredit(mod->grid01_cnt, ii)
  ├─ content->sndl += (ii-1)*sizeof(orec[0])
  └─ return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **조인/필터**: `CSC005R.FX_USER_NO = CSC004M.FX_USER_NO` 내부조인 + `CSC004M.USER_STTS_DCD IN ('1')` (사용 상태만) + `CSC005R.WBCS_RLNM_ALTR_NO = :vWbcsRlnmAltrNo` (입력 외환중개사ID).
- **페이징 커서**: `WHERE ... AND CSC005R.FX_SYS_LSMD_TS||CSC005R.FX_USER_NO > :vKeyv ORDER BY FX_SYS_LSMD_TS, FX_USER_NO`. 최초조회는 `vKeyv="0"`, 이후 마지막 fetch 행의 `TS||USER_NO` 를 NEXT_KEY 로 세팅. 한 페이지 최대 `max_row` 건, `max_row+1` 번째 fetch 성공 시 `i_isnext=1` 로 "다음 있음" 판정하고 그 행은 저장하지 않음.
- **트랜잭션**: AUTOCOMMIT 아님 — 진입함수에서 서브함수 결과에 따라 명시적 commit/rollback.
- **에러/메시지 코드** (`e_msg.code` → `fnSetMsg`):
  - `10118` — 자료 조회중 오류 (커서 OPEN/FETCH 실패, `eflg=1` 롤백)
  - `91001` — 해당하는 자료가 없습니다 (0건)
  - `91002` — 조회가 계속됩니다 (다음 페이지 존재, NEXT KEY 세팅)
  - `91003` — 조회가 완료되었습니다 (마지막 페이지)
  - default 분기 — "작업구분 오류." (`prGb≠'1'`)
