# 서비스 명세 — `W1411A01` (1회 최대거래 가능금액 등록)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며,
> `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|---|---|
| 서비스 코드 | `W1411A01` |
| 이름 | 1회 최대거래 가능금액 등록 (`.pc` Description = "1회 최대거래 가능금액 등록") |
| 유형 | **복합 등록/갱신 (UPSERT)** — grid 행별 UPDATE, 미존재 시 INSERT (SELECT 로 당사 기준 조회 선행) |
| 구현 | `win/src/trn/W1400/W1411A01.pc` (320 lines) |

## 2. I/O 전문 (`.h`)

**Input `W1411A01_I`** — 고정부 + 가변 grid(`orec`)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'2'`(등록수정) 만 처리, 그 외 "작업구분 오류" 후 return |
| `wbcsRlnmAltrNo` | 16 | 전행고객실명대체번호 (고객번호) — 필수 |
| `fxUserNo` | 30 | FX사용자번호 (이용자번호) — 필수 |
| `grid01_cnt` | 6 | 그리드 건수 — `l_n2i` 로 int 변환, 0 이면 "변경할 내역 없음" |
| `orec[].fxPdcd` | 3 | FX상품코드 — grid 행, 필수 |
| `orec[].crcd` | 3 | 통화코드 (한도설정통화) — grid 행, 필수 |
| `orec[].cusStupMxmmTrnAmt` | 22 | 고객설정1회최대거래가능금액 — `l_n2d` 로 double 변환 |

> `.h` 는 `orec[1]` 로 선언되어 있으나 `.pc` 는 `grid01_cnt` 만큼(`vCnt`) `orec[ii]` 를 순회하는
> 가변 grid. WTG 전문 조립 시 실제 grid 건수 = `grid01_cnt`.

**Output `W1411A01_O`** (6B)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 처리 결과 flag — 성공 시 `zredit(...,1)` 로 `1` 고정. 실제 처리 건수가 아님(§6). 상세 결과는 COMHDR(`eflg`/`mesg`) 로 전달 |

> 출력 grid(orec) 없음. 사실상 결과 데이터는 공통헤더로 나간다.

## 3. 구현 흐름 (`.pc`)

```
W1411A01()                      ← 진입 (콜백, 함수 주석은 "조회"지만 실제 dispatch)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '2' → retc = W1411A01_F2(...)     ← 실제 로직 (등록수정)
  │     그외 → comhdr_o->mesg="작업구분 오류." 후 즉시 return (commit/rollback 없음)
  └─ retc==RET_SUCCESS ? eflg"0"+l_dbcommit() : eflg"1"+l_dbrollback()  (명시적 커밋)
     → fnSetMsg(&e_msg, comhdr_o) 로 e_msg.code → mesg 반영

W1411A01_F2()                   ← 등록수정 (UPSERT)
  ├─ getmid/getmod, memset(mod)
  ├─ 입력 바인딩: yprintfrtrim/l_rtrim → vWbcsRlnmAltrNo, vFxUserNo, vCnt, hls_upd_id(=usid)
  ├─ 고정부 유효성:
  │     len(vWbcsRlnmAltrNo)<=0 → e_msg "32017", RET_FAILURE
  │     len(vFxUserNo)<=0       → e_msg "10774", RET_FAILURE
  │     vCnt==0                 → e_msg "42080"(변경 내역 없음), RET_FAILURE
  ├─ for ii in 0..vCnt-1        ← grid 행별 처리
  │     ├─ 행 바인딩: vFxPdcd, vCrcd, vCusStupMxmmTrnAmt
  │     ├─ 행 유효성: len(vFxPdcd)<=0 → "10202" / len(vCrcd)<=0 → "10190"
  │     ├─ SELECT FX_MXMM_TRN_AMT, FX_MNMM_TRN_AMT
  │     │   FROM TB_FXB_CMG008M WHERE FX_PDCD/CRCD/FX_CUS_LMT_DCD='1'(1회한도)
  │     │     → NO_DATA 포함 SQLCODE!=SUCCESS 시 e_msg "10122", RET_FAILURE
  │     │       (당사 기준 없으면 고객 한도 등록 불가)
  │     ├─ UPDATE TB_FXB_TRC001M SET 고객설정최대/RND1_MXMM/RND1_MNMM/LSMD_ID/TS
  │     │   WHERE WBCS_RLNM_ALTR_NO/FX_USER_NO/FX_PDCD/CRCD
  │     │     → SQLCODE!=SUCCESS && !=NO_DATA_FOUND → e_msg "10122", RET_FAILURE
  │     └─ if NO_DATA_FOUND → INSERT TB_FXB_TRC001M (USE_YN='Y', COMNT_CON='')
  │            → SQLCODE!=0 → e_msg "10204", RET_FAILURE
  └─ 정상 → mod->grid01_cnt=1, e_msg "10150", RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG008M` | 당사 상품·통화별 거래한도 기준 (한글명 미상) | — | **SELECT** | `(FX_PDCD, CRCD, FX_CUS_LMT_DCD='1')` 로 당사 1회 최대(`FX_MXMM_TRN_AMT`)/최소(`FX_MNMM_TRN_AMT`) 거래금액 조회. 미존재 시 오류(고객 한도 등록 차단) |
| `TB_FXB_TRC001M` | 고객별거래한도 | — | **UPDATE / INSERT** | `(WBCS_RLNM_ALTR_NO, FX_USER_NO, FX_PDCD, CRCD)` 키로 고객설정최대금액 + 당사 1회 최대/최소 한도 갱신, 없으면 신규 등록 |

*DML 요약: `TB_FXB_CMG008M` SELECT 1문(행별) + `TB_FXB_TRC001M` UPDATE→(미매칭 시) INSERT. grid 건수만큼 반복. DELETE 없음.*

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='2'`(등록수정) 단일 분기만 구현. 그 외는 처리 없이 "작업구분 오류" 로 즉시 응답(commit/rollback 미수행).
- **당사 기준 선행 조회**: 각 grid 행마다 `TB_FXB_CMG008M` 에서 당사 1회 한도(최대/최소)를 먼저 조회한다. 당사 기준이 없으면(NO_DATA 포함) 해당 행에서 실패 처리 — "당사 기준이 없는데 고객 한도를 등록할 수 없다".
- **UPSERT**: `TB_FXB_TRC001M` UPDATE 후 `SQLCODE==NO_DATA_FOUND` 이면 INSERT. UPDATE 는 고객설정금액(`CUS_STUP_MXMM_TRN_AMT`) + 당사 조회값(`RND1_MXMM_TRAB_AMT`/`RND1_MNMM_TRAB_AMT`) + 변경 ID(`FX_SYS_LSMD_ID`=COMHDR usid) + `SYSDATE` 스탬프. INSERT 는 추가로 `USE_YN='Y'`, `COMNT_CON=''`.
- **트랜잭션**: 모든 grid 행 처리 후 dispatch 에서 일괄 `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. 행 도중 실패 시 즉시 return → 전체 rollback (부분 커밋 없음). AUTOCOMMIT 아님.
- **에러(e_msg.code → mesg, eflg)**:
  - 성공: `10150` (조회/처리 완료 계열), `eflg="0"`.
  - 입력: `32017`(고객번호) / `10774`(이용자번호) / `42080`(변경 내역 없음) / `10202`(상품구분) / `10190`(한도설정통화).
  - 처리: `10122`(당사 한도 조회·UPDATE 오류) / `10204`(고객별거래한도 등록 오류), `eflg="1"`.
  - → WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
