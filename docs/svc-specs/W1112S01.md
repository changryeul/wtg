## 3. 구현 흐름 (`.pc`)
```
W1112S01()                                        // 콜백 진입
 ├ prthdr/getmid/getmod                           // 공통헤드·입력·출력 주소 획득
 ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp="M"
 ├ switch(prGb[0])
 │    case '1' → W1112S01_F1()                     // 조회
 │    default  → comhdr_o->mesg="작업구분 오류." → return (분기 즉시 반환, commit/rollback 없음)
 ├ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()
 └ return content->sndl

W1112S01_F1()                                      // 조회
 ├ content->sndl += W1112S01_O_SZ; memset(mod)
 ├ max_row = MAX_RRB / sizeof(orec[0])             // 페이지당 최대 행
 ├ keyc 분기: K_NEXT→content_get_nkey / K_PREV→content_get_pkey / 그외→"0"  → vKeyv
 ├ yprintf(vUseYn, mid->useYn); l_rtrim
 ├ DECLARE TB_FXB_COD_FXPAIR_C1 CURSOR (CMG004M ⋈ CMG006M)
 ├ OPEN → NO_DATA_FOUND 이면 e_msg 91001 + return RET_FAILURE
 └ for(ii=0; ;ii++)
       FETCH INTO 14개 컬럼 + cKeyv(RPSN_SQC)
        ├ NO_DATA_FOUND → break
        ├ SQL error → CLOSE + e_msg 10118 + return RET_FAILURE
        ├ ii==max_row → i_isnext++; break          // 다음 페이지 존재
        ├ l_rtrim 후 zsedit 로 mod->orec[ii] 채움
        └ vKeyv = cKeyv                             // 다음 KEY 후보
    CLOSE cursor
    ├ ii==0 && i_isnext==0 → e_msg 91001 (자료없음)
    ├ i_isnext==1 → content_set_nkey + e_msg 91002 (계속됨)
    └ else        → e_msg 91003 (완료) + grid01_cnt=ii
    ├ comhdr_o->eflg="0"; grid01_cnt=ii (무조건 재설정)
    └ content->sndl += (ii-1) * sizeof(orec[0]); return RET_SUCCESS
```

## 4. 사용 DB 테이블
| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-----|-----|-----|-----|-----|
| `TB_FXB_CMG004M` | 통화페어기본 | `CMG004M` | SELECT | 통화페어 마스터 (통화쌍, 소수점 자릿수/값, 손익페어, 표시순서 등) |
| `TB_FXB_CMG006M` | 통화별상품정보기본 | `CMG006M` | SELECT | `FX_XRT_FNNL_MTHD_DCD` (환율재정방식) 획득 + SPT 테너 필터 |

*DML 요약: SELECT 전용 (단일 커서 조회). INSERT/UPDATE/DELETE 없음. 콜백은 성공 시 `l_dbcommit`, 실패 시 `l_dbrollback` 호출하나 실제 변경 데이터는 없음.*

## 5. 로직 / 필터 규칙
- **조인/필터** (`TB_FXB_COD_FXPAIR_C1` 커서):
  - `CMG004M.USE_YN = NVL(:vUseYn, CMG004M.USE_YN)` — `useYn` 빈값이면 전체
  - `CMG006M.TNR_ID = 'SPT'` — 현물(SPOT) 테너만
  - `CMG004M.CRNC_PAIR_ID = CMG006M.CRNC_PAIR_ID` — 통화페어 조인
  - `CMG006M.CRNC_PAIR_ID NOT LIKE '%CNH%'` — CNH(역외위안) 제외
  - `ORDER BY CMG004M.RPSN_SQC` — 표시순서 정렬
- **소수점 값 편집**: `EMP_CLCL_DCPU_VL` / `CUS_CLCL_DCPU_VL` 이 1 미만이면 SQL 에서 `'0' || TO_CHAR(...)` 로 선행 0 부착.
- **페이징**: `max_row = MAX_RRB / sizeof(orec)`. `ii == max_row` 도달 시 그 행은 저장하지 않고 `i_isnext` 세팅 → 다음 페이지 존재 신호. FETCH 15번째 INTO 대상 `cKeyv` 는 `RPSN_SQC`(표시순서)를 KEY 로 받아 `content_set_nkey` 에 실음.
- **트랜잭션**: SELECT 전용이나 콜백 종료 시 명시적 commit/rollback (AUTOCOMMIT 아님).
- **에러/메시지 코드** (`e_msg.code` → `fnSetMsg`, `comhdr_o->eflg`):
  | code | 의미 | eflg | 흐름 |
  |-----|-----|-----|-----|
  | `10118` | SQL error (DECLARE/OPEN/FETCH 실패) | `1` | RET_FAILURE → rollback |
  | `91001` | 해당 자료 없음 | `0` | OPEN NO_DATA 또는 ii==0 |
  | `91002` | 조회 계속됨 (NEXT_KEY 존재) | `0` | i_isnext==1 |
  | `91003` | 조회 완료 | `0` | 정상 종료 |
  - `prGb` 미지정 값은 `comhdr_o->mesg="작업구분 오류."` 직접 세팅 후 반환 (e_msg 코드 미사용).

## 6. 주의 — 문서/코드 불일치
| 위치 | 헤더/주석 표기 | 실제(`.pc`) |
|-----|-----|-----|
| `.pc` L40~41 | `mid`/`mod` 둘 다 주석 "입력값 주소" | `mod` 는 출력값 주소 (getmod). 주석 복붙 오기 |
| `.pc` L156~169, L365, L382 | `keyc`(K_NEXT/K_PREV)로 `vKeyv` 획득·`content_set_nkey` 로 재설정 | **커서 WHERE 절에 `:vKeyv`(KEY) 조건이 전혀 없음.** NEXT/PREV 로 받은 KEY 가 쿼리에 반영되지 않아, 다음 페이지 요청 시에도 항상 처음부터 동일 결과를 재조회한다. 페이징 로직 실질 미동작 (버그) |
| `.pc` L221~227 | `EXEC SQL DECLARE ... CURSOR` 직후 `SQLCODE != 0` 검사 | Pro*C 에서 DECLARE 는 선언(비실행)이라 SQLCODE 가 갱신되지 않음 — 이 검사는 무의미 (실질 OPEN 시점 검사만 유효) |
| `.pc` L404 | `content->sndl += (ii - 1) * sizeof(orec[0])` | `ii==0`(자료없음) 이면 `(0-1)*size` 음수 가산. 앞의 `sndl += W1112S01_O_SZ`(orec 1개 포함)와 상쇄되어 순 orec 0개로 수치상 맞지만, 음수 곱 의존은 취약한 패턴 |
| `.pc` L371~396 vs L401~402 | 91001/91002/91003 분기 내에서 eflg·grid01_cnt 설정 | 분기 종료 후 L401~402 에서 `eflg="0"`, `grid01_cnt=ii` 를 무조건 재설정 → 91002(next) 경로에서도 grid01_cnt 최종값이 `ii`(=max_row)로 덮임 (분기 내 설정 상당수 무의미) |
| `.h` orec 크기 | `orec[1]` 고정 배열 선언 | 실제로는 `max_row` 까지 반복 append 하는 가변 그리드 (표준 grid 패턴) |
| `.pc` 미사용 변수 | — | `editb`, `cnt`, `retc`, `error_f`, `i_sndcnt`, `log_m`, `i`, `i_isnext` 초기값 등 다수 선언·미사용 (기능 영향 없음) |
