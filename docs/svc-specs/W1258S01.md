## 3. 구현 흐름 (`.pc`)

```
W1258S01()                       ← 진입 (콜백)
  ├─ prthdr / getmid / getmod    COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' → retc = W1258S01_F1(...)    ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? l_dbcommit()+eflg"0" : l_dbrollback()+eflg"1"
     → fnSetMsg(&e_msg, comhdr_o) → return content->sndl   (명시적 커밋, AUTOCOMMIT 아님)

W1258S01_F1()                    ← 조회 (커서)
  ├─ getmid/getmod, sndl += W1258S01_O_SZ, memset(mod)
  │  max_row = MAX_RRB / sizeof(orec[0])
  ├─ GRID key 처리 (content->keyc)  K_NEXT→content_get_nkey / K_PREV→content_get_pkey
  │     default(최초)→ vKeyv="0", l_rtrim(vKeyv)      ← 실제 WHERE 에 바인딩됨(페이징 동작)
  ├─ 입력 바인딩: yprintf + l_rtrim → vComCdNm/vCmcdVl/vCmcdVlNm/vUseYn (NULL/공백 거부 없음 §6)
  ├─ DECLARE W1258S01_C1 CURSOR (TB_FXB_CMC003D, 필터 4종 + key > :vKeyv, ORDER BY 코드명,값,값명)
  ├─ OPEN W1258S01_C1
  │     SQLCODE!=SUCCESS → NO_DATA_FOUND: e_msg"91001", RET_FAILURE (OPEN 단계 NO_DATA 는 사실상 dead §6)
  │                        기타 err     : e_msg"10118", RET_FAILURE
  ├─ for(ii=0;;ii++) FETCH → c* 지역변수
  │     NO_DATA_FOUND → break
  │     기타 err      → CLOSE + e_msg"10118" + RET_FAILURE
  │     ii==max_row   → i_isnext++; break   (이번 레코드는 저장 안 함 = 다음페이지 존재 표식)
  │     zsedit(mod->orec[ii].*) 저장 ; vKeyv = cKeyv (NEXT KEY 갱신)
  ├─ CLOSE W1258S01_C1
  ├─ ii==0 && i_isnext==0 → e_msg"91001"(자료없음), RET_FAILURE
  │  else i_isnext==1 → content_set_nkey(vKeyv), e_msg"91002"(계속됨)
  │       else        → e_msg"91003"(완료), grid01_cnt=ii
  └─ eflg"0", grid01_cnt=ii, sndl += (ii-1)*sizeof(orec[0]), RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **커서 필터** (`W1258S01_C1`):
  - `COM_CD_NM = NVL(UPPER(TRIM(:vComCdNm)), COM_CD_NM)` — 공통코드명 완전일치(대문자화), 빈값이면 전체.
  - `(CMCD_VL like '%'||TRIM(:vCmcdVl)||'%' OR CMCD_VL_NM like '%'||TRIM(:vCmcdVlNm)||'%')` — 값/값명 부분일치를 **OR** 로 묶음.
  - `USE_YN = NVL(:vUseYn, USE_YN)` — 사용여부 일치, 빈값이면 전체.
  - `TRIM(COM_CD_NM)||TRIM(CMCD_VL)||TRIM(CMCD_VL_NM) > :vKeyv` — 연결키 기준 페이징 커서.
  - `ORDER BY COM_CD_NM, CMCD_VL, CMCD_VL_NM`.
- **페이징**: `content->keyc` (K_NEXT/K_PREV/최초) 로 `vKeyv` 를 만들고 실제 WHERE `> :vKeyv` 에 바인딩. FETCH 루프가 `max_row` 도달 시 `i_isnext=1` 로 다음페이지 존재를 알리고 `content_set_nkey(vKeyv)` 로 NEXT KEY 를 실어 보냄 → **페이징 정상 동작** (W1119S01 의 dead vKeyv 와 다름). NEXT KEY 는 매 레코드 `cKeyv`(연결키) 로 갱신.
- **입력 유효성**: 3절 주석은 "입력항목 유효성 check" 이나 실제로는 `yprintf`+`l_rtrim` 만 수행, NULL/공백 거부 로직 없음(§6).
- **트랜잭션**: `RET_SUCCESS→l_dbcommit(eflg"0") / 실패→l_dbrollback(eflg"1")`. AUTOCOMMIT 아님. 실 데이터 변경은 없음(순수 조회).
- **에러/결과 코드** (`e_msg.code` → `fnSetMsg` → COMHDR `mesg`):
  - 성공(완료): `91003` "조회가 완료되었습니다.", `eflg=0`.
  - 성공(계속): `91002` "조회가 계속됩니다." — `max_row` 초과, NEXT KEY 세팅, `eflg=0` (상위 commit).
  - 자료없음: `91001` "해당 자료가 없습니다.", `RET_FAILURE` → 상위 rollback + `eflg=1`.
  - 커서 OPEN/FETCH 오류: `10118`, `RET_FAILURE`.
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (eflg 미세팅).
- **결과 전달**: grid(`grid01_cnt` + `orec[]`) + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).

## 6. 주의 — 문서/코드 불일치 (마이그레이션 시 확인)

| 위치 | 헤더/주석 표기 | 실제(`.pc`) |
|---|---|---|
| `.pc` L48 변수 주석 | `mod` = "W1258S01 의 입력값 주소" | 출력값(Output) 주소 — 복붙 오기 |
| `.pc` L68 섹션 주석 | "고객목록조회" | 실제는 공통코드 조회 — 타 서비스 복붙 잔재 |

| grid01_cnt 세팅 | 1회 | L324(완료 분기) + L332(공통) 로 중복 세팅 — 무해 |
| 미사용 지역변수 | — | `i,cnt,ret,editb,retc,error_f,i_sndcnt,log_m,max_row 외` 등 다수 선언되나 일부 미사용 (`max_row/i_isnext/vKeyv/c*` 는 사용) |

> 동작에 영향 없다. 단 클라이언트 전달용 설명은 `.h` 주석을 그대로 노출하므로,
> 정확한 의미가 필요하면 실제 의미로 교정하는 편이 좋다.
