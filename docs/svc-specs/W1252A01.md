# 서비스 명세 — `W1252A01` (코드정보상세 등록/수정/삭제)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며,
> `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|---|---|
| 서비스 코드 | `W1252A01` |
| 이름 | 코드정보상세 등록/수정/삭제 (`.pc` Description = "코드정보상세 등록/수정/삭제 변경") |
| 유형 | **복합 (INSERT / UPDATE / DELETE)** — `prGb` 작업구분에 따라 등록·수정·삭제 분기 |
| 구현 | `win/src/trn/W1250/W1252A01.pc` (518 lines) |

공통코드 상세값 마스터(`TB_FXB_CMC003D` — 공통코드값)를 관리하는 단건 트랜잭션 서비스.
운영 관리자 화면에서 공통그룹코드(`COM_CD_NM`) 하위의 코드값(`CMCD_VL`) 을 신규 등록하거나,
설명/참조내용/사용여부를 수정하거나, 물리 삭제한다.

## 3. 구현 흐름 (`.pc`)

```
W1252A01()                          ← 진입 (콜백)
  ├─ prthdr / getmid / getmod       COMHDR + input(mid) + output(mod) 준비
  ├─ e_msg 기본설정                  ltyp=KR, mtyp=S, rtyp=M
  └─ switch(mid->prGb[0])
       ├─ '1' → W1252A01_F1()  등록  INSERT TB_FXB_CMC003D
       ├─ '2' → W1252A01_F2()  수정  UPDATE TB_FXB_CMC003D
       ├─ '3' → W1252A01_F3()  삭제  DELETE TB_FXB_CMC003D
       └─ default → comhdr_o.mesg="작업구분 오류." ; return (commit/rollback 없이 즉시 반환)
     ↓ (retc 판정)
  ├─ retc==RET_SUCCESS → eflg="0" + l_dbcommit()
  └─ else              → eflg="1" + l_dbrollback()
  └─ fnSetMsg(&e_msg, comhdr_o)      e_msg.code → mesg/rcod 세팅 후 SEND

W1252A01_F1 (등록)
  ├─ yprintf/l_rtrim 입력 12개 host 변수 정리
  ├─ vComCdNm 공백 → e_msg "10311" (공통그룹코드 확인) → RET_FAILURE
  ├─ vCmcdVl  공백 → e_msg "10302" (공통코드 확인)     → RET_FAILURE
  ├─ INSERT TB_FXB_CMC003D (USE_YN='Y' 고정, FX_SYS_LSMD_TS=SYSDATE)
  ├─ SQLCODE==DUP_KEY → e_msg "20002" (중복 데이터) → RET_FAILURE
  └─ e_msg "10150" → RET_SUCCESS

W1252A01_F2 (수정)
  ├─ yprintf/l_rtrim 입력 12개 host 변수 정리
  ├─ vComCdNm 공백 → e_msg "10311" → RET_FAILURE
  ├─ vCmcdVl  공백 → e_msg "10302" → RET_FAILURE
  ├─ UPDATE TB_FXB_CMC003D SET (한글명/영문명/참조1~5/사용여부/설명/변경ID/변경TS)
  │        WHERE COM_CD_NM=:vComCdNm AND CMCD_VL=:vCmcdVl
  ├─ SQLCODE==NO_DATA_FOUND → e_msg "10221" (수정할 자료 없음) → RET_FAILURE
  └─ e_msg "10150" → RET_SUCCESS

W1252A01_F3 (삭제)
  ├─ yprintf/l_rtrim 입력 3개(vComCdNm/vCmcdVl/vFxSysLsmdId) 정리
  ├─ vComCdNm 공백 → e_msg "10301" (공통코드명 확인) → RET_FAILURE  (vCmcdVl 은 미검증, §6)
  ├─ #if 0 UPDATE USE_YN='N' (soft delete) ← 비활성 dead code (§6)
  ├─ DELETE TB_FXB_CMC003D WHERE COM_CD_NM=:vComCdNm AND CMCD_VL=:vCmcdVl  (물리 삭제)
  ├─ SQLCODE==NO_DATA_FOUND → e_msg "10307" (삭제대상 없음) → RET_FAILURE
  ├─ SQLCODE!=0             → e_msg "10122" (자료 처리중 오류) → RET_FAILURE
  └─ e_msg "10150" → RET_SUCCESS
```

트랜잭션 경계는 진입함수 `W1252A01()` 한 곳에서만 관리한다. 서브함수(F1/F2/F3)는 SQL 만 수행하고
`RET_SUCCESS`/`RET_FAILURE` 만 돌려주며, commit/rollback 은 진입함수가 retc 로 일괄 판정한다.
`default` 분기(작업구분 오류)는 commit/rollback 을 타지 않고 즉시 `content->sndl` 을 반환한다.

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC003D` | 공통코드값 (공통코드 상세) | `CMC003D` | INSERT / UPDATE / DELETE | 등록(F1) / 수정(F2) / 삭제(F3) 대상 마스터. 키=(COM_CD_NM, CMCD_VL) |

*DML 요약:*
- **F1 등록** — `INSERT INTO TB_FXB_CMC003D`: 13개 컬럼. `USE_YN` 은 입력값 무시하고 `'Y'` 리터럴,
  `FX_SYS_LSMD_TS`=`SYSDATE`, 키 컬럼 2개는 `TRIM()` 적용. 중복 시 `DUP_KEY`.
- **F2 수정** — `UPDATE ... WHERE COM_CD_NM=TRIM(:vComCdNm) AND CMCD_VL=TRIM(:vCmcdVl)`:
  한글명/영문명은 `TRIM()`, 참조1~5·사용여부·설명·변경ID 는 원값, 변경TS=`SYSDATE`.
- **F3 삭제** — `DELETE ... WHERE COM_CD_NM=TRIM(:vComCdNm) AND CMCD_VL=TRIM(:vCmcdVl)`: 물리 삭제.
  (soft-delete UPDATE 문은 `#if 0` 로 비활성.)

공통함수 경유 간접 테이블 접근 없음 (fnCustInfoQry 등 미사용). SELECT/커서 없음.

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `prGb[0]` = `'1'`/`'2'`/`'3'` 만 유효. 그 외는 "작업구분 오류" 메시지로 즉시 반환
  (DB 접근 없음, `MSG_NOWORK` 로그).
- **입력 검증**:
  - F1/F2: `vComCdNm` 공백 → `10311`, `vCmcdVl` 공백 → `10302`.
  - F3: `vComCdNm` 공백 → `10301`. `vCmcdVl` 은 검증하지 않으나 DELETE WHERE 절엔 포함(§6).
- **트랜잭션**: 진입함수가 서브함수 retc 로 `l_dbcommit()`/`l_dbrollback()` 단일 판정. AUTOCOMMIT 아님.
- **에러/메시지 코드** (`e_msg.code` → `fnSetMsg` 로 COMHDR 메시지 변환):
  | code | 의미 | 발생 지점 |
  |---|---|---|
  | `10150` | 정상 처리 (`MSG_REQUERY`) | F1/F2/F3 성공 |
  | `10311` | 입력값[공통그룹코드] 확인 | F1/F2 `vComCdNm` 공백 |
  | `10302` | 입력값[공통코드] 확인 | F1/F2 `vCmcdVl` 공백 |
  | `10301` | 입력값[공통코드명] 확인 | F3 `vComCdNm` 공백 |
  | `20002` | 중복된 데이터가 존재합니다 | F1 INSERT `DUP_KEY` |
  | `10221` | 수정할 자료가 없습니다 | F2 UPDATE `NO_DATA_FOUND` |
  | `10307` | 삭제대상이 없습니다 | F3 DELETE `NO_DATA_FOUND` |
  | `10122` | 자료 처리중 오류 | F3 DELETE `SQLCODE!=0` |
- **eflg**: 성공 `"0"` / 실패 `"1"` (진입함수에서 세팅).
