# 서비스 명세 — `W1358S01` (고객거래가능시간(원화대가통화) 조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1358S01` |
| 이름 | 고객거래가능시간(원화대가통화) 조회 |
| 유형 | 조회 (SELECT 전용, 단일 커서 grid) |
| 구현 | `win/src/trn/W1350/W1358S01.pc` (426 줄) |
| I/O 헤더 | `win/src/inc/trn/W1358S01.h` |
| 라우팅 alias | `W1358S01` |
| 채널 | 매매 엔진 transaction (`ChannelWeb`, `POST /v1/tx`) |

원화(KRW) 를 대가통화로 하는 통화쌍(base 통화 × FX 상품 × 테너) 조합별 지정가/시장가 거래가능시각을 조회한다. `TB_FXB_CMG004M`(COCU_CD='KRW') 를 기준통화 집합으로, 공통코드(`TB_FXB_CMC003D`) 의 FX_PDCD / TNR_ID 를 교차하여 `TB_FXB_CMG039M`(거래가능시간 마스터) 를 outer join 한다.

## 2. I/O 전문 (`.h`)

## 3. 구현 흐름 (`.pc`)

```
W1358S01()                              # 진입 dispatch
├─ prthdr / getmid / getmod            # COMHDR·입력·출력 주소 획득
├─ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
├─ switch(mid->prGb[0])
│    case '1' → W1358S01_F1()          # 원화대가통화 조회
│    default  → comhdr_o.mesg="작업구분 오류." → return sndl
├─ retc==RET_SUCCESS ? eflg=0 + l_dbcommit()
│                    : eflg=1 + l_dbrollback()
├─ fnSetMsg(&e_msg, comhdr_o)          # e_msg.code → mesg/rcod/pflg 세팅
└─ return content->sndl

W1358S01_F1()                          # 실 조회
├─ getmid / getmod, sndl += W1358S01_O_SZ, memset(mod)
├─ max_row = MAX_RRB / sizeof(orec[0])
├─ switch(content->keyc)              # K_NEXT/K_PREV → vKeyv 채움, default '0'
│    (※ vKeyv 는 쿼리에 미사용 — §6)
├─ baseYmd 유효성: 빈값 → e_msg 10159, eflg=1, return RET_FAILURE
├─ EXEC SQL DECLARE W1358S01_C1 CURSOR (아래 §4 조인)
├─ EXEC SQL OPEN → SQLCODE 체크 (NO_DATA_FOUND→91001, else→10118)
├─ for(ii=0; ;ii++)
│    ├─ FETCH INTO 9개 host var
│    │    NO_DATA_FOUND → break / 기타 오류 → CLOSE + 10118 + RET_FAILURE
│    ├─ ii==max_row → i_isnext++, break         # 다음 페이지 존재 표시
│    ├─ zsedit(mod->orec[ii].*, c*)             # 9개 컬럼 복사 (usdYn 제외)
│    └─ sprintf(vKeyv,"%.64s",cKeyv)            # ※ cKeyv 미초기화 — §6
├─ EXEC SQL CLOSE W1358S01_C1
├─ 결과 판정
│    ii==0 && i_isnext==0 → e_msg 91001 (자료없음)
│    i_isnext==1          → content_set_nkey(vKeyv) + e_msg 91002 (계속)
│    else                 → e_msg 91003 (완료)
├─ zredit(mod->grid01_cnt, ii)
├─ content->sndl += (ii-1)*sizeof(orec[0])      # ※ ii==0 시 언더플로 — §6
└─ return RET_SUCCESS
```

트랜잭션: 조회만 하지만 dispatch 말미에서 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 를 명시적으로 호출 (AUTOCOMMIT 아님).

## 5. 로직 / 필터 규칙

- **커서 조인 구조**: CMG004M(KRW 대가통화) × CMC003D_1(FX_PDCD) × CMC003D_2(TNR_ID) 데카르트 곱을 CMG039M 에 3키 outer join(CRCD, FX_PDCD, PDF_TNR_CLSF_ID). CMG039M 무 매칭이어도 시각은 NVL '000000' 으로 채운다.
- **상품×통화 정합 필터**:
  - `CMC003D_1.CMCD_VL = CASE WHEN BASE_CRCD='USD' THEN 그대로 ELSE DECODE(..,'MAR','',..)` — USD 가 아니면 'MAR'(마진) 상품 제외 (DECODE 결과 '' = NULL → 매칭 실패).
  - `CMC003D_2.CMCD_VL = CASE WHEN FX_PDCD NOT IN ('SPT','SWP') THEN 'ALL' ELSE DECODE(..,'ALL','',..)` — 현물환/스왑만 테너 세분, 나머지는 'ALL' 만.
- **기간 필터**: `:iBaseYmd BETWEEN NVL(CMG039M.APSR_YMD(+),'00000000') AND NVL(CMG039M.FNAP_YMD(+),'99991231')`.
- **정렬**: CMG004M.RPSN_SQC, CMC003D_1.RPSN_SQC(DECODE 순번), CMC003D_2.RPSN_SQC.
- **DECODE 표시명**: 상품코드명(BAR→바로환전, SPT→현물환, FWD→선물환, SWP→SWAP), 테너명(SPT→SPOT).
- **페이징**: `content->keyc`(K_NEXT/K_PREV/최초) 로 vKeyv 세팅, `ii==max_row` 도달 시 i_isnext 설정 후 91002(계속). *단, 커서 WHERE 에 키 필터가 없고 next key 소스(cKeyv)가 미초기화라 실제 페이지 진행은 동작하지 않는다 (§6).*
- **e_msg 코드**: `10159` 조회일자 미입력 / `91001` 자료없음 / `10118` 조회 오류 / `91002` 조회 계속 / `91003` 조회 완료. eflg 는 성공 0 / 실패 1.
