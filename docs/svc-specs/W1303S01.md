## 2. I/O 전문 (`.h`)

**Input `W1303S01_I`** (총 22B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 처리구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" 반환 |
| `prdCd` | 3 | 상품구분코드(FX_PDCD; MGN:FX margin, SPT:현물환, FWD:선물환). **`"ALL"` 이면 빈값 처리 → 전 상품 조회** |
| `pairId` | 7 | 통화 Pair (CRNC_PAIR_ID, 예 USDKRW) |

## 3. 구현 흐름 (`.pc`)

```
W1303S01()                                  // 콜백 진입 (L35)
 ├ prthdr / getmid / getmod                 // COMHDR·입력·출력 주소 획득
 ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M) // comhdr_o->rtyp="M"
 └ switch(mid->prGb[0])
     ├ '1' → W1303S01_F1()                  // 조회
     └ default → comhdr_o->mesg="작업구분 오류." + return (COMMIT/ROLLBACK 없이 즉시 반환)
   retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   // 읽기 전용이나 명시적 트랜잭션 종료

W1303S01_F1()                               // 조회 (L103)
 ├ content->sndl += O_SZ; memset(mod)
 ├ max_row = MAX_RRB / sizeof(orec[0])
 ├ switch(content->keyc): K_NEXT→nkey / K_PREV→pkey / else→"0"   // vKeyv 페이징 시작키
 ├ yprintf+l_rtrim 로 vPrdCd/vPairId/vTenorCd/vUseYn 세팅
 ├ vPrdCd=="ALL" → memset(vPrdCd,0) (전체)
 ├ DECLARE ST_W1303S01_C1 CURSOR FOR (인라인뷰 W1304S01 + CMC003D_1..4 아우터조인)
 ├ OPEN CURSOR → SQLCODE!=0 처리 (NO_DATA_FOUND=91001 / 기타=10118, RET_FAILURE)
 ├ for(ii=0;;ii++) FETCH …
 │    ├ NO_DATA_FOUND → break
 │    ├ 기타 SQLCODE → CLOSE + 10118 + RET_FAILURE
 │    ├ ii==max_row → i_isnext++; break         // 한 건 더 읽어 다음페이지 존재 판정
 │    ├ l_rtrim + zsedit 로 orec[ii] 채움
 │    └ vKeyv = cKeyv (ROW_NUM, 다음키 후보)
 ├ CLOSE ST_W1303S01_C1
 ├ ii==0 && i_isnext==0 → 91001(자료없음)
 │  else i_isnext==1 → content_set_nkey + 91002(계속) / else → 91003(완료) + grid01_cnt
 ├ (무조건) comhdr_o->eflg="0"; grid01_cnt=ii
 └ content->sndl += (ii-1)*sizeof(orec[0]); return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **동적 필터**: `prdCd`/`pairId`/`tenorCd`/`useYn` 모두 `NVL(:v, 컬럼)` (useYn 은 `NVL(RTRIM(:vUseYn),…)`) — 빈값이면 조건 무시. `prdCd=="ALL"` 은 코드에서 빈값으로 치환해 전체 조회.
- **커서 구조**: 인라인뷰(`W1304S01`)에서 `CMG006M × CMC003D(TNR_ID) × CMG004M` 을 조인하고 `ROW_NUMBER() OVER(ORDER BY FX_PDCD DESC, RPSN_SQC, CMCD_VL_REF_CON2)` 로 `ROW_NUM` 부여. 바깥에서 코드명 4종(`CMC003D_1..4`)을 `(+)` 아우터조인.
- **페이징**: `WHERE W1304S01.ROW_NUM > :vKeyv`. 시작키는 keyc(K_NEXT=nkey / K_PREV=pkey / 최초=0)에서 취득. `max_row+1` 건째를 읽으면 `i_isnext=1` 로 다음페이지 존재 표시하고 `content_set_nkey`.
- **트랜잭션**: 조회이나 콜백 종료 시 성공=`l_dbcommit()`, 실패=`l_dbrollback()`. `prGb` 오류 default 분기는 commit/rollback 없이 즉시 반환.
- **에러/메시지 (e_msg.code → fnSetMsg)**:
  - `10118` — 자료 조회중 오류 (`eflg="1"`, RET_FAILURE)
  - `91001` — 해당 자료 없음 (OPEN 시 NO_DATA_FOUND 또는 fetch 0건, `eflg="0"`)
  - `91002` — 조회 계속(다음 페이지 있음, nkey 세팅, `eflg="0"`)
  - `91003` — 조회 완료(`eflg="0"`)
  - 최종적으로 `eflg` 는 (L492) 무조건 `"0"` 로 덮어써짐 → 모든 경로 정상(eflg=0)으로 응답.
