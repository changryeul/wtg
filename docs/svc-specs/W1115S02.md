## 2. I/O 전문 (`.h`)

**Input `W1115S02_I`** (총 9B)
| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 처리구분 — `'1'` 조회만 유효 (그 외 "작업구분 오류") |
| `userId` | 8 | 유저 ID — 조회 대상 사용자(직원) 번호. `CSC004M.FX_USER_NO` 바인딩 |

**Output `W1115S02_O`** (grid, `orec[1]` 가변 — 실제로는 최대 `MAX_RRB/sizeof(orec[0])` 건 반복 append)
| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `grid01_cnt` | 6 | 그리드 건수 (`ii`, 실제 append 된 레코드 수) |
| `orec[].fdmId` | 10 | 외환중개사ID (`CMG002M.FDM_ID`, 활성 쿼리에서는 항상 `'FRX'`) |
| `orec[].fxPdcd` | 3 | FX상품코드 (`CMG002M.FX_PDCD`) |
| `orec[].crncPairId` | 8 | 통화페어ID (`CMG002M.CRNC_PAIR_ID`) |
| `orec[].stdTn` | 1 | STD_TN — SQL 에서 리터럴 `'S'` 고정 |
| `orec[].tnrId` | 3 | TENOR ID (`CMG002M.TNR_ID`) |
| `orec[].expiFnshYmd` | 8 | 만기종료년월일 (`CMG006M.EXPI_FNSH_YMD`) |
| `orec[].prvDt` | 8 | prvDt — SQL 에서 리터럴 `' '` (공백) 고정, 항상 blank |
| `orec[].nxtDt` | 8 | nxtDt — SQL 에서 리터럴 `' '` (공백) 고정, 항상 blank |
| `orec[].trnAblYn` | 1 | 거래가능여부 (`CMG002M.TRN_ABL_YN`, 필터상 항상 `'Y'`) |
| `orec[].symbol` | 60 | Market Data Symbol = `FDM_ID‖FX_PDCD‖CRNC_PAIR_ID‖TNR_ID‖EXPI_FNSH_YMD` 연결 |

> 커서 SELECT 는 위 10개 컬럼 + `ROW_NUM`(11번째, NEXT KEY 용) 을 fetch 한다. `ROW_NUM` 은 출력 전문에 실리지 않고 페이징 키로만 사용.

## 3. 구현 흐름 (`.pc`)

```
W1115S02()  // 콜백 진입 (L35)
  ├─ prthdr / getmid / getmod            // COMHDR·입력·출력 주소 획득
  ├─ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp='M'
  ├─ switch(mid->prGb[0])
  │    case '1' → W1115S02_F1()          // 조회
  │    default  → comhdr_o->mesg="작업구분 오류." + return  (커밋/롤백 없이 즉시 반환)
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   // 명시적 트랜잭션 종료

W1115S02_F1()  // 조회 (L103)
  ├─ getmid/getmod, content->sndl += W1115S02_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ switch(content->keyc)              // 페이징 키
  │    K_NEXT → content_get_nkey → vKeyv
  │    K_PREV → content_get_pkey → vKeyv
  │    default→ vKeyv="0"               // 최초
  ├─ vUserId = yprintf(mid->userId) + l_rtrim
  ├─ EXEC SQL DECLARE ST_W1115S02_C1 CURSOR FOR (SELECT … WHERE ROW_NUM > :vKeyv)
  │    └ SQLCODE!=0 → e_msg 10118, eflg=1 (※ return 없이 진행)
  ├─ EXEC SQL OPEN ST_W1115S02_C1
  │    ├ NO_DATA_FOUND → e_msg 91001, eflg=0 → return RET_FAILURE
  │    └ else err       → e_msg 10118, eflg=1 → return RET_FAILURE
  ├─ for(ii=0; ; ii++)
  │    ├ FETCH INTO cFdmId..cSymbol, cKeyv
  │    │   ├ NO_DATA_FOUND → break
  │    │   └ else err → CLOSE + e_msg 10118, eflg=1 → return RET_FAILURE
  │    ├ if(ii==max_row){ i_isnext++; break; }   // 페이지 초과 → 다음 페이지 있음
  │    ├ zsedit(mod->orec[ii].*)                  // 레코드 append
  │    └ vKeyv = cKeyv                            // NEXT KEY 갱신
  ├─ EXEC SQL CLOSE ST_W1115S02_C1
  ├─ if(ii==0 && i_isnext==0) → e_msg 91001 (자료없음), eflg=0
  │  else if(i_isnext==1)     → content_set_nkey + e_msg 91002 (조회 계속), eflg=0
  │       else               → e_msg 91003 (조회완료), eflg=0, grid01_cnt=ii
  ├─ eflg="0" (무조건 재설정), grid01_cnt=ii
  ├─ content->sndl += (ii-1)*sizeof(orec[0])
  └─ return RET_SUCCESS
```

## 4. 사용 DB 테이블
(활성 쿼리 L254~307 기준. L188~253 의 첫 SELECT 는 블록 주석 처리되어 미사용)

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-----|-----|-----|-----|-----|
| `TB_FXB_CSC004M` | 사용자정보 기본 | CSC004M | SELECT | `FX_USER_NO=:vUserId`, `USER_STTS_DCD='1'`, `FX_USER_DCD NOT IN ('TR','CS','RK')` 직원 자격 필터 |
| `TB_FXB_CMG002M` | 외환중개사상품기본 | CMG002M | SELECT | 상품 마스터 — `FDM_ID='FRX'`, `TRN_ABL_YN='Y'` 거래가능 상품 |
| `TB_FXB_CMG006M` | 통화별상품정보기본 | CMG006M | SELECT | FX_PDCD/CRNC_PAIR_ID/TNR_ID 조인, `USE_YN='Y'`, 만기일 |
| `TB_FXB_CMG004M` | 통화페어기본 | CMG004M | SELECT | CRNC_PAIR_ID 조인, 정렬키 `RPSN_SQC` 제공 |

*DML 요약: 커서 SELECT 1건(4개 테이블 카티전 조인 + 인라인뷰 + `ROW_NUMBER() OVER`)만 존재. INSERT/UPDATE/DELETE 없음. 조회 전용이나 콜백 종료 시 관례적으로 `l_dbcommit()` 호출.*

*(주석 처리된 미사용 첫 쿼리는 `TB_FXB_CSC005R`(사용자별고객번호관계) + `TB_FXB_CSC001M`(고객약정정보기본) + CMG002M/006M/004M 조합 — 고객 기준 버전. 실 구현은 이를 버리고 CSC004M 직원 기준 버전으로 교체함.)*

## 5. 로직 / 필터 규칙
- **작업구분**: `prGb='1'` 조회만 지원. 그 외는 즉시 "작업구분 오류." (트랜잭션 미수행).
- **페이징**: `content->keyc` (K_NEXT/K_PREV/최초) 로 `vKeyv` 결정 → 커서는 `WHERE W1115S02.ROW_NUM > :vKeyv` 로 이후 행만. 최초는 `vKeyv="0"`.
- **정렬/ROW_NUM**: `ORDER BY FDM_ID, FX_PDCD DESC, CMG004M.RPSN_SQC, EXPI_FNSH_YMD` 순으로 `ROW_NUMBER()` 부여. FETCH 된 `cKeyv`(=ROW_NUM) 를 다음 NEXT KEY 로 사용.
- **자격 필터**: 직원(userId) 이 `USER_STTS_DCD='1'` 이고 `FX_USER_DCD` 가 TR/CS/RK 가 아닌 경우에만 상품 목록 산출.
- **상품 필터**: `FDM_ID='FRX'` + `TRN_ABL_YN='Y'` + CMG006M `USE_YN='Y'`.
- **페이지 크기**: `max_row = MAX_RRB / sizeof(orec[0])`. `ii==max_row` 도달 시 그 행은 버리고 `i_isnext=1` 로 다음 페이지 존재 표시 (마지막으로 append 된 행의 키가 NEXT KEY 로 남아 재조회 시 이어짐 — 데이터 손실 없음).
- **트랜잭션**: F1 이 `RET_SUCCESS` → `l_dbcommit()`, `RET_FAILURE` → `l_dbrollback()`.
- **에러/메시지 코드 (e_msg.code → fnSetMsg)**:
  | code | 의미 | eflg |
  |-----|-----|-----|
  | `10118` | 커서 DECLARE/OPEN/FETCH SQL 오류 | 1 (비정상) |
  | `91001` | 자료 없음 (OPEN NO_DATA 또는 ii==0) | 0 |
  | `91002` | 조회 계속 (다음 페이지 있음, NEXT KEY set) | 0 |
  | `91003` | 조회 완료 | 0 |
