## 2. I/O 전문 (`.h`)

**Input `W1214S01_I`** (총 8B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" |
| `crncPairId` | 7 | 통화페어ID — 부분일치 검색어(LIKE `%…%`). 공백이면 전체 조회 |

**Output `W1214S01_O`** (grid — base 6B + `orec[]` 가변)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (조회된 행 수 `ii`, `zredit` 로 우측정렬 숫자편집) |
| `orec[].crncPairId` | 7 | 통화페어ID (`TB_FXB_CMG004M.CRNC_PAIR_ID`) |

> `.h` 는 `orec[1]` 로 선언되지만 실제로는 `max_row` 개까지 가변 반복. `content->sndl` 을
> `W1214S01_O_SZ + (ii-1)*sizeof(orec[0])` 로 늘려 실제 전송 길이를 결정한다.

## 3. 구현 흐름 (`.pc`)

```
W1214S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' → retc = W1214S01_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  ├─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()  (명시적 커밋)
  └─ fnSetMsg(&e_msg, comhdr_o) → return content->sndl

W1214S01_F1()                   ← 조회 (grid + 페이징)
  ├─ getmid/getmod, sndl += W1214S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])      한 페이지 최대 행수
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT → content_get_nkey → vKeyv
  │      K_PREV → content_get_pkey → vKeyv
  │      default(최초) → vKeyv="0"
  │      → l_rtrim(vKeyv)   (SQL 에 실제 바인딩됨 — 페이징 동작)
  ├─ 입력 바인딩: yprintf + l_rtrim → vCrncPairId  (NULL/공백 실검증 없음 — §6)
  ├─ DECLARE W1214S01_C1 CURSOR (아래 §5 SELECT)
  ├─ OPEN
  │      SQLCODE≠SUCCESS: NO_DATA_FOUND → e_msg "91001" + RET_SUCCESS
  │                       기타 err     → e_msg "10118" + RET_FAILURE
  ├─ for(ii=0; ; ii++)   FETCH → cCrncPairId, cKeyv
  │      NO_DATA_FOUND → break
  │      기타 err      → CLOSE + e_msg "10118" + RET_FAILURE
  │      ii==max_row   → i_isnext++, break (초과 1행 감지용 — orec 미기록)
  │      → mod->orec[ii].crncPairId = cCrncPairId
  │      → vKeyv = cKeyv (마지막 기록행 KEY 보관)
  ├─ CLOSE
  ├─ 결과 판정
  │      ii==0 && i_isnext==0 → e_msg "91001" (자료 없음)
  │      i_isnext==1          → content_set_nkey(vKeyv) + e_msg "91002" (계속)
  │      그 외                → e_msg "91003" (완료) + grid01_cnt=ii
  └─ eflg="0", grid01_cnt=ii, sndl += (ii-1)*sizeof(orec[0]) → RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG004M` | 통화페어(기본) 마스터 | `CMG004M` | **SELECT** (커서 `W1214S01_C1`) | `PL_CRNC_PAIR_ID='USD/KRW' AND USE_YN='Y'` 인 통화페어를 `CRNC_PAIR_ID` LIKE 부분일치 + `RPSN_SQC` 페이징 키로 목록 조회 |

*DML 요약: 커서 SELECT 1문(`TB_FXB_CMG004M`)뿐. INSERT/UPDATE/DELETE 없음. 규약상 성공 시 `l_dbcommit`. 공통함수 경유 간접 테이블 접근 없음.*

## 5. 로직 / 필터 규칙

- **커서 SELECT** (`W1214S01_C1`):
  ```sql
  SELECT CMG004M.CRNC_PAIR_ID
       , LPAD(CMG004M.RPSN_SQC,7,0) AS NEXT_KEY   -- 다음조회 KEY
    FROM TB_FXB_CMG004M CMG004M
   WHERE (    (TRIM(:vCrncPairId) is NULL     AND CMG004M.CRNC_PAIR_ID = CMG004M.CRNC_PAIR_ID)
           OR (TRIM(:vCrncPairId) is NOT NULL AND CMG004M.CRNC_PAIR_ID LIKE '%'||trim(:vCrncPairId)||'%'))
     AND CMG004M.PL_CRNC_PAIR_ID = 'USD/KRW'
     AND CMG004M.USE_YN          = 'Y'
     AND LPAD(CMG004M.RPSN_SQC,7,0) > :vKeyv
   ORDER BY RPSN_SQC
  ```
- **동적 필터**: `vCrncPairId` 공백/NULL 이면 전체(`CRNC_PAIR_ID = CRNC_PAIR_ID` 항상 참), 값 있으면 `LIKE '%값%'` 부분일치. `PL_CRNC_PAIR_ID='USD/KRW'` + `USE_YN='Y'` 고정 필터.
- **페이징 (실동작)**: `RPSN_SQC` 를 `LPAD(...,7,0)` 로 zero-fill 한 문자열을 KEY 로 사용. `vKeyv` 를 `> :vKeyv` 로 바인딩 → 최초 "0" 부터, NEXT 는 이전 페이지 마지막 기록행 KEY 이후부터 조회. `max_row+1` 번째 행을 FETCH 해 "다음 있음"(`i_isnext`)을 감지하고 그 초과 행은 grid 에 기록하지 않는다. `content_set_nkey` 로 다음 KEY 전달.
- **입력 유효성**: 3절 주석은 "입력항목 유효성 check" 이나 실제 `crncPairId` 를 `yprintf/l_rtrim` 만 하고 NULL/길이 거부 로직 없음(§6).
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님(변경 없어도 규약상 commit).
- **에러/결과 코드** (`e_msg.code` → `fnSetMsg` 로 COMHDR `mesg`):
  - `91001` "해당 자료가 없습니다." (조회 0건).
  - `91002` "조회가 계속됩니다." (다음 페이지 존재, `content_set_nkey` 세팅).
  - `91003` "조회가 완료되었습니다." (마지막 페이지).
  - `10118` 커서 OPEN/FETCH SQL 오류 → RET_FAILURE → rollback, eflg="1".
- **결과 전달**: `grid01_cnt` + `orec[]` 목록 + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
