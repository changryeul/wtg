## 3. 구현 흐름 (`.pc`)

```
W1402S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1402S01_F1(...)     ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? l_dbcommit() (eflg"0") : l_dbrollback() (eflg"1")
     → fnSetMsg(&e_msg, comhdr_o) → return content->sndl   (명시적 커밋, AUTOCOMMIT 아님)

W1402S01_F1()                   ← 조회 본체
  ├─ getmid/getmod, sndl += W1402S01_O_SZ, memset(mod)
  │  max_row = MAX_RRB / sizeof(orec[0])           ← 페이지당 최대 건수
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT→content_get_nkey / K_PREV→content_get_pkey / default→"0" → vKeyv, l_rtrim
  ├─ 입력 바인딩: XPRINTF+l_rtrim → vWbcsRlnmAltrNo / hls_upd_id(usid) / hls_upd_ip(loip)
  │      (유효성 실검증 없음 — §6, hls_upd_* 는 이후 미사용 — §6)
  ├─ DECLARE W1402S01_C1 CURSOR (5-table 조인, KEY > :vKeyv, ORDER BY FX_USER_NO,MNGM_BRCD,WBCS_RLNM_ALTR_NO)
  ├─ OPEN W1402S01_C1
  │      NO_DATA_FOUND → e_msg "91001", return SQL_SUCCESS (커밋 경로)
  │      기타 SQL err  → e_msg "10118", return RET_FAILURE (롤백)
  ├─ FETCH 루프 (ii=0..)
  │      NO_DATA_FOUND → break
  │      기타 err → CLOSE, e_msg "10118", return RET_FAILURE
  │      ii == max_row → i_isnext++, break (해당 행은 저장 안 함 = next 페이지 판정용)
  │      ccifDcd=="11"(개인)?
  │          ├─ TB_FXB_CMC003D(TRN_CNTL_TIM,'0') 조회가능시간 CASE 판정 → vSchYn
  │          │     SQL err → "10118" RET_FAILURE / vSchYn=='N' → "32713"(거래제한) RET_FAILURE
  │          └─ fnGetCusCpn(&stCusCpn)  ← CRM 연락처 인터페이스 (param=vWbcsRlnmAltrNo)
  │                실패 → SKIP(부가정보라 오류 무시), 성공 → email/domain/id/cpn override
  │      zsedit → mod->orec[ii] 16필드 세팅, vKeyv = cKeyv (다음키)
  ├─ CLOSE W1402S01_C1
  ├─ ii==0 && i_isnext==0 → e_msg "91001" (해당 자료 없음)
  │  i_isnext==1          → content_set_nkey(vKeyv), e_msg "91002" (조회 계속)
  │  else                 → e_msg "91003" (조회 완료), zredit(grid01_cnt, ii)
  ├─ zredit(grid01_cnt, ii)                         ← 무조건 1회 더 (§6 중복)
  └─ content->sndl += (ii-1) * sizeof(orec[0]) → return RET_SUCCESS
```

`W1402S01_F2()` 는 프로토타입만 선언(`//SOON 삭제`)되어 있고 본문 정의·호출이 없다 — dead code(§6).

## 5. 로직 / 필터 규칙

- **DML**: 실 데이터 변경 없음. 커서 SELECT + (개인 분기) 시간체크 SELECT + CRM I/F. 성공 시 `l_dbcommit`, 실패 시 `l_dbrollback`.
- **조회 필터**: `CSC005R.WBCS_RLNM_ALTR_NO = RTRIM(:vWbcsRlnmAltrNo)` 를 앵커로 CSC004M(FX_USER_NO)·CSC001M(WBCS_RLNM_ALTR_NO) 조인. 코드명 2건은 outer join `(+)`.
- **페이징(정상 동작)**: `KEY = FX_USER_NO||MNGM_BRCD||WBCS_RLNM_ALTR_NO`, 커서 조건 `KEY > :vKeyv` + `ORDER BY` 로 정렬. `content->keyc`(NEXT/PREV/최초)로 `vKeyv` 를 구성해 실제로 바인딩한다 → W1119S01 과 달리 페이징이 동작한다.
  - `ii == max_row` 도달 시 그 행은 저장하지 않고 `i_isnext` 만 올려 다음 페이지 존재를 표시(`91002` + `set_nkey`). 다음키는 마지막으로 **저장된** 행의 `cKeyv` → 페이지 경계 누락/중복 없음.
- **개인(`CIF_DCD='11'`) 특례**:
  - 조회가능시간 CASE 판정으로 `vSchYn='N'` 이면 `32713`(은행 일자전환 거래제한) RET_FAILURE.
  - `fnGetCusCpn()` 로 CRM 연락처(이메일주소/ID/도메인/휴대폰)를 override. CRM 실패는 부가정보라 오류 처리하지 않고 빈값/기존값 유지(SKIP).
  - CRM 호출 파라미터는 `crprsCusNo`(대표고객번호)가 아니라 `vWbcsRlnmAltrNo` 사용(§6).
- **입력 유효성**: 3절 주석은 "입력항목 유효성 check" 이나 실제 `wbcsRlnmAltrNo` 를 `XPRINTF/l_rtrim` 만 하고 NULL/공백 거부 없음(§6). 빈 키는 커서 `NO_DATA_FOUND`(`91001`)로 걸러짐.
- **에러/결과 코드**:
  - `91001` — 자료 없음(OPEN NO_DATA 또는 fetch 0건). OPEN 경로는 `return SQL_SUCCESS` → 커밋.
  - `91002` — 조회 계속(다음 페이지 있음), `set_nkey`.
  - `91003` — 조회 완료.
  - `10118` — 커서 OPEN/FETCH/시간체크 SQL 오류 → RET_FAILURE(롤백).
  - `32713` — (개인) 조회 불가 시간 → RET_FAILURE.
  - 작업구분 오류 — COMHDR `mesg="작업구분 오류."` (eflg 미세팅).
- **결과 전달**: `grid01_cnt` + `orec[0..ii-1]` + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200 + COMHDR 본문 동봉(레거시 규약).

> 직접 좌우하므로 마이그레이션·리팩터 시 원 SQL 의도를 반드시 확인할 것.
