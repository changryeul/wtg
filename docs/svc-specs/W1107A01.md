## 5. 로직 / 필터 규칙

- **처리 분기**: `prGb='1'` 만 유효. 그 외는 COMHDR `mesg="작업구분 오류."` 로 즉시 반환.
- **입력 유효성**: 공통 3개(`lgnId`/`fxUserNo`/`secuMdiaDcd`) 후 매체별 필수값 검증. `isNullSpace()==0`(공백/NULL) 이면 매체·항목별 e_msg code 로 거부(`43065`/`10555`/`10556`/`10564~10573`/`10559`).
- **사용자 조회**: `CSC004M`·`CSC005R`·`CSC001M` 3-테이블 조인으로 `CUS_ABBR_NM`/`CIF_NO`/`CIF_DCD` 확보. `NO_DATA_FOUND` 시 "미등록 사용자"(`10118`).
- **매체별 검증 위임**:
  - OTP(`'2'`): 개인(`cCifDcd="11"`)이면 `fnGetSccdCheck1(prGb=3)` 로 실명번호(`cRnn→cCifNo`) 선조회 후 `fnOtpLogin` 호출.
  - 보안카드(`'1'`): `fnGetSccdCheck1(prGb=2)` — 순번/앞뒤 비밀번호 검증.
  - 스마트보안카드(`'3'`): `fnChkScardInfo(prGb=2)` — 로그인IP(`comhdr_i->loip`)·인증번호·UniqueKey 로 검증. 비고문구 `"<고객약어명> 고객님의 [FXON] 로그인을 위해 인증합니다."` 세팅.
  - 외부함수 `ret!=0` 이면 e_msg 를 그대로 실어 `eflg="1"`, RET_FAILURE.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. 실제 변경 DML 은 없으나 관례상 명시적 커밋.
- **결과 전달**: 성공 시 `returnCd="S"`/`returnMsg="OK"` + COMHDR `eflg="0"`, `e_msg.code="10150"`. 실패는 `eflg="1"` + 해당 code→mesg.
  - → WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
