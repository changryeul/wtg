# 서비스 명세 — `W1113S01` (고객결제계좌조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 2. I/O 전문 (`.h`)

### Input `W1113S01_I`
| 필드 | 크기 | 설명 (실제 의미) |
|------|-----|------|
| `prGb` | 1 | 처리구분 — `'1'` 조회만 유효, 그 외는 "작업구분 오류" 반환 |
| `fxUserNo` | 30 | 로그인 ID (FX사용자번호). `vFxUserNo` 로 사용 |

> 이 외에 COMHDR 의 `fdid`(펀드번호/CustomerId) 를 입력으로 사용한다(`comhdr_i->fdid` → `vFndNo`). 헤더 구조체에는 없고 공통헤더에서 취한다.

### Output `W1113S01_O` (grid, 가변 orec)
| 필드 | 크기 | 설명 |
|------|-----|------|
| `grid01_cnt` | 6 | 그리드 건수 (실 fetch 카운트 `ii`) |
| `orec[].wbcsRlnmAltrNo` | 30 | 전행고객실명대체번호 (고객번호) |
| `orec[].fxSlactDcd` | 1 | FX결제계좌구분코드 (1:원화, 2:외화) |
| `orec[].fxStlmAcn` | 30 | 결제계좌번호 — `fnAcntConv` 계좌포맷 적용 후 세팅 |
| `orec[].crcd` | 3 | 통화코드 |
| `orec[].acntOpnYmd` | 8 | 계좌개설 년월일 |
| `orec[].fxSlactYn` | 1 | FX결제계좌여부 |
| `orec[].slactTrmtYmd` | 8 | 결제계좌해지년월일 |

> `.h` 의 `orec[1]` 은 가변 grid. 실제로는 `max_row = MAX_RRB / sizeof(orec[0])` 만큼 append 하며 `content->sndl` 를 record 단위로 늘린다.

## 3. 구현 흐름 (`.pc`)

```
W1113S01()                                    // 콜백 dispatch
  ├ prthdr / getmid / getmod                  // 입출력 주소 획득
  ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)  // comhdr_o->rtyp="M"
  ├ switch(mid->prGb[0])
  │   case '1' : W1113S01_F1()                // 조회 + 기본계좌 정합
  │   default  : "작업구분 오류." 즉시 return
  └ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()

W1113S01_F1()                                 // 조회 및 기본계좌 정합
  ├ 1. init: content->sndl += O_SZ, max_row 계산
  ├ 2. keyc(K_NEXT/K_PREV/최초) → vKeyv        // ※ 이후 쿼리에서 미사용 (§6)
  ├ 3. 입력 유효성: vFxUserNo, vFndNo(fdid)
  ├ 4. 고객ID 획득
  │     SELECT WBCS_RLNM_ALTR_NO INTO cWbcsRlnmAltrNo
  │       FROM TB_FXB_CSC005R WHERE FX_USER_NO = vFxUserNo
  │       (NO_DATA→91001/eflg0, else 10118/eflg1 → RET_FAILURE)
  │     fnCustInfoQry(&CUST) → 약정정보(원화/외화계좌 등) 로드 (실패 10165)
  │     memcmp(고객ID 재검증, 불일치 10591)
  │  ── 원화계좌 정합 (lWonAcn != 0x00) ──
  │     SELECT FX_SLACT_YN FROM TB_FXB_CSC003M (WBCS_RLNM_ALTR_NO,FX_SLACT_DCD='1',FX_STLM_ACN)
  │       NO_DATA_FOUND → UPDATE 기존'Y'→'N' + INSERT 신규 기본계좌('Y')
  │       존재 & FX_SLACT_YN!='Y' → UPDATE 기존'Y'→'N' + UPDATE 대상→'Y'
  │  ── 외화계좌 정합 (lFrcyAcn != 0x00) ──   // FX_SLACT_DCD='2'
  │     (원화와 동형) NO_DATA→UPDATE'N'+INSERT / 존재&!='Y'→UPDATE'N'+UPDATE'Y'
  │       + TB_FXB_CSC012M FRCY_ACNT_PDF_DCD 갱신 (실패 10173)
  ├ 5. 조회 커서
  │     DECLARE W1113S01_C1 CURSOR FOR
  │       SELECT ... FROM TB_FXB_CSC003M
  │        WHERE WBCS_RLNM_ALTR_NO=cWbcsRlnmAltrNo
  │          AND STLM_ACSS_DCD='1' AND TRN_USE_YN='Y' AND FX_SLACT_YN='Y'
  │     OPEN → FETCH loop
  │       vFndNo 존재 시: TB_FXB_CSC012M LEFT JOIN CSC003M 로 이용자별 결제계좌 대체
  │         (미존재 10232, 오류 10118)
  │       fnAcntConv 계좌포맷 → orec[ii] 세팅
  │       ii==max_row → i_isnext++ break
  │     CLOSE (중복 CLOSE 2회)
  └ 결과: 0건→91001 / next→91002+nkey / 완료→91003, grid01_cnt=ii, RET_SUCCESS
```

트랜잭션: 서브함수는 SELECT/INSERT/UPDATE 만 수행하고 commit/rollback 을 하지 않는다. 최상위 `W1113S01()` 이 반환코드로 `l_dbcommit()`/`l_dbrollback()` 을 1회 결정한다(비 AUTOCOMMIT).

## 4. 사용 DB 테이블
| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|--------|--------|------|------|------|
| `TB_FXB_CSC005R` | 사용자별고객번호관계 | — | SELECT | `FX_USER_NO` → `WBCS_RLNM_ALTR_NO`(고객ID) 매핑 |
| `TB_FXB_CSC003M` | 고객계좌기본 | `CSC003M` | SELECT / INSERT / UPDATE | 원화·외화 기본계좌 정합(신규 등록, 기본여부 Y/N 전환) + 결제계좌 조회 커서 |
| `TB_FXB_CSC012M` | 이용자별 결제계좌기본 | `CSC012M` | SELECT / UPDATE | 펀드(이용자)별 결제계좌 대체 조회 + 외화계좌 상품구분코드 갱신 |

*DML 요약*: `TB_FXB_CSC005R` (SELECT ×1) · `TB_FXB_CSC003M` (SELECT ×3, UPDATE ×4, INSERT ×2, CURSOR SELECT ×1) · `TB_FXB_CSC012M` (JOIN SELECT ×1, UPDATE ×1). 고객 약정정보는 외부 함수 `fnCustInfoQry()` 로 조회(테이블 직접 접근 아님).

## 5. 로직 / 필터 규칙
- **작업구분**: `prGb='1'` 만 처리. 그 외는 즉시 "작업구분 오류." 로 종료(커밋/롤백 없이 `content->sndl` 반환).
- **고객ID 확정**: 로그인ID(`FX_USER_NO`)로 `TB_FXB_CSC005R` 에서 `WBCS_RLNM_ALTR_NO` 를 얻고, `fnCustInfoQry` 결과의 고객ID 와 `memcmp` 재검증(불일치 시 10591).
- **기본계좌 자동 정합(side effect)**: 약정정보의 원화(`lWonAcn`)/외화(`lFrcyAcn`) 계좌가 있으면 `TB_FXB_CSC003M` 에 기본계좌(`FX_SLACT_YN='Y'`)로 존재하도록 보정 — 없으면 기존 Y→N 후 신규 INSERT, 있으나 Y 아니면 기존 Y→N 후 해당 계좌 Y. 외화는 추가로 `TB_FXB_CSC012M.FRCY_ACNT_PDF_DCD` 갱신. `FX_SYS_LSMD_ID` 는 `"W1113S01"`, `FX_SYS_LSMD_TS`=SYSDATE.
- **조회 커서 필터**: `STLM_ACSS_DCD='1'`(정상) · `TRN_USE_YN='Y'` · `FX_SLACT_YN='Y'` 인 결제계좌만, 고객ID 로 필터하여 (구분,계좌) 순 정렬.
- **이용자별 대체**: `vFndNo`(펀드번호)가 있으면 커서 fetch 행마다 `TB_FXB_CSC012M`↔`CSC003M` LEFT JOIN 으로 이용자별 결제계좌를 재조회해 출력값을 대체(`FX_SLACT_YN` 은 강제 `'Y'` 상수). 미존재 시 10232.
- **계좌 포맷**: 출력 계좌번호는 `fnAcntConv` 로 포맷 변환 후 세팅.
- **페이징**: `ii==max_row` 도달 시 `i_isnext=1`, next-key 세팅 후 91002. 완료 91003, 무자료 91001.
- **에러(e_msg code→eflg)**: 91001 무자료(eflg0)/91002 계속(eflg0)/91003 완료(eflg0) · 10118 조회오류(1) · 10165 고객약정정보 조회오류(1) · 10171 원화 update 오류(1) · 10172 외화 update 오류(1) · 10173 외화계좌상품구분코드 수정오류(1) · 10174 등록오류(1) · 10232 펀드이용자 결제계좌 미존재(1) · 10591 고객ID 불일치(1).
