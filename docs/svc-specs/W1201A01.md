## 2. I/O 전문 (`.h`)

**Input `W1201A01_I`** — *괄호 안은 `.pc` 기준 실제 의미*

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 만 처리(등록), 그 외 "작업구분 오류" |
| `crcd` | 3 | 통화코드 — 필수. 공백/NULL 이면 `32029` 로 거부 |
| `fxSysLsmdId` | 30 | FX시스템최종변경ID — **`.pc` 에서 미사용** (대신 `comhdr_i->usid` 사용, §6) |
| `grid01_cnt` | 6 | 내부 그리드(orec) 건수 — `l_n2i()` 로 파싱해 INSERT 반복 횟수로 사용 |
| `orec[].fdmId` | 10 | 외환중개사ID — **`.pc` 에서 바인딩만 하고 INSERT 미사용** (`'FRX'` 리터럴 고정, §6) |
| `orec[].dwkDcd` | 1 | 요일구분코드 (일월화수목금토) — INSERT `DWK_DCD` |
| `orec[].comntCon` | 500 | 코멘트내용 — INSERT `COMNT_CON` |

> `.h` 상 `orec[1]` 로 선언되나 실제로는 `grid01_cnt` 건수만큼 가변 반복 처리하는 그리드 입력이다.

**Output `W1201A01_O`** (1B)

| 필드 | 크기 | 설명 |
|---|---|---|
| `dum` | 1 | 더미 — 결과 데이터 없음. 성공/실패는 COMHDR (`eflg`/`mesg`) 로 전달 |

> 성공 시 `content->sndl += W1201A01_O_SZ` (1B) 만 붙는다. 실질 응답 본문은 비어 있고 결과는 공통헤더로 나간다.

## 3. 구현 흐름 (`.pc`)

```
W1201A01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' → retc = W1201A01_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->mesg="작업구분 오류." 후 즉시 return
  └─ retc==RET_SUCCESS ? (eflg="0", l_dbcommit()) : (eflg="1", l_dbrollback())
        (명시적 커밋, AUTOCOMMIT 아님) → fnSetMsg(&e_msg, comhdr_o)

W1201A01_F1()                   ← 등록 (delete-all → re-insert)
  ├─ getmid/getmod, memset(mod)
  ├─ 입력 바인딩: yprintf + l_rtrim
  │     vCrcd        ← mid->crcd
  │     vFxSysLsmdId ← comhdr_i->usid   (mid->fxSysLsmdId 아님, §6)
  ├─ 유효성: strlen(vCrcd)<=0 → e_msg "32029", RET_FAILURE
  ├─ [4-1] DELETE TB_FXB_CMG009M WHERE CRCD = :vCrcd    (통화 기준 전건 삭제)
  │        SQLCODE!=SUCCESS && !=NO_DATA_FOUND → e_msg "10122", RET_FAILURE
  ├─ [4-2] cnt = l_n2i(mid->grid01_cnt)
  │        for(ii=0; ii<cnt; ii++)
  │           vFdmId/vDwkDcd/vComntCon ← orec[ii] (yprintf+l_rtrim)
  │           INSERT TB_FXB_CMG009M
  │             (FDM_ID='FRX', CRCD=:vCrcd, DWK_DCD=:vDwkDcd,
  │              COMNT_CON=:vComntCon, FX_SYS_LSMD_ID=:vFxSysLsmdId, FX_SYS_LSMD_TS=SYSDATE)
  │        (루프 내 SQLCODE 미검사 — §6)
  ├─ 루프 종료 후 SQLCODE!=0 판정 (마지막 반복 결과만 반영, §6)
  │     DUP_KEY → e_msg "10223", RET_FAILURE
  │     그외    → e_msg "10122", RET_FAILURE
  └─ 정상 → e_msg "10150", sndl += W1201A01_O_SZ, RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG009M` | 통화별주말기본 | `CMG009M` | **DELETE** | 등록 전 해당 `CRCD` 의 기존 주말요일 행 전건 삭제 |
| `TB_FXB_CMG009M` | 통화별주말기본 | `CMG009M` | **INSERT** | 그리드(orec) 각 행을 `FDM_ID='FRX'` 로 재적재 |

*DML 요약: 단일 테이블 `TB_FXB_CMG009M` 에 DELETE 1문 + INSERT N문 (그리드 건수). SELECT/UPDATE 없음. 커서/조인 없음.*

## 5. 로직 / 필터 규칙

- **전체 교체 패턴**: `(CRCD)` 단위로 기존 행을 모두 삭제한 뒤 입력 그리드를 재삽입. 부분 갱신이 아니라 통화별 주말요일 목록 전체를 통째로 바꾼다.
- **입력 유효성**: `crcd` 공백/NULL 이면 `32029` 로 즉시 거부 (DELETE/INSERT 미수행). 그리드 요일/코멘트에 대한 별도 검증은 없다.
- **INSERT 고정값**: `FDM_ID` 는 입력 `orec.fdmId` 를 무시하고 리터럴 `'FRX'` 고정. `FX_SYS_LSMD_ID` 는 입력 `fxSysLsmdId` 가 아니라 COMHDR 의 `usid`(요청자 ID)를 사용. `FX_SYS_LSMD_TS` 는 `SYSDATE`.
- **그리드 건수 0**: `grid01_cnt=0` 이면 INSERT 루프가 돌지 않아 **DELETE 만 수행**(해당 통화 주말요일 전건 삭제)되고 정상 종료된다.
- **트랜잭션**: `RET_SUCCESS→eflg="0"+l_dbcommit / 실패→eflg="1"+l_dbrollback`. AUTOCOMMIT 아님 — 명시적 커밋 필수.
- **에러 (`e_msg.code`→`mesg`)**:
  - `32029` 입력값(통화코드) 확인
  - `10122` 자료 처리중 오류 (DELETE 오류 / INSERT 일반 오류)
  - `10223` 중복(DUP) 오류 (INSERT `DUP_KEY`)
  - `10150` 정상 처리 (성공)
  - → WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, `errn`/COMHDR 본문 동봉(레거시 규약).

## 6. 주의 — 문서/코드 불일치 (마이그레이션 시 확인)

| 위치 | 헤더/주석 표기 | 실제(`.pc`) |
|---|---|---|
| `.h` `fxSysLsmdId` 필드 | FX시스템최종변경ID (입력으로 받는 듯) | 미사용 — INSERT `FX_SYS_LSMD_ID` 는 `comhdr_i->usid` 를 사용 |
| `.h` `orec.fdmId` 필드 | 외환중개사ID (입력으로 받는 듯) | `vFdmId` 로 바인딩·로깅만 하고 INSERT 에는 리터럴 `'FRX'` 고정 (dead code) |
| `.pc` 루프 내 INSERT | 각 건별 결과 확인 의도 | 루프 안에서 `SQLCODE` 미검사 — 중간 건 실패해도 다음 건 계속 시도, 루프 종료 후 **마지막 INSERT 의 `SQLCODE` 만** 판정 (초기 건 실패 누락 가능성) |
| `mid` 로깅 (line 53) | `INDATA=[%-200.200s]` | `orec.comntCon`(500B)까지 포함하는 가변 입력을 200B 로만 로깅 (진단 정보 절단) |

> WTG 는 `.h` 를 파싱해 필드명/크기만 취하고 전문을 통과시키므로 위 오기가 동작에 영향을 주지 않는다. 다만 클라이언트가 `fxSysLsmdId`/`orec.fdmId` 를 채워 보내도 엔진이 무시(`usid`/`'FRX'` 로 대체)한다는 점, 그리드 부분 실패 시 진단이 어렵다는 점은 연동 개발자에게 전달할 가치가 있다.
