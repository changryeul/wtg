## 5. 로직 / 필터 규칙

- **작업 분기**: `prGb` 1문자로 F1/F2/F3 분기. 미매칭 시 COMHDR `mesg`="작업구분 오류." 로 즉시 종료(커밋/롤백·fnSetMsg 미경유).
- **선행 검증 (F1/F2)**: 계좌를 붙이려는 외환중개사(`FDM_ID`)가 `TB_FXB_CMG001M` 에 `USE_YN='Y'` 로 실재해야 함. `SELECT ... INTO :unUseYn` 는 존재 확인용이며 조회 값 자체는 사용하지 않음.
- **입력 유효성**: `fdmId` 공백/NULL → `10205`, `fdmAcn` 공백/NULL → `10557` (세 함수 공통).
- **키**: `CMG003M` 대상은 (`FDM_ID`, `FDM_ACN`) 복합키로 특정.
- **하드코딩/기본값**:
  - 등록(F1) INSERT 시 `USE_YN='Y'` 고정 (입력 `useYn` 무시), `FX_SYS_LSMD_TS=SYSDATE`.
  - 수정(F2) UPDATE 시 `USE_YN=NVL(:vUseYn,'Y')`, `FX_SYS_LSMD_TS=SYSDATE`.
  - 삭제(F3) UPDATE 시 `USE_YN='N'`, 변경자 ID 는 입력이 아닌 `comhdr_i->usid`.
- **트랜잭션**: 각 F 함수는 `RET_SUCCESS/RET_FAILURE` 만 반환하고, 상위 `W1357A01()` 가 결과에 따라 `l_dbcommit()`/`l_dbrollback()` 수행. AUTOCOMMIT 아님 — 명시적 커밋 필수.
- **에러/메시지 (e_msg.code → mesg, eflg 는 상위에서 0/1 세팅)**:
  - 성공: `10150` (MSG_REQUERY, 세 함수 공통).
  - 입력: `10205`(외환중개사ID) / `10557`(계좌번호).
  - F1/F2 선행: `10811`(미사용 중인 외환중개사ID) / `10118`(자료조회 오류).
  - F1 등록: `20002`(중복) / `10558`(등록 중 오류).
  - F2 수정: `10221`(수정할 자료 없음).
  - F3 삭제: `10208`(대상 없음/NOTFOUND) / `10122`(처리 중 오류).
  - → WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, `errn`/COMHDR 본문 동봉(레거시 규약).
