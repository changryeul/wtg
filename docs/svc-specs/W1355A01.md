# 서비스 명세 — `W1355A01` (외환중개사 거래중지/시작 (운영상태 변경))

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1355A01` |
| 이름 | 외환중개사 거래중지/시작 (외환중개사 운영상태 변경) |
| 유형 | 갱신 (UPDATE 단건) |
| 구현 | `win/src/trn/W1350/W1355A01.pc` (188 라인) |
| I/O 헤더 | `win/src/inc/trn/W1355A01.h` |
| 라우팅 alias | `W1355A01` |
| 채널 | `ChannelWeb` (매매 AP, `/v1/tx` passthrough) |
| 모듈 | W1350 (외환중개사 관리) |

외환중개사(FX 브로커/딜링하우스)의 운영여부(`OPRT_YN`) 플래그를 갱신해 해당 중개사의 거래를 중지/시작시키는 단건 UPDATE 서비스. 작업구분(`prGb`)은 현재 `'1'`(수정)만 처리한다.

## 2. I/O 전문 (`.h`)

**Input `W1355A01_I`** (총 42 byte)

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 작업구분 — `'1'`=수정만 처리, 그 외 "작업구분 오류" 반환 |
| `fmdId` | 10 | 외환중개사ID — UPDATE WHERE 절 키 (`FDM_ID`) |
| `oprtYn` | 1 | 운영여부 — SET `OPRT_YN` (거래 중지/시작 플래그) |
| `fxSysLsmdId` | 30 | FX시스템 최종변경자ID — SET `FX_SYS_LSMD_ID` |

**Output `W1355A01_O`** (총 1 byte)

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `dum` | 1 | 더미 — 실제 데이터 없음. 처리결과는 COMHDR(`eflg`/`rtyp`/`mesg`/`rcod`)로만 전달 |

grid/orec 없음. 갱신 결과는 공통헤더로만 회신한다.

## 3. 구현 흐름 (`.pc`)

```
W1355A01()  [dispatch]
  prthdr / getmid / getmod            // COMHDR·mid·mod 주소 획득
  memset(e_msg) + ltyp=KR mtyp=S rtyp=M  // E_MSG 기본설정
  switch(mid->prGb[0])
    '1' → W1355A01_F1()               // 수정
    default → comhdr_o.mesg="작업구분 오류.", 즉시 return(sndl)  // ★commit/rollback 미수행
  if retc == RET_SUCCESS
      comhdr_o.eflg="0"; l_dbcommit()
  else
      comhdr_o.eflg="1"; l_dbrollback()
  fnSetMsg(&e_msg, comhdr_o)          // e_msg.code → mesg/rcod/rtyp 세팅
  return(content->sndl)

W1355A01_F1()  [수정]
  getmid / getmod / memset(mod)
  yprintf + l_rtrim → vFmdId / vOprtYn / vFxSysLsmdId  // 입력 정규화
  EXEC SQL UPDATE TB_FXB_CMG001M
      SET OPRT_YN, FX_SYS_LSMD_ID, FX_SYS_LSMD_TS=SYSDATE
      WHERE FDM_ID = :vFmdId
  if SQLCODE == NO_DATA_FOUND → e_msg.code="10221" (수정할 자료 없음); return RET_FAILURE
  e_msg.code="10150"; return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-----|-----|-----|-----|-----|
| `TB_FXB_CMG001M` | 외환중개사기본 | `CMG001M` | UPDATE | `FDM_ID` 매칭 단건의 운영여부/최종변경자/변경일시 갱신 |

*DML 요약: `W1355A01_F1` 에서 `UPDATE TB_FXB_CMG001M SET OPRT_YN, FX_SYS_LSMD_ID, FX_SYS_LSMD_TS=SYSDATE WHERE FDM_ID=:vFmdId` 단건 1회. INSERT/DELETE/SELECT/커서 없음. 공통함수 경유 간접 테이블 접근 없음.*

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `switch(mid->prGb[0])` — `'1'`(수정)만 `W1355A01_F1` 호출. 그 외 값은 COMHDR `rtyp="M"`, `mesg="작업구분 오류."` 세팅 후 즉시 return (§6 참조 — 이 경로는 commit/rollback을 타지 않음).
- **입력 정규화**: `yprintf` 로 고정폭 필드를 문자열화 후 `l_rtrim` 으로 우측 공백 제거. `prGb` 자체는 검증만 하고 SQL 바인드에 사용하지 않는다 (`vPrGb`, `cDum` 은 선언만 되고 미사용 — dead variable).
- **UPDATE 대상**: `FDM_ID` = 입력 `fmdId` 인 단건. 갱신 컬럼은 `OPRT_YN`(운영여부), `FX_SYS_LSMD_ID`(최종변경자ID), `FX_SYS_LSMD_TS`(최종변경일시=SYSDATE).
- **트랜잭션**: 서비스는 AUTOCOMMIT 아님. dispatcher(`W1355A01`)가 서브함수 retc 로 판정 — `RET_SUCCESS` → `l_dbcommit()`, 그 외 → `l_dbrollback()`.
- **에러/메시지 (e_msg.code → fnSetMsg → mesg)**:
  - `10221` — 수정할 자료가 없습니다. 입력값을 확인하세요 (`SQLCODE == NO_DATA_FOUND`), `eflg="1"` rollback.
  - `10150` — 정상 처리 (재조회 요청, `MSG_REQUERY`), `eflg="0"` commit.
  - 작업구분 오류 — e_msg.code 미세팅, COMHDR `mesg` 직접 세팅 후 조기 return.
