## 2. I/O 전문 (`.h`)

**Input `W1358S02_I`** (총 10B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" |
| `apsrYmd` | 8 | 적용시작년월일 (기준일자, YYYYMMDD) — 유효 적용시작일 산정 기준 |

| `orec[].pdfTnrId` | 3 | 상품테너ID (`CMG002M.TNR_ID`) |
| `orec[].pdfTnrIdNm` | 100 | 상품테너명 — 실제로는 `CMG002M.TNR_ID` 값 그대로(테너명 아님, §6) |

## 3. 구현 흐름 (`.pc`)

```
W1358S02()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1'  → retc = W1358S02_F1(...)     ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  ├─ retc==RET_SUCCESS ? (eflg="0", l_dbcommit) : (eflg="1", l_dbrollback)
  │                             (명시적 커밋, AUTOCOMMIT 아님)
  └─ fnSetMsg(&e_msg, comhdr_o) → 결과 SEND (content->sndl)

W1358S02_F1()                   ← 달러대가통화 조회
  ├─ getmid/getmod, sndl += W1358S02_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])   (한 전문에 담을 최대 건수)
  ├─ GRID key 처리 (content->keyc)  K_NEXT→nkey / K_PREV→pkey / default→"0"
  │      → vKeyv 세팅 후 l_rtrim  (계산만 하고 SQL 엔 미사용 — §6 dead code)
  ├─ 입력 바인딩: yprintf + l_rtrim → vApsrYmd, vInqGb  (NULL/공백 실검증 없음 — §6)
  ├─ [4-1] 유효 적용시작일 vIngYmd 산정 (TB_FXB_CMG039M)
  │      vInqGb=='1' → SELECT MAX(APSR_YMD) WHERE APSR_YMD<=:apsr AND FNAP_YMD>=:apsr
  │      그외        → SELECT MAX(APSR_YMD) WHERE APSR_YMD = :apsr
  │      NO_DATA_FOUND → 무시(vIngYmd 공백 유지) / 기타 SQL err → e_msg "10118", RET_FAILURE
  │      → l_trim(vIngYmd)
  ├─ [4-2] DECLARE W1358S02_C1 CURSOR (CMG002M×CMG004M×CMG039M(+)×CMC003D1(+))
  │      OPEN → NO_DATA_FOUND: e_msg "91001", RET_FAILURE / 기타: "10118", RET_FAILURE
  ├─ FETCH 루프 (ii=0..)
  │      NO_DATA_FOUND → break / 기타 err → CLOSE, "10118", RET_FAILURE
  │      ii==max_row → i_isnext++, break (다음 페이지 있음 표시)
  │      zsedit → mod->orec[ii].*   (8필드)
  │      NEXT KEY: sprintf(vKeyv,"%.64s",cKeyv)  (cKeyv 미할당 — 항상 공백, §6)
  ├─ CLOSE W1358S02_C1
  ├─ 결과 코드 세팅
  │      ii==0 && i_isnext==0 → "91001" (해당 자료 없음)
  │      i_isnext==1          → content_set_nkey(vKeyv), "91002" (조회 계속 — vKeyv 공백, §6)
  │      그외                 → "91003" (조회 완료)
  │      strlen(vIngYmd)==0   → "10796" (등록된 거래가능시간 없음, 위 코드 덮어씀 §6)
  ├─ zredit(mod->grid01_cnt, ii)
  └─ content->sndl += (ii-1) * sizeof(orec[0]) → RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **2단계 조회**: ① 기준일(`apsrYmd`)·조회구분(`inqGb`)으로 고객거래가능시간기본에서 유효 적용시작일 `vIngYmd` 를 먼저 확정 → ② 메인 커서가 `CMG039M.APSR_YMD(+)=:vIngYmd` 로 outer join 해 해당 적용일자의 거래가능시간을 붙인다.
- **inqGb 분기**: `'1'` = 기준일이 적용기간에 포함되는(`APSR_YMD ≤ 기준일 ≤ FNAP_YMD`) 최신 적용시작일. 그 외 = 기준일과 정확히 같은 적용시작일. 둘 다 `MAX(APSR_YMD)` (단건).
- **대가통화 필터**: `CMG004M.COCU_CD <> 'KRW'` → 원화 대가가 아닌 통화페어(= "달러대가통화")만. 서비스명이 "달러대가통화" 이나 코드 필터는 "비KRW 대가" 전체(§6).
- **상품 필터**: `FDM_ID='FRX'`(NHB House 고객) AND `FX_PDCD='SPT'` AND `USE_YN='Y'` AND `TRN_ABL_YN='Y'`.
- **정렬**: `CMG004M.RPSN_SQC`(통화Pair 순서), `CMC003D1.CMCD_VL_REF_CON2`(TENOR 순서).
- **outer join 특성**: `CMG039M`(거래가능시간)·`CMC003D1`(TENOR) 이 outer 이므로 상품은 있으나 거래가능시간 미등록이면 시각 컬럼이 공백으로 반환될 수 있다.
- **페이징(그리드)**: `content->keyc`(NEXT/PREV/최초)로 `vKeyv` 를 만들지만 커서 SQL 에 바인딩되지 않고, `cKeyv` 도 끝까지 미할당이라 NEXT KEY 가 항상 공백 → **keyset 페이징 미동작**(§6). `ii==max_row` 시 `i_isnext`(다음 있음)만 세팅.
- **전송 길이**: 진입에서 `sndl += O_SZ`(orec 1개 포함) 후 종료에서 `sndl += (ii-1)*sizeof(orec)` → 총 orec `ii` 개. `ii==0` 이면 `-1*size` 로 orec 영역이 상쇄되어 헤더(`grid01_cnt`)만 전송.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드 (`e_msg.code` → `fnSetMsg` → COMHDR)**:
  - `91003` 조회 완료 (정상, 데이터 있고 다음 페이지 없음)
  - `91002` 조회 계속됨 (`i_isnext==1`, NEXT KEY 세팅 — 단 공백)
  - `91001` 해당 자료 없음 (커서 OPEN NO_DATA 또는 `ii==0 && i_isnext==0`)
  - `10796` 등록된 거래가능시간 없음 (`vIngYmd` 공백 시 — 위 91xxx 를 덮어씀)
  - `10118` 자료 조회중 오류 (사전 SELECT / 커서 OPEN·FETCH 의 예상외 SQLCODE)
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (eflg 미세팅)
- **결과 전달**: 그리드(`grid01_cnt`+`orec[]`) + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
