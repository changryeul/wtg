## 2. I/O 전문 (`.h`)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|

## 3. 구현 흐름 (`.pc`)

```
W1119S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  │                             comhdr_o->rtyp = "M"
  ├─ switch(mid->prGb)
  │     '1' → retc = W1119S01_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)

W1119S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1119S01_O_SZ, memset(mod)
  ├─ GRID key 처리 (content->keyc)  K_NEXT→nkey / K_PREV→pkey / default→"0"
  │      → vKeyv 세팅 후 l_rtrim  (계산만 하고 실제 SQL 엔 미사용 — §6 dead code)
  ├─ 입력 바인딩: yprintf + l_rtrim → vWbcsRlnmAltrNo  (유효성 실검증 없음 — §6)
  ├─ #if 0 fnGetCusClcd 인터페이스 조회 로직 (비활성 — 테이블 조회로 대체)
  ├─ CUST.wbcsRlnmAltrNo = vWbcsRlnmAltrNo
  │  retc = fnCustInfoQry(&CUST, &e_msg)   ← 고객약정정보기본(TB_FXB_CSC001M) 단건 조회
  │      실패 → RET_FAILURE (e_msg 는 fnCustInfoQry 내부에서 set: 10177/10165)
  │  → cWbcsRlnmAltrNo / cEdpsCsn(%.0lf) / cCusClcd 추출
  ├─ EXEC SQL SELECT CMCD_VL_NM INTO :cCusClcdNm
  │           FROM TB_FXB_CMC003D
  │           WHERE COM_CD_NM='CUS_CLCD' AND CMCD_VL=:cCusClcd
  │      NO_DATA_FOUND → e_msg "91001", eflg "0"(!), RET_FAILURE
  │      기타 SQL err  → e_msg "10118", eflg "1",  RET_FAILURE
  ├─ mod->{wbcsRlnmAltrNo,edpsCsn,cusClcd,cusClcdNm} 세팅
  └─ 정상 → eflg "0", e_msg "91003"(조회 완료), RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |

| `TB_FXB_CSC001M` | 고객약정정보기본 | — | **SELECT** (간접) | `fnCustInfoQry()`(WCM101.pc) 내부에서 `WBCS_RLNM_ALTR_NO` 로 고객정보(전산고객번호/고객분류코드 등) 단건 조회 |

*DML 요약: 본 `.pc` 직접 SELECT 1문(`TB_FXB_CMC003D`) + 공통함수 `fnCustInfoQry` 경유 SELECT 1문(`TB_FXB_CSC001M`). INSERT/UPDATE/DELETE/커서 없음.*

## 5. 로직 / 필터 규칙

- **DML**: SELECT 2건(1건은 공통함수 경유). 조인/커서 없음. 실제 데이터 변경 없음이나 규약상 성공 시 `l_dbcommit`.
- **조회 체인**: ① `fnCustInfoQry(전행고객번호)` → 고객약정정보기본에서 전산고객번호·고객분류코드 확보 → ② 그 고객분류코드로 코드정보상세에서 분류코드명 확보.
- **입력 유효성**: 3절 주석은 "입력항목 유효성 check" 이나 실제 `wbcsRlnmAltrNo` 를 `yprintf/l_rtrim` 만 하고 NULL/공백 거부 로직 없음(§6). 빈 키는 `fnCustInfoQry` 의 `NO_DATA_FOUND`(10177) 로 걸러짐.
- **GRID 키 처리**: `content->keyc`(NEXT/PREV/최초)로 `vKeyv` 를 만들지만 SELECT 에 바인딩되지 않는다 → 실질 단건 조회, 페이징 미동작(§6).
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드**:
  - 성공: `91003` "조회가 완료되었습니다.", `eflg=0`.
  - 분류코드명 없음: `91001`, `eflg=0`(정상 플래그인데 RET_FAILURE→rollback — §6), fnSetMsg 후 실패.
  - 분류코드명 SELECT 오류: `10118`, `eflg=1`.
  - 고객정보 없음/오류: `fnCustInfoQry` 내부 `10177`/`10165` (본 `.pc` 는 eflg 미세팅 후 즉시 RET_FAILURE 반환 — §6).
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (eflg 미세팅).
- **결과 전달**: 출력 레코드(4필드) + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
