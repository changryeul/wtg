# 서비스 명세 — `W1509S01` (고객정보 조회 / 헤더상 "상품설명서기초정보 조회")

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며,
> `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|---|---|

| 유형 | **조회 (SELECT)** — 단건 고객정보 조회, 데이터 변경 없음 |

| I/O 헤더 | `win/src/inc/trn/W1509S01.h` |
| 라우팅 alias | `W1509S01` → `POST /v1/tx {"alias":"W1509S01","data":{…}}` |
| 채널 | 진입부에 `ctyp→chnl` 매핑 없음 (COMHDR 그대로 사용) |

## 2. I/O 전문 (`.h`)

## 3. 구현 흐름 (`.pc`)

```
W1509S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              pname=rkey, ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ retc = W1509S01_F1(...)     ← 실제 로직 (작업구분 분기 없음, 조회 전용)
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)

W1509S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1509S01_O_SZ, memset(mod)
  ├─ comhdr_o->rtyp = "G"        (main 의 e_msg.rtyp="M" 와 별개로 여기서 "G"로 재설정 — §6)
  ├─ 입력 바인딩: yprintf(iCustId, mid->CustId) + l_rtrim
  ├─ 유효성: iCustId[0]==0x00 → e_msg "10201"(고객번호 입력), eflg "1", RET_FAILURE
  │
  ├─ ① SELECT WBCS_RLNM_ALTR_NO INTO :oWbcsRlnmAltrNo
  │       FROM TB_FXB_CSC005R WHERE FX_USER_NO = :iCustId     ← 사용자별고객번호관계
  │       NO_DATA_FOUND → e_msg "31008"(정보 없음), eflg "1", RET_FAILURE
  │       기타 SQL err  → e_msg "10118"(조회중 오류),  eflg "1", RET_FAILURE
  │   → l_rtrim(oWbcsRlnmAltrNo)
  │
  ├─ ② SELECT A.WBCS_RLNM_ALTR_NO, A.CSM, A.CIF_DCD, A.CIF_NO, A.MNGM_BRCD,
  │           B.MNGM_BRM, A.EDPS_CSN
  │       INTO :oWbcsRlnmAltrNo,:oCsm,:oCifDcd,:oCifNo,:oMngmBrcd,:oMngmBrm,:oEdpsCsn
  │       FROM TB_FXB_CSC001M A, TB_FXB_CMC001F B                ← 고객약정정보기본 + 부점정보
  │       WHERE A.MNGM_BRCD = B.MNGM_BRCD(+)                     (부점정보 outer join)
  │         AND A.WBCS_RLNM_ALTR_NO = :oWbcsRlnmAltrNo
  │       ORDER BY A.WBCS_RLNM_ALTR_NO                           (단건 equality 조회에 무의미 — §6)
  │       NO_DATA_FOUND → e_msg "31008", RET_FAILURE  (eflg 미세팅 — §6)
  │       기타 SQL err  → e_msg "10118", RET_FAILURE  (eflg 미세팅 — §6)
  │
  ├─ mod 채움: WbcsRlnmAltrNo/Csm/CifDcd/CifNo/MngmBrcd/MngmBrm (yprintfrtrim)
  │            oEdpsCsn 은 조회하나 mod 로 미전달 (주석 처리 — §6)
  ├─ /* (FXON)전자약정 고객정보조회 fnGetElecContrCustInfoChk 블록 전체 비활성(#주석) */
  └─ 정상 → eflg "0", e_msg "00000", comhdr_o->rcod=COD_SUCCESS, RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CSC005R` | 사용자별고객번호관계 | — | **SELECT** | `FX_USER_NO = :iCustId` 로 전행고객실명대체번호(`WBCS_RLNM_ALTR_NO`) 단건 확보 (①) |
| `TB_FXB_CSC001M` | 고객약정정보기본 | `A` | **SELECT** | `WBCS_RLNM_ALTR_NO` 로 고객명/CIF구분/CIF번호/관리부점/전산고객번호 조회 (②) |
| `TB_FXB_CMC001F` | 부점정보 | `B` | **SELECT** (outer join) | `A.MNGM_BRCD = B.MNGM_BRCD(+)` 로 부점명(`MNGM_BRM`) 조회 (②) |

*DML 요약: 직접 SELECT 2문(② 는 2테이블 outer join). INSERT/UPDATE/DELETE·커서 없음. 공통함수 경유 간접 테이블 접근 없음(전자약정 조회 `fnGetElecContrCustInfoChk` 는 주석 처리). 데이터 변경 없으나 규약상 성공 시 `l_dbcommit`.*

## 5. 로직 / 필터 규칙

- **조회 체인**: ① `FX_USER_NO`(입력 CustId) → 사용자별고객번호관계에서 전행고객실명대체번호 확보 → ② 그 실명대체번호로 고객약정정보기본 + 부점정보(outer join)에서 고객 기본정보 확보.
- **입력 유효성**: `CustId` 공백/NULL 이면 `10201` 로 즉시 거부. 그 외 검증 없음.
- **작업구분 분기 없음**: `prGb`/`switch` 없음 — 항상 조회 경로만 수행(순수 SELECT 서비스).
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드**:
  - 성공: `e_msg "00000"`, `eflg="0"`, `comhdr_o->rcod=COD_SUCCESS`.
  - 고객번호 미입력: `10201`, `eflg="1"`, RET_FAILURE.
  - ① 사용자별고객번호관계 없음: `31008`, `eflg="1"`, RET_FAILURE.
  - ② 고객약정정보기본 없음: `31008`, RET_FAILURE (eflg 미세팅 — §6).
  - SQL 오류(①/②): `10118` (①은 eflg="1", ②는 eflg 미세팅 — §6).
- **비활성 로직**: (FXON)전자약정 고객정보조회(`fnGetElecContrCustInfoChk`, `EAI_FXBO00081322_ST`)를 통한 실명번호/한글고객명/주소/투자성향 등 확장 필드 채움이 통째로 주석 처리 → 현재 산출 전문은 6개 필드만.
- **결과 전달**: 출력 레코드(6필드) + COMHDR(`eflg`/`rcod`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
