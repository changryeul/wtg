# 서비스 명세 — `W1101S02` (공인인증서 로그인 인증처리)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석 및 코드 내 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1101S02` |
| 서비스 이름 | 공인인증서 로그인 인증처리 (금융결제원 RA 인증 + 사용자 등록/갱신) |
| 유형 | **복합** — 인증서 검증(외부 RA 조회) + 고객 약정 조회 + 사용자정보/고객번호관계 UPDATE/INSERT |
| 구현 | `win/src/trn/W1101/W1101S02.pc` (892 lines) |
| I/O 헤더 | `win/src/inc/trn/W1101S02.h` |
| 라우팅 alias | `W1101S02` |
| 채널 | 매매 채널 (mymqd broker, `content->rkey` 기반 AP 콜백) |

W1101 모듈의 공인인증서 로그인 처리. 입력으로 받은 인증서명(signMsg)을 외부 RA 서버로 검증(`CertProcess`)해 CIF/USER_ID 를 추출하고, 고객 약정(`TB_FXB_CSC001M`)을 확인한 뒤 사용자정보기본(`TB_FXB_CSC004M`)과 사용자별고객번호관계(`TB_FXB_CSC005R`)에 로그인 흔적을 UPDATE/INSERT 한다. 즉 헤더가 "조회(prGb=1)"로 표기하지만 실제로는 DML 을 수행하는 등록/갱신 트랜잭션이다(§6).

## 2. I/O 전문 (`.h`)

### Input `W1101S02_I`

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 작업구분 (헤더상 "1 조회") — **코드에서 읽지 않음/미분기** (§6) |
| `signMsg` | 10240 | 인증서명(서명 데이터). `CertProcess` 입력으로 그대로 전달 |

`W1101S02_I_SZ = sizeof(W1101S02_I)` = 10241 바이트.

### Output `W1101S02_O`

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `cifNo` | 20 | 실명번호 — `cert_out->cifNo` 복사 |
| `lgnId` | 20 | 로그인ID — `cert_out->user_id` 복사 |
| `otinYn` | 1 | 타기관발급여부. `ra_flag=='1'`(자행) → `'N'`, 그 외 → `'Y'` |
| `svrCert` | 5120 | 서버key — **실제로는 공백(0x20)으로만 채워지고 값이 세팅되지 않음** (§6) |

단건(고정) 출력. grid/orec 가변부 없음.

## 3. 구현 흐름 (`.pc`)

```
W1101S02(mymq, content, recv_b, recv_l)                         // 콜백 진입
  ├ CertInit()                                                  // 인증 LIB 초기화
  ├ ContKey = content->rkey                                     // cert.c 용 전역
  ├ prthdr / getmid / getmod                                   // COMHDR·입력·출력 주소 획득
  ├ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp="M"
  ├ retc = W1101S02_F1(...)                                     // 실제 처리
  │    ├ getmid/getmod, content->sndl += W1101S02_O_SZ (①)      // ← 여기서 1차 가산
  │    ├ memset(mod), memset(cert_out)
  │    ├ memcpy(sign_data, mid->signMsg); sign_len=strlen(...)
  │    ├ CertProcess(sign_data, sign_len, cert_out)             // 외부 RA 서버 인증서 검증
  │    │    └ rtn<0 → (rtn<=-100: e_msg.code="88888" RA오류 / else "10808") → RET_FAILURE
  │    ├ cifNo=cert_out->cifNo, lgnId=cert_out->user_id (rtrim)
  │    ├ mod->cifNo/lgnId 세팅, otinYn = ra_flag=='1'?'N':'Y'
  │    ├ memset(mod->svrCert, 0x20, ...)                        // 공백 채움만
  │    ├ CheckUserTable(mymq, content, cifNo, lgnId)            // ↓ 약정확인 + DML
  │    │    ├ SELECT TB_FXB_CSC001M (약정확인, FX_CUS_STTS_DCD IN '1')
  │    │    │    └ NO_DATA/error → e_msg.code="10165" RET_FAILURE
  │    │    ├ SELECT COUNT TB_FXB_CSC004M → cExistYn (미사용, §6)
  │    │    ├ Update_CSC004M(...)                               // UPDATE→(NO_DATA)INSERT
  │    │    ├ Update_CSC005R(...)                               // INSERT (UPDATE 블록 #if 0)
  │    │    └ l_dbcommit() (②)                                  // 내부 커밋
  │    │        └ rtn!=SUCCESS → e_msg.code="10165" RET_FAILURE
  │    ├ comhdr_o->eflg="0", e_msg.code="91003"(정상)
  │    └ content->sndl += W1101S02_O_SZ (③)  → RET_SUCCESS      // ← 2차 가산(중복, §6)
  ├ retc==SUCCESS → eflg="0", l_dbcommit()  (④)                 // 외부 커밋(내부와 이중)
  │  else          → eflg="1", l_dbrollback()
  ├ fnSetMsg(&e_msg, comhdr_o)                                  // e_msg → 응답 헤더
  ├ CertEnd()
  └ return content->sndl
```

보조 함수:
- `Update_CSC004M` : `TB_FXB_CSC004M` UPDATE 성공 시 return, `NO_DATA_FOUND` 이면 INSERT. (INSERT 성공 경로 return 누락 — §6)
- `Update_CSC005R` : `TB_FXB_CSC005R` INSERT (UPDATE 경로는 `#if 0` 로 비활성). `SQLCODE==-1`(중복)은 무시하고 성공 처리.
- `OID_Check`, `CSC001M_Print`, `CSC004M_Print` : 정의만 있고 **호출 없음(dead code)** — §6.

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-----|-----|-----|-----|-----|
| `TB_FXB_CSC001M` | 고객기본(FX고객) | `csc001m` | **SELECT** | 약정 확인 — `WBCS_RLNM_ALTR_NO = :cid AND FX_CUS_STTS_DCD IN ('1')`, 42개 컬럼 단건 조회 |
| `TB_FXB_CSC004M` | 사용자정보기본 | — | **SELECT / UPDATE / INSERT** | ①존재확인 COUNT ②로그인 상태 UPDATE(USER_STTS_DCD/변경ID/변경일시) ③없으면 신규 INSERT |
| `TB_FXB_CSC005R` | 사용자별고객번호관계(사용자고객번호등록) | — | **INSERT** | 사용자–고객번호 매핑 등록 (중복 SQLCODE=-1 무시). UPDATE 는 `#if 0` |

간접 접근: `CertProcess`(cert.c)는 DB 가 아닌 외부 RA 서버 조회(인증서 검증)로, 테이블 접근 아님.

*DML 요약:*
- SELECT ×2 — `TB_FXB_CSC001M`(약정), `TB_FXB_CSC004M`(존재 COUNT)
- UPDATE ×1 — `TB_FXB_CSC004M`(상태/최종변경)
- INSERT ×2 — `TB_FXB_CSC004M`(UPDATE 대상 없을 때), `TB_FXB_CSC005R`(사용자고객번호)
- COMMIT — `CheckUserTable` 내부 1회 + `W1101S02` dispatch 성공 시 1회 (이중, §6). 실패 시 `l_dbrollback`.

## 5. 로직 / 필터 규칙

- **인증서 검증**: `CertProcess` 리턴 < 0 이면 실패. `rtn <= -100` 은 RA 서버 오류로 `e_msg.code="88888"` + `result/msg` 를 조립해 mesg 에 세팅, 그 외 음수는 `"10808"`(잠시 후 재시도).
- **약정 필터**: `TB_FXB_CSC001M WHERE WBCS_RLNM_ALTR_NO = :cid AND FX_CUS_STTS_DCD IN ('1')`. `NO_DATA_FOUND` 또는 SQL 오류 → `e_msg.code="10165"`(약정없음) → RET_FAILURE.
- **타기관 여부**: `cert_out->ra_flag[0]=='1'`(자행/금결원 발급)이면 `otinYn='N'`, 아니면 `'Y'`(타기관).
- **사용자 등록/갱신 (upsert)**:
  - `Update_CSC004M` : 활성(`#else`) 경로는 UPDATE 시 `USER_STTS_DCD='1'`, `FX_SYS_LSMD_ID='W1101S02'`, `FX_SYS_LSMD_TS=SYSDATE` 3개만 갱신. `NO_DATA_FOUND` 이면 14개 컬럼 신규 INSERT(`FX_USER_DCD='TR'`, `SCRE_ACS_ATHR_DCD='XX'`, `USER_LVL_DCD='XX'`, `SECU_MDIA_DCD='0'`, `USER_STTS_DCD='1'` 등 하드코딩).
  - `Update_CSC005R` : INSERT 만 수행, 중복(`SQLCODE=-1`)은 정상 처리.
- **트랜잭션**: 정상 종료 시 `comhdr_o->eflg="0"` + COMMIT, 실패 시 `eflg="1"` + ROLLBACK. `fnSetMsg` 로 `e_msg`(code/mtyp/mesg) 를 응답 COMHDR 에 반영.
- **에러 코드 요약**: `91003`(정상), `10165`(약정없음/조회오류), `10808`(인증서 확인 실패-재시도), `88888`(RA 서버 오류, mesg 동적 조립). 주석처리된 `10118/10422/10574/10575/10576` 은 미사용.

| 위치 | 헤더/주석 표기 | 실제(.pc) |
|-----|-----|-----|
| 서비스 성격 (`W1101S02_F1` 함수명 "조회", 입력 `prGb` "1 조회") | 조회성 서비스로 표기 | 실제로는 `TB_FXB_CSC004M` UPDATE/INSERT + `TB_FXB_CSC005R` INSERT 수행하는 **등록/갱신(복합)** 트랜잭션 |

| `.h` 파일 Description | "공인인증검증처리" | `.pc` 헤더는 "공인인증서로그인 인증처리" — 명칭 상이(둘 다 인증서 로그인) |

| `Update_CSC004M` INSERT 경로 (line 663~676) | — | INSERT 성공 후 **명시적 `return` 누락** — 비-void 함수가 반환값 없이 종료(UB). 호출측 `if(rtn!=0)` 검증이 쓰레기값에 의존 |

| `Update_CSC005R` UPDATE 블록 | 사용자별고객번호 UPDATE 존재 | `#if 0` 로 비활성 — INSERT-only, 중복(-1) 무시 |
