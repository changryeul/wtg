## 3. 구현 흐름 (`.pc`)

```
W1701S02()                        ← 진입 (콜백)
  ├─ getmid / getmod             COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               pname=rkey, ltyp=comhdr_i->ltyp, mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(comhdr_i->ctyp[0])   HTS/MTS/WTS/EMP/API/SPI → comhdr_i->chnl = ctyp (default: 로그만)
  ├─ retc = W1701S02_F1(...)      ← 실제 조회 로직
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)
     └─ fnSetMsg(&e_msg, comhdr_o) → return content->sndl

W1701S02_F1()                     ← 조회
  ├─ getmid/getmod, sndl += W1701S02_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ comhdr_o->rtyp = "G"         (해외선물 메시지코드 구분 — main 의 e_msg.rtyp="M" 과 별개, §6)
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT → content_get_nkey → v_keyv
  │      K_PREV → content_get_pkey → v_keyv
  │      default(최초) → v_keyv="0"      후 l_rtrim
  ├─ 입력 바인딩: yprintf + l_rtrim → ifXuSerDcd/ilRdvMenuId/imDdvMenuId/isMdvMenuId
  │      (주석은 "입력항목 유효성 check" 이나 실제 NULL/공백 거부 없음 — §6)
  ├─ DECLARE W1701S02_C1 CURSOR   TB_FXB_CMC011M(CMC011M) + TB_FXB_CMC010M 스칼라 서브쿼리 ×4
  │      WHERE LRDV_MENU_ID LIKE '%'||:il... AND NVL(FX_USER_DCD,' ') LIKE '%'||:if...
  │            AND MDDV/SMDV (값 있을때만 정확일치) AND MENU_ID > :v_keyv
  │      ORDER BY MENU_ID
  ├─ OPEN C1  (실패 → e_msg "95000", RET_FAILURE)
  ├─ for(ii=0; ; ii++)
  │      FETCH C1 INTO :o... (24 컬럼)
  │      SQLCODE==NO_DATA_FOUND → break
  │      ii==max_row → i_isnext++, break  (한 행 더 읽었지만 저장 안함 = 다음페이지 존재 신호)
  │      zsedit(mod->orec[ii].*, o...)     24개 필드 저장
  │      v_keyv = MENU_ID(6)               NEXT_KEY 갱신
  ├─ CLOSE C1
  ├─ 결과 판정
  │      ii==0 && i_isnext==0 → e_msg "91001" (해당 자료 없음)
  │      i_isnext==1          → content_set_nkey(v_keyv), e_msg "53102"(내역 남음), eflg="0"
  │      그외                 → e_msg "91003"(정상 조회), eflg="0"
  ├─ zredit(mod->grid01_cnt, ii)
  └─ content->sndl += (ii-1) * sizeof(orec[0]) ; return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC011M` | 권한별 메뉴상세 (메뉴 구성) | `CMC011M` | **SELECT** | 커서 `W1701S02_C1` 의 FROM 본체 — 권한그룹/대·중·소분류·화면·프로그램 속성 조회, `MENU_ID > :v_keyv` 페이징 |
| `TB_FXB_CMC010M` | 메뉴(기본) | — | **SELECT** (스칼라 서브쿼리 ×4) | 각 `MENU_ID` / `LRDV`/`MDDV`/`SMDV_MENU_ID` 에 대응하는 `MENU_NM`(메뉴명/대·중·소분류명) 조회 |

*DML 요약: 조회 전용. 커서 1개(`W1701S02_C1`) — 본체 `TB_FXB_CMC011M` 1건 + 스칼라 서브쿼리로 `TB_FXB_CMC010M` 4회 참조. INSERT/UPDATE/DELETE 없음. 규약상 성공 시 `l_dbcommit`.*

## 5. 로직 / 필터 규칙

- **DML**: SELECT(커서) 1건. 데이터 변경 없음이나 규약상 `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`.
- **동적 필터** (WHERE):
  - `LRDV_MENU_ID LIKE '%' || :ilRdvMenuId` — **suffix(접미) LIKE**. 공백 입력 시 `LIKE '%'` 로 전체 매칭.
  - `NVL(FX_USER_DCD,' ') LIKE '%' || :ifXuSerDcd` — 권한그룹 suffix LIKE. 공백 시 전체.
  - `MDDV_MENU_ID` : `imDdvMenuId` NULL/공백 → 조건 무시, 값 있으면 정확일치.
  - `SMDV_MENU_ID` : `isMdvMenuId` NULL/공백 → 조건 무시, 값 있으면 정확일치.
  - `MENU_ID > :v_keyv` : 페이징 커서 키 (최초 `"0"`, 이후 마지막 저장행 `MENU_ID`).
- **페이징**: `content->keyc`(K_NEXT/K_PREV/최초)로 `v_keyv` 결정 → `MENU_ID > :v_keyv` 로 다음 페이지. 루프에서 `max_row` 를 초과하는 한 행을 추가로 FETCH 하면 `i_isnext=1` → `content_set_nkey` + 코드 `53102`. (K_PREV 는 `content_get_pkey` 로 pkey 를 받지만, 정렬/조건은 NEXT 와 동일 `MENU_ID >` 이므로 역방향 페이징은 실질 미동작 — §6)
- **건수/길이**: `grid01_cnt = ii`(저장행수). `sndl += W1701S02_O_SZ` 로 1행 선반영 후 `(ii-1)*orec` 만큼 가산 → 총 `ii` 행 길이. `ii==0` 이면 `-1*orec` 로 헤더(grid01_cnt)만 남김.
- **트랜잭션**: 명시적 commit/rollback, AUTOCOMMIT 아님.
- **에러/결과 코드** (`e_msg.code` → `mesg`, `rtyp="M"`):
  - `91003` 정상 조회, `eflg="0"`.
  - `53102` 조회 내역 남음(다음 페이지 존재), `eflg="0"`, NEXT_KEY 세팅.
  - `91001` 해당 자료 없음 (`ii==0 && i_isnext==0`).
  - `95000` 커서 DECLARE/OPEN 오류 → `RET_FAILURE` → rollback.
- **결과 전달**: grid(`grid01_cnt`+`orec[]`) + COMHDR(`eflg`/`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200 + COMHDR 동봉(레거시 규약).

## 6. 주의 — 문서/코드 불일치 (마이그레이션 시 확인)

| 위치 | 헤더/주석 표기 | 실제(`.pc`) |
|---|---|---|
| `.pc` L34 변수 주석 | `mod` = "W1701S02 의 입력값 주소" | 출력값(Output) 주소 — 복붙 오기 |
| `.pc` L113 함수 주석 | `W1701S02_F1` = "나의 미래 예상현금흐름 입력-조회" | 실제는 메뉴상세조회 — 다른 서비스에서 복붙한 오기 |
| L199 유효성 섹션 주석 | "입력항목 유효성 check" | 실제 NULL/공백 검증 없음 — `yprintf/l_rtrim` 만 수행 (빈값은 LIKE '%' 로 전체매칭) |
| L261~262 LIKE 필터 | (대분류/권한 조건 매칭) | `'%' || :val` = **접미(suffix) LIKE**. 접두 부분매칭 의도였다면 `:val || '%'` 여야 함. 인덱스 미활용 + 의도치 않은 매칭 가능 |
| L268~273 DECLARE 후 SQLCODE 체크 | 커서 DECLARE 오류 처리 | DECLARE 는 precompiler 지시자로 SQLCODE 무의미 + `return` 도 주석처리 → dead code |
| L283~290 OPEN 후 `NO_DATA_FOUND` 분기 | OPEN 시 무자료 처리 | `NO_DATA_FOUND` 는 FETCH 에서만 발생 — OPEN 뒤 이 분기는 절대 진입 안됨 (dead code) |
| L346 fetch 로그 | `APLog(rkey, APLOG_DEBUG, fmt...)` 패턴 | 로그레벨 인자 누락 — `APLog(content->rkey, "[%-15s]...")` 로 포맷/가변인자 어긋남 (로그 깨질 수 있음) |
| K_PREV 페이징 | 이전 페이지 조회 | `content_get_pkey` 로 키만 받고 정렬·조건은 NEXT 와 동일(`MENU_ID > :v_keyv` / `ORDER BY MENU_ID`) → 역방향 페이징 실질 미동작 |
| `comhdr_o->rtyp` | main 에서 `e_msg.rtyp="M"` | F1 에서 `comhdr_o->rtyp="G"` 로 별도 세팅 — 출력 COMHDR rtyp 은 "G"(해외선물 코드 구분) |
| 미사용 지역변수 | — | `i,cnt,editb,ii(main),max_row 외 log_m,error_f,i_sndcnt,xval,retc(F1)` 등 다수 미사용 |

> WTG 는 `.h` 를 파싱해 필드명/크기만 취하고 전문을 통과시키므로 위 데드코드/주석 오기가
> 동작에 영향 없다. 단 suffix LIKE(L261~262)는 조회 결과에 실제 영향을 줄 수 있어
> 마이그레이션·재작성 시 접두/접미 매칭 의도를 반드시 확인해야 한다.
