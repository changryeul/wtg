## 3. 구현 흐름 (`.pc`)

```
W1105A01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  │                             comhdr_o->rtyp = "M"
  ├─ switch(mid->prGb)
  │     '1' → retc = W1105A01_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->mesg="작업구분 오류." 후 즉시 return
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)

W1105A01_F1()                   ← 펀드 선택 (함수 주석은 "조회"지만 실제 UPDATE)
  ├─ getmid/getmod, sndl += W1105A01_O_SZ, memset(mod)
  ├─ 입력 바인딩: yprintf + l_rtrim → vFxUserNo, vFndNo
  ├─ 유효성: isNullSpace(vFxUserNo)==0 → e_msg "10555", eflg "1", RET_FAILURE
  │          (2번째 체크도 vFxUserNo 재검사 — fndNo 미검증, §6 주의)
  ├─ EXEC SQL UPDATE TB_FXB_CSC011M SET LAST_TS=SYSDATE
  │           WHERE USE_YN='Y' AND FX_USER_NO=:vFxUserNo AND FND_NO=:vFndNo
  ├─ SQLCODE!=0 → e_msg "10174"(등록중 오류), eflg "1", RET_FAILURE
  └─ 정상 → eflg "0", e_msg "32156"(등록 완료), RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **DML**: 조건부 UPDATE 1문. 조인/커서 없음. 매칭 조건 = `USE_YN='Y' AND FX_USER_NO=:x AND FND_NO=:y`.
  - `LAST_TS=SYSDATE` 만 갱신 → "이 사용자가 이 펀드를 마지막으로 선택한 시각" 스탬프.
- **입력 유효성**: `fxUserNo` 공백/NULL 이면 `10555` 로 거부. (`fndNo` 는 §6 버그로 사실상 미검증)
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님 — 명시적 커밋 필수.
- **결과 전달**: 데이터 대신 COMHDR — `eflg`(0 정상/1 오류) + `e_msg.code`→`mesg`.
  - 성공: `32156` "등록 완료", 실패: `10555`(입력 확인) / `10174`(처리 오류).
  - → WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, `errn`/COMHDR 본문 동봉(레거시 규약).
