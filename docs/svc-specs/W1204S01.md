# 서비스 명세 — `W1204S01` (통화Pair 코드 목록 조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며,
> `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 2. I/O 전문 (`.h`)

**Input `W1204S01_I`** (총 201B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 처리구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" 후 즉시 return |
| `comCdNm` | 100 | 공통코드명 필터 — SQL `COM_CD_NM = NVL(TRIM(:vComCdNm), COM_CD_NM)` (빈값이면 전체) |
| `cmcdVl` | 100 | 공통코드값 필터 — SQL `CMCD_VL LIKE RTRIM(:vCmcdVl)||'%'` (전방일치 prefix) |

**Output `W1204S01_O`** — grid (가변 `orec[]`)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (`zredit(ii)` — 실제 채운 레코드 수) |
| `orec[].comCdNm` | 100 | 공통코드명 (`CMC003D.COM_CD_NM`) |
| `orec[].cmcdVl` | 100 | 공통코드값 (`CMC003D.CMCD_VL` = 통화페어ID) |
| `orec[].cmcdVlNm` | 200 | 공통코드값명 (`CMC003D.CMCD_VL_NM`) |
| `orec[].cmcdVlEnsnNm` | 100 | 공통코드값영문명 (`CMC003D.CMCD_VL_ENSN_NM`) |
| `orec[].cmcdVlRefCon1` | 32 | 공통코드값참조내용1 (`CMC003D.CMCD_VL_REF_CON1`) |
| `orec[].cmcdVlRefCon2` | 32 | 공통코드값참조내용2 (`…REF_CON2`) |
| `orec[].cmcdVlRefCon3` | 32 | 공통코드값참조내용3 (`…REF_CON3`) |
| `orec[].cmcdVlRefCon4` | 32 | 공통코드값참조내용4 (`…REF_CON4`) |
| `orec[].cmcdVlRefCon5` | 32 | 공통코드값참조내용5 (`…REF_CON5`) |

> `.h` 는 `orec[1]` 로 1건만 선언하나 실제로는 `content->sndl` 를 늘리며 최대 `MAX_RRB/sizeof(orec[0])` 건까지 append 하는 가변 grid. `orec[0]` 크기 = 660B (100+100+200+100+32×5). NEXT_KEY 는 COMHDR nkey(`content_set_nkey`)로 별도 전달, orec 필드엔 없음.

## 3. 구현 흐름 (`.pc`)

```
W1204S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1204S01_F1(...)     ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? l_dbcommit()+eflg"0" : l_dbrollback()+eflg"1"
     → fnSetMsg(&e_msg, comhdr_o)  (e_msg.code → mesg/rcod)  → return content->sndl

W1204S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1204S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])           ← 페이지당 최대 건수
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT → content_get_nkey → vKeyv
  │      K_PREV → content_get_pkey → vKeyv          (§6: PREV 도 '>' 조건 → 실질 forward only)
  │      default(최초) → vKeyv = "0"
  ├─ 입력 바인딩: yprintf + l_rtrim → vComCdNm / vCmcdVl  (유효성 실검증 없음 — §6)
  ├─ EXEC SQL DECLARE W1204S01_C1 CURSOR FOR
  │      SELECT … FROM TB_FXB_CMC003D, TB_FXB_CMG004M
  │      WHERE CMCD_VL = CRNC_PAIR_ID
  │        AND COM_CD_NM = NVL(TRIM(:vComCdNm), COM_CD_NM)
  │        AND CMCD_VL LIKE RTRIM(:vCmcdVl)||'%'
  │        AND LPAD(RPSN_SQC,5,0) > :vKeyv
  │      ORDER BY CMG004M.RPSN_SQC
  ├─ EXEC SQL OPEN C1
  │      SQLCODE≠SUCCESS: NO_DATA→e_msg"91001"+RET_FAILURE / 기타→"10118"+RET_FAILURE
  │                       (§6: OPEN 에 NO_DATA_FOUND 는 실질 미발생)
  ├─ for(ii=0;;ii++)
  │      FETCH C1 INTO :cComCdNm … :cCmcdVlRefCon5, :cKeyv
  │      SQLCODE: NO_DATA→break / 기타err→CLOSE+e_msg"10118"+RET_FAILURE
  │      if (ii == max_row) { i_isnext++; break; }   ← 1건 초과분 감지(미저장) = 다음 페이지 有
  │      l_rtrim(모든 컬럼)
  │      zsedit(mod->orec[ii].* = c*)                 ← 레코드 적재
  │      sprintf(vKeyv, "%.64s", cKeyv)               ← NEXT_KEY = 마지막 적재행 RPSN_SQC
  ├─ EXEC SQL CLOSE C1
  ├─ 결과판정
  │      ii==0 && i_isnext==0 → e_msg"91001"(자료없음)
  │      i_isnext==1          → content_set_nkey(vKeyv) + e_msg"91002"(조회 계속)
  │      그외                  → e_msg"91003"(조회 완료)
  ├─ zredit(mod->grid01_cnt, ii)
  ├─ content->sndl += (ii-1) * sizeof(orec[0])        ← §6: ii==0 시 언더플로우
  └─ return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC003D` | 코드정보상세 | `CMC003D` | **SELECT** (커서 `W1204S01_C1`) | 공통코드명/값/값명/영문명/참조내용1~5 조회. `COM_CD_NM` NVL 필터 + `CMCD_VL` prefix LIKE 필터 |
| `TB_FXB_CMG004M` | 통화페어기본 | `CMG004M` | **SELECT** (커서, 조인) | `CMCD_VL = CRNC_PAIR_ID` 조인으로 유효 통화페어 코드로 한정 + `RPSN_SQC`(표시순서)로 정렬/페이징 키 산출 |

*DML 요약: 커서 `W1204S01_C1` 의 2-테이블 조인 SELECT 1건 뿐. INSERT/UPDATE/DELETE 없음. 데이터 변경은 없으나 규약상 성공 시 `l_dbcommit`, 실패 시 `l_dbrollback`.*

## 5. 로직 / 필터 규칙

- **조인/필터**: `CMC003D.CMCD_VL = CMG004M.CRNC_PAIR_ID` 로 코드정보상세를 통화페어기본에 매칭 → 통화페어로 등록된 코드값만 결과에 포함. `COM_CD_NM = NVL(TRIM(:vComCdNm), COM_CD_NM)`(빈값이면 전체), `CMCD_VL LIKE RTRIM(:vCmcdVl)||'%'`(전방일치).
- **정렬/페이징 키**: `LPAD(CMG004M.RPSN_SQC,5,0)`(대표시퀀스 5자리 zero-pad). `ORDER BY RPSN_SQC`, 커서 조건 `LPAD(RPSN_SQC,5,0) > :vKeyv`. 최초 조회는 `vKeyv="0"`.
- **페이징 방식**: `max_row = MAX_RRB / sizeof(orec[0])` 만큼 적재하고, `ii==max_row` 시점에 1건을 추가 FETCH 해 보고 존재하면 저장하지 않고 `i_isnext=1` → "다음 있음" 판정. NEXT_KEY(=마지막 적재행 RPSN_SQC)를 `content_set_nkey` 로 세팅.
- **입력 유효성**: 3절 주석은 "입력항목 유효성 check" 이나 실제 `yprintf/l_rtrim` 만 수행, NULL/공백 거부 없음(§6).
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드 (e_msg.code → fnSetMsg)**:
  - `91001` — 자료없음 (OPEN NO_DATA, 또는 `ii==0 && i_isnext==0`).
  - `91002` — 조회가 계속됩니다 (`i_isnext==1`, NEXT_KEY 동봉).
  - `91003` — 조회가 완료되었습니다 (마지막 페이지).
  - `10118` — 커서 OPEN/FETCH SQL 오류 → RET_FAILURE → rollback, `eflg=1`.
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (eflg 미세팅).
- **결과 전달**: grid(`grid01_cnt` + `orec[ii]`) + COMHDR(`eflg`/`e_msg.code`→`mesg`/`rcod`) + NEXT_KEY(nkey). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).

> 노출되므로, 실제 의미(통화Pair 공통코드 목록/공통코드명)로 교정하는 편이 좋다.
