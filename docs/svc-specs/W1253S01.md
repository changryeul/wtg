## 2. I/O 전문 (`.h`)

**Input `W1253S01_I`** (총 4B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'`(조회) 만 처리, 그 외 "작업구분 오류" |

## 3. 구현 흐름 (`.pc`)

```
W1253S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' → retc = W1253S01_F1(...)     ← 실제 조회 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (MSG_NOWORK 로그; eflg 미설정)
  ├─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
  │                                     (조회이나 명시적 커밋, AUTOCOMMIT 아님)
  └─ fnSetMsg(&e_msg, comhdr_o)  e_msg.code → comhdr_o 메시지 반영 후 send

W1253S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1253S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])   페이지 크기 (orec 이 2364B 라 소수)
  ├─ GRID key 처리 (content->keyc)
  │      K_NEXT → content_get_nkey → vKeyv
  │      K_PREV → content_get_pkey → vKeyv   (커서는 항상 '>' ASC — PREV 미동작, §6)
  │      default(최초) → vKeyv = "0"
  │      l_rtrim(vKeyv)
  ├─ 입력 바인딩: yprintfrtrim → vFxMsgPtrnDcd / vUseYn
  │      atoi(vFxMsgPtrnDcd)>0 이면 "%02d" 재포맷 (예 "1"→"01")
  │      (필수 유효성 체크는 전부 주석처리 — 두 필터 모두 선택, §6)
  ├─ DECLARE W1253S01_C1 CURSOR ... OPEN
  │      SQLCODE != SUCCESS →
  │          NO_DATA_FOUND → e_msg "91001", RET_SUCCESS (실질 dead — OPEN 은 NDF 안냄, §6)
  │          기타          → e_msg "10118", RET_FAILURE
  ├─ for (ii=0; ; ii++)
  │      각 컬럼 host 변수 memset
  │      FETCH INTO ...
  │          NO_DATA_FOUND → break
  │          기타 err → CLOSE + e_msg "10118" + RET_FAILURE
  │      if (ii == max_row) { i_isnext++; break; }   ← 다음페이지 있음 표시
  │      mod->orec[ii] 각 필드 zsedit (본문만 sprintf)
  │      vKeyv = cKeyv (= FX_MSG_PTRN_DCD)           ← NEXT KEY 갱신
  ├─ CLOSE W1253S01_C1
  ├─ if (ii==0 && i_isnext==0) → e_msg "91001" (자료 없음)
  │  else if (i_isnext==1)     → content_set_nkey(vKeyv) + e_msg "91002" (계속)
  │  else                      → e_msg "91003" (완료) + grid01_cnt=ii
  └─ grid01_cnt = ii (재세팅)
     sndl += (ii-1) * sizeof(orec[0])                ← 페이지 길이 보정 (§6)
     return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMC004M` | SMS메시지설정기본 | `CMC004M` | **SELECT** (커서) | 메인 조회 대상. `FX_MSG_PTRN_DCD`/`USE_YN` 필터 + `FX_MSG_PTRN_DCD > :vKeyv` keyset 페이징, `ORDER BY FX_MSG_PTRN_DCD` |
| `TB_FXB_CMC003D` | 코드정보상세 | `CMC003D1` | **SELECT** (LEFT OUTER JOIN) | `SMSS_TGT_DCD = CMCD_VL AND COM_CD_NM='SMSS_TGT_DCD'` 로 대상구분코드명(`CMCD_VL_NM`) 조인 (`NVL` fallback = 코드값) |

*DML 요약: 커서 `W1253S01_C1` 1개(SELECT + LEFT OUTER JOIN)뿐. INSERT/UPDATE/DELETE 없음. 조회 전용이나 진입부에서 명시적 commit/rollback 수행.*

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='1'` (조회) 만 유효. 그 외는 COMHDR `mesg="작업구분 오류."` 로 즉시 반환.
- **필터 (모두 선택)**: `FX_MSG_PTRN_DCD = NVL(:vFxMsgPtrnDcd, FX_MSG_PTRN_DCD)`, `USE_YN = NVL(:vUseYn, USE_YN)`. 빈 문자열은 Oracle 에서 NULL 로 취급되어 `NVL` 이 전체조회로 동작. `fxMsgPtrnDcd` 는 숫자 입력 시 `%02d` 로 zero-pad 정규화.
- **페이징**: keyset 방식 — 정렬키 `FX_MSG_PTRN_DCD` 를 NEXT KEY(`vKeyv`) 로 사용, `> :vKeyv` 로 다음 페이지. 한 페이지 `max_row = MAX_RRB / sizeof(orec[0])`. `ii == max_row` 도달 시 그 행은 버리고 `i_isnext` 표시 후 break.
- **트랜잭션**: `W1253S01()` 이 `retc` 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` (조회지만 커서 정리를 위해 명시적).
- **에러/메시지 (e_msg.code)**:
  - `91001` — 자료 없음 (건수 0 & 다음페이지 없음) / OPEN NO_DATA_FOUND(사실상 미발생)
  - `91002` — 조회 계속 (다음 페이지 존재, NEXT KEY 설정)
  - `91003` — 조회 완료 (마지막 페이지)
  - `10118` — 커서 OPEN/FETCH SQL 오류 → RET_FAILURE
  - `10527`/`10577` — 입력 필수값 오류 코드가 코드상 존재하나 **주석처리되어 미사용**
- **eflg**: 진입부에서 `retc` 기준 "0"(성공)/"1"(실패) 세팅.

| 위치 | 헤더/주석 표기 | 실제(`.pc`) |

| `.pc` L337 zsedit 주석 | `smssTgtDcd` 세팅에 "발송매체구분코드" 주석 | 값은 `SMSS_TGT_DCD`(발송/수신대상구분코드). SQL 주석은 "수신대상코드"·헤더는 "발송대상구분코드" — 세 곳 명칭 불일치 (발송/수신/발송매체 혼용) |
| `.pc` L330-332 | 본문내용 세팅 | `fxMsgCon` 은 길이 2100 이라 `zsedit()` 미동작(먹통) → `sprintf("%s")` 로 세팅. NUL 종료 문자열 가정이라 CP949 바이너리에는 취약 |
| `.pc` L204-217 | 필수 입력값 유효성 검사(`10527`/`10577`) | 전부 주석처리 — `fxMsgPtrnDcd`/`useYn` 모두 선택 필터로만 동작 |

| `.pc` L381 vs L388 | grid 건수 세팅 | `zredit(grid01_cnt, ii)` 가 완료 분기(L381)와 무조건(L388) 두 번 수행 — 중복 (기능상 무해) |
| `.h` orec 주석 | `sndgMdiaDcd`/`sndgMdiaDcdNm`/`replyGroup` 필드 | 모두 주석처리되어 전문에 미포함 (`.pc` SELECT/FETCH 에서도 주석) — 실제 orec 은 위 §2 의 10개 필드만 |
