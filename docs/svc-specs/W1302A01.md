## 3. 구현 흐름 (`.pc`)

```
W1302A01()                      ← 진입 (콜백, 함수 주석은 "조회")
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '2' | '3' → retc = W1302A01_F1(...)     ← 실제 로직 (등록/수정 공용)
  │     그외       → comhdr_o->mesg="작업구분 오류." 후 즉시 return
  └─ retc==RET_SUCCESS ? l_dbcommit()+eflg"0" : l_dbrollback()+eflg"1"
       → fnSetMsg(&e_msg, comhdr_o) 로 COMHDR 응답 메시지 세팅   (AUTOCOMMIT 아님)

W1302A01_F1()                   ← 등록/수정 (함수 주석 "등록/수정")
  ├─ getmid/getmod, memset(mod)
  ├─ 입력 바인딩: yprintf + l_rtrim (문자형) / l_n2i·l_n2d (숫자형)
  │              vFxSysLsmdId ← comhdr_i->usid  (입력 fxSysLsmdId 아님)
  ├─ 유효성 검사 (실패 시 e_msg.code 세팅 후 RET_FAILURE):
  │     vCrncPairId  공백 → "10183"
  │     vBaseCrcd    공백 → "10184"
  │     vCocuCd      공백 → "10185"
  │     vPlCrncPairId 공백 → "10186"
  │     (vFxPrlocDcd "10187" 검사 → 전체 주석처리 dead code, §6)
  │     vDcpnPcsnDcd 공백 → "10188"
  │     vUseYn       공백 → "32517"
  │     vRpsnSqc == 0     → "10189"
  ├─ prGb=='3' → EXEC SQL UPDATE TB_FXB_CMG004M ... WHERE CRNC_PAIR_ID=:vCrncPairId
  │     SQLCODE==NO_DATA_FOUND → "10221"(수정할 데이터 없음)
  │     그외 SQLCODE!=0        → "10181"(수정 중 오류)  → RET_FAILURE
  ├─ prGb=='2' → EXEC SQL INSERT INTO TB_FXB_CMG004M ...
  │     SQLCODE==DUP_KEY → "10223"(중복)
  │     그외 SQLCODE!=0  → "10180"(등록 중 오류)  → RET_FAILURE
  └─ 정상 → e_msg.code "10150", RET_SUCCESS
```

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |

## 5. 로직 / 필터 규칙

- **작업 분기**: `W1302A01()` 의 `switch(mid->prGb[0])` 로 `'2'`(INSERT)/`'3'`(UPDATE) 만 `W1302A01_F1` 진입. 나머지는 즉시 "작업구분 오류" 반환.
- **입력 유효성**: `_F1` 앞단에서 필수 필드 공백 검사 — 통화페어ID/기준통화/상대통화/손익통화페어/소수점처리구분/사용여부 및 표시순서(≠0). 위반 시 각각 `10183~10189`, `32517` 코드로 `RET_FAILURE`.
  - `fxPrlocDcd`(손익계산방법) 검사는 소스에서 **전부 주석처리** 되어 실제로는 검증하지 않음.
- **DML 세부**:
  - UPDATE(prGb='3'): `BASE_CRCD/COCU_CD/PL_CRNC_PAIR_ID` 는 `TRIM()`, `FX_PCRR_UT/DCPN_PCSN_DCD/SPOT_DT_CDNC_VL/RPSN_SQC/USE_YN` 은 `NVL(...)` 기본값 적용, `FX_SYS_LSMD_ID=:usid`, `FX_SYS_LSMD_TS=SYSDATE`.
  - INSERT(prGb='2'): 동일 컬럼 세트 삽입. `USE_YN` 기본값이 UPDATE는 `'N'`, INSERT는 `'Y'` 로 서로 다름(§6).
- **트랜잭션**: `RET_SUCCESS→l_dbcommit()+eflg"0"` / 실패→`l_dbrollback()+eflg"1"`. AUTOCOMMIT 아님 — 명시적 커밋 필수.
- **결과 전달**: 데이터 대신 COMHDR — `eflg`(0 정상/1 오류) + `e_msg.code`→`fnSetMsg`→`mesg`.
  - 성공: `10150`, 실패: 입력검사 `10183~10189`/`32517`, UPDATE `10221`(데이터 없음)/`10181`, INSERT `10223`(중복)/`10180`.
  - → WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
