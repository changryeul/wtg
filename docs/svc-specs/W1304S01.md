# 서비스 명세 — `W1304S01` (표준상품정보 조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1304S01` |
| 서비스 이름 | 표준상품정보 조회 |
| 유형 | 조회 (SELECT 전용, 그리드 + NEXT/PREV 페이징) |
| 구현 | `win/src/trn/W1300/W1304S01.pc` (504 라인) |
| I/O 헤더 | `win/src/inc/trn/W1304S01.h` |
| 라우팅 alias | `W1304S01` |
| 채널 | 매매(web) — COMHDR 기반 전문 |

통화별 상품정보(상품구분 / 통화Pair / Tenor / Market Data source·symbol / 환율재정방식 / 사용여부 등)를 그리드로 조회한다. 작업구분(`prGb`)은 `1`(조회)만 지원.

## 2. I/O 전문 (`.h`)

### Input `W1304S01_I`

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 처리구분 — `.pc`는 `'1'`(조회)만 처리, 그 외는 "작업구분 오류." 반환 |
| `prdCd` | 3 | 상품구분코드 (MGN:FX margin / SPT:현물환 / FWD:선물환). `"ALL"` 입력 시 전체(필터 해제) |
| `pairTp` | 3 | 통화Pair 대가구분 — `LIKE '%'||pairTp||'%'` 부분일치. `USD` 지정 시 `%KRW` 로 끝나는 Pair 제외 |
| `pairId` | 7 | 통화Pair (예: USDKRW). 빈값이면 전체 |
| `tenorCd` | 10 | 테너기간코드 (TOD/TOM/SPT/S/N/W01/W02/M01/M02 등). 빈값이면 전체 |
| `useYn` | 1 | 사용여부 (Y/N). 빈값이면 전체 |

빈 문자열 필드는 `NVL(:v, 컬럼)` 로 처리되어 자동으로 전체 조회가 된다.

## 3. 구현 흐름 (`.pc`)

```
W1304S01()  [진입 콜백]
  ├─ prthdr / getmid / getmod  — COMHDR·입력·출력 주소 확보
  ├─ E_MSG 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp="M"
  ├─ switch(mid->prGb[0])
  │     case '1' → W1304S01_F1()   (조회)
  │     default  → comhdr_o->mesg="작업구분 오류." 후 즉시 return(sndl)
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()
        → 출력 로그 후 return(content->sndl)

W1304S01_F1()  [조회]
  ├─ getmid/getmod, sndl += O_SZ, mod 초기화
  ├─ max_row = MAX_RRB / sizeof(orec[0])   (한 응답 최대 건수)
  ├─ 페이징 키 결정 switch(content->keyc)
  │     K_NEXT → content_get_nkey → vKeyv
  │     K_PREV → content_get_pkey → vKeyv
  │     default(최초) → vKeyv = "0"
  ├─ 입력 필터 편집 (yprintf + l_rtrim): prdCd/pairTp/pairId/tenorCd/useYn
  │     prdCd=="ALL" → vPrdCd 비움 (전체)
  ├─ DECLARE ST_W1304S01_C1 CURSOR FOR (인라인뷰 W1304S01 + CMC003D_1~4 outer join)
  ├─ OPEN ST_W1304S01_C1
  │     NO_DATA_FOUND → e_msg 91001, eflg 0, return RET_FAILURE
  │     기타 오류      → e_msg 10118, eflg 1, return RET_FAILURE
  ├─ for(ii=0; ; ii++)  FETCH → 호스트변수
  │     NO_DATA_FOUND → break
  │     기타 오류      → CLOSE, e_msg 10118, return RET_FAILURE
  │     ii==max_row    → i_isnext++, break  (다음 페이지 존재 표시, 초과행 미저장)
  │     l_rtrim 후 zsedit 로 mod->orec[ii] 채움, vKeyv = cKeyv (NEXT KEY 갱신)
  ├─ CLOSE ST_W1304S01_C1
  ├─ 결과 분기
  │     ii==0 && i_isnext==0 → e_msg 91001 (자료없음)
  │     i_isnext==1          → content_set_nkey, e_msg 91002 (계속)
  │     그 외                → e_msg 91003 (완료), zredit grid01_cnt
  ├─ eflg="0", zredit(grid01_cnt, ii)   (분기와 무관하게 최종 재설정)
  └─ sndl += (ii-1) * sizeof(orec[0]); return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분 분기**: `switch(mid->prGb[0])` — `'1'`(조회)만 구현. 그 외는 커밋/롤백 없이 "작업구분 오류." 메시지 반환.
- **동적 필터** (인라인뷰 WHERE):
  - `FX_PDCD = NVL(:vPrdCd, FX_PDCD)`, `CRNC_PAIR_ID = NVL(:vPairId, ...)`, `TNR_ID = NVL(:vTenorCd, ...)`, `USE_YN = NVL(:vUseYn, ...)` — 빈값이면 전체.
  - `CRNC_PAIR_ID LIKE '%'||:vPairTp||'%'` + `vPairTp='USD'` 일 때 `NOT LIKE '%KRW'` (달러대가에서 원화대가 Pair 배제).
  - `prdCd="ALL"` 입력 시 vPrdCd 를 비워 상품 전체 조회.
  - 고정 조건: `CMC003D.COM_CD_NM='TNR_ID'`, `CMC003D.USE_YN='Y'`.
- **정렬**: `FX_PDCD DESC, CMG004M.RPSN_SQC, CMC003D.CMCD_VL_REF_CON2` — 동일 기준으로 `ROW_NUMBER()` 부여, 이 값이 페이징 키(ROW_NUM).
- **페이징**: 인라인뷰에서 `ROW_NUM > :vKeyv` 로 이전 페이지 이후만 조회. 최초는 vKeyv="0". FETCH 가 `max_row`(`MAX_RRB / orec 크기`) 째 도달하면 `i_isnext` 표시 후 break — 초과행은 저장 안 하고 마지막 저장행의 ROW_NUM 을 NEXT KEY(`content_set_nkey`)로 반환.
- **트랜잭션**: 조회만 하지만 진입 콜백이 성공 시 명시적 `l_dbcommit()`, 실패 시 `l_dbrollback()`.
- **에러/메시지 코드** (`e_msg.code` → `fnSetMsg`):
  - `10118` — 자료 조회중 오류 (SQL 오류), `eflg=1`.
  - `91001` — 해당 자료 없음, `eflg=0`.
  - `91002` — 조회 계속(다음 페이지 존재), `eflg=0`, NEXT KEY 세팅.
  - `91003` — 조회 완료, `eflg=0`.

## 6. 주의 — 문서/코드 불일치

| 위치 | 헤더/주석 표기 | 실제(.pc) |
|-----|--------------|-----------|
| `.pc:313` DECLARE CURSOR 직후 `if (SQLCODE != 0)` | 조회 오류 검사 의도 | DECLARE CURSOR 는 실행문이 아니라 SQLCODE 를 갱신하지 않는 선언문 — 이 검사는 사실상 dead check 이며, 참이 되어도 `return` 없이 그대로 OPEN 으로 진행 (무의미). |
| `.pc:469~500` 결과 분기 | 91001/91002/91003 별로 eflg·grid01_cnt 설정 | 분기 끝 `499~500` 에서 `eflg="0"` 와 `zredit(grid01_cnt, ii)` 를 무조건 재설정 → 91001(자료없음)·91002(계속) 분기의 eflg 및 91003 분기의 grid01_cnt 세팅이 최종적으로 덮어써짐(중복/무력화). 자료없음(91001)도 eflg 는 항상 0 으로 반환. |
| `.pc:492` vs `.pc:500` `zredit(grid01_cnt, ii)` | — | 91003 분기 내부(492)와 함수 말미(500) 두 번 호출 — 중복. |
| `.pc:502` `content->sndl += (ii-1) * sizeof(orec[0])` | — | `ii==0`(무건수) 시 `(0-1)`로 orec 한 건분을 빼는 음수 연산 idiom. 진입부(155)에서 이미 `O_SZ`(orec[1] 1건 포함)를 더했기에 결과적으로 헤더만 남아 정합하나, 의도가 불명확하고 부호에 취약. |
| `.h:41` 주석 `mod` 설명 | "서비스 코드 W1304S01 의 **입력값** 주소" | 실제 `mod` 는 출력값 주소 — 주석 복붙 오기 (`.pc:41` 동일 오기). |
| `.h:30` `pairTp` 주석 `(KRW, USD))` / `.pc:125` `vPairId` 주석 | `pairTp`=통화Pair구분, `vPairId`=통화Pair구분(KRW:원화대가...) | DECLARE 의 `vPairId` 주석이 `pairTp` 설명을 복붙 — 실제 `vPairId` 는 통화Pair(CRNC_PAIR_ID) 필터. 괄호 오타 `))` 포함. |
| `.h:47` `finishYmd` 주석 "Value Date" | Value Date | 실제 컬럼은 `EXPI_FNSH_YMD`(만기종료년월일) — 명칭 상 만기일이나 화면 라벨은 Value Date 로 사용. |
