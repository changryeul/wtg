## 2. I/O 전문 (`.h`)

**Input `W1101T01_I`** (`W1101T01_I_SZ` = 1 byte)

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 작업구분 (`1`=조회). **`.pc` 에서 읽지도 분기하지도 않음 — dead field** |

**Output `W1101T01_O`** (`W1101T01_O_SZ` = 5120 byte)

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `svrCert` | 5120 | 서버 key (서버 인증서). **`.pc` 에서 `memset(0)` 만 하고 값을 채우지 않음 — 항상 공백 반환** |

> 실제 인증 결과(usid 등)는 `svrCert` 가 아니라 `content->cookie`(cookie_t, `WITH_COOKIE`)로 전달된다. 출력 전문 본문에는 유효 데이터가 없다.

## 3. 구현 흐름 (`.pc`)

```
W1101T01(콜백)
  ├─ ContKey = content->rkey                         // cert.c 전역
  ├─ prthdr / getmid / getmod                        // 입출력 헤더·구조체 주소 획득
  ├─ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M), comhdr_o->rtyp="M"
  ├─ retc = W1101T01_F1(...)                          // 실제 작업 (아래)
  ├─ if retc == RET_SUCCESS → comhdr_o->eflg="0" + l_dbcommit()
  │  else                    → comhdr_o->eflg="1" + l_dbrollback()
  ├─ fnSetMsg(&e_msg, comhdr_o)                       // e_msg → 응답 메시지 세팅
  ├─ [DEV] 로그인 cookie 발급 블록:
  │     s_cookie(static) memset → comhdr_i->usid 를 공백/NUL 전까지 복사
  │     s_cookie.name="WTGDEV"
  │     content->cookie=&s_cookie; content->with |= WITH_COOKIE
  └─ return content->sndl

W1101T01_F1(조회 — 실제로는 무처리)
  ├─ getmid / getmod
  ├─ content->sndl += W1101T01_O_SZ                   // (1회차)
  ├─ memset(mod, 0, W1101T01_O_SZ)                    // svrCert 초기화만
  ├─ comhdr_o->eflg="0"
  ├─ e_msg.code="91003"                               // 무조건 세팅
  ├─ content->sndl += W1101T01_O_SZ                   // (2회차 — §6 참조)
  └─ return RET_SUCCESS                               // 항상 성공
```

`EXEC SQL` 문·커서·DML 이 **하나도 없다.** `l_dbcommit()` / `l_dbrollback()` 은 호출되나 실제 트랜잭션 대상 SQL 이 없어 no-op 에 가깝다.

## 5. 로직 / 필터 규칙
- **커서/DML**: 없음.
- **동적 필터**: 없음. `prGb` 작업구분 입력을 읽지 않으므로 조건 분기 자체가 없다.
- **트랜잭션**: `W1101T01_F1` 이 항상 `RET_SUCCESS` 를 반환 → 항상 `eflg="0"` + `l_dbcommit()`. rollback 경로는 사실상 도달 불가(dead branch).
- **에러(e_msg)**: `e_msg.code` 를 성공/실패와 무관하게 `"91003"` 로 고정 세팅 후 `fnSetMsg` 로 응답 메시지에 실음. 별도 오류코드 분기 없음.
- **cookie 발급(핵심 실동작)**: `comhdr_i->usid` 를 공백/NUL 이전까지 복사해 `cookie_t.usid` 구성, `name="WTGDEV"`, `WITH_COOKIE` 세팅. 개발 단계로 usid 를 무검증 신뢰(주석에 실계정 id/pw 검증 도입 시 인증 성공 분기로 이동하라 명시). WTG `mci-api` 가 이 cookie_t 로 세션/JWT 를 생성하는 경로의 진입점.
- **출력**: `svrCert` 는 0 으로만 초기화되어 항상 빈 값.
