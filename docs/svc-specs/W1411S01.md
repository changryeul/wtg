## 2. I/O 전문 (`.h`)

**Input `W1411S01_I`** (총 50B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류." |
| `wbcsRlnmAltrNo` | 16 | 전행고객실명대체번호 — 조회 키(고객 식별) |
| `fxUserNo` | 30 | FX사용자번호 — 커서 조인 조건에만 바인딩. 첫 쿼리는 `'ALL'` 강제 (데이터가 `ALL` 로만 관리됨, 화면도 `ALL` 만 내림) |
| `crcd` | 3 | 통화코드 — 선택적 필터. 커서에서 `NVL(RTRIM(:vCrcd), CMG008M.CRCD)` (빈값이면 전체 통화) |

## 3. 구현 흐름 (`.pc`)

```
W1411S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1411S01_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  ├─ retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()   (명시적 커밋)
  └─ fnSetMsg(&e_msg, comhdr_o) → COMHDR 에 결과코드/메시지 반영 후 sndl 반환

W1411S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1411S01_O_SZ, memset(mod), max_row = MAX_RRB/sizeof(orec[0])
  ├─ GRID key 처리 (content->keyc) K_NEXT→nkey / K_PREV→pkey / default→"0"
  │      → vKeyv 세팅 후 l_rtrim   (커서 WHERE 에 미바인딩 → 페이징 미동작, §6)
  ├─ 입력 바인딩: yprintfrtrim → vWbcsRlnmAltrNo / vFxUserNo / vCrcd (NULL/공백 검증 없음, §6)
  ├─ [SELECT 1] 고객약정 1일 USD 최대금액
  │      TB_FXB_TRC001M WHERE 고객=vWbcsRlnmAltrNo, FX_USER_NO='ALL', FX_PDCD='SPT', CRCD='USD'
  │      NO_DATA_FOUND → e_msg "10231"(현물환 1일 거래한도 미등록), RET_FAILURE
  │      기타 SQL err  → e_msg "10118", RET_FAILURE
  │      성공 → mod->dd1MxmmTrabAmt = "%.2f"
  ├─ [CURSOR W1411S01_C1] OPEN
  │      CMG008M(상품별거래한도) LEFT JOIN TRC001M(고객별거래한도) 을 BAR/SPT/FWD 3중 UNION ALL
  │      FX_CUS_LMT_DCD='1'(1회한도), CRCD=NVL(vCrcd, CMG008M.CRCD), GROUP BY CRCD, ORDER BY NEXT_KEY
  │      OPEN NO_DATA_FOUND → e_msg "91001", RET_SUCCESS / 기타 err → "10118", RET_FAILURE
  ├─ FETCH 루프 (ii)
  │     ├─ crcd != "USD" → [SELECT 2] 당사현물환최대금액 환산 재산출
  │     │      TB_FXB_CMG014F(시장환율) × TRC001M.DD1_MXMM_TRAB_AMT(USD 1일한도, FX_USER_NO='ALL')
  │     │      영업일 = TB_FXB_CMG012M(FDM_ID='FRX') APLL_BSOP_YMD, BLTN_TOB='1'(1회차), JPY 는 ×100
  │     │      SQL err → CLOSE + e_msg "10118", RET_FAILURE
  │     ├─ if (ii == max_row) → i_isnext++, break  (해당 행은 orec 에 미기록 — §6 경계행 유실)
  │     ├─ mod->orec[ii] 각 금액 "%.2f" 세팅, crcd zsedit
  │     └─ vKeyv = cKeyv(NEXT_KEY)   (다음 호출에서 커서가 참조 안 함 — §6)
  ├─ CLOSE W1411S01_C1
  ├─ 결과 판정
  │     ii==0 && i_isnext==0 → e_msg "91001" (자료 없음)
  │     i_isnext==1          → content_set_nkey(vKeyv), e_msg "91002" (계속)
  │     그 외                → e_msg "91003" (완료), grid01_cnt=ii
  └─ grid01_cnt = ii (재설정), sndl += (ii-1)*sizeof(orec[0]), RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_TRC001M` | 고객별거래한도기본 | `TRC001M` | **SELECT** | ① 고객약정 현물환 1일 USD 최대금액 단건, ② 커서 LEFT JOIN(고객 등록금액 `CUS_STUP_MXMM_TRN_AMT`), ③ SELECT 2 의 USD 1일한도 서브쿼리 |
| `TB_FXB_CMG008M` | 상품별거래한도기본 | `CMG008M` | **SELECT** | 커서 주 테이블. BAR/SPT/FWD 별 당사 최대/최소(`FX_MXMM_TRN_AMT`/`FX_MNMM_TRN_AMT`), `FX_CUS_LMT_DCD='1'`(1회한도) |
| `TB_FXB_CMG014F` | 시장환율인터페이스 | `CMG014F` | **SELECT** | 비 USD 통화 당사현물환최대금액 환산용 고시환율(`FX_TUSA_XRT`, `BLTN_TOB='1'` 1회차, JPY ×100) |
| `TB_FXB_CMG012M` | 영업일자/기준일 관리 (추정 — `FDM_ID='FRX'`) | — | **SELECT** | SELECT 2 서브쿼리, 현재영업일(`APLL_BSOP_YMD`) 취득 |

*DML 요약: 전부 **SELECT** (단건 INTO 1 + 커서 1 + 루프 내 통화별 환산 SELECT 1). INSERT/UPDATE/DELETE 없음. 커서 `W1411S01_C1` (BAR/SPT/FWD 3중 UNION ALL → GROUP BY CRCD). 데이터 변경 없으나 규약상 성공 시 `l_dbcommit`.*

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='1'`(조회) 만 유효. 그 외 COMHDR `mesg="작업구분 오류."` 후 즉시 반환.
- **선행 검증**: 고객약정 현물환 1일 USD 최대금액(`DD1_MXMM_TRAB_AMT`)이 없으면(`NO_DATA_FOUND`) `10231` 로 조회 자체를 실패시킨다 — 현물환 최대금액 산출의 필수 기준값.
- **당사현물환최대금액 산출**: USD 는 등록된 1일 USD 최대금액을 그대로, 그 외 통화는 `조회일 1회차 고시환율(CMG014F) × 1일 USD 최대금액` 로 환산(JPY 는 ×100). 즉 1일 최대금액은 USD 로만 약정하고 타 통화는 실시간 환산.
- **커서 필터**: `FX_CUS_LMT_DCD='1'`(1회한도) 고정. 통화 필터 `crcd` 는 선택적(`NVL`). 고객 등록금액은 `TRC001M` LEFT JOIN 이라 미등록 시 `NVL(...,0)`.
- **FX_USER_NO 처리**: 첫 쿼리는 `'ALL'` 강제, 커서 조인은 입력 `vFxUserNo` 바인딩(화면이 `ALL` 로 내려준다는 전제).
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/결과 코드**(`e_msg.code` → `fnSetMsg` → COMHDR `mesg`):
  - `10231` 현물환 1일 거래한도 미등록(선행 SELECT NO_DATA) → RET_FAILURE.
  - `10118` 조회 오류(SELECT/커서 OPEN/FETCH/환산 SQL 오류) → RET_FAILURE.
  - `91001` 해당 자료 없음(커서 결과 0건) → RET_SUCCESS.
  - `91002` 조회 계속(NEXT 존재, `content_set_nkey`) → RET_SUCCESS.
  - `91003` 조회 완료 → RET_SUCCESS.
- **결과 전달**: `dd1MxmmTrabAmt` + `grid01_cnt` + `orec[ii]` + COMHDR(`eflg`/`mesg`). `sndl` 은 `O_SZ`(orec 1개 포함) 에 `(ii-1)*orec` 을 더해 정확히 `ii` 레코드 길이로 맞춘다.

## 6. 주의 — 문서/코드 불일치 (마이그레이션 시 확인)

| 위치 | 헤더/주석 표기 | 실제(`.pc`) |
|---|---|---|
| `.pc` L48 변수 주석 | `mod` = "W1411S01 의 입력값 주소" | 출력값(Output) 주소 — 복붙 오기 |
| 페이징(`vKeyv`) | `content->keyc` NEXT/PREV 로 시작 키 계산 | 커서 WHERE 에 `vKeyv` 미바인딩. 다음 페이지도 처음부터 재조회 → **페이징 미동작(무한 재조회 성격)**. 시작부 계산은 dead code, 루프 끝 `content_set_nkey(vKeyv)` 도 실효 없음 |
| L421 `if (ii == max_row)` | 다음 페이지 존재 표시 | 해당 경계 행을 fetch 후 `orec` 에 기록하지 않고 break(`i_isnext++`). 페이징이 깨져 있어 다음 호출에서 재조회되지 않으므로 **경계 행 1건이 출력에서 유실** |
| 유효성 섹션 주석(L187) | "입력항목 유효성 check" | 실제 NULL/공백 거부 없음 — `yprintfrtrim` trim 만 수행 |
| `.h` `fxUserNo`(30) | "FX사용자번호" | 첫 쿼리는 `'ALL'` 강제라 무시됨. 커서 조인에만 사용, 화면이 `ALL` 로만 내려주는 전제(`.pc` DECLARE 주석) |
| `.h` `orec[1]` | 배열 크기 1 | 실제 `max_row`(=`MAX_RRB/sizeof(orec[0])`) 개까지 기록 — grid 관례(가변 반복부), 선언 크기와 무관 |
| L474 / L481 `grid01_cnt` | — | 91003 분기에서 한 번, 함수 말미에서 또 한 번 `zredit` — 중복 설정(동작 무해) |
| 미사용 지역변수 | — | `i, cnt, editb, ret, ii(진입부), retc(진입부), error_f, i_sndcnt, log_m, vBaseCrcd/vCocuCd 는 사용` — `error_f/i_sndcnt/log_m/editb/cnt/i/ret` 등 다수 미사용 |

> WTG 는 `.h` 를 파싱해 필드명/크기만 취하고 전문을 통과시키므로 위 주석/데드코드가
> 동작에 영향 없다. 단 **페이징 미동작 + 경계행 유실**은 통화 수가 `max_row` 를
> 초과할 때 실제 데이터 누락을 유발하므로, 마이그레이션 시 커서에 `WHERE NEXT_KEY > :vKeyv`
> 바인딩(또는 전건 반환)으로 교정 필요.
