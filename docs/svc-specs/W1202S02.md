# 서비스 명세 — `W1202S02` (시장환율인터페이스 (회차 조회))

## 2. I/O 전문 (`.h`)

**Input `W1202S02_I`** (총 9B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 처리구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" |
| `bltnYmd` | 8 | 고시년월일 (YYYYMMDD) — 조회 키. WHERE `BLTN_YMD = :vBltnYmd` 에 바인딩 |

**Output `W1202S02_O`** (grid — 헤더 + 가변 orec[])

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (조회된 회차 수, `zredit` 우측정렬 정수) |
| `orec[].bltnTob` | 4 | 고시회차 (`CMG014F.BLTN_TOB`) — 반복 레코드 |

> grid 출력. `.h` 는 `orec[1]` 로 선언되나 `.pc` 가 `content->sndl` 을 `(건수-1)*sizeof(orec[0])`
> 만큼 늘려 실제 N건을 가변 전송. 성공/실패는 COMHDR (`eflg`/`mesg`) 로 함께 전달.

## 3. 구현 흐름 (`.pc`)

```
W1202S02()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' → retc = W1202S02_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return
  │            (eflg 미설정, MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? eflg="0",l_dbcommit() : eflg="1",l_dbrollback()
        → fnSetMsg(e_msg, comhdr_o)   (명시적 커밋, AUTOCOMMIT 아님)

W1202S02_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1202S02_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])        (페이지당 최대 건수)
  ├─ GRID key 처리 (content->keyc)  K_NEXT→nkey / K_PREV→pkey / default→"0"
  │      → vKeyv 세팅 후 l_rtrim   (계산만 하고 실제 SQL 엔 미사용 — §6 dead code)
  ├─ 입력 바인딩: yprintf + l_rtrim → vBltnYmd  (유효성 실검증 없음 — §6)
  ├─ EXEC SQL DECLARE W1202S02 CURSOR FOR
  │       SELECT DISTINCT CMG014F.BLTN_TOB
  │         FROM TB_FXB_CMG014F CMG014F
  │        WHERE CMG014F.BLTN_YMD = :vBltnYmd     (-- AND ... > :vKeyv 는 주석처리)
  │        ORDER BY CMG014F.BLTN_TOB
  │      DECLARE err → e_msg "10118", RET_FAILURE
  ├─ EXEC SQL OPEN
  │      NO_DATA_FOUND → e_msg "91001", RET_FAILURE
  │      기타 err      → e_msg "10118", RET_FAILURE
  ├─ FETCH 루프 (ii=0..)
  │      NO_DATA_FOUND → break
  │      기타 err → CLOSE + e_msg "10118" + RET_FAILURE
  │      ii == max_row → i_isnext++ , break        (다음 페이지 있음 표시)
  │      → zsedit(mod->orec[ii].bltnTob, cBltnTob)
  │      → vKeyv = cBltnTob (NEXT KEY 후보)
  ├─ EXEC SQL CLOSE
  ├─ 결과 판정
  │      ii==0 && i_isnext==0 → e_msg "91001" (자료 없음)
  │      i_isnext==1          → content_set_nkey(vKeyv), e_msg "91002" (조회 계속)
  │      else                 → e_msg "91003" (조회 완료), grid01_cnt=ii
  └─ zredit(mod->grid01_cnt, ii)
     content->sndl += (ii - 1) * sizeof(orec[0])   (ii==0 시 -4B — §6)
     return RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_CMG014F` | 시장환율 고시 정보 (추정) | `CMG014F` | **SELECT** (커서) | `BLTN_YMD = :vBltnYmd` 로 해당 일자의 고시회차(`BLTN_TOB`) 를 `DISTINCT` + `ORDER BY BLTN_TOB` 로 목록 조회 |

*DML 요약: 커서 기반 SELECT 1문(`TB_FXB_CMG014F`) 뿐. INSERT/UPDATE/DELETE 없음, 공통함수 경유 간접 접근도 없음. 조인 없음(단일 테이블). 데이터 변경 없으나 규약상 성공 시 `l_dbcommit`.*

## 5. 로직 / 필터 규칙

- **DML**: 단일 테이블 커서 SELECT (`DECLARE`/`OPEN`/`FETCH`/`CLOSE`). `DISTINCT`로 회차 중복 제거, `ORDER BY BLTN_TOB` 오름차순.
- **필터**: 고시년월일(`bltnYmd`) 단일 조건. 다음 페이지 이어받기용 `AND BLTN_YMD > :vKeyv` 조건은 소스에 주석 처리되어 실제로는 미적용(§6).
- **입력 유효성**: 3절 주석은 "입력항목 유효성 check" 이나 실제 `bltnYmd` 를 `yprintf/l_rtrim` 만 하고 NULL/공백/형식 거부 로직 없음(§6). 빈 일자는 매칭 0건 → `91001` 로 귀결.
- **페이징**: `content->keyc`(NEXT/PREV/최초)로 `vKeyv` 를 만들고 `max_row` 초과 시 `i_isnext`+`content_set_nkey` 로 NEXT KEY 를 반환하지만, 커서 WHERE 에 `vKeyv` 가 바인딩되지 않아 다음 페이지 요청 시에도 항상 같은 첫 페이지가 반환됨 → **페이징 미동작(§6)**.
- **트랜잭션**: `RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님. 조회 전용이나 규약 준수.
- **에러/결과 코드**:
  - 성공(완료): `91003` "조회가 완료되었습니다.", `eflg=0`, `grid01_cnt=ii`.
  - 계속: `91002` "조회가 계속됩니다.", NEXT KEY 설정, `eflg=0`.
  - 자료 없음: `91001` "해당하는 자료가 없습니다." (ii==0), `eflg=0` 이지만 F1 은 `RET_SUCCESS` 반환 → 상위서 commit (§6).
  - 조회중 오류: `10118` (DECLARE/OPEN/FETCH SQL 오류), `RET_FAILURE→eflg=1,rollback`.
  - 작업구분 오류: COMHDR `mesg="작업구분 오류."` (eflg 미세팅).
- **결과 전달**: grid 헤더(`grid01_cnt`) + orec[] + COMHDR(`eflg`/`e_msg.code`→`mesg`). WTG `/v1/tx` 는 `errn`≠0 이어도 전문 있으면 200, COMHDR 본문 동봉(레거시 규약).
