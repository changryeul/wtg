# 서비스 명세 — `W1351S01` (외환중개사조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1351S01` |
| 이름 | 외환중개사조회 |
| 유형 | 조회 (그리드, SELECT 전용 — DML 없음) |
| 구현 | `win/src/trn/W1350/W1351S01.pc` (397 라인) |
| I/O 헤더 | `win/src/inc/trn/W1351S01.h` |
| 라우팅 alias | `W1351S01` |
| 채널 | 매매 (COMHDR 기반 고정폭 전문) |
| 모듈 | W1350 |

외환중개사기본 마스터(`TB_FXB_CMG001M`)를 조건 필터 + 키셋 페이징으로 조회해 그리드로 반환한다. 작업구분(`prGb`)은 `'1'`(조회) 하나만 처리한다.

## 2. I/O 전문 (`.h`)

**Input `W1351S01_I`** (총 112 byte)

| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| `prGb` | 1 | 작업구분. `'1'`=조회. 그 외는 "작업구분 오류" 반환 |
| `fdmId` | 10 | 외환중개사ID 필터 |
| `fdmNm` | 100 | 외환중개사명 필터 (코드검색화면용) |
| `useYn` | 1 | 사용여부 필터 (Y/N, 빈값이면 전체) |

**Output `W1351S01_O`** (그리드 — `grid01_cnt` + `orec[]` 가변, 실제 채워지는 건수만큼 재계산)

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `grid01_cnt` | 6 | 그리드 건수 (반환 레코드 수) |
| `orec[].fdmId` | 10 | 외환중개사ID |

## 3. 구현 흐름 (`.pc`)

```
W1351S01()                                  진입/dispatch
  ├ prthdr / getmid / getmod               COMHDR·mid·mod 주소 확보
  ├ E_MSG 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  ├ switch(mid->prGb[0])
  │    case '1' → W1351S01_F1()             조회
  │    default  → comhdr_o.mesg="작업구분 오류." → return sndl (commit/rollback 미수행)
  ├ retc==RET_SUCCESS → eflg="0" + l_dbcommit()
  │            else    → eflg="1" + l_dbrollback()
  ├ fnSetMsg(&e_msg, comhdr_o)              e_msg.code → 응답 mesg/rcod 세팅
  └ return content->sndl

W1351S01_F1()                               조회 본체
  ├ getmid / getmod, sndl += O_SZ, memset(mod)
  ├ max_row = MAX_RRB / sizeof(orec[0])     한 전문에 실을 최대 행수
  ├ switch(content->keyc)                   페이징 키 획득
  │    K_NEXT → content_get_nkey → vKeyv
  │    K_PREV → content_get_pkey → vKeyv
  │    default(최초) → vKeyv="0"
  ├ yprintf/l_rtrim 로 vFdmId/vFdmNm/vUseYn 정규화
  ├ DECLARE W1351S01_C1 CURSOR (아래 §5 WHERE)
  ├ OPEN → SQLCODE 검사
  │    NO_DATA_FOUND → e_msg.code="91001", return RET_FAILURE
  │    기타 오류     → e_msg.code="10118", return RET_FAILURE
  ├ for(ii=0;;ii++) FETCH INTO c*
  │    NO_DATA_FOUND → break
  │    기타 오류     → CLOSE + e_msg.code="10118", return RET_FAILURE
  │    ii==max_row  → i_isnext++, break      (다음 페이지 존재 표시)
  │    zsedit 로 orec[ii] 채움, vKeyv = cKeyv(=FDM_ID)  다음키 갱신
  ├ CLOSE W1351S01_C1
  ├ ii==0 && i_isnext==0 → e_msg.code="91001" (자료 없음)
  │    else i_isnext==1  → content_set_nkey + e_msg.code="91002" (계속)
  │    else              → e_msg.code="91003" (완료)
  ├ zredit(grid01_cnt, ii), sndl += (ii-1)*sizeof(orec[0])
  └ return RET_SUCCESS
```

트랜잭션: 조회 전용이나 상위 함수가 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()`을 명시적으로 호출한다(엔진 AUTOCOMMIT 아님).

## 5. 로직 / 필터 규칙

- **커서 `W1351S01_C1`** — `TB_FXB_CMG001M` 단일 테이블, `ORDER BY FDM_ID`, `FDM_ID` 기준 키셋 페이징(`FDM_ID > :vKeyv`).
- **동적 필터 (WHERE)**:
  - `trim(:vFdmNm) IS NULL` (메인화면조회) → `FDM_ID = NVL(:vFdmId, FDM_ID)` — ID 완전일치, 빈값이면 전체.
  - `trim(:vFdmNm) IS NOT NULL` (코드검색화면조회) → `FDM_ID LIKE '%'||trim(:vFdmId)||'%' OR FDM_NM LIKE '%'||trim(:vFdmNm)||'%'` — ID/명 부분일치.
  - `USE_YN = NVL(:vUseYn, USE_YN)` — 사용여부 필터, 빈값이면 전체.
  - `FDM_ID > :vKeyv` — 페이징 커서 (최초 조회는 `vKeyv="0"`).
- **페이징**: 한 전문당 `max_row = MAX_RRB / sizeof(orec[0])` 행. `ii == max_row` 도달 시 `i_isnext=1` 로 표시하고 break → `content_set_nkey`로 다음키(마지막 FDM_ID) 전달.
- **전문 길이 재계산**: `sndl += O_SZ` 후 `sndl += (ii-1)*sizeof(orec[0])` — 실제 채운 행수(ii)만큼 보정 (`O_SZ`에 orec[1] 1건이 포함되어 있으므로 ii=0이면 그리드 헤더만 남김).
- **에러 코드 (e_msg.code → fnSetMsg)**:
  - `91001` 해당하는 자료가 없습니다 (OPEN 시 NO_DATA 또는 fetch 0건).
  - `91002` 조회가 계속됩니다 (다음 페이지 존재).
  - `91003` 조회가 완료되었습니다 (마지막 페이지).
  - `10118` 자료 조회중 오류가 발생하였습니다 (OPEN/FETCH SQL 오류).
  - `eflg`: 성공 "0" / 실패 "1". `rtyp="M"`(메시지 전송), `mtyp="S"`(FXON), `ltyp="KR"`.
