# 서비스 명세 — `W1358S03` (고객거래가능시간 적용년월일 검색)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며, `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1358S03` |
| 이름 | 고객거래가능시간 적용년월일 검색 |
| 유형 | 조회 (SELECT only — grid 목록 반환) |
| 구현 | `win/src/trn/W1350/W1358S03.pc` (289 라인) |
| I/O 헤더 | `win/src/inc/trn/W1358S03.h` |
| 라우팅 alias | `W1358S03` |
| 모듈 | W1350 |
| 채널 | ChannelWeb (`/v1/tx`) |

거래가능시간 관리 테이블(`TB_FXB_CMG039M`)에서 적용 년월일 구간(시작/종료)과 최종 변경 ID 를 distinct 목록으로 조회해 그리드로 돌려주는 단순 조회 TR.

## 2. I/O 전문 (`.h`)

**Input `W1358S03_I`** (1 byte)
| 필드 | 크기 | 설명 (실제 의미) |
|-----|-----|-----|
| prGb | 1 | 작업구분. `.pc` 는 `'1'` 만 처리(→ F1 검색), 그 외는 "작업구분 오류" 반환 |

**Output `W1358S03_O`** (grid — `orec` 가변)
| 필드 | 크기 | 설명 |
|-----|-----|-----|
| grid01_cnt | 6 | 그리드 반환 건수 (반환한 orec 개수 `ii`) |
| orec[].apsrYmd | 8 | 적용시작년월일 (CMG039M.APSR_YMD) |
| orec[].fnapYmd | 8 | 적용종료년월일 (CMG039M.FNAP_YMD) |
| orec[].fxSysLsmdId | 30 | FX 최종변경 ID (MAX(CMG039M.FX_SYS_LSMD_ID)) |

`.h` 상 `orec[1]` 로 선언되어 있으나 `.pc` 는 `max_row = MAX_RRB / sizeof(orec[0])` 만큼 반복 적재하는 가변 grid (전형적 orec[1] 관용 표현).

## 3. 구현 흐름 (`.pc`)

```
W1358S03()  진입/디스패치
  ├─ prthdr / getmid / getmod        // COMHDR·mid·mod 주소 확보
  ├─ e_msg 기본설정 (ltyp=KR, mtyp=S, rtyp=M)
  ├─ switch(mid->prGb[0])
  │     case '1' → W1358S03_F1()      // 검색
  │     default  → comhdr_o.mesg="작업구분 오류." 즉시 return(sndl)
  ├─ retc==RET_SUCCESS ? eflg="0" + l_dbcommit()
  │                    : eflg="1" + l_dbrollback()
  ├─ fnSetMsg(&e_msg, comhdr_o)       // e_msg.code → rtyp/rcod/mesg 세팅
  └─ return(content->sndl)

W1358S03_F1()  고객거래가능시간 적용년월일 검색
  ├─ getmid/getmod, content->sndl += W1358S03_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ switch(content->keyc)            // K_NEXT/K_PREV → vKeyv 추출, default → "0"
  ├─ DECLARE/OPEN W1358S03_C1         // 적용일자 distinct 조회 커서
  │     OPEN 실패: NO_DATA_FOUND → e_msg 91001 / else → 10118, return FAILURE
  ├─ for(ii=0;;ii++) FETCH
  │     NO_DATA_FOUND → break
  │     FETCH 오류 → CLOSE + e_msg 10118, return FAILURE
  │     ii==max_row → i_isnext++, break        // 다음페이지 존재 표시
  │     zsedit(orec[ii].*) 적재
  ├─ CLOSE W1358S03_C1
  ├─ 결과 판정
  │     ii==0 && i_isnext==0 → e_msg 91001 (자료 없음)
  │     i_isnext==1          → content_set_nkey + e_msg 91002 (계속)
  │     else                 → e_msg 91003 (완료)
  ├─ zredit(mod->grid01_cnt, ii)
  ├─ content->sndl += (ii-1) * sizeof(orec[0])
  └─ return(RET_SUCCESS)
```

트랜잭션: 조회 전용이나 공통 규약대로 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 명시 호출(AUTOCOMMIT 아님).

## 4. 사용 DB 테이블
| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-----|-----|-----|-----|-----|
| TB_FXB_CMG039M | 고객거래가능시간 관리 (적용일자) | CMG039M | SELECT | 적용시작/종료 년월일 + MAX(FX최종변경ID) 를 (APSR_YMD, FNAP_YMD) GROUP BY 로 distinct 조회 |

*DML 요약: SELECT 1건 (커서 `W1358S03_C1`). INSERT/UPDATE/DELETE 없음. 공통함수 경유 간접 테이블 접근 없음.*

## 5. 로직 / 필터 규칙
- **커서 `W1358S03_C1`**: `TB_FXB_CMG039M` 를 `GROUP BY APSR_YMD, FNAP_YMD` 로 묶고 `MAX(FX_SYS_LSMD_ID)` 를 선택, `ORDER BY APSR_YMD, FNAP_YMD` 정렬. **WHERE 절 없음** — 전체 적용일자 구간을 반환.
- **작업구분 분기**: `prGb='1'` 만 검색 실행. 그 외 값은 즉시 "작업구분 오류." 메시지로 반환(SQL 미실행).
- **페이징(의도)**: `content->keyc` 로 K_NEXT/K_PREV 시 vKeyv 를 세팅하고, fetch 가 `max_row` 도달 시 `i_isnext` 로 다음페이지 존재를 표시 → `content_set_nkey` + e_msg 91002. (단 실제 재개는 미동작 — §6 참조)
- **건수/전문길이**: `grid01_cnt=ii`, `content->sndl` 은 base(`+W1358S03_O_SZ`) 후 `(ii-1)*sizeof(orec[0])` 가산 → 순수 헤더 + ii 레코드 길이.
- **에러/메시지 코드 (e_msg.code → fnSetMsg)**:
  | code | 의미 | 발생 |
  |-----|-----|-----|
  | 91001 | 해당하는 자료가 없습니다 | OPEN NO_DATA_FOUND, 또는 fetch 0건 |
  | 91002 | 조회가 계속됩니다 | max_row 초과(다음페이지 존재) |
  | 91003 | 조회가 완료되었습니다 | 마지막 페이지까지 반환 완료 |
  | 10118 | 자료 조회중 오류가 발생하였습니다 | CURSOR OPEN/FETCH SQL 오류 |
- **eflg**: 성공 "0" / 실패 "1" (comhdr_o).

## 6. 주의 — 문서/코드 불일치
| 위치 | 헤더/주석 표기 | 실제 (`.pc`) |
|-----|-----|-----|
| `.pc` L36 함수 프로토타입 주석 | `W1358S03_F1 ... // 달러대가통화 조회` | 실제 F1 은 "고객거래가능시간 적용년월일 검색" (복붙 오기) |
| `.pc` L68 주석 | `//통화 목록 조회` | 실제 CMG039M 적용일자 목록 조회 (복붙 오기) |
| `.pc` L253-254, L258 NEXT KEY | `sprintf(vKeyv, "%.64s", cKeyv)` 로 다음키 세팅 | **버그**: `cKeyv` 는 DECLARE 만 되고 memset/대입이 전혀 없음(미초기화 garbage). 커서 SELECT 도 키 컬럼을 선택하지 않음. 따라서 vKeyv 로 세팅되는 next-key 가 무의미. |
| 페이징 전체 | K_NEXT/K_PREV 로 vKeyv 를 받아 이어서 조회하는 구조로 보임 | **버그(페이징 미동작)**: 커서에 vKeyv 를 쓰는 WHERE 절이 없어, 다음 호출 시에도 항상 처음부터 동일 첫 페이지를 재조회. 91002(계속)로 안내해도 실제 후속 페이지 이동 불가. |
| `.pc` L48 주석 | `mod ... 의 입력값 주소` | 실제 mod 는 출력값(Output) 주소 (주석 오기) |
| `.h` L36 prGb 주석 | `작업구분` (값 정의 없음) | 유효값은 `'1'` 뿐, 나머지는 오류 처리 |
