# 엔진 인증/로그인 사슬 ↔ WTG 매핑 (조사 노트)

MyMQ 매매 엔진(`~/mywork/nh`)의 실제 인증·로그인·등급·마진 서비스를 소스에서 추적한
결과와, 그로부터 도출한 WTG 인증/마진 설계 방향 정리.

- 조사일: 2026-07-18
- 대상 소스: `nh/win/src/trn/W1100`, `W1101` (Pro*C `.pc` + `inc/trn/*.h`)
- 관련 기존 문서: `docs/auth.md` (기존 가정), `docs/margin-policy.md`, `docs/margin-business-spec.md`

---

## 1. 요약 (TL;DR)

1. **엔진 로그인은 단일 서비스가 아니라 인증 → 세션개설의 다단계 사슬**이다.
   공인인증서 인증(W1101S02) → 보안매체/OTP 검증(W1107A01) → 로그인처리·세션개설(W1130A02).
2. **어느 인증 서비스도 등급(tier)이나 권한을 반환하지 않는다.** 신원(실명번호/사용자번호)만
   확정한다. → 등급은 로그인 응답에서 못 받으므로 **별도 조회/미러링**이 필수.
3. 엔진이 발급하는 "세션 핸들"의 실체는 암호화 `cookie_t` 가 아니라
   **`lgnIdntCon`(일시+MAC+IP+사용자번호) 문자열**이다.
4. **마진의 실체는 "VIP/GOLD 등급"이 아니라 고객별·통화페어별 실제 스프레드 테이블**이다
   (W1114S01). 등급(고객분류코드)은 마진을 그룹으로 묶는 계층(마진적용그룹)일 뿐,
   최종 마진은 고객/그룹 단위 스프레드로 떨어진다.
5. → **WTG 권고**: JWT 에 거친 tier 를 박는 대신 **고객 식별자만** 담고, 마진 시세는
   mci-price 가 **고객별 스프레드를 조회·적용**하는 구조가 실제 업무에 부합. 미러링 원본은
   "등급 코드"가 아니라 **고객 스프레드(및 마진적용그룹)** 로 잡는다.

## 2. 엔진 인증/로그인 사슬

```
① 공인인증서 인증 (W1101S02)   → cifNo(실명번호) + lgnId 확보
        │
② 보안매체/OTP 검증 (W1107A01) → 성공/실패 (fxUserNo 기준)
        │
③ 로그인처리·세션개설 (W1130A02) → lgnIdntCon(세션 열쇠) + 영업일 반환
```

"너 누구야(인증)"는 ①②에서 끝나고, ③은 순수하게 세션을 여는 단계다.
①의 `cifNo/lgnId` → ③의 `fxUserNo` 로 잇는 사용자 매핑 조회가 중간에 한 번 더 있을 것으로
추정 (미확인 — §6 참조).

## 3. 각 서비스 상세

### ① W1101S02 — 공인인증서 로그인 인증처리
`nh/win/src/trn/W1101/W1101S02.pc`

| 방향 | 필드 | 내용 |
|-----|------|-----|
| 입력 | signMsg | 인증서명 (공인인증서 서명값, 10240B) |
| 출력 | cifNo | 실명번호 |
| 출력 | lgnId | 로그인ID |
| 출력 | otinYn | 타기관 발급여부 |
| 출력 | svrCert | 서버 key |

인증서 서명 검증 후 **실명번호 + 로그인ID** 반환. 등급/권한 없음.
(내부적으로 `CSC001M.CUS_CLCD` 를 읽기는 하나 출력엔 싣지 않음.)

### ② W1107A01 — 이용자비밀번호체크 (보안매체/OTP 검증)
`nh/win/src/trn/W1100/W1107A01.pc`

| 방향 | 필드 | 내용 |
|-----|------|-----|
| 입력 | fxUserNo, lgnId | 사용자 식별 |
| 입력 | secuMdiaDcd | 보안매체구분 (1:보안카드 2:OTP 3:스마트보안카드) |
| 입력 | otpSrn, otpSeqno, secuPswd1/2, otpCopCd, cdGrpDataCon | OTP/보안카드 값 |
| 출력 | returnCd / returnMsg | "S" / "OK" — 성공·실패만 |

내부 `fnChkScardInfo()`(보안매체 검증)를 호출해 결과만 반환. 사용자 데이터·등급 없음.

### ③ W1130A02 — 로그인처리 (세션 개설)
`nh/win/src/trn/W1100/W1130A02.pc`

| 방향 | 필드 | 내용 |
|-----|------|-----|
| 입력 | prGb | 처리구분 ('1'=로그인) |
| 입력 | fxUserNo, fxUserNm, lgnId | 이미 확정된 사용자 |
| 입력(COMHDR) | loip, maca, cont(F직원/H고객), fdid | IP/MAC/구분/펀드번호 |
| 출력 | lgnIdntCon | **로그인 식별번호 = 일시+MAC+IP+사용자번호** (세션 열쇠) |
| 출력 | fwdPreChkPopYn | 선물환 사전점검 팝업 여부 |
| 출력 | apllBsopYmd/wlbrYmd/nxtBsopYmd | 당/전/익 영업일 |
| 출력 | lgnTs | 최종 로그아웃 일시 |

처리 순서(F1):
1. 입력검증(FX사용자번호 필수, **비번 검증 없음**)
2. 영업일 조회 (`TB_FXB_CMG012M`)
3. 최종 로그아웃 시각 조회 (`TB_FXB_CSC015L`, LGN_YN='N')
4. **중복 로그인 처리** — 기존 접속중(LGN_YN='Y') 세션을 커서로 찾아 강제 로그아웃
   (UPDATE LGN_YN='N', LGOT_SCD='1'). 단 **기존 세션 강제종료 broadcast + SMS 알람 코드는
   현재 통째 주석 처리** (DB 상태만 변경, 실제 밀어내기 통보는 비활성).
5. 세션 식별번호 `lgnIdntCon` 생성
6. 사용자 정보 조회 (`TB_FXB_CSC004M`: 별칭/로그인ID/사용자명)
7. **새 로그인 기록 INSERT** (`TB_FXB_CSC015L`, LGN_YN='Y', 매체 '1X', '정상접속') ← 세션 개설
8. 성공 반환 (eflg='0', 91003)

## 4. 등급/마진의 실체

### 4.1 등급 성격 값 — 두 갈래

**고객 단위 — 고객분류코드 (W1119S01)** `inc/trn/W1119S01.h`

| 출력 | 내용 |
|-----|-----|
| cusClcd | 고객분류코드 (3자리 **코드값**) |
| cusClcdNm | 코드명 |

코드명은 코드 마스터에서 해석:
```sql
SELECT CMCD_VL_NM FROM TB_FXB_CMC003D   -- 코드정보상세
 WHERE COM_CD_NM = 'CUS_CLCD' AND CMCD_VL = :cCusClcd
```
등급 체계는 `CMC003D` 의 `CUS_CLCD` 코드그룹에 정의됨.

**사용자 단위 — 사용자레벨/권한 (W1111S01)** `inc/trn/W1111S01.h`

| 출력 | 내용 |
|-----|-----|
| userLvlDcd | 사용자레벨구분코드 (`CSC004M.USER_LVL_DCD`) |
| screAcsAthrDcd | 화면접근권한 (리스크약정 + 선물환거래범위 조합) |
| fxUserDcd | FX사용자구분코드 |
| fwexTrnRngDcd | 선물환거래범위구분코드 |
| userSttsDcd | 사용자상태구분코드 |

→ 이쪽은 tier 가 아니라 **권한(authorization)**. WTG 는 건드리지 말고 엔진에 위임 (auth.md §1 원칙과 일치).

### 4.2 마진의 실체 — 고객별·통화페어별 스프레드 (W1114S01)
`inc/trn/W1114S01.h`

| 방향 | 필드 | 내용 |
|-----|------|-----|
| 입력 | wbcsRlnmAltrNo | 전행고객실명대체번호 (고객 단위 키) |
| 출력(grid) | crncPairId | 통화페어ID |
| | sptFxBymg / sptFxSlmg | SPOT FX 매입/매도 마진 |
| | fwdFxBymg / fwdFxSlmg | FWD FX 매입/매도 마진 |
| | sptBobmgPtrnDcd / fwdBobmgPtrnDcd | 영업점 마진유형 |

마진은 **고객 × 통화쌍 × SPOT/FWD** 단위로 실제 매입·매도 마진값이 저장된다.
거친 등급이 아니라 고객별 스프레드 테이블.

### 4.3 마진 적용 그룹 계층 (W2006S01 / W2010S01 / W2002S01)

`CUS_CLCD` 는 단독 등급이 아니라 마진 그룹과 엮임:
```
고객분류 본점마진적용그룹 유형코드 (hdomAplyGrpTcd)
고객분류 본점마진적용그룹 ID       (hdomAplyGrpId)
```
실제 모델: **고객분류 → 마진적용그룹 → 마진값**. 등급은 마진을 그룹으로 묶는 도구.

## 5. WTG 매핑 및 설계 권고

| WTG 개념 (auth.md 가정) | 엔진 실제 | 매핑/권고 |
|----------------------|----------|----------|
| 로그인 = id/pw 검증 + cookie_t | 인증 사슬(①②) + 세션개설(③) 분리 | WTG login 은 ①②③를 순차 호출하는 오케스트레이션 |
| cookie_t (348B 바이너리) | `lgnIdntCon` (일시+MAC+IP+사용자번호 문자열) | Redis `wtg:sess:<sid> → lgnIdntCon` 저장 |
| JWT claim `tier` (VIP/GOLD/STD) | 로그인 응답에 등급 **없음** | JWT 에 tier 대신 **고객 식별자**만 담기 |
| Profile 기반 마진 fan-out | 고객별·통화쌍별 스프레드 (W1114S01) | mci-price 가 **고객 스프레드 조회·적용** |
| 등급 카탈로그 미러링 | 원본 = 고객 스프레드/마진적용그룹 | fx-sync 패턴으로 **스프레드** 미러링 (등급 아님) |
| 권한(거래범위/화면접근) | userLvlDcd/screAcsAthrDcd 등 | WTG 미개입, 엔진 위임 (auth.md §1 유지) |

핵심: `auth.md` 의 "JWT tier 한 개" 모델은 단순화였고, 실제 업무는 훨씬 세밀하다.
WTG 는 등급을 흉내내기보다 **고객 식별자 + 고객별 스프레드 미러링**으로 가는 것이 정확.

## 6. 남은 확인 항목 (소스만으로 불가 — DB/추가 조사 필요)

1. **`CMC003D` 의 `CUS_CLCD` 실제 코드값** — 등급이 몇 단계이고 각 뜻이 무엇인지.
   소스에 시드/DDL 없음. 런타임 DB 조회 필요.
2. **`cifNo/lgnId` → `fxUserNo` 매핑 서비스** — ①인증과 ③세션개설 사이의 사용자 변환.
   (`CSC005R` 사용자별고객번호관계 / `CSC004M` 사용자정보기본 후보.)
3. **중복접속 강제종료 broadcast + SMS 주석 처리**가 의도인지 (W1130A02 §3.4).
4. **마진 최종 원본 테이블** — W1114S01 이 읽는 스프레드 테이블명 및 마진적용그룹(W2006/W2010)과의
   실제 조인 경로. WTG 미러링 대상 확정에 필요.
