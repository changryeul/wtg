# 서비스 명세 — `W1500A01` (에러코드 조회/등록/수정/삭제)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며,
> `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|---|---|
| 서비스 코드 | `W1500A01` |
| 이름 | 에러코드 조회/등록/수정/삭제 (`.pc` Description = "에러코드조회/등록/수정/삭제") |
| 유형 | **복합 (CRUD)** — `prGb` 로 조회(SELECT)/등록(INSERT)/수정(UPDATE)/삭제(DELETE) 분기 |
| 구현 | `win/src/trn/W1500/W1500A01.pc` (628 lines) |
| I/O 헤더 | `win/src/inc/trn/W1500A01.h` |
| 라우팅 alias | `W1500A01` → `POST /v1/tx {"alias":"W1500A01","data":{…}}` |
| 채널 | 진입부에 `ctyp→chnl` 매핑 없음 (COMHDR 그대로 사용) |

## 2. I/O 전문 (`.h`)

**Input `W1500A01_I`** (총 263B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 처리(작업)구분 — `'1'`조회 `'2'`등록 `'3'`수정 `'4'`삭제, 그 외 "작업구분 오류" |
| `errCd` | 5 | 에러코드 (조회: LIKE 부분검색 / 등록·수정·삭제: 키. 등록·수정 시 5자리 정확히 요구) |
| `lnggDcd` | 1 | 언어구분 — 조회에서 `'0'` 이면 전체, 그 외 `'1'`한글 `'2'`영어. 등록·수정·삭제 시 필수 |
| `errMsg` | 256 | 에러메시지 — 조회: LIKE 부분검색(대문자화) / 등록·수정: 저장할 메시지(필수). 삭제: 미사용 |

**Output `W1500A01_O`** (grid — 가변)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (조회 시 반환행 수, 등록·수정 시 `1`) |
| `orec[]` | 262 × N | 반복 레코드 (아래) — 헤더 정의는 `orec[1]` 이나 실제로 `max_row` 개까지 채운다 |
| └ `errCd` | 5 | 에러코드 |
| └ `lnggDcd` | 1 | 언어구분 |
| └ `errMsg` | 256 | 에러메시지 (DB `ERR_CD_CON`) |

> 등록/수정/삭제(F2/F3/F4)는 `grid01_cnt` 만 세팅하고 `orec` 는 비운다(memset).
> 처리 결과는 공통헤더 `eflg`/`mesg` 로 전달.

## 3. 구현 흐름 (`.pc`)

```
W1500A01()                       ← 진입 (콜백)
  ├─ prthdr / getmid / getmod    COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정               ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb[0])
  │     '1' → W1500A01_F1()  조회 (커서)
  │     '2' → W1500A01_F2()  등록 (INSERT)
  │     '3' → W1500A01_F3()  수정 (UPDATE)
  │     '4' → W1500A01_F4()  삭제 (DELETE)
  │     그외 → comhdr_o->mesg="작업구분 오류." 후 즉시 return
  ├─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)
  └─ fnSetMsg(&e_msg, comhdr_o)  → e_msg.code 를 mesg 로 편집해 응답

W1500A01_F1()  조회
  ├─ sndl += O_SZ, memset(mod), max_row = MAX_RRB / sizeof(orec[0])
  ├─ 페이징 키: content->keyc → K_NEXT(nkey) / K_PREV(pkey) / 최초("0") → vKeyv
  ├─ 입력 바인딩: yprintf+l_rtrim → vErrCd / vLnggDcd / vErrMsg  (유효성 검증 없음)
  ├─ DECLARE C1 CURSOR: TB_FXB_SYG001M LIKE 필터 3종 + 키 페이징(FX_ERR_CD||LNGG_DCD > :vKeyv)
  ├─ OPEN → NO_DATA_FOUND면 91001/RET_FAILURE, 그 외 오류 10118/RET_FAILURE
  ├─ FETCH 루프 (ii=0..):
  │     NO_DATA_FOUND → break / 기타오류 → CLOSE+10118+RET_FAILURE
  │     ii==max_row → i_isnext++, break (NEXT 페이지 존재)
  │     zsedit(orec[ii]) 채움, vKeyv = cKeyv(6자리)
  ├─ CLOSE
  ├─ 결과 판정: 무데이터 91001 / NEXT 있음 91002(+set_nkey) / 완료 91003
  └─ eflg="0", grid01_cnt=ii, sndl += (ii-1)*sizeof(orec[0]), RET_SUCCESS

W1500A01_F2()  등록
  ├─ 바인딩 vErrCd/vLnggDcd/vErrMsg, vSysLastUno = comhdr_i->usid(6)
  ├─ 유효성: errCd 길이≠5 → 32468 / errMsg 공백 → 32469 / lnggDcd 공백 → 00001  (모두 eflg="1", RET_FAILURE)
  ├─ INSERT TB_FXB_SYG001M (FX_ERR_CD,LNGG_DCD,ERR_CD_CON,FX_SYS_LSMD_ID,FX_SYS_LSMD_TS=SYSDATE)
  ├─ SQLCODE==-1(unique 위반) → 00108 "이미 등록", 기타 → 00105 "등록중 오류" (RET_FAILURE)
  └─ 정상 → grid01_cnt=1, eflg="0", e_msg 10150, RET_SUCCESS

W1500A01_F3()  수정
  ├─ 바인딩 + 동일 유효성 3종 (errCd 길이≠5 / errMsg 공백 / lnggDcd 공백)
  ├─ UPDATE TB_FXB_SYG001M SET ERR_CD_CON=:vErrMsg WHERE FX_ERR_CD=:x AND LNGG_DCD=:y
  ├─ SQLCODE==NO_DATA_FOUND → 00107 (※ UPDATE 는 NO_DATA_FOUND 미발생, §6) / SQLCODE!=0 → 00106
  └─ 정상 → grid01_cnt=1, eflg="0", e_msg 10150, RET_SUCCESS

W1500A01_F4()  삭제
  ├─ 바인딩 + 유효성: errCd 공백 → 32468 / lnggDcd 공백 → 00001 (errMsg 미검증)
  ├─ DELETE TB_FXB_SYG001M WHERE FX_ERR_CD=:x AND LNGG_DCD=:y   (SQLCODE 검증 없음, §6)
  └─ eflg="0", e_msg 10150, RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|---|---|---|---|---|
| `TB_FXB_SYG001M` | FX 에러코드 기본(마스터) | `SYG001M` | **SELECT / INSERT / UPDATE / DELETE** | 에러코드 마스터. 컬럼: `FX_ERR_CD`(에러코드), `LNGG_DCD`(언어구분), `ERR_CD_CON`(에러메시지 내용), `FX_SYS_LSMD_ID`(최종수정자=usid), `FX_SYS_LSMD_TS`(최종수정일시=SYSDATE) |

*DML 요약: 단일 테이블 `TB_FXB_SYG001M` 에 대해 prGb 분기별로 커서 SELECT(F1) / INSERT(F2) / UPDATE(F3) / DELETE(F4). 조인·다중테이블·공통함수 경유 접근 없음.*

## 5. 로직 / 필터 규칙

- **조회 필터 (F1)**:
  - `FX_ERR_CD LIKE '%'||:vErrCd||'%'` — 에러코드 부분일치.
  - `LNGG_DCD LIKE DECODE(:vLnggDcd,'0','',:vLnggDcd)||'%'` — `'0'` 이면 전체, 아니면 해당 언어 prefix.
  - `UPPER(ERR_CD_CON) LIKE '%'||UPPER(:vErrMsg)||'%'` — 메시지 대소문자 무시 부분일치.
  - 정렬 `ORDER BY FX_ERR_CD, LNGG_DCD`. 페이징 키 = `FX_ERR_CD||LNGG_DCD`(6자리) 기준 `> :vKeyv`.
  - `max_row = MAX_RRB / sizeof(orec[0])`. `ii==max_row` 도달 시 `i_isnext=1` + `set_nkey` → 다음 페이지 유도.
- **페이징 방향**: `content->keyc` 가 `K_NEXT`/`K_PREV`/기타(최초)로 nkey/pkey/"0" 세팅. (단 커서는 `> :vKeyv` 단방향이라 K_PREV 도 사실상 forward — §6)
- **입력 유효성**:
  - 조회(F1): 검증 없음 — 빈 조건이면 전건 조회.
  - 등록/수정(F2/F3): `errCd` 5자리 정확(32468), `errMsg` 필수(32469), `lnggDcd` 필수(00001).
  - 삭제(F4): `errCd` 필수(32468, 길이검증 없음), `lnggDcd` 필수(00001), `errMsg` 미검증.
- **트랜잭션**: 진입부에서 `retc==RET_SUCCESS→l_dbcommit / 실패→l_dbrollback`. AUTOCOMMIT 아님.
- **에러/메시지 코드 (e_msg.code → mesg)**:
  - 조회: `91001` 무데이터 / `91002` 계속(NEXT) / `91003` 완료 / `10118` 커서 오류.
  - 등록: `10150` 완료 / `00108` 중복 / `00105` 등록오류.
  - 수정: `10150` 완료 / `00107` 대상없음 / `00106` 수정오류.
  - 삭제: `10150` 완료.
  - 유효성: `32468`(에러코드) / `32469`(에러메시지) / `00001`(언어구분).
  - `eflg` 0 정상 / 1 오류. → WTG `/v1/tx` 는 errn≠0 이어도 전문 있으면 200, COMHDR 동봉(레거시 규약).

## 6. 주의 — 문서/코드 불일치 (마이그레이션 시 확인)

| 위치 | 헤더/주석 표기 | 실제(`.pc`) |
|---|---|---|
| `.h` Program Name / Rev | "에러코드조회" | 조회/등록/수정/삭제 CRUD 전부 (`prGb` 분기) |
| `.h` `orec[1]` | 1건 배열로 정의 | 실제 `max_row` 건까지 채우는 가변 grid (레거시 관용 패턴) |
| F3 UPDATE `SQLCODE==NO_DATA_FOUND` (517-530) | "등록된 에러코드 정보가 없습니다"(00107) | **UPDATE 는 매칭 0행이어도 NO_DATA_FOUND(1403) 미발생 → 이 분기 dead code**. 존재하지 않는 키 수정도 SQLCODE 0 → 10150 "완료" 로 응답(무동작 성공) |
| F4 DELETE (614-618) | — | **DELETE 후 SQLCODE 검증 전무**. 0행 삭제/오류여도 항상 eflg="0", 10150 "완료" 반환 |
| F1 OPEN `SQLCODE==NO_DATA_FOUND` (229) | 건수없음 처리(91001) | OPEN CURSOR 는 NO_DATA_FOUND 를 반환하지 않음 → 실질 dead branch (무데이터는 FETCH 루프 후 `ii==0` 경로에서 처리) |
| F1 DECLARE 직후 `if(SQLCODE!=0)` (219) | 커서 declare 오류 처리 | DECLARE CURSOR 는 런타임 no-op → SQLCODE 미갱신, dead check |
| F1 FETCH 오류 로그 (268) | "W3300S99_C1 CURSOR Fetch Error" | 타 서비스(W3300S99)에서 복붙된 로그 문자열 (동작 무관) |
| F1 결과판정 후 (330-331) | 91001/91002/91003 분기별 grid 세팅 | 분기 후 무조건 `eflg="0"`, `grid01_cnt=ii` 재설정 → 앞선 분기의 grid 세팅과 중복(무해) |
| F1 `keyc` K_PREV 처리 (171-172) | 이전 페이지 | 커서가 `> :vKeyv` 단방향이라 이전 방향 페이징 미지원 — pkey 로도 forward 조회 |
| F2/F3/F4 `vPrGb`, `vSysLastPrcssYms` | 선언됨 | 미사용 dead 변수 (F2/F3 는 `vSysLastUno` 만 INSERT 에 사용, F3/F4 는 usid 도 미사용) |

> WTG 는 `.h` 를 파싱해 필드명/크기만 취하고 전문을 통과시키므로 위 dead code/오기가
> WTG 동작에 영향을 주진 않는다. 다만 F3(존재하지 않는 키 수정)·F4(삭제 검증 부재)는
> 엔진 측 "무동작 성공" 응답 가능성이 있어, 클라이언트 UX(수정/삭제 성공 표시) 설계 시 유의.
