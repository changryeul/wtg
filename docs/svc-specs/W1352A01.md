## 3. 구현 흐름 (`.pc`)

```
W1352A01()                         ← 진입 (콜백)
  ├─ prthdr / getmid / getmod      COMHDR + input(mid) + output(mod) 준비
  ├─ e_msg 기본설정                ltyp=KR, mtyp=S, rtyp=M
  └─ switch(mid->prGb[0])          처리구분 분기
       ├─ '2','3' → W1352A01_F1()  등록/수정
       ├─ '4'     → W1352A01_F2()  삭제(논리)
       └─ default → comhdr_o.mesg="작업구분 오류." + return (commit/rollback 없음)
     ↓ retc 판정
     retc==RET_SUCCESS → eflg="0" + l_dbcommit()
     else              → eflg="1" + l_dbrollback()
     fnSetMsg(&e_msg, comhdr_o)     e_msg.code → 응답 메시지 세팅
     return content->sndl

W1352A01_F1()  등록/수정
  ├─ getmid/getmod, mod 초기화
  ├─ yprintf + l_rtrim 으로 입력값 정리 (vFdmId/vFxPdcd/vCrncPairId/vTnrID/
  │    vTrnAblYn/vRfrcCon/vUseYn), vFxSysLsmdId ← comhdr_i->usid
  ├─ 필수값 검증 (fdmId/fxPdcd/crncPairId/tnrID) — 실패 시 RET_FAILURE
  ├─ SELECT '1' FROM TB_FXB_CMC003D  상품별 유효 TENOR 존재 검증
  │    NO_DATA_FOUND → 10222 "가능 TENOR 아님", 기타 SQLCODE≠0 → 10118
  ├─ prGb=='3' → UPDATE TB_FXB_CMG002M (trn_abl_yn/rfrc_con/use_yn/변경자/변경시각)
  │    NO_DATA_FOUND → 10221, 기타 오류 → 10122
  └─ prGb=='2' → INSERT INTO TB_FXB_CMG002M
       DUP_KEY → 10223 (중복), 기타 오류 → 10122
     성공 → e_msg.code="10150" + RET_SUCCESS

W1352A01_F2()  삭제(논리)
  ├─ getmid/getmod, mod 초기화
  ├─ yprintf + l_rtrim (vFdmId/vFxPdcd/vCrncPairId/vTnrID), vFxSysLsmdId ← usid
  ├─ 필수값 검증 (fdmId/fxPdcd/crncPairId/tnrID) — 실패 시 RET_FAILURE
  └─ UPDATE TB_FXB_CMG002M SET USE_YN='N' (+ 변경자/변경시각)
       NO_DATA_FOUND → 10208 "NOTFOUND", 기타 오류 → 10122
     성공 → e_msg.code="10150" + RET_SUCCESS
```

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |

| `TB_FXB_CMC003D` | 공통상세코드 (TENOR 코드) | `CMC003D` | SELECT | `COM_CD_NM='TNR_ID'` + `CMCD_VL=tnrID` + `CMCD_VL_REF_CON3=fxPdcd` 로 상품별 유효 TENOR 존재검증 (F1) |
| `TB_FXB_CMG002M` | 외환중개사상품 마스터 | `CMG002M` | INSERT / UPDATE | 등록(INSERT, F1 prGb='2') / 수정(UPDATE, F1 prGb='3') / 논리삭제(UPDATE USE_YN='N', F2 prGb='4') |

*DML 요약: SELECT 1건(TENOR 유효성) + INSERT/UPDATE 1건(CMG002M). 물리 DELETE 없음 — 삭제는 `USE_YN='N'` UPDATE. 트랜잭션은 진입함수에서 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()`.*

## 5. 로직 / 필터 규칙

- **처리구분 분기**: `prGb` `'2'`=등록, `'3'`=수정 → `W1352A01_F1`, `'4'`=삭제 → `W1352A01_F2`. 그 외는 "작업구분 오류" 로 즉시 return (commit/rollback 미수행).
- **필수 입력 검증** (F1/F2 공통): `fdmId`(10205) / `fxPdcd`(10209) / `crncPairId`(10183) / `tnrID`(10210) 중 하나라도 공백이면 실패.
- **TENOR 유효성** (F1 등록/수정만): `TB_FXB_CMC003D` 에서 상품코드(`fxPdcd`)에 매핑된 TENOR(`tnrID`) 존재 여부 검증. 미존재 시 `10222`, 조회오류 시 `10118`.
- **기본값 처리**: `trnAblYn`, `useYn` 빈값 시 SQL `NVL(...,'Y')` 로 `'Y'` 저장. `rfrcCon` 은 `RTRIM`.
- **키 구성**: `FDM_ID + FX_PDCD + CRNC_PAIR_ID + TNR_ID` 4개 컬럼이 대상 레코드 식별 키 (UPDATE/DELETE WHERE, INSERT PK).
- **변경 이력**: 모든 write 에 `FX_SYS_LSMD_ID = comhdr_i->usid`, `FX_SYS_LSMD_TS = SYSDATE` 스탬프.
- **에러 코드 (e_msg.code → fnSetMsg)**:
  - `10150` — 정상 처리 (RET_SUCCESS)
  - `10205` / `10209` / `10183` / `10210` — 필수값 누락 (외환중개사ID / 상품코드 / 통화페어ID / 표준Tenor)
  - `10222` — 해당 상품의 가능 TENOR 아님 / `10118` — TENOR 공통코드 조회 오류
  - `10221` — 수정 대상 데이터 없음 (F1 UPDATE NO_DATA)
  - `10223` — 등록 중복 (INSERT DUP_KEY) / `10122` — 등록·수정·삭제 SQL 일반 오류
  - `10208` — 삭제 대상 없음 (F2 UPDATE NO_DATA)
- **eflg**: RET_SUCCESS → `"0"`, 실패 → `"1"` (진입함수에서 세팅, commit/rollback 동반).
