# 서비스 명세 — `W1210S01` (표준 TENOR 목록조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 `.pc` 실 구현 기준이며,
> `.h` 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요

| 항목 | 값 |
|---|---|

| 이름 | 표준 TENOR 목록조회 (`.pc` Description = "표준TENOR목록조회") |
| 유형 | **조회 (SELECT)** — 공통코드 `TNR_ID`(표준 TENOR) 목록을 grid 로 페이징 조회 |

## 2. I/O 전문 (`.h`)

**Input `W1210S01_I`** (총 201B)

| 필드 | 크기 | 설명 (실제) |
|---|---|---|
| `prGb` | 1 | 작업구분 — `'1'` 만 처리(조회), 그 외 "작업구분 오류" |
| `cmcdVl` | 100 | 코드값 검색어 — `CMCD_VL LIKE %값%` (UPPER 적용) |
| `cmcdEnsnNm` | 100 | 코드명 검색어 — `CMCD_VL_NM LIKE %값%` (UPPER 적용) |

> 두 검색어는 **OR** 로 결합된다 (§5 참조). 페이징 NEXT/PREV 키는 전문 필드가 아니라
> `content` 의 nkey/pkey(=`CMCD_VL`) 로 전달된다.

**Output `W1210S01_O`** — grid (가변 `orec[]`)

| 필드 | 크기 | 설명 |
|---|---|---|
| `grid01_cnt` | 6 | 그리드 건수 (`zredit(ii)`) |
| `orec[].cmcdVl` | 100 | 공통코드값 (`CMC003D.CMCD_VL`) |
| `orec[].cmcdEnsnNm` | 100 | 코드명 (`CMC003D.CMCD_VL_NM`) — 헤더 주석은 "영문명" 이나 실제는 코드값명 (§6) |

> `.h` 는 `orec[1]` 로 선언되나 실제로는 `max_row = MAX_RRB / sizeof(orec[0])` 만큼 채워지는
> 가변 grid. 한 레코드 = 200B.

## 3. 구현 흐름 (`.pc`)

```
W1210S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  ├─ switch(mid->prGb)
  │     '1' → retc = W1210S01_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류." 후 즉시 return (MSG_NOWORK)
  └─ retc==RET_SUCCESS ? (eflg="0", l_dbcommit()) : (eflg="1", l_dbrollback())
     └─ fnSetMsg(&e_msg, comhdr_o)   e_msg.code → COMHDR mesg/rcod 반영

W1210S01_F1()                   ← 조회 본체
  ├─ getmid/getmod, sndl += O_SZ, mod 0-clear
  ├─ max_row = MAX_RRB / sizeof(orec[0])
  ├─ content->keyc 로 페이징 키 획득
  │     K_NEXT → content_get_nkey → vKeyv
  │     K_PREV → content_get_pkey → vKeyv
  │     그외   → vKeyv = "0" (최초)
  ├─ 입력 검색어 정리 (yprintf + l_rtrim: vCmcdVl, vCmcdEnsnNm)
  ├─ DECLARE/OPEN W1210S01_C1  (TB_FXB_CMC002M ⨝ TB_FXB_CMC003D, COM_CD_NM='TNR_ID')
  │     OPEN 실패 → NO_DATA면 e_msg 91001, else 10118 → RET_FAILURE
  ├─ for(ii=0; ; ii++) FETCH
  │     NO_DATA → break
  │     기타 오류 → CLOSE + e_msg 10118 → RET_FAILURE
  │     ii == max_row → i_isnext++, break (해당 행 미기록)
  │     행 기록: orec[ii].cmcdVl / cmcdEnsnNm, vKeyv = cCmcdVl (NEXT KEY)
  ├─ CLOSE W1210S01_C1
  ├─ (ii==0 && i_isnext==0) → e_msg 91001 (자료없음)
  │   else i_isnext==1 → content_set_nkey(vKeyv), e_msg 91002 (계속)
  │        else        → e_msg 91003 (완료), grid01_cnt=ii
  └─ grid01_cnt = ii, sndl += (ii-1)*sizeof(orec[0]), return RET_SUCCESS
```

| `TB_FXB_CMC002M` | 공통코드 마스터 | `CMC002M` | SELECT | `COM_CD_NM='TNR_ID'` 코드 그룹 조인 필터 |
| `TB_FXB_CMC003D` | 공통코드 상세 | `CMC003D` | SELECT | 표준 TENOR 코드값/코드명/사용여부/정렬키 조회 |

## 5. 로직 / 필터 규칙

- **커서 `W1210S01_C1`** — CMC002M ⨝ CMC003D (`COM_CD_NM` 조인) + `COM_CD_NM='TNR_ID'` + `USE_YN='Y'`.
- **검색 필터 (OR)**:
  `CMCD_VL LIKE '%'||UPPER(RTRIM(:vCmcdVl))||'%' OR CMCD_VL_NM LIKE '%'||UPPER(RTRIM(:vCmcdEnsnNm))||'%'`
  — 코드값 검색어와 코드명 검색어가 **OR** 로 묶이며, 컬럼과 바인드 모두 UPPER 처리(대소문자 무시). 두 검색어 모두 비면 `LIKE '%%'` 로 전건.
- **페이징 (keyset)**: `AND CMCD_VL > :vKeyv`, 최초 `vKeyv="0"`. NEXT 시 마지막 `CMCD_VL` 를 nkey 로 저장.
  정렬은 `ORDER BY CMC003D.CMCD_VL_REF_CON2` (정렬용 참조 컬럼).
- **페이지 크기**: `max_row = MAX_RRB / sizeof(orec[0])`. `ii==max_row` 도달 시 해당 행은 기록하지 않고 `i_isnext=1` 로 "다음 있음" 표시.
- **트랜잭션**: 조회 전용이나 진입부에서 결과에 따라 commit/rollback 명시 (AUTOCOMMIT 아님).
- **에러/상태 코드** (`e_msg.code`, `fnSetMsg` 로 COMHDR 반영):
  | code | 의미 | 조건 |
  |---|---|---|
  | `91001` | 해당하는 자료가 없습니다 | 조회 결과 0건 |
  | `91002` | 조회가 계속됩니다 | `max_row` 초과(다음 페이지 존재), nkey 세팅 |
  | `91003` | 조회가 완료되었습니다 | 마지막 페이지 |
  | `10118` | 자료 조회중 오류 | 커서 OPEN/FETCH SQL 오류 |
  - eflg: 성공 `"0"`, 실패 `"1"` (진입 함수에서 retc 기준 세팅).
