# 서비스 명세 — `W1201S02` (통화별고정휴일조회)

> 작성 양식: `docs/svc-spec-template.md`. 본 명세는 .pc 실 구현 기준이며, .h 헤더 주석과 어긋나는 부분은 §6 주의에 정리했다.

## 1. 개요
| 항목 | 값 |
|-----|-----|
| 서비스 코드 | `W1201S02` |
| 서비스 이름 | 통화별고정휴일조회 |
| 유형 | 조회 (단일 그리드, NEXT 페이징) |
| 구현 | `win/src/trn/W1200/W1201S02.pc` (319 lines) |
| I/O 헤더 | `win/src/inc/trn/W1201S02.h` |
| 모듈 | W1200 |
| 라우팅 alias | `W1201S02` |
| 채널 | `ChannelWeb` (mci-api `/v1/tx`) |

FX 고정휴일 마스터(`TB_FXB_CMG010M`, `FDM_ID='FRX'`)에서 입력 통화코드에 해당하는 고정휴일(월일 + 휴일명) 목록을 조회한다. `FDM_ID||CRCD||FXNG_HLDD_MD` 를 키로 하는 커서 기반 NEXT 페이징이다.

## 2. I/O 전문 (`.h`)

**Input `W1201S02_I`** (총 4 byte)

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `prGb` | 1 | 작업구분 — `.pc` 는 `'1'` (조회) 만 처리, 그 외는 "작업구분 오류" |
| `crcd` | 3 | 통화코드 — 필수 (빈값이면 32029 반환) |

**Output `W1201S02_O`** (grid — `orec` 가변, 헤더 정의는 `orec[1]`)

| 필드 | 크기 | 설명 |
|-----|-----|-----|
| `grid01_cnt` | 6 | 그리드 건수 (반환 행수 `ii`, zredit) |
| `orec[].crcd` | 3 | 통화코드 |
| `orec[].fxngHlddMd` | 4 | 고정휴일월일 (MMDD) |
| `orec[].fxHlddNm` | 50 | FX휴일명 |

`orec` 는 `MAX_RRB / sizeof(orec[0])` 만큼 반복 가능. 전송 길이는 `W1201S02_O_SZ + (ii-1)*sizeof(orec[0])` 로 확장.

## 3. 구현 흐름 (`.pc`)

```
W1201S02()                              # 진입 dispatch
  prthdr / getmid / getmod             # 공통헤드 + 입력/출력 주소
  e_msg 기본설정 (KR / S / M)
  switch(mid->prGb[0])
    case '1': W1201S02_F1()             # 조회
    default : comhdr_o->mesg="작업구분 오류." → return sndl
  retc==RET_SUCCESS ? eflg="0"+l_dbcommit() : eflg="1"+l_dbrollback()
  fnSetMsg(&e_msg, comhdr_o)            # e_msg.code → mesg/rcod 편집
  return content->sndl

W1201S02_F1()                           # 조회 본체
  getmid/getmod; sndl += O_SZ; memset(mod)
  max_row = MAX_RRB / sizeof(orec[0])
  switch(content->keyc)                 # NEXT/PREV → vKeyv, 최초는 "0"
    K_NEXT: content_get_nkey
    K_PREV: content_get_pkey
    default: vKeyv="0"
  yprintf(vCrcd, mid->crcd); l_rtrim
  if strlen(vCrcd)==0 → e_msg.code="32029" → RET_FAILURE
  DECLARE W1201S02_C1 CURSOR (SELECT ... TB_FXB_CMG010M WHERE FDM_ID='FRX' AND CRCD=:vCrcd AND key>:vKeyv)
  OPEN → SQLCODE 검사 (NO_DATA→91001 / else→10118)
  for(ii=0;;ii++)
     FETCH → NO_DATA break / err→CLOSE+10118 RET_FAILURE
     if ii==max_row → i_isnext++; break     # 한 행 더 읽어 NEXT 판단 (미저장)
     zsedit orec[ii]; vKeyv = cKeyv          # NEXT KEY 갱신
  CLOSE W1201S02_C1
  if ii==0 && i_isnext==0 → 91001 RET_FAILURE
  else if i_isnext==1 → content_set_nkey(vKeyv); 91002 (계속)
       else → 91003 (완료); grid01_cnt=ii
  grid01_cnt=ii; sndl += (ii-1)*sizeof(orec[0])
  return RET_SUCCESS
```

트랜잭션: 조회 전용이나 상위 dispatch 가 성공 시 `l_dbcommit()`, 실패 시 `l_dbrollback()` 을 명시적으로 호출한다.

## 4. 사용 DB 테이블

| 테이블 | 한글명 | 별칭 | 접근 | 용도 |
|-------|-------|-----|-----|-----|
| `TB_FXB_CMG010M` | FX 고정휴일 마스터 (통화별) | `CMG010M` | SELECT | `FDM_ID='FRX'` + 통화코드 필터로 고정휴일(월일/휴일명) 조회 |

*DML 요약: SELECT 1건 (커서 `W1201S02_C1`). INSERT/UPDATE/DELETE 없음. 간접 공통함수 경유 테이블 접근 없음.*

## 5. 로직 / 필터 규칙

- **작업구분**: `prGb='1'` 만 조회 수행. 그 외는 comhdr 에 "작업구분 오류" 세팅 후 즉시 반환(커밋/롤백 분기 미경유).
- **필수 입력**: `crcd` 빈값이면 `32029` (입력값[통화코드]을 확인하세요) → RET_FAILURE.
- **페이징**: 키 `FDM_ID||CRCD||FXNG_HLDD_MD`. 최초는 `vKeyv="0"`, K_NEXT/K_PREV 는 content nkey/pkey 사용. 커서 조건 `key > :vKeyv` 로 다음 페이지 진행.
- **NEXT 판정**: 페이지당 `max_row` 행 저장 후 `ii==max_row` 인 (max_row+1 번째) 행을 한 번 더 FETCH 하면 `i_isnext++` 하고 저장 없이 break → `content_set_nkey` + `91002`(조회가 계속됩니다). 그렇지 않으면 `91003`(조회가 완료되었습니다).
- **건수**: `grid01_cnt = ii` (실제 저장 행수).
- **에러/메시지 코드 (e_msg.code → fnSetMsg 로 mesg 편집)**:
  - `32029` 입력값[통화코드] 확인 (필수 누락)
  - `10118` 자료 조회중 오류 (커서 OPEN/FETCH SQL 오류)
  - `91001` 해당하는 자료가 없습니다 (0건)
  - `91002` 조회가 계속됩니다 (NEXT 존재)
  - `91003` 조회가 완료되었습니다 (마지막 페이지)
- **eflg**: dispatch 에서 성공 `"0"`, 실패 `"1"`.

## 6. 주의 — 문서/코드 불일치

| 위치 | 헤더/주석 표기 | 실제(.pc) |
|-----|-------------|----------|
| `.pc` L47 | `mod` 주석 "서비스 코드 W1201S02 의 입력값 주소" | 실제로는 출력값(Output) 주소 — 복붙 오기 |
| `.pc` L214-221 | 커서 OPEN 직후 `SQLCODE==NO_DATA_FOUND → 91001` 분기 | Oracle 은 OPEN 시 NO_DATA_FOUND 를 리턴하지 않음 → 사실상 dead code. 0건 판정은 L287 (`ii==0 && i_isnext==0`) 에서 처리됨 |
| `.pc` L306, L313 | 완료 분기에서 `zredit(mod->grid01_cnt, ii)` | 완료 분기(L306)와 공통(L313)에서 중복 세팅 — 무해하나 중복 |
| `.pc` L283 | 로그 "다음데이터= [%s], cKeyv" | 루프 마지막 FETCH(NO_DATA) 시 `cKeyv` 는 memset 후 빈값 — 로그에 NEXT KEY 미표시. 실제 NEXT KEY 는 `vKeyv` |
| `.pc` L119-131 | 지역변수 다수 선언 (`cnt/ret/editb/i_sndcnt/log_m/error_f` 등) | 미사용 dead variable |
| `.pc` L166-167 | `K_PREV` 도 `content_get_pkey` 로 vKeyv 취득 | 커서 조건이 `key > :vKeyv` 단방향뿐이라 진정한 '이전' 역방향 페이징은 아님 (NEXT KEY 기반 forward-only) |
