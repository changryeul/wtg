## 3. 구현 흐름 (`.pc`)

```
W1215S01()                      ← 진입 (콜백)
  ├─ prthdr / getmid / getmod   COMHDR + input(mid) + output(mod) 준비
  ├─ E_MSG 기본설정              ltyp="KR", mtyp="S"(FXON), rtyp="M"(Message전송)
  │                             comhdr_o->rtyp = "M"
  ├─ switch(mid->prGb[0])
  │     '1' → retc = W1215S01_F1(...)     ← 실제 로직
  │     그외 → comhdr_o->rtyp="M", mesg="작업구분 오류.", eflg="1" 후 즉시 return
  │            (MSG_NOWORK 로그)
  └─ retc==RET_SUCCESS ? l_dbcommit() : l_dbrollback()   (명시적 커밋, AUTOCOMMIT 아님)

W1215S01_F1()                   ← 조회
  ├─ getmid/getmod, sndl += W1215S01_O_SZ, memset(mod)
  ├─ max_row = MAX_RRB / sizeof(orec[0])   페이지 최대 건수
  ├─ GRID key 처리 (content->keyc)  K_NEXT→nkey / K_PREV→pkey / default→"0"
  │      → vKeyv 세팅 후 l_rtrim  (계산만 하고 실제 SQL 엔 미사용 — §6 페이징 미동작)
  ├─ 입력 바인딩: yprintf + l_rtrim → vPairID / vPrdCd / vUseYn  (유효성 실검증 없음)
  ├─ DECLARE TB_FXB_COD_GENERAL_C1 CURSOR   (CMG006M × CMC003D join)
  ├─ OPEN
  │     NO_DATA_FOUND → e_msg 91001, eflg="0", return RET_FAILURE (→ rollback)
  │     기타 SQLCODE≠0 → e_msg 10118, eflg="1", return RET_FAILURE
  ├─ FETCH 루프 (ii=0..)
  │     NO_DATA_FOUND → break
  │     기타 오류 → CLOSE, e_msg 10118, eflg="1", return RET_FAILURE
  │     ii == max_row → i_isnext++ , break  (해당 행은 orec 미기록, next 표시용)
  │     orec[ii] 채움 + vKeyv = cKeyv (next key)
  ├─ CLOSE
  ├─ 결과 판정
  │     ii==0 && i_isnext==0 → e_msg 91001 (자료없음), eflg="0"
  │     i_isnext==1          → content_set_nkey(vKeyv), e_msg 91002 (계속)
  │     그외                 → e_msg 91003 (완료), grid01_cnt=ii
  ├─ eflg="0", grid01_cnt=ii (무조건 재설정)
  └─ sndl += (ii-1)*orec_sz , return RET_SUCCESS
```

## 5. 로직 / 필터 규칙

- **작업구분**: `mid->prGb[0]=='1'` (조회) 만 처리, 그 외 "작업구분 오류" + `eflg="1"` 즉시 반환.
- **동적 필터 (NVL 바인딩)**: `CRNC_PAIR_ID = NVL(:vPairID, CRNC_PAIR_ID)`, `FX_PDCD = NVL(:vPrdCd, FX_PDCD)`, `USE_YN = NVL(:vUseYn, USE_YN)` — 입력 빈값이면 해당 조건 무시(전체). 단, 호스트 변수는 `yprintf` 로 채워지므로 빈 입력 시 실제 NULL 여부는 바인딩 규약에 의존.
- **조인 고정 조건**: `CMG006M.TNR_ID = CMC003D.CMCD_VL AND CMC003D.COM_CD_NM='TNR_ID' AND CMC003D.USE_YN='Y'`.
- **정렬**: `ORDER BY CRNC_PAIR_ID, CMC003D.CMCD_VL_REF_CON2`.
- **페이징**: `content->keyc` (K_NEXT/K_PREV/default) 로 `vKeyv` 산출 후 `l_rtrim` — 그러나 커서 WHERE 절에 `vKeyv` 를 **전혀 사용하지 않음**. next key(`content_set_nkey`)만 세팅될 뿐 다음 조회에 반영 안 됨 → 실질 페이징 미동작 (§6).
- **max_row 초과**: `ii==max_row` 시 마지막 FETCH 행은 기록하지 않고 `i_isnext=1` 로 "계속" 표시.
- **에러/메시지 코드**:
  - `10118` — CURSOR DECLARE/OPEN/FETCH 오류, `eflg="1"`.
  - `91001` — 자료 없음 (OPEN NO_DATA 또는 ii==0), `eflg="0"`.
  - `91002` — 조회 계속(다음 페이지 있음), `eflg="0"`, next key 세팅.
  - `91003` — 조회 완료, `eflg="0"`, `grid01_cnt=ii`.
  - 모든 메시지는 `fnSetMsg(&e_msg, comhdr_o)` 로 COMHDR 에 반영.
